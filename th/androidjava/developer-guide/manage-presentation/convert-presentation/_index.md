## **ภาพรวม**

Aspose.Slides for Android via Java สามารถโหลดไฟล์นำเสนอ PowerPoint และ OpenDocument ได้และบันทึกหรือเรนเดอร์เป็นรูปแบบอื่นๆ อีกหลายรูปแบบโดยไม่ต้องใช้ Microsoft PowerPoint, OpenOffice หรือ LibreOffice คุณสามารถแปลงไฟล์ PPT แบบเก่าเป็น PPTX สมัยใหม่ ส่งออกไฟล์นำเสนอเป็นเอกสารรูปแบบคงที่เช่น PDF และ XPS เผยแพร่สไลด์เป็น HTML หรือเรนเดอร์สไลด์เป็นไฟล์รูปภาพสำหรับพรีวิว, รูปย่อ และการเก็บถาวร

การแปลงเอกสารส่วนใหญ่ใช้ขั้นตอนการทำงานทั่วไปเดียวกัน: โหลดไฟล์ต้นฉบับ, เลือกรูปแบบผลลัพธ์ที่ต้องการ, และใช้ตัวเลือกเฉพาะรูปแบบตามความจำเป็น สำหรับรูปแบบภาพ, แต่ละสไลด์จะถูกเรนเดอร์แยกกันแล้วบันทึกเป็นภาพแรสเตอร์หรือเวกเตอร์ บทความเฉพาะที่เชื่อมโยงด้านล่างให้รายละเอียดการทำงานสำหรับแต่ละกรณี

## **การแปลง PowerPoint และ OpenDocument**

Aspose.Slides for Android via Java รองรับการแปลงจากรูปแบบการนำเสนอที่ใช้บ่อย เช่น PPT, PPTX, PPS, PPSX, POT, POTX, และ ODP API การแปลงเดียวกันใช้กับไฟล์ PowerPoint และ OpenDocument ดังนั้นกระบวนการที่บันทึกไฟล์ PPTX เป็น PDF มักจะสามารถใช้กับไฟล์ ODP ได้โดยเปลี่ยนเพียงไฟล์อินพุต

เมื่อทำการแปลงไฟล์ ODP อย่าลืมว่าแอปพลิเคชัน PowerPoint และ OpenDocument ไม่รองรับทุกคุณลักษณะของการจัดหน้าและรูปแบบในแบบเดียวกัน หากไฟล์ ODP ถูกสร้างใน LibreOffice หรือ OpenOffice Impress ให้ตรวจสอบผลลัพธ์และใช้ตัวเลือกที่อธิบายไว้ใน [แปลงการนำเสนอ OpenDocument](/slides/th/androidjava/convert-openoffice-odp/) เมื่อคุณต้องการคำแนะนำเฉพาะรูปแบบ

## **การแปลง PPT เป็น PPTX**

PPT คือรูปแบบ PowerPoint แบบไบนารีเก่า, ในขณะที่ PPTX คือรูปแบบ Office Open XML สมัยใหม่ Aspose.Slides for Android via Java รองรับการแปลง PPT เป็น PPTX ด้วยความแม่นยำสูงโดยคงรักษาโครงสร้างการนำเสนอที่ซับซ้อน เช่น มาสเตอร์, เลย์เอาต์, สไลด์, ชาร์ต, รูปร่างที่กลุ่ม, พื้นที่จุ, เฟรมข้อความ, เทกเจอร์, และการเติมรูปภาพ

สำหรับข้อมูลรายละเอียด ดูที่ [แปลง PPT เป็น PPTX](/slides/th/androidjava/convert-ppt-to-pptx/) และ [PPT vs PPTX](/slides/th/androidjava/ppt-vs-pptx/).

## **การส่งออกรูปแบบคงที่**

PDF, XPS, และ TIFF มีประโยชน์เมื่อผลลัพธ์ต้องดูเหมือนเดิมบนอุปกรณ์ต่างๆ และไม่ควรแก้ไขเป็นการนำเสนอ บทความเฉพาะของ PDF, XPS, และ TIFF อธิบายวิธีควบคุมการปฏิบัติตาม, สไลด์ที่ซ่อน, โน้ต, คุณภาพภาพ, การบีบอัด, รูปแบบพิกเซล, และขนาดผลลัพธ์

## **การส่งออก HTML และ Image**

การส่งออก HTML และ HTML5 มีประโยชน์สำหรับการดูในเบราว์เซอร์, การเผยแพร่บนเว็บ, และการแชร์ที่เบา การส่งออกรูปภาพมีประโยชน์เมื่อแต่ละสไลด์ต้องเป็นพรีวิว, รูปย่อ, หรือทรัพยากรแรสเตอร์แยก ใช้บทความ PNG, JPG, และ SVG เพื่อคำแนะนำการเรนเดอร์เฉพาะรูปแบบ

## **คำถามที่พบบ่อย**

**ฉันต้องการ Microsoft PowerPoint เพื่อแปลงการนำเสนอหรือไม่?**

ไม่ Aspose.Slides for Android via Java เป็นไลบรารีอิสระและไม่ต้องการ Microsoft PowerPoint หรือการทำงานอัตโนมัติของ Office

**ฉันสามารถแปลงการนำเสนอหลายไฟล์พร้อมกันได้หรือไม่?**

ได้ โหลดการนำเสนอแต่ละไฟล์, บันทึกเป็นรูปแบบที่ต้องการ, และทำลายอ็อบเจ็กต์การนำเสนอหลังจากประมวลผล สำหรับการประมวลผลแบบขนาน ใช้ตัวอย่างการนำเสนอแยกกันและปฏิบัติตามแนวทาง [การทำงานหลายเธรด](/slides/th/androidjava/multithreading/)

**ฉันสามารถส่งออกเฉพาะสไลด์ที่เลือกได้หรือไม่?**

ได้ มีหลายวิธีการส่งออกที่อนุญาตให้ส่งค่าดัชนีสไลด์หรือเรนเดอร์สไลด์แต่ละหน้า ตามรูปแบบผลลัพธ์ ดูบทความเฉพาะสำหรับรูปแบบเป้าหมาย

**ฉันสามารถรวมสไลด์ที่ซ่อนเมื่อส่งออกเป็น PDF หรือ XPS ได้หรือไม่?**

ได้ ใช้การตั้งค่าการส่งออกสไลด์ที่ซ่อนที่อธิบายในบทความการแปลง [PDF](/slides/th/androidjava/convert-powerpoint-to-pdf/) และ [XPS](/slides/th/androidjava/convert-powerpoint-to-xps/)

**ฉันสามารถสร้างผลลัพธ์ PDF/A ได้หรือไม่?**

ได้ มีการตั้งค่าการปฏิบัติตามมาตรฐาน PDF สำหรับการส่งออก PDF ดูที่ [แปลง PowerPoint เป็น PDF](/slides/th/androidjava/convert-powerpoint-to-pdf/) สำหรับรายละเอียด

**ฟอนต์จัดการอย่างไรระหว่างการแปลง?**

Aspose.Slides สามารถใช้ฟอนต์ที่ฝังไว้, ฟอนต์สำรอง, และการตั้งค่าการแทนที่ฟอนต์ ดูที่ [ฟอนต์ที่ฝัง](/slides/th/androidjava/embedded-font/), [ฟอนต์สำรอง](/slides/th/androidjava/fallback-font/), และ [การแทนที่ฟอนต์](/slides/th/androidjava/font-substitution/).
