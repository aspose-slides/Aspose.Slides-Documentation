## **ภาพรวม**

Aspose.Slides อนุญาตให้คุณแทนที่แบบอักษรหนึ่งด้วยอีกแบบหนึ่งทั่วทั้งงานนำเสนอ เมื่อแบบอักษรถูกแทนที่ ตัวอย่างทั้งหมดของแบบอักษรเดิมจะถูกเปลี่ยนเป็นแบบอักษรใหม่

เพื่อดำเนินการแทนที่แบบอักษร ให้โหลดงานนำเสนอ กำหนดแบบอักษรต้นทางและแบบอักษรที่ต้องการแทนที่ เรียกเมธอดการแทนที่แบบอักษร และบันทึกงานนำเสนอที่แก้ไขเป็นไฟล์ PPTX วิธีนี้มีประโยชน์เมื่อคุณต้องการสลับแบบอักษรจากตระกูลหนึ่งไปยังอีกตระกูลหนึ่งทั่วทั้งงานนำเสนอโดยเจตนา

## **แทนที่แบบอักษร**

หากคุณเปลี่ยนใจเกี่ยวกับการใช้แบบอักษร คุณสามารถแทนที่แบบอักษรนั้นด้วยแบบอักษรอื่นได้ ตัวอย่างทั้งหมดของแบบอักษรเก่าจะถูกแทนที่ด้วยแบบอักษรใหม่

Aspose.Slides อนุญาตให้คุณแทนที่แบบอักษรโดยวิธีนี้:

1. โหลดงานนำเสนอที่เกี่ยวข้อง
2. โหลดแบบอักษรที่ต้องการจะแทนที่
3. โหลดแบบอักษรใหม่
4. แทนที่แบบอักษร
5. บันทึกงานนำเสนอที่แก้ไขเป็นไฟล์ PPTX

```java
// โหลดงานนำเสนอ
Presentation pres = new Presentation("Fonts.pptx");
try {
    // โหลดแบบอักษรต้นทางที่จะแทนที่
    IFontData sourceFont = new FontData("Arial");
    
    // โหลดแบบอักษรใหม่
    IFontData destFont = new FontData("Times New Roman");
    
    // แทนที่แบบอักษร
    pres.getFontsManager().replaceFont(sourceFont, destFont);
    
    // บันทึกงานนำเสนอ
    pres.save("UpdatedFont_out.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```

## **คำถามที่พบบ่อย**

**ความแตกต่างระหว่าง "การแทนที่แบบอักษร", "การทดแทนแบบอักษร", และ "แบบอักษรสำรอง" คืออะไร?**

การแทนที่เป็นการสลับโดยเจตนาจากตระกูลหนึ่งไปยังอีกตระกูลหนึ่งทั่วทั้งเอกสาร [การทดแทน](/slides/th/androidjava/font-substitution/) คือกฎเช่น “หากแบบอักษรไม่พร้อมใช้งาน ให้ใช้ X” [แบบอักษรสำรอง](/slides/th/androidjava/fallback-font/) จะถูกนำไปใช้เฉพาะกราฟฟิกที่หายไปเมื่อแบบอักษรพื้นฐานติดตั้งแล้วแต่ไม่มีตัวอักษรที่ต้องการ

**การแทนที่ใช้กับสไลด์แม่, เค้าโครง, โน้ต, และคอมเมนต์หรือไม่?**

ใช่ การแทนที่ส่งผลต่อออบเจ็กต์ทั้งหมดในงานนำเสนอที่ใช้แบบอักษรเดิม รวมถึงสไลด์แม่และโน้ต; คอมเมนต์ก็เป็นส่วนหนึ่งของเอกสารและจะถูกนำมาพิจารณาโดยเอนจินแบบอักษร

**แบบอักษรจะเปลี่ยนภายในออบเจ็กต์ OLE ที่ฝังอยู่ (เช่น Excel) หรือไม่?**

ไม่ [เนื้อหา OLE](/slides/th/androidjava/manage-ole/) ถูกควบคุมโดยแอปพลิเคชันของตนเอง การแทนที่ในงานนำเสนอไม่ได้ทำการฟอร์แมตใหม่ให้ข้อมูล OLE ภายใน; มันอาจแสดงเป็นภาพหรือเป็นเนื้อหาที่แก้ไขได้จากภายนอก

**ฉันสามารถแทนที่แบบอักษรได้เฉพาะส่วนของงานนำเสนอ (ตามสไลด์หรือภูมิภาค) หรือไม่?**

การแทนที่แบบมีเป้าหมายทำได้ หากคุณเปลี่ยนแบบอักษรในระดับออบเจ็กต์/ช่วงที่ต้องการแทนที่จะใช้การแทนที่ทั่วทั้งเอกสารตรรกะการเลือกแบบอักษรโดยรวมระหว่างการแสดงผลยังคงเหมือนเดิม

**ฉันจะทราบล่วงหน้าว่างานนำเสนอใช้แบบอักษรอะไรบ้าง?**

ใช้ [ตัวจัดการแบบอักษร](https://reference.aspose.com/slides/th/androidjava/com.aspose.slides/fontsmanager/) ของงานนำเสนอ: มันให้รายการของ [ตระกูลที่ใช้งาน](https://reference.aspose.com/slides/th/androidjava/com.aspose.slides/fontsmanager/#getFonts--) และข้อมูลเกี่ยวกับ [การทดแทน/"ไม่ทราบ" แบบอักษร](https://reference.aspose.com/slides/th/androidjava/com.aspose.slides/fontsmanager/#getSubstitutions--) ซึ่งช่วยวางแผนการแทนที่

**การแทนที่แบบอักษรทำงานเมื่อแปลงเป็น PDF/รูปภาพหรือไม่?**

ใช่ ระหว่างการส่งออก Aspose.Slides ใช้ [ลำดับการเลือก/การทดแทนแบบอักษร](/slides/th/androidjava/font-selection-sequence/) เหมือนกัน ดังนั้นการแทนที่ที่ทำล่วงหน้าจะได้รับการยอมรับในระหว่างการแปลง

**ฉันต้องติดตั้งแบบอักษรเป้าหมายในระบบหรือไม่ หรือสามารถแนบโฟลเดอร์แบบอักษรได้หรือไม่?**

ไม่จำเป็นต้องติดตั้ง: ไลบรารีอนุญาตให้ [โหลดแบบอักษรภายนอก](/slides/th/androidjava/custom-font/) จากโฟลเดอร์ของผู้ใช้เพื่อใช้ระหว่าง [การเรนเดอร์และการส่งออก](/slides/th/androidjava/convert-powerpoint/)

**การแทนที่จะทำให้ "tofu" (สี่เหลี่ยม) แทนที่อักขระได้หรือไม่?**

ได้เฉพาะเมื่อแบบอักษรเป้าหมายมี glyph ที่ต้องการจริง ๆ หากไม่มี ให้ [กำหนดค่าแบบอักษรสำรอง](/slides/th/androidjava/fallback-font/) เพื่อครอบคลุมอักขระที่หายไป
