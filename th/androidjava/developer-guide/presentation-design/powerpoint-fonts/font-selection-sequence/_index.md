## **Overview**

เมื่อโหลด การแสดงผล หรือแปลงงานนำเสนอเป็นรูปแบบอื่น Aspose.Slides จะตรวจสอบว่าแบบอักษรที่ใช้ในงานนำเสนอมีอยู่ในระบบปฏิบัติการหรือไม่ หากแบบอักษรที่จำเป็นหายไป Aspose.Slides จะเลือกแบบอักษรทดแทนที่ใกล้เคียงที่สุดกับแบบอักษรที่ PowerPoint จะใช้  

Aspose.Slides จะค้นหาแบบอักษรที่เลือกในระบบปฏิบัติการก่อน หากพบก็จะใช้ หากไม่พบจะใช้แบบอักษรทดแทนที่เหมาะสม เมื่อมีการกำหนดกฎการแทนที่แบบอักษรผ่าน `FontSubstRule` ก็จะนำกฎเหล่านั้นมาพิจารณาด้วย  

คุณยังสามารถเพิ่มแบบอักษรในช่วงเวลาที่แอปพลิเคชันทำงาน ใช้แบบอักษรฝังจากงานนำเสนอ หรือโหลดแบบอักษรภายนอกสำหรับเอกสารผลลัพธ์เช่นไฟล์ PDF ได้อีกด้วย  

## **Font Selection**

มีข้อกำหนดบางประการที่เกี่ยวกับแบบอักษรในงานนำเสนอเมื่อทำการโหลด การแสดงผล หรือแปลงเป็นรูปแบบอื่น ตัวอย่างเช่น เมื่อคุณพยายามแปลงงานนำเสนอ (สไลด์) เป็นภาพ ระบบจะตรวจสอบแบบอักษรของงานนำเสนอเพื่อยืนยันว่ามีแบบอักษรที่เลือกอยู่ในระบบปฏิบัติการหรือไม่ หากแบบอักษรถูกยืนยันว่าหายไป จะถูกแทนที่ — ดูที่ [**การแทนที่แบบอักษร**](https://docs.aspose.com/slides/th/androidjava/font-replacement/) และ [**การสับเปลี่ยนแบบอักษร**](https://docs.aspose.com/slides/th/androidjava/font-substitution/)  

นี่คือขั้นตอนที่ Aspose.Slides ปฏิบัติตามเมื่อต้องจัดการกับแบบอักษร:

1. Aspose.Slides ค้นหาแบบอักษรในระบบปฏิบัติการเพื่อหาแบบอักษรที่ตรงกับแบบอักษรที่เลือกในงานนำเสนอ  
2. หากพบแบบอักษรที่เลือก Aspose.Slides จะใช้มัน มิฉะนั้น Aspose.Slides จะใช้แบบอักษรทดแทนที่ใกล้เคียงที่สุดกับสิ่งที่ PowerPoint จะใช้  
3. หากมีการตั้งกฎการแทนที่แบบอักษรผ่าน [FontSubstRule](https://reference.aspose.com/slides/th/androidjava/com.aspose.slides/fontsubstrule/) จะนำกฎเหล่านั้นไปใช้  

Aspose.Slides อนุญาตให้คุณเพิ่มแบบอักษรในช่วงเวลาที่แอปพลิเคชันทำงานและแล้วจึงใช้แบบอักษรเหล่านั้น ดูที่ [**แบบอักษรกำหนดเอง**](https://docs.aspose.com/slides/th/androidjava/custom-font/)  

เมื่อแบบอักษรเพิ่มเติมถูกเก็บไว้ภายในงานนำเสนอ พวกมันจะถูกเรียกว่า [**แบบอักษรฝัง**](https://docs.aspose.com/slides/th/androidjava/embedded-font/)  

Aspose.Slides อนุญาตให้คุณเพิ่มแบบอักษรที่ใช้กับเอกสารผลลัพธ์ *เท่านั้น* ตัวอย่างเช่น หากงานนำเสนอที่คุณต้องการแปลงเป็น PDF มีแบบอักษรที่หายจากระบบของคุณและแบบอักษรฝังอยู่ คุณสามารถเพิ่มหรือโหลดแบบอักษรที่จำเป็นเป็น **แบบอักษรภายนอก**  

{{% alert title="Note" color="primary" %}} 
เราไม่ได้จัดจำหน่ายแบบอักษรใด ๆ ทั้งแบบที่ต้องชำระค่าใช้จ่ายหรือแบบฟรี API ของเราช่วยให้คุณสามารถโหลดแบบอักษรภายนอกและฝังลงในเอกสารได้ แต่คุณต้องทำเช่นนั้นด้วยแบบอักษรตามดุลยพินิจและความรับผิดชอบของคุณเอง. 
{{% /alert %}}

## **FAQ**

**ฉันจะกำหนดว่าแบบอักษรใดบ้างที่ถูกใช้จริงในงานนำเสนอก่อนการแปลงได้อย่างไร?**  

Aspose.Slides ให้คุณตรวจสอบแบบอักษรที่ใช้ผ่าน [font manager](https://reference.aspose.com/slides/th/androidjava/com.aspose.slides/fontsmanager/) เพื่อให้คุณตัดสินใจว่าจะ [ฝัง](/slides/th/androidjava/embedded-font/), [แทนที่](/slides/th/androidjava/font-replacement/) หรือเพิ่ม [แหล่งที่มาภายนอก](/slides/th/androidjava/custom-font/) ซึ่งช่วยป้องกันการแทนที่ที่ไม่ต้องการระหว่างการแสดงผลและการส่งออก  

**ฉันสามารถเพิ่มไดเร็กทอรีแบบอักษรเพิ่มเติมโดยไม่ต้องติดตั้งบนระบบปฏิบัติการได้หรือไม่?**  

ได้ คุณสามารถลงทะเบียน [แหล่งแบบอักษรภายนอก](/slides/th/androidjava/custom-font/) เช่น โฟลเดอร์หรือสตรีมในหน่วยความจำสำหรับการแสดงผลและการส่งออก ซึ่งจะลดการพึ่งพาแบบอักษรของระบบโฮสต์และทำให้การจัดวางคาดการณ์ได้  

**ฉันจะป้องกันการเปลี่ยนไปใช้แบบอักษรที่ไม่เหมาะสมโดยอัตโนมัติเมื่อตัวอักษรขาดหายได้อย่างไร?**  

กำหนด [การแทนที่แบบอักษร](/slides/th/androidjava/font-replacement/) และ [กฎการสำรองแบบอักษร](/slides/th/androidjava/fallback-font/) อย่างชัดเจนล่วงหน้า โดยการวิเคราะห์แบบอักษรที่ใช้และตั้งค่าความสำคัญที่ควบคุมสำหรับแบบอักษรทดแทน คุณจะได้ความสอดคล้องของการพิมพ์และหลีกเลี่ยงผลลัพธ์ที่ไม่คาดคิด.
