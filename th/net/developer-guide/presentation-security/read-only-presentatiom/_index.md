## **บทนำ**

ใน PowerPoint 2019, Microsoft ได้นำเสนอการตั้งค่า **Always Open Read-Only** เป็นหนึ่งในตัวเลือกที่ผู้ใช้สามารถใช้เพื่อปกป้องงานนำเสนอของตน คุณอาจต้องการใช้การตั้งค่า Read-Only นี้เพื่อปกป้องงานนำเสนอเมื่อ

- คุณต้องการป้องกันการแก้ไขโดยบังเอิญและรักษาเนื้อหาของงานนำเสนอให้ปลอดภัย  
- คุณต้องการแจ้งให้ผู้คนทราบว่างานนำเสนอที่คุณให้เป็นเวอร์ชันสุดท้าย  

หลังจากที่คุณเลือกตัวเลือก **Always Open Read-Only** สำหรับงานนำเสนอ เมื่อผู้ใช้เปิดงานนำเสนอ พวกเขาจะเห็นคำแนะนำ **Read-Only** และอาจเห็นข้อความในรูปแบบต่อไปนี้: *เพื่อป้องกันการเปลี่ยนแปลงโดยบังเอิญ ผู้เขียนได้ตั้งค่าไฟล์นี้ให้เปิดแบบอ่านอย่างเดียว*  

คำแนะนำ Read-Only เป็นมาตรการเตือนที่เรียบง่ายแต่มีประสิทธิภาพ ซึ่งทำให้ผู้ใช้ไม่แก้ไขได้เนื่องจากต้องดำเนินการลบออกก่อนจึงจะสามารถแก้ไขงานนำเสนอได้ หากคุณไม่ต้องการให้ผู้ใช้ทำการเปลี่ยนแปลงงานนำเสนอและต้องการบอกพวกเขาในทางสุภาพ คำแนะนำ Read-Only อาจเป็นตัวเลือกที่ดีสำหรับคุณ  

> หากงานนำเสนอที่มีการป้องกัน **Read-Only** ถูกเปิดใน Microsoft PowerPoint เวอร์ชันเก่า—ซึ่งไม่รองรับฟังก์ชันที่เพิ่งแนะนำ—คำแนะนำ **Read-Only** จะถูกละเลย (งานนำเสนอนั้นจะเปิดตามปกติ)

## **ใช้โหมด Read-Only**

Aspose.Slides for .NET ให้คุณตั้งค่างานนำเสนอเป็น **Read-Only** ซึ่งหมายความว่าผู้ใช้ (หลังจากเปิดงานนำเสนอ) จะเห็นคำแนะนำ **Read-Only** ตัวอย่างโค้ดนี้แสดงวิธีตั้งค่างานนำเสนอเป็น **Read-Only** ใน C# ด้วย Aspose.Slides:

```c#
using (Presentation pres = new Presentation())
{
    pres.ProtectionManager.ReadOnlyRecommended = true;
    pres.Save("ReadOnlyPresentation.pptx", SaveFormat.Pptx);
}
```

{{% alert color="primary" %}} 
**หมายเหตุ**: คำแนะนำ **Read-Only** มีจุดมุ่งหมายเพื่อไม่ให้แก้ไขหรือป้องกันการเปลี่ยนแปลงโดยบังเอิญในงานนำเสนอ PowerPoint หากบุคคลที่มีความรู้และตั้งใจแก้ไขงานของคุณ พวกเขาสามารถลบการตั้งค่า Read-Only ได้ง่าย หากคุณต้องการป้องกันการแก้ไขโดยไม่ได้รับอนุญาตอย่างจริงจัง คุณควรใช้ [การป้องกันที่เข้มงวดขึ้นซึ่งรวมถึงการเข้ารหัสและรหัสผ่าน](https://docs.aspose.com/slides/th/net/password-protected-presentation/). 
{{% /alert %}} 

## **คำถามที่พบบ่อย**

**'Read-Only recommended' แตกต่างจากการป้องกันด้วยรหัสผ่านเต็มอย่างไร?**  
'Read-Only recommended' เพียงแสดงคำแนะนำให้เปิดไฟล์ในโหมดอ่านอย่างเดียวและสามารถข้ามได้ง่าย [การป้องกันด้วยรหัสผ่าน](/slides/th/net/password-protected-presentation/) จริงๆ แล้วจำกัดการเปิดหรือแก้ไขและเหมาะสมเมื่อคุณต้องการการควบคุมความปลอดภัยที่แท้จริง.  

**'Read-Only recommended' สามารถรวมกับลายน้ำเพื่อยับยั้งการแก้ไขได้หรือไม่?**  
ใช่ คำแนะนำสามารถจับคู่กับ [ลายน้ำ](/slides/th/net/watermark/) เป็นการเตือนด้วยภาพ; พวกมันเป็นกลไกแยกกันและทำงานร่วมกันได้ดี.  

**แมโครหรือเครื่องมือภายนอกยังสามารถแก้ไขไฟล์ได้หรือไม่เมื่อเปิดใช้งานคำแนะนำ?**  
ใช่ คำแนะนำไม่บล็อกการเปลี่ยนแปลงโดยโปรแกรม หากต้องการป้องกันการแก้ไขอัตโนมัติ ให้ใช้ [รหัสผ่านและการเข้ารหัส](/slides/th/net/password-protected-presentation/).  

**'Read-Only recommended' มีความสัมพันธ์กับแฟล็ก 'IsEncrypted' และ 'IsWriteProtected' อย่างไร?**  
พวกมันเป็นสัญญาณที่แตกต่างกัน 'Read-Only recommended' เป็นคำเตือนที่นุ่มนวลและเป็นทางเลือก; [IsWriteProtected](https://reference.aspose.com/slides/th/net/aspose.slides/protectionmanager/iswriteprotected/) และ [IsEncrypted](https://reference.aspose.com/slides/th/net/aspose.slides/protectionmanager/isencrypted/) แสดงข้อจำกัดการเขียนหรือการอ่านที่แท้จริงซึ่งขึ้นอยู่กับรหัสผ่านหรือการเข้ารหัส.
