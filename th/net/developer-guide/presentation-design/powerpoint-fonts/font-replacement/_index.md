## **ภาพรวม**

Aspose.Slides อนุญาตให้คุณแทนที่ฟอนต์หนึ่งด้วยฟอนต์อื่นทั่วทั้งงานนำเสนอ เมื่อฟอนต์ถูกแทนที่ ตัวอย่างทั้งหมดของฟอนต์ต้นฉบับจะถูกเปลี่ยนเป็นฟอนต์ใหม่

เพื่อทำการแทนที่ฟอนต์ ให้โหลดงานนำเสนอ กำหนดฟอนต์ต้นฉบับและฟอนต์ที่จะแทนที่ เรียกเมธอดการแทนที่ฟอนต์ แล้วบันทึกงานนำเสนอที่แก้ไขเป็นไฟล์ PPTX วิธีนี้มีประโยชน์เมื่อคุณต้องการสลับจากครอบครัวฟอนต์หนึ่งไปยังอีกครอบครัวหนึ่งทั่วทั้งงานนำเสนออย่างตั้งใจ

## **แทนที่ฟอนต์**

หากคุณเปลี่ยนใจเกี่ยวกับการใช้ฟอนต์ คุณสามารถแทนที่ฟอนต์นั้นด้วยฟอนต์อื่นได้ ตัวอย่างทั้งหมดของฟอนต์เก่าจะถูกแทนที่ด้วยฟอนต์ใหม่

Aspose.Slides อนุญาตให้คุณแทนที่ฟอนต์ด้วยวิธีนี้:

1. โหลดงานนำเสนอที่เกี่ยวข้อง  
2. โหลดฟอนต์ที่ต้องการแทนที่  
3. โหลดฟอนต์ใหม่  
4. แทนที่ฟอนต์  
5. บันทึกงานนำเสนอที่แก้ไขเป็นไฟล์ PPTX  

โค้ด C# นี้แสดงการแทนที่ฟอนต์:

```c#
// โหลดงานนำเสนอ
Presentation presentation = new Presentation("Fonts.pptx");

// โหลดฟอนต์ต้นทางที่จะถูกแทนที่
IFontData sourceFont = new FontData("Arial");

// โหลดฟอนต์ใหม่
IFontData destFont = new FontData("Times New Roman");

// แทนที่ฟอนต์
presentation.FontsManager.ReplaceFont(sourceFont, destFont);

// บันทึกงานนำเสนอ
presentation.Save("UpdatedFont_out.pptx", SaveFormat.Pptx);
```

เพื่อกำหนดกฎที่บ่งบอกว่าจะเกิดอะไรขึ้นในเงื่อนไขบางอย่าง (เช่น หากไม่สามารถเข้าถึงฟอนต์ได้) ดูที่ [**การทดแทนฟอนต์**](/slides/th/net/font-substitution/).

## **คำถามที่พบบ่อย**

**ความแตกต่างระหว่าง "การแทนที่ฟอนต์", "การทดแทนฟอนต์", และ "ฟอนต์สำรอง" คืออะไร?**

การแทนที่เป็นการสลับโดยเจตนาจากครอบครัวฟอนต์หนึ่งไปยังอีกครอบครัวหนึ่งทั่วทั้งเอกสาร. [การทดแทน](/slides/th/net/font-substitution/) คือกฎเช่น “ถ้าฟอนต์ไม่พร้อมใช้งาน ให้ใช้ X.” [ฟอนต์สำรอง](/slides/th/net/fallback-font/) จะถูกนำมาใช้แบบเจาะจงสำหรับ glyph ที่หายไปแต่ฟอนต์หลักติดตั้งอยู่แต่ไม่มีอักขระที่ต้องการ.

**การแทนที่ทำงานกับมาสเตอร์สไลด์, เลย์เอาต์, โน้ต, และคอมเมนต์หรือไม่?**

ใช่. การแทนที่มีผลต่อวัตถุทั้งหมดในงานนำเสนอที่ใช้ฟอนต์ต้นฉบับ รวมถึงมาสเตอร์สไลด์และโน้ต; คอมเมนต์ก็เป็นส่วนหนึ่งของเอกสารและจะถูกพิจารณาโดยเอนจินฟอนต์.

**ฟอนต์จะเปลี่ยนภายในวัตถุ OLE ที่ฝังอยู่ (เช่น Excel) หรือไม่?**

ไม่. [เนื้อหา OLE](/slides/th/net/manage-ole/) ถูกควบคุมโดยแอปพลิเคชันของมันเอง การแทนที่ในงานนำเสนอไม่ได้ทำการจัดรูปแบบข้อมูล OLE ภายในใหม่; มันอาจจะแสดงเป็นรูปภาพหรือเป็นเนื้อหาที่สามารถแก้ไขจากภายนอกได้.

**ฉันสามารถแทนที่ฟอนต์เฉพาะบางส่วนของงานนำเสนอ (ตามสไลด์หรือบริเวณ) ได้หรือไม่?**

การแทนที่แบบเจาะจงเป็นไปได้หากคุณเปลี่ยนฟอนต์ในระดับของวัตถุ/ช่วงที่ต้องการแทนที่จะใช้การแทนที่ทั่วถึงทั้งเอกสารตรรกะการเลือกฟอนต์โดยรวมระหว่างการเรนเดอร์ยังคงเหมือนเดิม.

**ฉันจะทราบล่วงหน้าว่างานนำเสนอใช้ฟอนต์ใดบ้างทั้งหมดได้อย่างไร?**

ใช้ [ผู้จัดการฟอนต์]https://reference.aspose.com/slides/th/net/aspose.slides/fontsmanager/: มันให้รายการของ [ครอบครัวฟอนต์ที่ใช้อยู่]https://reference.aspose.com/slides/th/net/aspose.slides/fontsmanager/getfonts/ และข้อมูลเกี่ยวกับ [การทดแทน/ฟอนต์ที่ไม่รู้จัก]https://reference.aspose.com/slides/th/net/aspose.slides/fontsmanager/getsubstitutions/, ซึ่งช่วยวางแผนการแทนที่.

**การแทนที่ฟอนต์ทำงานเมื่อแปลงเป็น PDF/รูปภาพหรือไม่?**

ใช่. ระหว่างการส่งออก Aspose.Slides จะใช้ [ลำดับการเลือก/การทดแทนฟอนต์](/slides/th/net/font-selection-sequence/) ดังนั้นการแทนที่ที่ทำล่วงหน้าจะได้รับการเคารพในระหว่างการแปลง.

**ฉันจำเป็นต้องติดตั้งฟอนต์เป้าหมายในระบบหรือสามารถแนบโฟลเดอร์ฟอนต์ได้หรือไม่?**

ไม่จำเป็นต้องติดตั้ง: ไลบรารีอนุญาตให้ [โหลดฟอนต์ภายนอก](/slides/th/net/custom-font/) จากโฟลเดอร์ผู้ใช้เพื่อใช้ในระหว่าง [การเรนเดอร์และส่งออก](/slides/th/net/convert-powerpoint/).

**การแทนที่จะทำให้ “tofu” (สี่เหลี่ยม) แทนที่อักขระได้หรือไม่?**

เฉพาะเมื่อฟอนต์เป้าหมายมี glyph ที่ต้องการจริง ๆ หากไม่มี ให้ [กำหนดค่าฟอนต์สำรอง](/slides/th/net/fallback-font/) เพื่อครอบคลุมอักขระที่หายไป.
