## **ภาพรวม**

Aspose.Slides ช่วยให้คุณระบุฟอนต์เริ่มต้นที่ใช้เมื่อการนำเสนอถูกเรนเดอร์ ซึ่งเป็นประโยชน์เมื่อสร้างภาพตัวอย่างสไลด์หรือส่งออกการนำเสนอเป็นรูปแบบเช่น PDF และ XPS ฟอนต์เริ่มต้นจะถูกกำหนดผ่าน `LoadOptions` ก่อนโหลดการนำเสนอ

คุณสมบัติ `DefaultRegularFont` กำหนดฟอนต์เริ่มต้นสำหรับข้อความปกติ ส่วน `DefaultAsianFont` กำหนดฟอนต์เริ่มต้นสำหรับข้อความเอเชีย หลังจากตั้งค่าตัวเลือกเหล่านี้แล้ว การนำเสนอสามารถโหลดและเรนเดอร์โดยใช้ฟอนต์ที่ระบุ

## **ใช้ฟอนต์เริ่มต้นสำหรับการเรนเดอร์การนำเสนอ**
Aspose.Slides ให้คุณตั้งค่าฟอนต์เริ่มต้นสำหรับการเรนเดอร์การนำเสนอเป็น PDF, XPS หรือภาพตัวอย่าง บทความนี้แสดงวิธีกำหนด DefaultRegularFont และ DefaultAsianFont เพื่อใช้เป็นฟอนต์เริ่มต้น โปรดทำตามขั้นตอนด้านล่างเพื่อโหลดฟอนต์จากไดเรกทอรีภายนอกโดยใช้ Aspose.Slides for .NET API:

1. สร้างอินสแตนซ์ของ LoadOptions
1. ตั้งค่า DefaultRegularFont เป็นฟอนต์ที่คุณต้องการ ในตัวอย่างต่อไปนี้เราใช้ Wingdings
1. ตั้งค่า DefaultAsianFont เป็นฟอนต์ที่คุณต้องการ เราใช้ Wingdings ในตัวอย่างต่อไปนี้
1. โหลดการนำเสนอโดยใช้ Presentation พร้อมกำหนดตัวเลือกการโหลด
1. จากนั้นสร้างภาพตัวอย่างสไลด์, PDF และ XPS เพื่อตรวจสอบผลลัพธ์

การดำเนินการของข้างต้นมีดังนี้

```c#
// ใช้ตัวเลือกการโหลดเพื่อระบุฟอนต์ปกติและฟอนต์เอเชียเริ่มต้น
LoadOptions loadOptions = new LoadOptions(LoadFormat.Auto);
loadOptions.DefaultRegularFont = "Wingdings";
loadOptions.DefaultAsianFont = "Wingdings";

using (Presentation pptx = new Presentation("DefaultFonts.pptx", loadOptions))
{
    using (IImage image = pptx.Slides[0].GetImage(1, 1))
    {
        image.Save("DefaultFonts_out.png", ImageFormat.Png);
    }

    pptx.Save("DefaultFonts_out.pdf", SaveFormat.Pdf);
    pptx.Save("DefaultFonts_out.xps", SaveFormat.Xps);
}
```

## **คำถามที่พบบ่อย**

**DefaultRegularFont และ DefaultAsianFont มีผลต่ออะไรบ้าง—เพียงการส่งออก หรือรวมถึงภาพตัวอย่างสไลด์, PDF, XPS, HTML, และ SVG ด้วย?**

พวกเขามีส่วนร่วมในกระบวนการเรนเดอร์สำหรับผลลัพธ์ที่รองรับทั้งหมด รวมถึงภาพตัวอย่างสไลด์, [PDF](/slides/th/net/convert-powerpoint-to-pdf/), [XPS](/slides/th/net/convert-powerpoint-to-xps/), [raster images](/slides/th/net/convert-powerpoint-to-png/), [HTML](/slides/th/net/convert-powerpoint-to-html/), และ [SVG](/slides/th/net/render-a-slide-as-an-svg-image/) เนื่องจาก Aspose.Slides ใช้ตรรกะการจัดเลย์เอาต์และการแก้ไข glyph เดียวกันสำหรับเป้าหมายเหล่านี้

**ฟอนต์เริ่มต้นจะถูกนำไปใช้เมื่อเพียงแค่อ่านและบันทึกไฟล์ PPTX โดยไม่มีการเรนเดอร์หรือไม่?**

ไม่มี ฟอนต์เริ่มต้นมีความสำคัญเมื่อข้อความต้องถูกวัดและวาด การเปิด‑บันทึกอย่างตรงไปตรงมาของการนำเสนอไม่ทำให้การกำหนดฟอนต์ที่เก็บไว้หรือโครงสร้างของไฟล์เปลี่ยนแปลง ฟอนต์เริ่มต้นจะมีผลในระหว่างการดำเนินการที่ทำการเรนเดอร์หรือจัดเรียงข้อความใหม่

**หากฉันเพิ่มโฟลเดอร์ฟอนต์ของตัวเองหรือให้ฟอนต์จากหน่วยความจำ จะถูกพิจารณาเมื่อเลือกฟอนต์เริ่มต้นหรือไม่?**

ใช่ [Custom font sources](/slides/th/net/custom-font/) จะขยายแคตาล็อกของฟอนต์ฟา มีลและ glyph ที่พร้อมใช้งานสำหรับเอนจิน ฟอนต์เริ่มต้นและ [fallback rules](/slides/th/net/fallback-font/) ใด ๆ จะตรวจสอบแหล่งข้อมูลเหล่านี้ก่อน ทำให้มีการครอบคลุมที่เชื่อถือได้มากขึ้นบนเซิร์ฟเวอร์และในคอนเทนเนอร์

**ฟอนต์เริ่มต้นจะมีผลต่อเมตริกของข้อความ (kerning, advances) และทำให้การตัดบรรทัดและการห่อหุ้มเปลี่ยนแปลงหรือไม่?**

ใช่ การเปลี่ยนฟอนต์จะทำให้เมตริกของ glyph เปลี่ยนแปลงและอาจทำให้การตัดบรรทัด, การห่อหุ้ม, และการแบ่งหน้าเปลี่ยนแปลงระหว่างการเรนเดอร์ เพื่อความเสถียรของการจัดวาง ให้ [embed the original fonts](/slides/th/net/embedded-font/) หรือเลือกฟอนต์เริ่มต้นและฟอนต์สำรองที่เข้ากันทางเมตริก

**มีประโยชน์ใดในการตั้งค่าฟอนต์เริ่มต้นหากฟอนต์ทั้งหมดที่ใช้ในการนำเสนอถูกฝังไว้หรือไม่?**

บ่อยครั้งไม่จำเป็น เพราะ [embedded fonts](/slides/th/net/embedded-font/) จะทำให้ลักษณะการแสดงผลคงที่อยู่แล้ว ฟอนต์เริ่มต้นยังคงเป็นประโยชน์เป็นเครือข่ายความปลอดภัยสำหรับอักขระที่ไม่ได้ครอบคลุมโดยส่วนย่อยที่ฝังไว้ หรือเมื่อไฟล์มีข้อความที่ผสมระหว่างที่ฝังและไม่ฝัง
