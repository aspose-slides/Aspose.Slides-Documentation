## **ภาพรวม**

เมื่อการนำเสนอถูกโหลด, เรนเดอร์, หรือแปลงเป็นรูปแบบอื่น, Aspose.Slides ตรวจสอบว่าแบบอักษรที่ใช้ในงานนำเสนอมีอยู่ในระบบปฏิบัติการหรือไม่ หากแบบอักษรที่ต้องการหายไป, Aspose.Slides จะเลือกแบบอักษรทดแทนที่ใกล้ที่สุดกับแบบอักษรที่ PowerPoint จะใช้

Aspose.Slides จะค้นหาแบบอักษรที่เลือกในระบบปฏิบัติการก่อน หากพบแบบอักษรนั้นจะใช้งาน หากไม่พบจะใช้แบบอักษรทดแทนที่เหมาะสม เมื่อมีกฎการทดแทนแบบอักษรที่กำหนดผ่าน `FontSubstRule` กฎเหล่านั้นก็จะถูกพิจารณาด้วย

คุณยังสามารถเพิ่มแบบอักษรในระหว่างการทำงานของแอปพลิเคชัน, ใช้แบบอักษรที่ฝังอยู่ในงานนำเสนอ, หรือโหลดแบบอักษรภายนอกสำหรับเอกสารผลลัพธ์เช่นไฟล์ PDF

## **การเลือกแบบอักษร**

กฎบางอย่างจะนำไปใช้กับแบบอักษรในงานนำเสนอเมื่อทำการโหลด, เรนเดอร์, หรือแปลงเป็นรูปแบบอื่น ตัวอย่างเช่น เมื่อคุณพยายามแปลงงานนำเสนอ (สไลด์) เป็นรูปภาพ, ระบบจะตรวจสอบแบบอักษรของงานนำเสนอเพื่อยืนยันว่าแบบอักษรที่เลือกมีอยู่ในระบบปฏิบัติการหรือไม่ หากพบว่าแบบอักษรหายไป จะทำการแทนที่ — ดู [**Font Replacement**](https://docs.aspose.com/slides/th/net/font-replacement/) และ [**Font Substitution**](https://docs.aspose.com/slides/th/net/font-substitution/)

นี่คือขั้นตอนที่ Aspose.Slides ปฏิบัติตามเมื่อจัดการกับแบบอักษร:

1. Aspose.Slides ค้นหาแบบอักษรในระบบปฏิบัติการเพื่อค้นหาแบบอักษรที่ตรงกับแบบอักษรที่เลือกในงานนำเสนอ. 
2. หากพบแบบอักษรที่เลือก, Aspose.Slides จะใช้แบบอักษรนั้น มิฉะนั้น Aspose.Slides จะใช้แบบอักษรทดแทนที่ใกล้ที่สุดกับสิ่งที่ PowerPoint จะใช้.
3. หากมีกฎการแทนที่แบบอักษรที่ตั้งค่าผ่าน [FontSubstRule](https://reference.aspose.com/slides/th/net/aspose.slides/fontsubstrule/), จะนำไปใช้. 

Aspose.Slides อนุญาตให้คุณเพิ่มแบบอักษรในระหว่างการทำงานของแอปพลิเคชันและจากนั้นใช้แบบอักษรเหล่านั้น. ดู [**Custom fonts**](https://docs.aspose.com/slides/th/net/custom-font/). 

เมื่อมีการวางแบบอักษรเพิ่มเติมภายในงานนำเสนอ, แบบอักษรเหล่านั้นจะเรียกว่า [**Embedded fonts**](https://docs.aspose.com/slides/th/net/embedded-font/).

Aspose.Slides อนุญาตให้คุณเพิ่มแบบอักษรที่นำไปใช้กับเอกสารผลลัพธ์ *เท่านั้น* ตัวอย่างเช่น, หากงานนำเสนอที่คุณต้องการแปลงเป็น PDF มีแบบอักษรที่หายจากระบบของคุณและแบบอักษรที่ฝังอยู่, คุณสามารถเพิ่มหรือโหลดแบบอักษรที่จำเป็นเป็น **external fonts**. 

{{% alert title="Note" color="primary" %}} 
เราไม่ได้จัดจำหน่ายแบบอักษรใด ๆ ไม่ว่าจะเป็นแบบที่ต้องจ่ายหรือแบบฟรี API ของเรายอมให้คุณโหลดแบบอักษรภายนอกและฝังลงในเอกสาร, แต่คุณต้องทำเช่นนั้นด้วยแบบอักษรที่คุณเลือกและรับผิดชอบเอง.
{{% /alert %}}

## **คำถามที่พบบ่อย**

**ฉันจะตรวจสอบได้อย่างไรว่าแบบอักษรใดที่ใช้จริงในงานนำเสนอก่อนการแปลง?**

Aspose.Slides ให้คุณตรวจสอบแบบอักษรที่ใช้ผ่าน [font manager](https://reference.aspose.com/slides/th/net/aspose.slides/presentation/fontsmanager/), เพื่อให้คุณตัดสินใจว่าจะ [embed](/slides/th/net/embedded-font/), [replace](/slides/th/net/font-replacement/), หรือเพิ่ม [external sources](/slides/th/net/custom-font/) หรือไม่ สิ่งนี้ช่วยป้องกันการทดแทนที่ไม่ต้องการระหว่างการเรนเดอร์และการส่งออก.

**ฉันสามารถเพิ่มไดเรกทอรีแบบอักษรเพิ่มเติมโดยไม่ต้องติดตั้งบนระบบปฏิบัติการได้หรือไม่?**

ได้. คุณสามารถลงทะเบียน [external font sources](/slides/th/net/custom-font/) เช่น โฟลเดอร์หรือสตรีมในหน่วยความจำสำหรับการเรนเดอร์และการส่งออก สิ่งนี้จะตัดการพึ่งพาแบบอักษรของระบบโฮสต์และทำให้การจัดวางคาดเดาได้.

**ฉันจะป้องกันการ fallback ที่เงียบไปยังแบบอักษรที่ไม่เหมาะสมเมื่อ glyph หายได้อย่างไร?**

กำหนด [font replacement](/slides/th/net/font-replacement/) และกฎ [fallBack](/slides/th/net/fallback-font/) ของแบบอักษรอย่างชัดเจนล่วงหน้า โดยการวิเคราะห์แบบอักษรที่ใช้และตั้งค่าลำดับความสำคัญที่ควบคุมสำหรับการทดแทน คุณจะมั่นใจได้ว่าการพิมพ์แบบเป็นสม่ำเสมอและหลีกเลี่ยงผลลัพธ์ที่ไม่คาดคิด.
