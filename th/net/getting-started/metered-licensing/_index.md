## **บทนำ**

การให้สิทธิ์แบบมีการวัดเป็นกลไกการให้สิทธิ์ที่สามารถใช้ควบคู่กับวิธีการให้สิทธิ์ที่มีอยู่ได้ หากคุณต้องการให้เรียกเก็บค่าใช้จ่ายตามการใช้งานคุณลักษณะของ Aspose.Slides API คุณจะเลือกการให้สิทธิ์แบบมีการวัด

## **ใช้คีย์แบบมีการวัด**

เมื่อคุณซื้อใบอนุญาตแบบมีกระบวนการวัด คุณจะได้รับคีย์ (ไม่ใช่ไฟล์ใบอนุญาต) คีย์แบบมีกระบวนการวัดนี้สามารถใช้ได้โดยใช้คลาส [Metered](https://reference.aspose.com/slides/th/net/aspose.slides/metered/) ของ Aspose ที่จัดเตรียมสำหรับการดำเนินการมิดีระ หากต้องการรายละเอียดเพิ่มเติม ดูที่ [Metered Licensing FAQ](https://purchase.aspose.com/faqs/licensing/metered).

1. สร้างอินสแตนซ์ของคลาส [Metered](https://reference.aspose.com/slides/th/net/aspose.slides/metered/)
1. ส่งคีย์สาธารณะและคีย์ส่วนตัวของคุณไปยังเมธอด [SetMeteredKey](https://reference.aspose.com/slides/th/net/aspose.slides/metered/setmeteredkey/)
1. ทำการประมวลผลบางอย่าง (ดำเนินการงาน).
1. เรียกเมธอด [GetConsumptionQuantity](https://reference.aspose.com/slides/th/net/aspose.slides/metered/getconsumptionquantity/) ของคลาส `Metered`.

คุณควรเห็นจำนวน/ปริมาณของคำขอ API ที่คุณได้ใช้ไปจนถึงขณะนี้

โค้ดตัวอย่างนี้แสดงให้คุณเห็นวิธีใช้การให้สิทธิ์แบบมีกระบวนการวัด:

```cs
// สร้างอินสแตนซ์ของคลาส Metered
Aspose.Slides.Metered metered = new Aspose.Slides.Metered();

// ส่งคีย์สาธารณะและคีย์ส่วนตัวไปยังอ็อบเจ็กต์ Metered
metered.SetMeteredKey("<valid public key>", "<valid private key>");

// รับปริมาณข้อมูลที่วัดได้ก่อนเรียก API
decimal amountBefore = Aspose.Slides.Metered.GetConsumptionQuantity();
Console.WriteLine("Amount consumed before: " + amountBefore.ToString());

// ทำบางอย่างกับ Aspose.Slides API ที่นี่
// ...

// รับปริมาณข้อมูลที่วัดได้หลังจากเรียก API
decimal amountAfter = Aspose.Slides.Metered.GetConsumptionQuantity();
Console.WriteLine("Amount consumed after: " + amountAfter.ToString());
```

{{% alert color="warning" title="NOTE"  %}} 
เพื่อใช้การให้สิทธิ์แบบมีกระบวนการวัด คุณจำเป็นต้องมีการเชื่อมต่ออินเทอร์เน็ตที่เสถียร เนื่องจากกลไกการให้สิทธิ์ใช้อินเทอร์เน็ตเพื่อโต้ตอบกับบริการของเราและทำการคำนวณอย่างต่อเนื่อง.
{{% /alert %}} 

## **คำถามที่พบบ่อย**

**ฉันสามารถใช้ใบอนุญาตแบบมีกระบวนการวัดร่วมกับใบอนุญาตปกติ (ถาวรหรือชั่วคราว) ในแอปพลิเคชันเดียวกันได้หรือไม่?**

ใช่. การให้สิทธิ์แบบมีกระบวนการวัดเป็นกลไกการให้สิทธิ์เพิ่มเติมที่สามารถใช้ควบคู่กับ[วิธีการให้สิทธิ์](/slides/th/net/licensing/) ที่มีอยู่ คุณเลือกกลไกที่ต้องการใช้เมื่อแอปพลิเคชันเริ่มทำงาน.

**อะไรเป็นการนับการใช้ภายใต้ใบอนุญาตแบบมีกระบวนการวัด: การดำเนินการหรือไฟล์?**

การใช้งาน API จะถูกนับหมายถึงจำนวนคำขอหรือการดำเนินการ คุณสามารถรับการใช้งานปัจจุบันผ่าน [consumption-tracking methods](https://reference.aspose.com/slides/th/net/aspose.slides/metered/).

**การให้สิทธิ์แบบมีกระบวนการวัดเหมาะกับสถาปัตยกรรมไมโครเซอร์วิสและสภาพแวดล้อมแบบ serverless ที่อินสแตนซ์รีสตาร์ทบ่อยหรือไม่?**

ใช่. เนื่องจากการบันทึกบัญชีทำที่ระดับการเรียก API ดังนั้นสถานการณ์ที่มีการเริ่มต้นใหม่บ่อย ๆ (cold start) จึงเข้ากันได้ หากมีการเชื่อมต่อเครือข่ายที่เสถียรสำหรับการคำนวณแบบมีกระบวนการวัด.

**ฟังก์ชันการทำงานของไลบรารีแตกต่างกันหรือไม่เมื่อใช้ใบอนุญาตแบบมีกระบวนการวัดเทียบกับใบอนุญาตถาวร?**

ไม่. สิ่งนี้เป็นเพียงเกี่ยวกับกลไกการให้สิทธิ์และการเรียกเก็บเงิน; ความสามารถของผลิตภัณฑ์ยังคงเหมือนเดิม.

**การให้สิทธิ์แบบมีกระบวนการวัดสัมพันธ์กับรุ่นทดลองและใบอนุญาตชั่วคราวอย่างไร?**

รุ่นทดลองมีข้อจำกัดและลายน้ำ, [ใบอนุญาตชั่วคราว](https://purchase.aspose.com/temporary-license/) จะลบข้อจำกัดออกเป็นเวลา 30 วัน, และการให้สิทธิ์แบบมีกระบวนการวัดจะลบข้อจำกัดและคิดค่าใช้จ่ายตามการใช้งานจริง.

**ฉันสามารถควบคุมงบประมาณโดยตอบสนองอัตโนมัติเมื่อเกินเกณฑ์การใช้งานหรือไม่?**

ใช่. วิธีปฏิบัติทั่วไปคือการอ่านการใช้แบบปัจจุบันเป็นระยะ ๆ ผ่าน [tracking methods](https://reference.aspose.com/slides/th/net/aspose.slides/metered/) และกำหนดขีดจำกัดหรือการแจ้งเตือนของคุณเองในระดับแอปพลิเคชันหรือการเฝ้าติดตาม.
