## **ข้อกำหนดซอฟต์แวร์**
ก่อนที่คุณจะดาวน์โหลดและเรียกใช้ตัวอย่าง โปรดตรวจสอบและยืนยันว่าการตั้งค่าของคุณตรงตามข้อกำหนดเหล่านี้: 

- Visual Studio 2010 หรือสูงกว่า.
- NuGet Package Manager ถูกติดตั้งใน Visual Studio ตรวจสอบว่าเวอร์ชันล่าสุดของ NuGet API ได้รับการติดตั้งใน Visual Studio แล้ว. 

สำหรับคำแนะนำในการติดตั้ง NuGet package manager ไปที่หน้านี้: https://docs.microsoft.com/en-us/nuget/install-nuget-client-tools

1. ไปที่ **Tools** > **Options** > **NuGet Package Manager**.

1. ขยาย **NuGet Package Manager** (โดยการคลิกสองครั้ง) แล้วเลือก **Package Sources**. 

1. ตรวจสอบและยืนยันว่าได้เลือกพารามิเตอร์ nuget.org. 

   โครงการตัวอย่างใช้ฟีเจอร์ NuGet Automatic Package Restore ดังนั้นคุณจึงต้องมีการเชื่อมต่ออินเทอร์เน็ตที่พร้อมใช้งาน. 

   หากคุณไม่มีการเชื่อมต่ออินเทอร์เน็ตที่พร้อมใช้งานบนเครื่องที่คุณตั้งใจจะเรียกใช้ตัวอย่าง โปรดตรวจสอบ [Installation](https://docs.aspose.com/slides/th/net/installation/) และเพิ่มการอ้างอิงไปยัง Aspose.Slides.dll ในโครงการตัวอย่างด้วยตนเอง.
## **ดาวน์โหลด Aspose.Slides จาก GitHub**
ตัวอย่าง Aspose.Slides สำหรับ .NET ทั้งหมดถูกโฮสต์บน [GitHub](https://github.com/aspose-slides/Aspose.Slides-for-.NET).

คุณสามารถโคลนรีโพเจอรีด้วยไคลเอนต์ GitHub ที่คุณชื่นชอบ หรือดาวน์โหลดไฟล์ ZIP [ที่นี่](https://github.com/aspose-slides/Aspose.Slides-for-.NET/archive/master.zip).

1. หากคุณดาวน์โหลดไฟล์ ZIP คุณต้องแตกเนื้อหาออกไปยังโฟลเดอร์บนคอมพิวเตอร์ของคุณ. 

ตัวอย่างทั้งหมดถูกเก็บอยู่ในโฟลเดอร์ **Examples**.

มีไฟล์โซลูชัน Visual Studio ของ C#. โปรเจกต์ถูกสร้างใน Visual Studio 2013 แต่ไฟล์โซลูชันนั้นเข้ากันได้กับ Visual Studio 2010 SP1 และสูงกว่า.

2. เปิดไฟล์โซลูชันใน Visual Studio แล้วสร้างโปรเจกต์.

   ในการรันครั้งแรก การพึ่งพาจะถูกดาวน์โหลดโดยอัตโนมัติผ่าน NuGet.

โฟลเดอร์ **Data** ที่อยู่ในโฟลเดอร์รากของ **Examples** มีไฟล์อินพุตที่ใช้ในตัวอย่าง C#. คุณต้องดาวน์โหลดโฟลเดอร์ **Data** ร่วมกับโครงการตัวอย่าง.

3. เปิดไฟล์ RunExamples.cs ตัวอย่างทั้งหมดถูกเรียกใช้จากไฟล์นี้.

4. ยกเลิกการคอมเมนต์ตัวอย่างที่คุณต้องการรันในโครงการ.

หากคุณมีปัญหาในการตั้งค่าหรือรันตัวอย่าง โปรดติดต่อเราผ่านฟอรัมตามสะดวก.
## **มีส่วนร่วม**
คุณสามารถมีส่วนร่วมในโครงการโดยการเพิ่มหรือปรับปรุงตัวอย่าง ตัวอย่างและโครงการแสดงผลทั้งหมดในรีโพเจอรีเป็นโอเพนซอร์ส ดังนั้นคุณ (และผู้อื่น) สามารถใช้ได้อย่างอิสระในแอปพลิเคชัน.

เพื่อมีส่วนร่วม คุณสามารถ fork รีโพเจอรี แก้ไขซอร์สโค้ด และสร้าง pull request เราจะตรวจสอบการเปลี่ยนแปลง หากพบว่ามีประโยชน์ เราจะเพิ่มเข้าไปในรีโพเจอรี.
