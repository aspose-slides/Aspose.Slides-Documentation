## **แนะนำ**

การให้สิทธิ์แบบมีการวัดจำนวนเป็นกลไกการให้สิทธิ์ที่สามารถใช้ร่วมกับวิธีการให้สิทธิ์ที่มีอยู่ได้ หากคุณต้องการเรียกเก็บค่าบริการตามการใช้ฟีเจอร์ของ Aspose.Slides API คุณจะเลือกใช้การให้สิทธิ์แบบมีการวัดจำนวน

## **ใช้คีย์แบบมีการวัดจำนวน**

เมื่อคุณซื้อใบอนุญาตแบบมีการวัดจำนวน คุณจะได้รับคีย์ (และไม่มีไฟล์ใบอนุญาต) คีย์แบบมีการวัดจำนวนนี้สามารถนำไปใช้ได้โดยใช้คลาส [Metered](https://reference.aspose.com/slides/th/nodejs-java/aspose.slides/metered/) ที่ Aspose จัดให้สำหรับการวัดจำนวน สำหรับรายละเอียดเพิ่มเติม ดูที่ [Metered Licensing FAQ](https://purchase.aspose.com/faqs/licensing/metered).

1. สร้างอินสแตนซ์ของคลาส [Metered](https://reference.aspose.com/slides/th/nodejs-java/aspose.slides/metered/).

1. ส่งคีย์สาธารณะและคีย์ส่วนตัวของคุณไปยังเมธอด [setMeteredKey](https://reference.aspose.com/slides/th/nodejs-java/aspose.slides/metered/#setMeteredKey).

1. ทำการประมวลผลบางอย่าง (ทำงานต่าง ๆ).

1. เรียกเมธอด [getConsumptionQuantity](https://reference.aspose.com/slides/th/nodejs-java/aspose.slides/metered/#getConsumptionQuantity) ของคลาส `Metered`.

คุณควรเห็นจำนวน/ปริมาณของคำร้องขอ API ที่คุณได้ใช้ไปจนถึงตอนนี้

ตัวอย่างโค้ดนี้แสดงวิธีการใช้การให้สิทธิ์แบบมีการวัดจำนวน:

```js
var aspose = aspose || {};
aspose.slides = require("aspose.slides.via.java");

// สร้างอินสแตนซ์ของคลาส Metered
var metered = new aspose.slides.Metered();

// ส่งคีย์สาธารณะและคีย์ส่วนตัวไปยังอ็อบเจกต์ Metered
metered.setMeteredKey("<valid public key>", "<valid private key>");

// ดึงค่าปริมาณที่ใช้ก่อนการเรียก API
var amountBefore = aspose.slides.Metered.getConsumptionQuantity();
console.log("Amount consumed before:", amountBefore);

// ทำบางอย่างด้วย API ของ Aspose.Slides ที่นี่
// ...

// ดึงค่าปริมาณที่ใช้หลังการเรียก API
var amountAfter = aspose.slides.Metered.getConsumptionQuantity();
console.log("Amount consumed after:", amountAfter);
```

{{% alert color="warning" title="หมายเหตุ" %}} 
เพื่อใช้การให้สิทธิ์แบบมีการวัดจำนวน คุณจำเป็นต้องมีการเชื่อมต่ออินเทอร์เน็ตที่เสถียร เนื่องจากกลไกการให้สิทธิ์ใช้อินเทอร์เน็ตในการโต้ตอบกับบริการของเราอย่างต่อเนื่องและทำการคำนวณ
{{% /alert %}} 

## **คำถามที่พบบ่อย**

**คุณสามารถใช้ใบอนุญาตแบบมีการวัดจำนวนร่วมกับใบอนุญาตปกติ (แบบถาวรหรือแบบชั่วคราว) ในแอปพลิเคชันเดียวกันได้หรือไม่?**

ใช่ การให้สิทธิ์แบบมีการวัดจำนวนเป็นกลไกเพิ่มเติมที่สามารถใช้ร่วมกับ [วิธีการให้สิทธิ์](/slides/th/nodejs-java/licensing/) ที่มีอยู่ได้ คุณเลือกใช้กลไกใดเมื่อแอปพลิเคชันเริ่มทำงาน

**อะไรคือสิ่งที่นับเป็นการใช้ในใบอนุญาตแบบมีการวัดจำนวน: การทำงานหรือไฟล์?**

การใช้งาน API จะถูกนับหมายถึงจำนวนคำร้องขอหรือการทำงาน คุณสามารถรับข้อมูลการใช้ปัจจุบันได้ผ่าน [เมธอดการติดตามการใช้](https://reference.aspose.com/slides/th/nodejs-java/aspose.slides/metered/)

**การให้สิทธิ์แบบมีการวัดจำนวนเหมาะกับไมโครเซอร์วิสและสภาพแวดล้อมแบบเซิร์ฟเวอร์เลสที่อินสแตนซ์รีสตาร์ทบ่อยหรือไม่?**

ใช่ เนื่องจากการคำนวณทำที่ระดับการเรียก API สถานการณ์ที่มีการเริ่มต้นใหม่บ่อยจึงเข้ากันได้ ตราบใดที่มีการเข้าถึงเครือข่ายที่เสถียรสำหรับการคำนวณแบบมีการวัดจำนวน

**ฟังก์ชันการทำงานของไลบรารีจะแตกต่างกันอย่างไรเมื่อใช้ใบอนุญาตแบบมีการวัดจำนวนเทียบกับใบอนุญาตถาวร?**

ไม่มี นี่เป็นเพียงเรื่องของกลไกการให้สิทธิ์และการเรียกเก็บเงิน; ความสามารถของผลิตภัณฑ์ยังคงเหมือนเดิม

**การให้สิทธิ์แบบมีการวัดจำนวนเกี่ยวข้องอย่างไรกับรุ่นทดลองและใบอนุญาตชั่วคราว?**

รุ่นทดลองมีข้อจำกัดและลายน้ำ, [ใบอนุญาตชั่วคราว](https://purchase.aspose.com/temporary-license/) จะลบข้อจำกัดออกเป็นเวลา 30 วัน, และการให้สิทธิ์แบบมีการวัดจำนวนจะลบข้อจำกัดและเรียกเก็บตามการใช้งานจริง

**ฉันสามารถควบคุมงบประมาณโดยอัตโนมัติเมื่อเกินเกณฑ์การใช้หรือไม่?**

ใช่ การปฏิบัติทั่วไปคือการอ่านการใช้ปัจจุบันเป็นระยะ ๆ ผ่าน [เมธอดการติดตาม](https://reference.aspose.com/slides/th/nodejs-java/aspose.slides/metered/) และกำหนดขีดจำกัดหรือการแจ้งเตือนของคุณเองที่ระดับแอปพลิเคชันหรือระบบมอนิเตอร์
