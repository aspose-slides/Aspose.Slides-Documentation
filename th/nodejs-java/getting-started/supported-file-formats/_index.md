## **ภาพรวม**

Aspose.Slides รองรับไฟล์งานนำเสนอจาก Microsoft PowerPoint 97 จนถึง Office 365 รวมถึง Microsoft PowerPoint สำหรับ Mac บทความนี้ระบุรุ่น PowerPoint ที่ไลบรารีสนับสนุนและให้ตารางรูปแบบไฟล์ที่สามารถโหลด บันทึก หรือทำได้ทั้งสองอย่าง

บทความยังตอบคำถามทั่วไปเกี่ยวกับการปฏิบัติตามมาตรฐาน PDF การฝังฟอนต์ ไฟล์ที่ป้องกันด้วยรหัสผ่าน ฟอนต์แบบกำหนดเอง การสำรองฟอนต์ และตัวเลือกการส่งออก XPS

## **รุ่น Microsoft PowerPoint ที่รองรับ**
- Microsoft PowerPoint 97
- Microsoft PowerPoint 2000
- Microsoft PowerPoint XP
- Microsoft PowerPoint 2003
- Microsoft PowerPoint 2007
- Microsoft PowerPoint 2010
- Microsoft PowerPoint 2013
- Microsoft PowerPoint 2016
- Microsoft PowerPoint 2019
- Microsoft PowerPoint สำหรับ MAC
- Office 365

## **คำถามที่พบบ่อย**

**ฉันสามารถบันทึกงานนำเสนอเป็น PDF ที่เป็นไปตามมาตรฐานการเก็บรักษาและการเข้าถึง (PDF/A และ PDF/UA) ได้หรือไม่?**

ใช่ Aspose.Slides รองรับการส่งออกเป็น PDF พร้อมระดับการปฏิบัติตามเช่น PDF/A-2a, PDF/A-2b, PDF/A-2u, PDF/A-3a, PDF/A-3b รวมถึง PDF/UA ผ่านการตั้งค่า [การปฏิบัติตาม](https://reference.aspose.com/slides/th/nodejs-java/aspose.slides/pdfoptions/setcompliance/) ใน [ตัวเลือกการส่งออก PDF](https://reference.aspose.com/slides/th/nodejs-java/aspose.slides/pdfoptions/)

**ไลบรารีสนับสนุนการฝังฟอนต์เมื่อส่งออกเป็น PDF และสามารถควบคุมอย่างละเอียดได้หรือไม่?**

ใช่ คุณสามารถควบคุมได้ว่าฟอนต์จะถูกฝังเต็มหรือเป็นชุดย่อย (เฉพาะ glyph ที่ใช้) ระบุวิธีการจัดการฟอนต์ระบบทั่วไป และกำหนดพฤติกรรมสำหรับข้อความ ASCII ผ่าน [ตัวเลือกการส่งออก PDF](https://reference.aspose.com/slides/th/nodejs-java/aspose.slides/pdfoptions/)

**ฉันสามารถตรวจจับได้ว่าไฟล์ถูกป้องกันด้วยรหัสผ่านก่อนโหลดหรือไม่?**

ใช่ ด้วยการใช้ [API การตรวจสอบแบบ factory](https://reference.aspose.com/slides/th/nodejs-java/aspose.slides/presentationfactory/) คุณสามารถสอบถามไฟล์งานนำเสนอเพื่อกำหนดว่ามีการป้องกันด้วยรหัสผ่านโดยไม่ต้องเปิดไฟล์เต็ม

**มีกลไกการสำรองฟอนต์และการสนับสนุนฟอนต์แบบกำหนดเองหรือไม่?**

ใช่ ไลบรารีสนับสนุนการ [โหลด](/slides/th/nodejs-java/custom-font/) และการ [ฝัง](/slides/th/nodejs-java/embedded-font/) ฟอนต์แบบกำหนดเองและให้กฎ [การสำรองฟอนต์](/slides/th/nodejs-java/fallback-font/) เพื่อป้องกัน glyph ที่หายไประหว่างการเรนเดอร์และการแปลง

**ฉันสามารถส่งออกสไลด์เป็น XPS ได้หรือไม่ และมีตัวเลือกในการปรับแต่งผลลัพธ์ XPS หรือไม่?**

ใช่ การ [ส่งออกเป็น XPS](/slides/th/nodejs-java/convert-powerpoint-to-xps/) ได้รับการสนับสนุนและคุณสามารถปรับ [ตัวเลือกการบันทึก](https://reference.aspose.com/slides/th/nodejs-java/aspose.slides/xpsoptions/) ที่เกี่ยวข้องเพื่อควบคุคุณภาพและเนื้อหาของเอกสาร XPS
