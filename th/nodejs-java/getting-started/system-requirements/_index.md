## **บทนำ**

Aspose.Slides สำหรับ Node.js ผ่าน Java ไม่จำเป็นต้องติดตั้งผลิตภัณฑ์ของบุคคลที่สามใด ๆ เช่น Microsoft PowerPoint. Aspose.Slides เองเป็นเอนจิ๋นสำหรับการสร้าง, แก้ไข, แปลง, และแสดงผลเอกสารในรูปแบบต่าง ๆ รวมถึงรูปแบบการนำเสนอของ Microsoft PowerPoint.

## **ระบบปฏิบัติการที่รองรับ**

Aspose.Slides สำหรับ Node.js ผ่าน Java รองรับระบบปฏิบัติการ 32‑บิตหรือ 64‑บิตใดก็ได้ที่รัน Java runtime รวมถึงแต่ไม่จำกัดเพียง:

### **Windows**
- Microsoft Windows 2003 Server ( x64, x86)
- Microsoft Windows 2008 Server ( x64, x86)
- Microsoft Windows 2012 Server ( x64, x86)
- Microsoft Windows 2012 R2 Server ( x64, x86)
- Microsoft Windows 2016 Server ( x64, x86)
- Microsoft Windows 2019 Server ( x64, x86)
- Microsoft Windows Vista ( x64, x86)
- Microsoft Windows XP ( x64, x86)
- Microsoft Windows 7 ( x64, x86)
- Microsoft Windows 8, 8.1 ( x64, x86)
- Microsoft Windows 10 ( x64, x86)

### **Linux**
- Linux (Ubuntu, CentOS และอื่น ๆ)

### **Mac**
- Mac OS X

## **คำถามที่พบบ่อย**

**ฉันจำเป็นต้องติดตั้ง Microsoft PowerPoint เพื่อการแปลงและการแสดงผลหรือไม่?**

ไม่, ไม่จำเป็นต้องใช้ PowerPoint; Aspose.Slides เป็นเอนจิ๋นแบบสแตนด์อโลนสำหรับ [การสร้าง](/slides/th/nodejs-java/create-presentation/), การแก้ไข, [การแปลง](/slides/th/nodejs-java/convert-presentation/), และ [การแสดงผล](/slides/th/nodejs-java/convert-powerpoint-to-png/) การนำเสนอ.

**ฟอนท์ใดที่จำเป็นสำหรับการแสดงผลที่ถูกต้อง?**

โดยปฏิบัติ, ฟอนท์ที่ใช้ในงานนำเสนอหรือ [ตัวแทนที่เหมาะสม](/slides/th/nodejs-java/font-substitution/) ต้องมีอยู่. เพื่อให้การแสดงผลสม่ำเสมอบน Linux/macOS, ควรติดตั้งแพคเกจฟอนท์ที่ทั่วไป.

**ทำไมฟอนท์ที่กำหนดเองจึงถูกแสดงเป็นฟอนท์สำรองหรือข้อความหายบน Linux?**

หากไฟล์ฟอนท์มีรายการตารางชื่อที่ไม่สอดคล้องหรือเสียหาย, ระบบจับคู่ฟอนท์ของ Linux (FreeType/fontconfig) อาจเลือกบันทึกที่ไม่ถูกต้อง, ทำให้ฟอนท์ไม่สามารถระบุตัวได้. การใช้เวอร์ชันฟอนท์ที่มีการแก้ไขรายการตารางชื่อหรือการติดตั้งการทดแทนที่สอดคล้องกันจะช่วยแก้ปัญหา.
