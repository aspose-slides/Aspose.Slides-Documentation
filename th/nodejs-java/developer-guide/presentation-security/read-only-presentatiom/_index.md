## **บทนำ**

ใน PowerPoint 2019, Microsoft ได้นำเสนอการตั้งค่า **Always Open Read-Only** เป็นหนึ่งในตัวเลือกที่ผู้ใช้สามารถใช้เพื่อปกป้องงานนำเสนอของตน คุณอาจต้องการใช้การตั้งค่าอ่านอย่างเดียวนี้เพื่อปกป้องงานนำเสนอเมื่อ

- คุณต้องการป้องกันการแก้ไขโดยไม่ได้ตั้งใจและรักษาเนื้อหาของงานนำเสนอให้ปลอดภัย
- คุณต้องการแจ้งให้ผู้คนทราบว่างานนำเสนอที่คุณให้เป็นเวอร์ชันสุดท้าย

หลังจากคุณเลือกตัวเลือก **Always Open Read-Only** สำหรับงานนำเสนอ เมื่อผู้ใช้เปิดงานนำเสนอ พวกเขาจะเห็นคำแนะนำ **Read-Only** และอาจเห็นข้อความในรูปแบบนี้: *เพื่อป้องกันการเปลี่ยนแปลงโดยไม่ได้ตั้งใจ ผู้เขียนได้ตั้งค่าไฟล์นี้ให้เปิดเป็นแบบอ่านอย่างเดียว*  

คำแนะนำ **Read-Only** เป็นการขัดจังหวะที่เรียบง่ายแต่มีประสิทธิภาพซึ่งทำให้ผู้ใช้ไม่แก้ไข เพราะผู้ใช้ต้องทำขั้นตอนเพื่อเอาออกก่อนที่จะสามารถแก้ไขงานนำเสนอได้ หากคุณไม่ต้องการให้ผู้ใช้ทำการเปลี่ยนแปลงงานนำเสนอและต้องการบอกพวกเขาอย่างสุภาพ คำแนะนำ **Read-Only** อาจเป็นตัวเลือกที่ดีสำหรับคุณ  

> หากงานนำเสนอที่มีการป้องกัน **Read-Only** ถูกเปิดในแอปพลิเคชัน Microsoft PowerPoint รุ่นเก่าที่ไม่รองรับฟังก์ชันที่เพิ่งแนะนำนี้ คำแนะนำ **Read-Only** จะถูกละเลย (งานนำเสนอจะเปิดตามปกติ)

## **ใช้โหมดอ่านอย่างเดียว**

Aspose.Slides สำหรับ Node.js ผ่าน Java ให้คุณตั้งค่างานนำเสนอให้เป็น **Read-Only** ซึ่งหมายความว่าผู้ใช้ (หลังจากเปิดงานนำเสนอ) จะเห็นคำแนะนำ **Read-Only** ตัวอย่างโค้ดนี้จะแสดงวิธีตั้งค่างานนำเสนอให้เป็น **Read-Only** ใน JavaScript ด้วย Aspose.Slides:

```javascript
var pres = new aspose.slides.Presentation();
try {
    pres.getProtectionManager().setReadOnlyRecommended(true);
    pres.save("ReadOnlyPresentation.pptx", aspose.slides.SaveFormat.Pptx);
} finally {
    if (pres != null) {
        pres.dispose();
    }
}
```

{{% alert color="primary" %}} 

**หมายเหตุ**: คำแนะนำ **Read-Only** มีจุดประสงค์เพียงเพื่อไม่ให้ผู้ใช้แก้ไขหรือป้องกันไม่ให้เกิดการเปลี่ยนแปลงโดยไม่ได้ตั้งใจในงานนำเสนอ PowerPoint หากบุคคลที่มีความตั้งใจ—ที่รู้ว่าตนทำอะไร—ตัดสินใจแก้ไขงานนำเสนอของคุณ พวกเขาสามารถลบการตั้งค่า Read-Only ได้ง่าย หากคุณต้องการป้องกันการแก้ไขโดยไม่ได้รับอนุญาตอย่างจริงจัง คุณควรใช้ [การป้องกันที่เข้มงวดขึ้นที่เกี่ยวข้องกับการเข้ารหัสและรหัสผ่าน](https://docs.aspose.com/slides/th/nodejs-java/password-protected-presentation/).

## **คำถามที่พบบ่อย**

**ความแตกต่างระหว่าง 'Read-Only recommended' กับการป้องกันด้วยรหัสผ่านเต็มรูปแบบคืออะไร?**

'Read-Only recommended' จะแสดงเพียงข้อเสนอแนะให้เปิดไฟล์ในโหมดอ่านอย่างเดียวและง่ายต่อการข้ามผ่าน. [การป้องกันด้วยรหัสผ่าน](/slides/th/nodejs-java/password-protected-presentation/) จริงๆ แล้วจำกัดการเปิดหรือแก้ไขและเหมาะเมื่อคุณต้องการการควบคุมความปลอดภัยจริง.

**สามารถผสาน 'Read-Only recommended' กับลายน้ำเพื่อเพิ่มการขัดจังหวะการแก้ไขได้หรือไม่?**

ได้. คำแนะนำสามารถจับคู่กับ [ลายน้ำ](/slides/th/nodejs-java/watermark/) เป็นการขัดจังหวะด้วยภาพ; ทั้งสองเป็นกลไกที่แยกจากกันและทำงานร่วมกันได้ดี.

**แมโครหรือเครื่องมือภายนอกยังสามารถแก้ไขไฟล์ได้เมื่อคำแนะนำถูกเปิดใช้งานหรือไม่?**

ได้. คำแนะนำไม่บล็อกการเปลี่ยนแปลงโดยโปรแกรม. เพื่อป้องกันการแก้ไขอัตโนมัติ ใช้ [รหัสผ่านและการเข้ารหัส](/slides/th/nodejs-java/password-protected-presentation/).

**'Read-Only recommended' มีความสัมพันธ์อย่างไรกับแฟล็ก 'IsEncrypted' และ 'IsWriteProtected'?**

พวกมันเป็นสัญญาณที่ต่างกัน. 'Read-Only recommended' เป็นการแจ้งเตือนแบบอ่อนและเป็นทางเลือก; [isWriteProtected](https://reference.aspose.com/slides/th/nodejs-java/aspose.slides/protectionmanager/iswriteprotected/) และ [isEncrypted](https://reference.aspose.com/slides/th/nodejs-java/aspose.slides/protectionmanager/isencrypted/) แสดงข้อจำกัดการเขียนหรืออ่านจริงที่ขึ้นอยู่กับรหัสผ่านหรือการเข้ารหัส.
