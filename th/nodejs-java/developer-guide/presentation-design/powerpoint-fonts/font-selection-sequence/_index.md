## **ภาพรวม**

เมื่อพรีเซนเทชันถูกโหลด, เรนเดอร์, หรือแปลงเป็นรูปแบบอื่น, Aspose.Slides ตรวจสอบว่าแบบอักษรที่ใช้ในพรีเซนเทชันมีอยู่ในระบบปฏิบัติการหรือไม่ หากแบบอักษรที่ต้องการขาดหายไป, Aspose.Slides จะเลือกแบบอักษรทดแทนที่ใกล้เคียงกับที่ PowerPoint จะใช้ที่สุด

Aspose.Slides จะค้นหาแบบอักษรที่เลือกในระบบปฏิบัติการก่อน หากพบแบบอักษรนั้นก็จะใช้ หากไม่พบจะใช้แบบอักษรทดแทนที่เหมาะสม เมื่อมีการกำหนดกฎการแทนที่แบบอักษรผ่าน `FontSubstRule` กฎเหล่านั้นก็จะถูกนำมาพิจารณาด้วย

คุณยังสามารถเพิ่มแบบอักษรในเวลาที่แอปพลิเคชันทำงาน, ใช้แบบอักษรที่ฝังไว้จากพรีเซนเทชัน, หรือโหลดแบบอักษรภายนอกสำหรับเอกสารผลลัพธ์เช่นไฟล์ PDF ได้อีกด้วย

## **การเลือกแบบอักษร**

มีกฎบางประการที่ใช้กับแบบอักษรในพรีเซนเทชันเมื่อพรีเซนเทชันถูกโหลด, เรนเดอร์, หรือแปลงเป็นรูปแบบอื่น เช่น เมื่อคุณพยายามแปลงพรีเซนเทชัน (สไลด์) เป็นภาพ, แบบอักษรของพรีเซนเทชันจะถูกตรวจสอบเพื่อยืนยันว่ามีแบบอักษรที่เลือกอยู่ในระบบปฏิบัติการหรือไม่ หากยืนยันว่าไม่มีแบบอักษรเหล่านั้น, จะมีการแทนที่ — ดู [**Font Replacement**](https://docs.aspose.com/slides/th/nodejs-java/font-replacement/) และ [**Font Substitution**](https://docs.aspose.com/slides/th/nodejs-java/font-substitution/)

นี่คือกระบวนการที่ Aspose.Slides ปฏิบัติตามเมื่อจัดการกับแบบอักษร:

1. Aspose.Slides ค้นหาแบบอักษรในระบบปฏิบัติการเพื่อหาแบบอักษรที่ตรงกับแบบอักษรที่พรีเซนเทชันเลือกไว้  
2. หากพบแบบอักษรที่เลือก, Aspose.Slides จะใช้แบบอักษรนั้น มิฉะนั้น Aspose.Slides จะใช้แบบอักษรทดแทนที่ใกล้เคียงกับที่ PowerPoint จะใช้มากที่สุด  
3. หากมีกฎการแทนที่แบบอักษรที่ตั้งค่าผ่าน [FontSubstRule](https://reference.aspose.com/slides/th/nodejs-java/aspose.slides/fontsubstrule/), กฎเหล่านั้นจะถูกนำมาใช้  

Aspose.Slides อนุญาตให้คุณเพิ่มแบบอักษรในเวลาที่แอปพลิเคชันทำงานและนำแบบอักษรเหล่านั้นมาใช้ ดู [**Custom fonts**](https://docs.aspose.com/slides/th/nodejs-java/custom-font/)  

เมื่อมีการวางแบบอักษรเพิ่มเติมภายในพรีเซนเทชัน, แบบอักษรเหล่านั้นจะเรียกว่า [**Embedded fonts**](https://docs.aspose.com/slides/th/nodejs-java/embedded-font/)  

Aspose.Slides อนุญาตให้คุณเพิ่มแบบอักษรที่ใช้กับ *เฉพาะ* เอกสารผลลัพธ์ ตัวอย่างเช่น หากพรีเซนเทชันที่คุณต้องการแปลงเป็น PDF มีแบบอักษรที่ขาดหายจากระบบและแบบอักษรที่ฝังไว้, คุณสามารถเพิ่มหรือโหลดแบบอักษรที่จำเป็นเป็น **external fonts** ได้  

{{% alert title="Note" color="primary" %}} 
เราไม่ได้แจกจ่ายแบบอักษรใด ๆ ไม่ว่าจะเป็นแบบที่ต้องชำระเงินหรือแบบฟรี API ของเราช่วยให้คุณโหลดแบบอักษรภายนอกและฝังลงในเอกสาร, แต่คุณต้องทำเช่นนั้นด้วยแบบอักษรตามดุลยพินิจและความรับผิดชอบของคุณเอง
{{% /alert %}}

## **คำถามที่พบบ่อย**

**ฉันจะกำหนดว่าแบบอักษรใดบ้างที่ถูกใช้จริงในพรีเซนเทชันก่อนการแปลงได้อย่างไร?**

Aspose.Slides ให้คุณตรวจสอบแบบอักษรที่ใช้ผ่าน [font manager](https://reference.aspose.com/slides/th/nodejs-java/aspose.slides/presentation/getfontsmanager/), เพื่อให้คุณตัดสินใจว่าจะ [embed](/slides/th/nodejs-java/embedded-font/), [replace](/slides/th/nodejs-java/font-replacement/), หรือเพิ่ม [external sources](/slides/th/nodejs-java/custom-font/) หรือไม่ วิธีนี้ช่วยป้องกันการแทนที่ที่ไม่ต้องการระหว่างการเรนเดอร์และการส่งออก  

**ฉันสามารถเพิ่มไดเรกทอรีแบบอักษรเพิ่มเติมโดยไม่ต้องติดตั้งบนระบบปฏิบัติการได้หรือไม่?**

ได้ คุณสามารถลงทะเบียน [external font sources](/slides/th/nodejs-java/custom-font/) เช่น โฟลเดอร์หรือสตรีมในหน่วยความจำสำหรับการเรนเดอร์และการส่งออก สิ่งนี้ช่วยลบการพึ่งพาแบบอักษรของระบบโฮสต์และทำให้การจัดวางคาดการณ์ได้  

**ฉันจะป้องกันการโFallback เงียบๆ ไปยังแบบอักษรที่ไม่เหมาะสมเมื่อ glyph ขาดหายได้อย่างไร?**

กำหนด [font replacement](/slides/th/nodejs-java/font-replacement/) อย่างชัดเจนและกฎ [fallBack](/slides/th/nodejs-java/fallback-font/) ล่วงหน้า โดยการวิเคราะห์แบบอักษรที่ใช้และตั้งลำดับความสำคัญที่ควบคุมสำหรับตัวทดแทน, คุณจะทำให้การจัดพิมพ์สอดคล้องและหลีกเลี่ยงผลลัพธ์ที่ไม่คาดคิด
