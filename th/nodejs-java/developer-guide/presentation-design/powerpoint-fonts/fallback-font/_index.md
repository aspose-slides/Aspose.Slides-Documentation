## **บทนำ**

ฟอนต์สำรองจะถูกใช้เมื่อฟอนต์ที่ระบุสำหรับข้อความมีอยู่ในระบบแต่ไม่มี glyph ที่ต้องการ ในกรณีนี้ Aspose.Slides สามารถใช้หนึ่งในฟอนต์สำรองที่ระบุเพื่อแทนที่ glyph ที่หายไป

## **ฟอนต์สำรอง**

Aspose.Slides อนุญาตให้สร้างฟอนต์สำรอง, เพิ่มลงในคอลเลกชันฟอนต์สำรอง, ตั้งค่าคอลเลกชันฟอนต์สำรองสำหรับงานนำเสนอที่กำหนด, ลบฟอนต์สำรองออกจากงานนำเสนอ, ระบุกฎที่จะใช้ฟอนต์สำรองและอื่น ๆ

เพื่อทำความคุ้นเคยกับคุณลักษณะเหล่านี้, ใช้ลิงก์ต่อไปนี้:

- [สร้างฟอนต์สำรอง](/slides/th/nodejs-java/create-fallback-font)
- [สร้างคอลเลกชันฟอนต์สำรอง](/slides/th/nodejs-java/create-fallback-fonts-collection)
- [เรนเดอร์งานนำเสนอด้วยฟอนต์สำรอง](/slides/th/nodejs-java/render-presentation-with-fallback-font)

## **คำถามที่พบบ่อย**

**ฟอนต์สำรองแตกต่างจากการแทนที่ฟอนต์อย่างไร?**

ฟอนต์สำรองจะถูกใช้ต่ออักขระหรือช่วงของ Unicode เมื่อฟอนต์หลักไม่มี glyph เฉพาะ; มันจะเติมเฉพาะอักขระที่หายไปเท่านั้น. [การแทนที่](/slides/th/nodejs-java/font-substitution/) จะเปลี่ยนฟอนต์ที่หายไปหรือไม่พร้อมใช้งานสำหรับชุดข้อความทั้งหมดหรือส่วนของข้อความด้วยฟอนต์อื่น. สามารถใช้ร่วมกันได้ แต่ขอบเขตและตรรกะการเลือกต่างกัน.

**การตั้งค่าฟอนต์สำรองถูกบันทึกภายในไฟล์งานนำเสนอหรือไม่?**

ไม่. การกำหนดค่าฟอนต์สำรองอยู่ในช่วงการประมวลผล/เรนเดอร์ในไลบรารีและไม่ได้ถูกซีเรียลไลซ์เข้าสู่ PPTX. งานนำเสนอจะไม่เก็บกฎฟอนต์สำรองของคุณ.

**ฟอนต์สำรองมีผลต่อองค์ประกอบที่สร้างโดยวัตถุ PowerPoint (SmartArt, แผนภูมิ, WordArt) หรือไม่?**

มี. ข้อความภายในวัตถุเหล่านี้จะผ่านไปยัง pipeline การเรนเดอร์เดียวกัน ดังนั้นกฎฟอนต์สำรองเดียวกันจะถูกใช้กับข้อความเหล่านี้เช่นเดียวกับข้อความปกติ.
