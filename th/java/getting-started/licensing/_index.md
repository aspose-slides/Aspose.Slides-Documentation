## **ภาพรวม**

Aspose.Slides สามารถใช้ได้ในโหมดประเมินหรือติดตั้งด้วยใบอนุญาตที่ถูกต้อง เวอร์ชันประเมินให้ฟังก์ชันเดียวกับเวอร์ชันที่มีใบอนุญาต แต่จะเพิ่มลายน้ำการประเมินเมื่อเปิดหรือบันทึกการนำเสนอและจำกัดการสกัดข้อความให้เหลือหนึ่งสไลด์

บทความนี้อธิบายว่าการจัดการใบอนุญาตทำงานอย่างไรใน Aspose.Slides และวิธีการใช้ใบอนุญาตก่อนใช้งานไลบรารี ใบอนุญาตสามารถโหลดจากไฟล์, สตรีม หรือทรัพยากรฝังตัวโดยใช้คลาส `License` บทความนี้ยังแสดงวิธีการตรวจสอบว่าใบอนุญาตได้ถูกนำไปใช้อย่างถูกต้องหรือไม่

## **การจัดการใบอนุญาตใน Aspose.Slides**

* เวอร์ชันประเมินจะกลายเป็นเวอร์ชันที่มีใบอนุญาตหลังจากคุณซื้อใบอนุญาตและเพิ่มโค้ดสองสามบรรทัด (เพื่อใช้ใบอนุญาต).  
* ใบอนุญาตเป็นไฟล์ XML แบบข้อความธรรมดาที่บรรจุรายละเอียดเช่น ชื่อผลิตภัณฑ์ จำนวนผู้พัฒนาที่ได้รับการอนุญาต วันที่หมดอายุการสมัครสมาชิก เป็นต้น.  
* ไฟล์ใบอนุญาตได้รับการลงลายเซ็นดิจิทัล ดังนั้นคุณต้องไม่แก้ไขไฟล์ แม้แต่การเพิ่มบรรทัดว่างโดยไม่ได้ตั้งใจในเนื้อหาไฟล์ก็จะทำให้ไฟล์ไม่เป็นธรรมนูญ.  
* Aspose.Slides for Java มักจะค้นหาใบอนุญาตจากตำแหน่งต่อไปนี้:
  * เส้นทางที่ระบุโดยตรง
  * โฟลเดอร์ที่บรรจุ Aspose.Slides.jar  
* เพื่อหลีกเลี่ยงข้อจำกัดที่เกี่ยวข้องกับเวอร์ชันประเมิน คุณจำเป็นต้องตั้งค่าใบอนุญาตก่อนใช้ **Aspose.Slides** คุณต้องตั้งค่าใบอนุญาตเพียงครั้งเดียวต่อแอปพลิเคชันหรือกระบวนการ

## **การใช้ใบอนุญาต**

ใบอนุญาตสามารถโหลดจาก **ไฟล์** หรือ **สตรีม**.

{{% alert color="primary" %}}

Aspose.Slides มีคลาส[License](https://reference.aspose.com/slides/th/java/com.aspose.slides/License)สำหรับการทำงานเกี่ยวกับใบอนุญาต.

{{% /alert %}} 

{{% alert color="warning" %}}

ใบอนุญาตใหม่สามารถเปิดใช้งาน Aspose.Slides ได้เฉพาะกับเวอร์ชัน 21.4 หรือใหม่กว่าถัดไป เวอร์ชันก่อนหน้าจะใช้ระบบใบอนุญาตที่แตกต่างและจะไม่รับรู้ใบอนุญาตเหล่านี้.

{{% /alert %}}

### **ไฟล์**

วิธีที่ง่ายที่สุดในการตั้งค่าใบอนุญาตคือการวางไฟล์ใบอนุญาตไว้ในโฟลเดอร์ที่บรรจุ Aspose.Slides.jar หรือ jar ของแอปพลิเคชันของคุณ.

โค้ด Java นี้แสดงวิธีตั้งค่าไฟล์ใบอนุญาต:

``` java
// สร้างอินสแตนซ์ของคลาส License
com.aspose.slides.License license = new com.aspose.slides.License();

// ตั้งค่าพาธไฟล์ใบอนุญาต
license.setLicense("Aspose.Slides.Java.lic");
```

{{% alert color="warning" %}} 

หากคุณวางไฟล์ใบอนุญาตในไดเรกทอรีอื่น เมื่อคุณเรียกเมธอด[SetLicense](https://reference.aspose.com/slides/th/java/com.aspose.slides/License#setLicense-java.lang.String-) ชื่อไฟล์ใบอนุญาตที่ส่วนท้ายของพาธที่ระบุต้องตรงกับไฟล์ใบอนุญาตของคุณ.

ตัวอย่างเช่น คุณสามารถเปลี่ยนชื่อไฟล์ใบอนุญาตเป็น*Aspose.Slides.Java.lic.xml* แล้วในโค้ดของคุณต้องส่งพาธไปยังไฟล์ (จบด้วย*Aspose.Slides.Java.lic.xml*) ไปยังเมธอด[SetLicense](https://reference.aspose.com/slides/th/java/com.aspose.slides/License#setLicense-java.lang.String-).

{{% /alert %}}

### **สตรีม**

คุณสามารถโหลดใบอนุญาตจากสตรีมได้ โค้ด Java นี้แสดงวิธีใช้ใบอนุญาตจากสตรีม:

``` java
// สร้างอินสแตนซ์ของคลาส License
com.aspose.slides.License license = new com.aspose.slides.License();

// ตั้งค่าใบอนุญาตผ่านสตรีม
license.setLicense(new java.io.FileInputStream("Aspose.Slides.Java.lic"));
```

### **PHP/Java Bridge**

หากคุณใช้ Aspose.Slides for PHP ผ่าน Java คุณสามารถตั้งค่าใบอนุญาตผ่าน PHP/Java bridge ได้ bridge นี้ทำให้คุณใช้คลาส Java ในไวยากรณ์ PHP สำหรับข้อมูลเพิ่มเติม ดู[License in PHP](/slides/th/php-java/licensing/).

## **การตรวจสอบใบอนุญาต**

เพื่อตรวจสอบว่าใบอนุญาตถูกตั้งค่าอย่างถูกต้องหรือไม่ คุณสามารถตรวจสอบได้ โค้ด Java นี้แสดงวิธีตรวจสอบใบอนุญาต:

```java
License license = new License();
license.setLicense("Aspose.Slides.Java.lic");

if (License.isLicensed()) 
{
    System.out.println("License is good!");
}
```

## **ความปลอดภัยของเธรด**

{{% alert title="Note" color="warning" %}} 

เมธอด[SetLicense](https://reference.aspose.com/slides/th/java/com.aspose.slides/License#setLicense-java.io.InputStream-) ไม่ปลอดภัยต่อการใช้ในหลายเธรด หากเมธอดนี้ต้องถูกเรียกพร้อมกันจากหลายเธรด คุณอาจต้องใช้กลไกการซิงโครไนซ์ (เช่น lock) เพื่อหลีกเลี่ยงปัญหา.

## **คำถามที่พบบ่อย**

**ฉันสามารถใช้ใบอนุญาตในสภาพแวดล้อมที่ไม่มีการเชื่อมต่ออินเทอร์เน็ตโดยสิ้นเชิงได้หรือไม่?**

ได้. การตรวจสอบใบอนุญาตทำโดยการใช้ไฟล์ใบอนุญาตในเครื่องเท่านั้น ไม่จำเป็นต้องเชื่อมต่ออินเทอร์เน็ต.

**อะไรจะเกิดขึ้นหลังจากการสมัครสมาชิกหนึ่งปีหมดอายุ? ไลบรารีจะหยุดทำงานหรือไม่?**

ไม่. ใบอนุญาตเป็นแบบถาวร: คุณสามารถใช้เวอร์ชันที่เผยแพร่ก่อนวันที่สิ้นสุดการสมัครสมาชิกของคุณต่อไปได้; แต่คุณจะไม่สามารถใช้เวอร์ชันใหม่ที่ออกหลังจากนั้นได้จนกว่าจะต่ออายุ.
