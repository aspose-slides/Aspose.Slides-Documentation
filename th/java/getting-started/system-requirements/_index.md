## **ภาพรวม**
Aspose.Slides for Java ไม่จำเป็นต้องติดตั้ง Microsoft PowerPoint เนื่องจาก Aspose.Slides เองเป็นเอนจินการสร้างเอกสาร Microsoft PowerPoint, การแปลง, การจัดหน้าและการแสดงผล

## **ระบบปฏิบัติการที่รองรับ**
Aspose.Slides for Java รองรับระบบปฏิบัติการ 32‑bit หรือ 64‑bit ใด ๆ ที่รัน Java runtime รวมถึงแต่ไม่จำกัดเพียง:

### **วินโดวส์**
- Microsoft Windows 2003 Server ( x64, x86)
- Microsoft Windows 2008 Server ( x64, x86)
- Microsoft Windows 2012 Server ( x64, x86)
- Microsoft Windows 2012 R2 Server ( x64, x86)
- Microsoft Windows 2016 Server ( x64, x86)
- Microsoft Windows 2019 Server ( x64, x86)
- Microsoft Windows Vista ( x64, x86)
- Microsoft Windows XP ( x64, x86)
- Microsoft Windows 7 ( x64, x86)
- Microsoft Windows 8, 8.1 ( x64, x86)
- Microsoft Windows 10 ( x64, x86)

### **ลีนุกซ์**
- Linux (Ubuntu, OpenSUSE, CentOS and others)

### **แมค**
- Mac OS X

## **เวอร์ชัน Java ที่รองรับ**
Aspose.Slides for Java รองรับ J2SE 6.0 (Java 1.6) ขึ้นไป

## **คำถามที่พบบ่อย**

**ฉันจำเป็นต้องติดตั้ง Microsoft PowerPoint เพื่อทำการแปลงและแสดงผลหรือไม่?**

ไม่, ไม่จำเป็นต้องใช้ PowerPoint; Aspose.Slides เป็นเอนจินแบบสแตนด์อโลนสำหรับการ[สร้าง](/slides/th/java/create-presentation/), การแก้ไข, การ[แปลง](/slides/th/java/convert-presentation/), และการ[แสดงผล](/slides/th/java/convert-powerpoint-to-png/) งานนำเสนอ

**ฟอนต์ใดที่จำเป็นสำหรับการแสดงผลที่ถูกต้อง?**

โดยปฏิบัติ, ฟอนต์ที่ใช้ในงานนำเสนอหรือ [ทางเลือก](/slides/th/java/font-substitution/) ที่เหมาะสมต้องพร้อมใช้งาน เพื่อให้การแสดงผลสอดคล้องบน Linux/macOS แนะนำให้ติดตั้งแพคเกจฟอนต์ทั่วไป

**ทำไมฟอนต์ที่กำหนดเองจึงแสดงเป็นฟอนต์สำรองหรือข้อความหายบน Linux?**

หากไฟล์ฟอนต์มีรายการในตารางชื่อที่ไม่สอดคล้องหรือเสียหาย, ระบบจับคู่ฟอนต์ของ Linux (FreeType/fontconfig) อาจเลือกบันทึกที่ไม่ถูกต้อง ทำให้ฟอนต์ไม่สามารถ resolve ได้ การใช้เวอร์ชันฟอนต์ที่แก้ไขตารางชื่อหรือการติดตั้งฟอนต์ทดแทนที่สอดคล้องจะช่วยแก้ไขปัญหาได้
