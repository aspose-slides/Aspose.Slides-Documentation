## **ภาพรวม**

บทความนี้อธิบายว่าเมื่อใดที่นักพัฒนาอาจเลือกใช้ Open XML SDK หรือ Aspose.Slides สำหรับการทำงานกับเอกสารการนำเสนอ มันอธิบายว่า Open XML SDK เป็นไลบรารีสำหรับจัดการแพ็กเกจ OOXML และองค์ประกอบ XML ที่อยู่ภายใน ส่วน Aspose.Slides นำเสนอเป็นไลบรารีการประมวลผลการนำเสนอที่มีโมเดลออบเจกต์ระดับสูงและสนับสนุนหลายงานที่เกี่ยวกับ PowerPoint

บทความเปรียบเทียบตัวเลือกทั้งสองโดยดูจากรูปแบบที่รองรับ โมเดลการเขียนโปรแกรม ความสามารถในการเรนเดอร์และพิมพ์ การสนับสนุนแพลตฟอร์ม และกรณีการใช้งานทั่วไป มันยังชี้แจงว่า Open XML SDK อาจเหมาะสำหรับการดำเนินการพื้นฐานกับ PPTX หรือการเข้าถึงองค์ประกอบ OOXML โดยตรง ขณะที่ Aspose.Slides เหมาะกับงานการนำเสนอที่ซับซ้อนเช่น การทำงานกับหลายรูปแบบ PowerPoint การคัดลอกหรือทำสำเนา Shape การแทนที่ข้อความ การประยุกต์ใช้งานอนิเมชัน และการแปลงการนำเสนอเป็น PDF, TIFF หรือ XPS

## **Open XML SDK คืออะไร?**
ตามที่อธิบายใน [MSDN Library](https://docs.microsoft.com/en-us/office/open-xml/open-xml-sdk) Open XML SDK ได้รับการกำหนดว่า:

Open XML SDK 2.0 ทำให้การจัดการแพ็กเกจ Open XML และองค์ประกอบสคีม่า Open XML ที่อยู่ภายในแพ็กเกจเป็นเรื่องง่ายขึ้น Open XML SDK 2.0 รวมงานทั่วไปหลายอย่างที่นักพัฒนามักทำกับแพ็กเกจ Open XML ไว้แล้ว เพื่อให้คุณสามารถทำงานที่ซับซ้อนได้ด้วยเพียงไม่กี่บรรทัดของโค้ด

เอกสาร OOXML โดยพื้นฐานคือไฟล์ XML ที่บีบอัดเป็น zip และ Open XML SDK เป็นชุดคลาสที่ช่วยให้คุณทำงานกับเนื้อหาเอกสาร OOXML ในรูปแบบที่มีการกำหนดประเภทอย่างเข้มงวด ซึ่งหมายความว่าแทนที่จะต้องแตกไฟล์เพื่อดึง XML โหลด XML ไปยังโครงสร้าง DOM แล้วทำงานกับองค์ประกอบและแอตทริบิวต์ XML โดยตรง Open XML SDK จะให้คลาสที่ทำเช่นนั้นให้คุณ

## **Aspose.Slides คืออะไร?**
Aspose.Slides เป็นคลาสไลบรารีที่ช่วยให้แอปพลิเคชันของคุณทำงานประมวลผลการนำเสนอได้ตามรายการต่อไปนี้:

- การเขียนโปรแกรมด้วยโมเดลออบเจกต์ **Presentation**
- การแปลงคุณภาพสูงระหว่างรูปแบบการนำเสนอ PowerPoint ที่รองรับทั้งหมด รวมถึงการแปลงเป็น PDF, XPS และ TIFF
- ความสามารถในการสร้างภาพย่อของสไลด์ในรูปแบบที่รู้จักกันดี เช่น PNG, JPEG และ BMP พร้อมการส่งออกสไลด์เป็น SVG
- ความสามารถในการสร้างการนำเสนอจากศูนย์หรือโดยการรวมจากเอกสารหนึ่งหรือหลายเอกสาร
- การสนับสนุนการเพิ่มอนิเมชัน, Ole Frames, ตาราง, การสร้างและจัดการแผนภูมิ
- การให้การควบคุมอย่างละเอียดสำหรับการจัดการรูปแบบข้อความในระดับ TextFrames, Paragraphs และ Portions

สำหรับรายละเอียดเพิ่มเติมเกี่ยวกับคุณสมบัติที่รองรับ โปรดเยี่ยมชม [Aspose.Slides Features](/slides/th/java/product-overview/)

## **สรุป**
{{% alert color="primary" %}} 

Open XML SDK และ Aspose.Slides ไม่ได้แข่งขันกันโดยตรง เนื่องจากพวกมันตอบสนองต่อความต้องการและกลุ่มผู้ใช้ที่แตกต่างกัน Open XML SDK เป็นไลบรารีคลาสที่ให้วิธีการทำงานกับเอกสาร OOXML แบบมีการกำหนดประเภทอย่างเข้มงวด Aspose.Slides เป็นไลบรารีการประมวลผลการนำเสนอที่มีประโยชน์อย่างยิ่งและให้การสนับสนุนที่ครอบคลุมเกือบทั้งหมดของรูปแบบไฟล์ Microsoft PowerPoint

หากสิ่งที่คุณต้องการทำคือการดำเนินการเขียนโปรแกรมพื้นฐานกับเอกสาร PPTX อย่างง่าย Open XML SDK อาจเป็นตัวเลือกที่เหมาะสม ด้วย Open XML SDK คุณจะทำงานง่าย ๆ เช่น การสร้างเอกสาร PPTX อย่างง่าย การลบคอมเมนต์, หัวกระดาษ/ท้ายกระดาษ, การสกัดรูปภาพ หรืออื่น ๆ บางงานสามารถทำได้ด้วย Open XML SDK แต่ไม่สามารถทำได้ด้วย Aspose.Slides ตัวอย่างเช่น หากคุณต้องการเข้าถึงองค์ประกอบ XML และแอตทริบิวต์ของเอกสาร OOXML โดยตรง คุณควรใช้ Open XML SDK อย่างไรก็ตาม หากคุณต้องการทำงานที่ซับซ้อนกับเอกสาร เช่น งานต่อไปนี้ การใช้ Aspose.Slides จะเป็นตัวเลือกที่ดีที่สุด:

- สนับสนุนรูปแบบ PowerPoint เก่าเพิ่มเติมนอกเหนือจาก PPTX
- คัดลอกหรือทำสำเนา Shape ภายในสไลด์โดยวิธีที่รวมวัตถุ, สไตล์และการจัดรูปแบบอื่น ๆ อย่างเหมาะสม
- แทนที่ข้อความที่มีรูปแบบหรือไม่มีรูปแบบ
- ประยุกต์ใช้งานอนิเมชันและการใช้คอนเน็กเตอร์กับ Shape
- แปลงเอกสารเป็น PDF, TIFF หรือ XPS ให้แสดงผลเทียบเท่ากับที่ Microsoft PowerPoint ทำได้
- พัฒนาแอปพลิเคชัน .NET หรือ Java ในสภาพแวดล้อมทั้งบนเดสก์ท็อปและเว็บ
