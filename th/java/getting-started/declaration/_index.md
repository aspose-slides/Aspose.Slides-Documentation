{{% alert color="primary" %}}

ส่วนประกอบ Aspose Java ทั้งหมดต้องการชุดสิทธิ์ Full Trust เหตุผลคือส่วนประกอบ Aspose Java จำเป็นต้องเข้าถึงการตั้งค่าที่รีจิสทรี ไฟล์ระบบที่อยู่นอกไดเรกทอรีเสมือนสำหรับการดำเนินการบางอย่างเช่นการวิเคราะห์แบบอักษร เป็นต้น นอกจากนี้ส่วนประกอบ Aspose Java ยังอิงจากคลาสระบบหลักของ Java ซึ่งหลายกรณีก็ต้องการชุดสิทธิ์ Full Trust ด้วย

{{% /alert %}}

ผู้ให้บริการอินเทอร์เน็ต (Internet Service Providers) ที่โฮสต์หลายแอปพลิเคชันจากบริษัทต่าง ๆ มักบังคับใช้ระดับความปลอดภัย Medium Trust:

- OleDbPermission ไม่พร้อมใช้งาน ซึ่งหมายความว่าคุณไม่สามารถใช้ ADO.NET managed OLE DB data provider เพื่อเข้าถึงฐานข้อมูลได้
- EventLogPermission ไม่พร้อมใช้งาน ซึ่งหมายความว่าคุณไม่สามารถเข้าถึง Windows event log ได้
- ReflectionPermission ไม่พร้อมใช้งาน ซึ่งหมายความว่าคุณไม่สามารถใช้ reflection ได้
- RegistryPermission ไม่พร้อมใช้งาน ซึ่งหมายความว่าคุณไม่สามารถเข้าถึงรีจิสทรีได้
- WebPermission ถูกจำกัด ซึ่งหมายความว่าแอปพลิเคชันของคุณสามารถสื่อสารได้เฉพาะกับที่อยู่หรือช่วงที่อยู่ที่คุณกำหนดในองค์ประกอบ <trust>
- FileIOPermission ถูกจำกัด ซึ่งหมายความว่าคุณสามารถเข้าถึงไฟล์ได้เฉพาะในลำดับชั้นไดเรกทอรีเสมือนของแอปพลิเคชันของคุณเท่านั้น

{{% alert color="primary" %}}

เนื่องจากเหตุผลที่ระบุข้างต้น ส่วนประกอบ Aspose Java ไม่สามารถใช้บนเซิร์ฟเวอร์ที่ให้ชุดสิทธิ์ที่ไม่ใช่ Full Trust
