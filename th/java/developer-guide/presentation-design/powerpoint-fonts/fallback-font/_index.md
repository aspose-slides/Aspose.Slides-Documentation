## **บทนำ**

ฟอนต์สำรอง (fallback fonts) จะถูกใช้เมื่อฟอนต์ที่ระบุสำหรับข้อความมีอยู่ในระบบ แต่ไม่มี glyph ที่ต้องการ ในกรณีนี้ Aspose.Slides สามารถใช้ฟอนต์สำรองที่กำหนดไว้หนึ่งตัวเพื่อแทนที่ glyph ที่หายไป.

## **ฟอนต์สำรอง**

Aspose.Slides อนุญาตให้สร้างฟอนต์สำรอง, เพิ่มลงในคอลเลกชันฟอนต์สำรอง, ตั้งค่าคอลเลกชันฟอนต์สำรองสำหรับงานนำเสนอบางรายการ, ลบฟอนต์สำรองออกจากงานนำเสนอ, ระบุกฎเพื่อใช้ฟอนต์สำรองและอื่นๆ.

เพื่อทำความคุ้นเคยกับคุณลักษณะเหล่านี้, ให้ใช้ลิงก์ต่อไปนี้:
- [สร้างฟอนต์สำรอง](/slides/th/java/create-fallback-font)
- [สร้างคอลเลกชันฟอนต์สำรอง](/slides/th/java/create-fallback-fonts-collection)
- [เรนเดอร์งานนำเสนอด้วยฟอนต์สำรอง](/slides/th/java/render-presentation-with-fallback-font)

## **คำถามที่พบบ่อย**

**ฟอนต์สำรองต่างจากการแทนที่ฟอนต์อย่างไร?**

ฟอนต์สำรองจะถูกนำไปใช้ต่ออักขระหรือช่วงของ Unicode เมื่อฟอนต์หลักไม่มี glyph เฉพาะ; จะเติมเฉพาะอักขระที่หายไปเท่านั้น. [การแทนที่](/slides/th/java/font-substitution/) จะเปลี่ยนฟอนต์ที่หายไปหรือใช้ไม่ได้สำหรับช่วงข้อความหรือรันทั้งหมดเป็นฟอนต์อื่น. สามารถใช้ร่วมกันได้ แต่ขอบเขตและตรรกะการเลือกจะแตกต่างกัน.

**การตั้งค่าฟอนต์สำรองถูกบันทึกภายในไฟล์งานนำเสนอหรือไม่?**

ไม่. การกำหนดค่าฟอนต์สำรองทำงานในช่วงการประมวลผล/เรนเดอร์ตามไลบรารีและไม่ถูกจัดเก็บเป็นส่วนหนึ่งของไฟล์ PPTX. งานนำเสนอไม่ได้บันทึกกฎฟอนต์สำรองของคุณ.

**ฟอนต์สำรองส่งผลต่อองค์ประกอบที่สร้างโดยวัตถุ PowerPoint (SmartArt, แผนภูมิ, WordArt) หรือไม่?**

ใช่. ข้อความภายในวัตถุเหล่านี้จะผ่านกระบวนการเรนเดอร์เดียวกัน ดังนั้นกฎฟอนต์สำรองเดียวกันจะถูกใช้กับมันเช่นเดียวกับข้อความทั่วไป.
