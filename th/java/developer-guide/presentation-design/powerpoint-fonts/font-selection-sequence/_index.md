## **ภาพรวม**

เมื่อโหลด การแสดงผล หรือแปลงงานนำเสนอเป็นรูปแบบอื่น Aspose.Slides จะตรวจสอบว่าฟอนต์ที่ใช้ในงานนำเสนอมีอยู่ในระบบปฏิบัติการหรือไม่ หากฟอนต์ที่จำเป็นหายไป Aspose.Slides จะเลือกฟอนต์ทดแทนที่ใกล้เคียงที่สุดกับฟอนต์ที่ PowerPoint จะใช้  
Aspose.Slides จะค้นหาฟอนต์ที่เลือกในระบบปฏิบัติการเป็นอันดับแรก หากพบฟอนต์นั้นจะใช้มัน หากไม่พบจะใช้ฟอนต์ทดแทนที่เหมาะสม เมื่อมีการกำหนดกฎการแทนที่ฟอนต์ผ่าน `FontSubstRule` กฎเหล่านั้นก็จะถูกนำมาพิจารณาเช่นกัน  
คุณยังสามารถเพิ่มฟอนต์ในระหว่างการทำงานของแอปพลิเคชัน ใช้ฟอนต์ฝังจากงานนำเสนอ หรือโหลดฟอนต์ภายนอกสำหรับเอกสารผลลัพธ์เช่นไฟล์ PDF ได้

## **การเลือกฟอนต์**

มีกฎบางอย่างที่ใช้กับฟอนต์ในงานนำเสนอเมื่อโหลด แสดงผล หรือแปลงเป็นรูปแบบอื่น ตัวอย่างเช่นเมื่อคุณพยายามแปลงงานนำเสนอ (สไลด์) เป็นรูปภาพ ฟอนต์ของงานนำเสนอจะถูกตรวจสอบเพื่อยืนยันว่าฟอนต์ที่เลือกมีอยู่ในระบบปฏิบัติการหรือไม่ หากฟอนต์ถูกยืนยันว่าไม่มีอยู่ จะทำการแทนที่ — ดู [**การแทนที่ฟอนต์**](https://docs.aspose.com/slides/th/java/font-replacement/) และ [**การทดแทนฟอนต์**](https://docs.aspose.com/slides/th/java/font-substitution/)  

นี่คือกระบวนการที่ Aspose.Slides ปฏิบัติตามเมื่อต้องจัดการกับฟอนต์:

1. Aspose.Slides ค้นหาฟอนต์ในระบบปฏิบัติการเพื่อหาฟอนต์ที่ตรงกับฟอนต์ที่เลือกในงานนำเสนอ  
2. หากพบฟอนต์ที่เลือก Aspose.Slides จะใช้ฟอนต์นั้น มิฉะนั้น Aspose.Slides จะใช้ฟอนต์ทดแทนที่ใกล้เคียงที่สุดกับฟอนต์ที่ PowerPoint จะใช้  
3. หากมีการกำหนดกฎการแทนที่ฟอนต์ผ่าน [FontSubstRule](https://reference.aspose.com/slides/th/java/com.aspose.slides/fontsubstrule/) จะนำกฎเหล่านั้นไปใช้  

Aspose.Slides อนุญาตให้คุณเพิ่มฟอนต์ในระหว่างการทำงานของแอปพลิเคชันและจากนั้นใช้ฟอนต์เหล่านั้น ดู [**ฟอนต์แบบกำหนดเอง**](https://docs.aspose.com/slides/th/java/custom-font/)  

เมื่อมีการใส่ฟอนต์เพิ่มเติมภายในงานนำเสนอ ฟอนต์เหล่านั้นจะเรียกว่า [**ฟอนต์ฝัง**](https://docs.aspose.com/slides/th/java/embedded-font/)  

Aspose.Slides อนุญาตให้คุณเพิ่มฟอนต์ที่ใช้กับเอกสารผลลัพธ์ *เท่านั้น* ตัวอย่างเช่น หากงานนำเสนอที่คุณต้องการแปลงเป็น PDF มีฟอนต์ที่ขาดหายจากระบบและฟอนต์ฝังอยู่ คุณสามารถเพิ่มหรือโหลดฟอนต์ที่จำเป็นเป็น **ฟอนต์ภายนอก**  

{{% alert title="หมายเหตุ" color="primary" %}} 
เราจะไม่จัดจำหน่ายฟอนต์ใด ๆ ทั้งแบบที่ต้องชำระค่าใช้จ่ายหรือแบบฟรี API ของเราสามารถให้คุณโหลดฟอนต์ภายนอกและฝังลงในเอกสารได้ แต่คุณต้องทำเช่นนั้นด้วยฟอนต์ตามดุลยพินิจและความรับผิดชอบของคุณ  
{{% /alert %}}

## **คำถามที่พบบ่อย**

**ฉันจะตรวจสอบได้อย่างไรว่าฟอนต์ใดบ้างที่ใช้จริงในงานนำเสนอก่อนการแปลง?**  
Aspose.Slides ให้คุณตรวจสอบฟอนต์ที่ใช้ผ่าน [ผู้จัดการฟอนต์](https://reference.aspose.com/slides/th/java/com.aspose.slides/fontsmanager/) เพื่อให้คุณตัดสินใจว่าจะ [ฝัง](/slides/th/java/embedded-font/), [แทนที่](/slides/th/java/font-replacement/), หรือเพิ่ม [แหล่งฟอนต์ภายนอก](/slides/th/java/custom-font/) การทำเช่นนี้ช่วยป้องกันการแทนที่ฟอนต์ที่ไม่ต้องการระหว่างการแสดงผลและการส่งออก  

**ฉันสามารถเพิ่มไดเรกทอรีฟอนต์เพิ่มเติมโดยไม่ต้องติดตั้งลงในระบบปฏิบัติการได้หรือไม่?**  
ได้ คุณสามารถลงทะเบียน [แหล่งฟอนต์ภายนอก](/slides/th/java/custom-font/) เช่น โฟลเดอร์หรือสตรีมในหน่วยความจำสำหรับการแสดงผลและการส่งออก การทำเช่นนี้จะลดการพึ่งพาฟอนต์ของระบบโฮสต์และทำให้การจัดวางคาดการณ์ได้  

**ฉันจะป้องกันการเปลี่ยนไปใช้ฟอนต์ที่ไม่เหมาะสมโดยไม่มีการแจ้งเตือนเมื่ออักขระหายไปอย่างไร?**  
กำหนด [การแทนที่ฟอนต์](/slides/th/java/font-replacement/) และ [กฎการ fallback ฟอนต์](/slides/th/java/fallback-font/) อย่างชัดเจนล่วงหน้า โดยการวิเคราะห์ฟอนต์ที่ใช้และตั้งค่าความสำคัญที่ควบคุมสำหรับฟอนต์ทดแทน คุณจะได้รับการจัดเรียงตัวอักษรที่สม่ำเสมอและหลีกเลี่ยงผลลัพธ์ที่ไม่คาดคิด
