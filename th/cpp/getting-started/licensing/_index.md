## **ภาพรวม**

Aspose.Slides สามารถใช้ได้ในโหมดประเมินหรือด้วยใบอนุญาตที่ถูกต้อง เวอร์ชันประเมินให้ฟังก์ชันการทำงานเดียวกับเวอร์ชันที่มีใบอนุญาต แต่จะเพิ่มลายน้ำการประเมินเมื่อเปิดหรือบันทึกงานนำเสนอและจำกัดการสกัดข้อความไว้หนึ่งสไลด์

บทความนี้อธิบายวิธีการทำงานของการให้ใบอนุญาตใน Aspose.Slides และวิธีการใช้ใบอนุญาตก่อนใช้งานไลบรารี ใบอนุญาตสามารถโหลดจากไฟล์, สตรีม หรือทรัพยากรที่ฝังอยู่โดยใช้คลาส `License` บทความยังแสดงวิธีการตรวจสอบว่าใบอนุญาตได้ถูกนำไปใช้ถูกต้องหรือไม่

## **การให้ใบอนุญาตใน Aspose.Slides**

* เวอร์ชันประเมินจะกลายเป็นเวอร์ชันที่มีใบอนุญาตหลังจากคุณซื้อใบอนุญาตและนำไปใช้โดยเพิ่มบรรทัดโค้ดไม่กี่บรรทัด
* ใบอนุญาตเป็นไฟล์ XML แบบข้อความธรรมดาซึ่งประกอบด้วยรายละเอียดต่าง ๆ เช่น ชื่อผลิตภัณฑ์ จำนวนผู้พัฒนาที่ได้รับใบอนุญาต วันที่หมดอายุการสมัครสมาชิก เป็นต้น
* ไฟล์ใบอนุญาตถูกเซ็นดิจิทัล ดังนั้นห้ามแก้ไข แม้แต่การเปลี่ยนแปลงโดยบังเอิญ เช่น การใส่บรรทัดใหม่—ก็จะทำให้ไฟล์เป็นโมฆะ
* Aspose.Slides for C++ จะมองหาไฟล์ใบอนุญาตในตำแหน่งต่อไปนี้โดยทั่วไป:
  * เส้นทางที่ระบุโดยชัดเจนในโค้ดของคุณ
  * โฟลเดอร์ที่บรรจุ DLL ของคอมโพเนนท์ (ที่รวมอยู่ใน Aspose.Slides)
  * โฟลเดอร์ที่บรรจุแอสเซมบลีที่เรียก DLL ของคอมโพเนนท์
* เพื่อหลีกเลี่ยงข้อจำกัดของเวอร์ชันประเมิน คุณต้องตั้งค่าใบอนุญาตก่อนใช้ Aspose.Slides ใบอนุญาตต้องตั้งค่าเพียงครั้งเดียวต่อแอปพลิเคชันหรือกระบวนการ

## **ตั้งค่าใบอนุญาต**

ใบอนุญาตสามารถโหลดได้จาก **ไฟล์**, **สตรีม**, หรือ **ทรัพยากรที่ฝังอยู่**.

{{% alert color="primary" %}}

Aspose.Slides มีคลาส [License](https://reference.aspose.com/slides/th/cpp/class/aspose.slides.license/) สำหรับการทำงานเกี่ยวกับใบอนุญาต

{{% /alert %}} 

{{% alert color="warning" %}}

ใบอนุญาตใหม่สามารถเปิดใช้งาน Aspose.Slides ได้เฉพาะกับเวอร์ชัน 21.4 หรือใหม่กว่า เวอร์ชันก่อนหน้านี้ใช้ระบบการให้ใบอนุญาตที่แตกต่างและจะไม่รับรู้ใบอนุญาตเหล่านี้

{{% /alert %}}

### **ไฟล์**

วิธีที่ง่ายที่สุดในการตั้งค่าใบอนุญาตคือการวางไฟล์ใบอนุญาตในโฟลเดอร์เดียวกับ DLL ของคอมโพเนนท์ (ที่รวมอยู่ใน Aspose.Slides) และระบุเพียงชื่อไฟล์โดยไม่ต้องใส่เส้นทาง

โค้ด C++ ด้านล่างแสดงวิธีการตั้งค่าไฟล์ใบอนุญาต:

```c++
#include <Util/License.h>

using namespace Aspose::Slides;

int main()
{
    auto license = MakeObject<License>();
    license->SetLicense(u"Aspose.Slides.lic");

    return 0;
}
```

{{% alert color="warning" %}} 

หากคุณวางไฟล์ใบอนุญาตในไดเรกทอรีอื่น จากนั้นเมื่อเรียกเมธอด [License::SetLicense](https://reference.aspose.com/slides/th/cpp/aspose.slides/license/setlicense/) ชื่อไฟล์ที่อยู่ท้ายเส้นทางที่ระบุอย่างชัดเจนต้องตรงกับชื่อไฟล์ใบอนุญาตของคุณอย่างเต็มที่

ตัวอย่าง หากคุณเปลี่ยนชื่อไฟล์ใบอนุญาตเป็น *Aspose.Slides.lic.xml* คุณต้องส่งเส้นทางเต็มที่ลงท้ายด้วย *Aspose.Slides.lic.xml* ไปยังเมธอด [License::SetLicense](https://reference.aspose.com/slides/th/cpp/aspose.slides/license/setlicense/) ในโค้ดของคุณ

{{% /alert %}}

### **สตรีม**

คุณสามารถโหลดใบอนุญาตจากสตรีมได้ โค้ด C++ ด้านล่างแสดงวิธีการใช้ใบอนุญาตจากสตรีม:

```c++
auto license = MakeObject<License>();

auto stream = File::OpenRead(u"Aspose.Slides.lic");

license->SetLicense(stream);
```

## **ตรวจสอบใบอนุญาต**

เพื่อตรวจสอบว่าใบอนุญาตได้ถูกตั้งค่าอย่างถูกต้องหรือไม่ คุณสามารถทำการตรวจสอบได้ โค้ด C++ ด้านล่างแสดงวิธีการตรวจสอบใบอนุญาต:

```c++
auto license = MakeObject<License>();

license->SetLicense(u"Aspose.Slides.lic");

if (license->IsLicensed())
{
    Console::WriteLine(u"License is good!");
    Console::ReadKey();
}
```

## **ความปลอดภัยของเธรด**

{{% alert title="Note" color="warning" %}} 

เมธอด [License::SetLicense](https://reference.aspose.com/slides/th/cpp/aspose.slides/license/setlicense/) ไม่ **ปลอดภัยต่อเธรด** หากคุณจำเป็นต้องเรียกเมธอดนี้จากหลายเธรดพร้อมกัน แนะนำให้ใช้ primitive การซิงโครไนซ์ (เช่น lock) เพื่อป้องกันปัญหาที่อาจเกิดขึ้น

{{% /alert %}}

## **คำถามที่พบบ่อย**

**ฉันสามารถใช้ใบอนุญาตในสภาพแวดล้อมที่ไม่มีการเชื่อมต่ออินเทอร์เน็ตโดยสมบูรณ์ (ไม่มีการเข้าถึงอินเทอร์เน็ต) ได้หรือไม่?**

​ใช่ การตรวจสอบใบอนุญาตทำงานแบบออฟไลน์โดยใช้ไฟล์ใบอนุญาต; ไม่ต้องเชื่อมต่ออินเทอร์เน็ต

**จะเกิดอะไรขึ้นเมื่อการสมัครสมาชิกหนึ่งปีหมดอายุ? ไลบรารีจะหยุดทำงานหรือไม่?**

​ไม่ ใบอนุญาตเป็นแบบถาวร: คุณสามารถใช้เวอร์ชันที่ออกก่อนวันที่หมดอายุการสมัครสมาชิกของคุณต่อไปได้; แต่คุณจะไม่สามารถใช้เวอร์ชันใหม่ที่ออกหลังจากนั้นได้หากไม่ได้ต่ออายุ
