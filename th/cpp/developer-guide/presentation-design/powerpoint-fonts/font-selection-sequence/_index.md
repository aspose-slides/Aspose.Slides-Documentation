## **ภาพรวม**

เมื่อมีการโหลด การเรนเดอร์ หรือการแปลงงานนำเสนอเป็นรูปแบบอื่น Aspose.Slides จะตรวจสอบว่าแบบอักษรที่ใช้ในงานนำเสนอมีอยู่ในระบบปฏิบัติการหรือไม่ หากแบบอักษรที่ต้องการไม่มีอยู่ Aspose.Slides จะเลือกแบบอักษรแทนที่ที่ใกล้เคียงที่สุดกับแบบอักษรที่ PowerPoint จะใช้

Aspose.Slides จะค้นหาแบบอักษรที่เลือกในระบบปฏิบัติการก่อน หากพบก็จะใช้ หากไม่พบจะใช้แบบอักษรทดแทนที่เหมาะสม เมื่อกฎการแทนที่แบบอักษรถูกกำหนดผ่าน `FontSubstRule` กฎเหล่านั้นก็จะนำมาพิจารณาด้วย

คุณยังสามารถเพิ่มแบบอักษรในช่วงเวลาการทำงานของแอปพลิเคชัน ใช้แบบอักษรฝังในงานนำเสนอ หรือนำเข้าแบบอักษรภายนอกสำหรับเอกสารผลลัพธ์เช่นไฟล์ PDF

## **การเลือกแบบอักษร**

มีกฎบางประการที่ใช้กับแบบอักษรในงานนำเสนอเมื่อทำการโหลด เรนเดอร์ หรือแปลงเป็นรูปแบบอื่น ตัวอย่างเช่น เมื่อคุณพยายามแปลงงานนำเสนอ (สไลด์) เป็นภาพ แบบอักษรของงานนำเสนอนั้นจะถูกตรวจสอบเพื่อยืนยันว่ามีแบบอักษรที่เลือกอยู่ในระบบปฏิบัติการหรือไม่ หากแบบอักษรถูกยืนยันว่าไม่มีอยู่ จะถูกแทนที่ — ดู [**การแทนที่แบบอักษร**](https://docs.aspose.com/slides/th/cpp/font-replacement/) และ [**การแทนที่แบบอักษร**](https://docs.aspose.com/slides/th/cpp/font-substitution/)

นี่คือขั้นตอนที่ Aspose.Slides ปฏิบัติตามเมื่อจัดการกับแบบอักษร:

1. Aspose.Slides ค้นหาแบบอักษรในระบบปฏิบัติการเพื่อหารูปแบบที่ตรงกับแบบอักษรที่เลือกในงานนำเสนอ  
2. หากพบแบบอักษรที่เลือก Aspose.Slides จะใช้มัน มิฉะนั้น Aspose.Slides จะใช้แบบอักษรแทนที่ที่ใกล้เคียงที่สุดกับสิ่งที่ PowerPoint จะใช้  
3. หากมีกฎการแทนที่แบบอักษรที่ตั้งค่าไว้ผ่าน [FontSubstRule](https://reference.aspose.com/slides/th/cpp/aspose.slides/fontsubstrule/) จะนำมาใช้  

Aspose.Slides อนุญาตให้คุณเพิ่มแบบอักษรในช่วงเวลาการทำงานของแอปพลิเคชันและใช้แบบอักษรเหล่านั้น ดู [**แบบอักษรกำหนดเอง**](https://docs.aspose.com/slides/th/cpp/custom-font/)  

เมื่อแบบอักษรเพิ่มเติมถูกใส่ไว้ภายในงานนำเสนอ จะเรียกว่า [**แบบอักษรฝัง**](https://docs.aspose.com/slides/th/cpp/embedded-font/)

Aspose.Slides อนุญาตให้คุณเพิ่มแบบอักษรที่ใช้ *เฉพาะ* ในเอกสารผลลัพธ์ ตัวอย่างเช่น หากงานนำเสนอที่คุณต้องการแปลงเป็น PDF มีแบบอักษรที่ไม่มีในระบบของคุณและไม่มีแบบอักษรฝัง คุณสามารถเพิ่มหรือโหลดแบบอักษรที่จำเป็นเป็น **แบบอักษรภายนอก**  

{{% alert title="หมายเหตุ" color="primary" %}} 
เราไม่ได้จัดจำหน่ายแบบอักษรใด ๆ ไม่ว่าจะเป็นแบบที่ต้องชำระเงินหรือแบบฟรี API ของเราช่วยให้คุณโหลดแบบอักษรภายนอกและฝังลงในเอกสารได้ แต่คุณต้องทำเช่นนั้นด้วยแบบอักษรที่คุณเลือกและรับผิดชอบเอง
{{% /alert %}}

## **คำถามที่พบบ่อย**

**ฉันจะกำหนดได้อย่างไรรูปแบบอักษรที่ใช้จริงในงานนำเสนอก่อนการแปลง?**

Aspose.Slides ให้คุณตรวจสอบแบบอักษรที่ใช้ผ่าน [font manager](https://reference.aspose.com/slides/th/cpp/aspose.slides/presentation/get_fontsmanager/) เพื่อให้คุณตัดสินใจว่าจะ [embed](/slides/th/cpp/embedded-font/), [replace](/slides/th/cpp/font-replacement/), หรือเพิ่ม [external sources](/slides/th/cpp/custom-font/) วิธีนี้ช่วยป้องกันการแทนที่ที่ไม่ต้องการระหว่างการเรนเดอร์และการส่งออก

**ฉันสามารถเพิ่มไดเรกทอรีแบบอักษรเพิ่มเติมโดยไม่ต้องติดตั้งบนระบบปฏิบัติการได้หรือไม่?**

ได้ คุณสามารถลงทะเบียน [external font sources](/slides/th/cpp/custom-font/) เช่น โฟลเดอร์หรือสตรีมในหน่วยความจำสำหรับการเรนเดอร์และการส่งออก วิธีนี้จะลดการพึ่งพาแบบอักษรของระบบโฮสต์และทำให้การจัดวางคงที่

**ฉันจะป้องกันการ fallback เงียบต่อแบบอักษรที่ไม่เหมาะสมเมื่อ glyph ขาดหายได้อย่างไร?**

กำหนด [font replacement](/slides/th/cpp/font-replacement/) และกฎ [fallBack](/slides/th/cpp/fallback-font/) อย่างชัดเจนล่วงหน้า โดยวิเคราะห์แบบอักษรที่ใช้และตั้งลำดับความสำคัญที่ควบคุมสำหรับตัวทดแทน คุณจะได้การพิมพ์ที่สอดคล้องและหลีกเลี่ยงผลลัพธ์ที่ไม่คาดคิด
