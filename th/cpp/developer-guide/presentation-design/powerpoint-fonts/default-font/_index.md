## **ภาพรวม**

Aspose.Slides อนุญาตให้คุณระบุแบบอักษรเริ่มต้นที่ใช้เมื่อการนำเสนอถูกแสดงผล นี่เป็นประโยชน์เมื่อสร้างภาพย่อของสไลด์หรือส่งออกการนำเสนอเป็นรูปแบบเช่น PDF และ XPS แบบอักษรเริ่มต้นสามารถกำหนดค่าได้ผ่าน `LoadOptions` ก่อนที่การนำเสนอจะถูกโหลด.

`set_DefaultRegularFont` วิธีกำหนดแบบอักษรเริ่มต้นสำหรับข้อความทั่วไป, ส่วน `set_DefaultAsianFont` กำหนดแบบอักษรเริ่มต้นสำหรับข้อความเอเชีย หลังจากตั้งค่าตัวเลือกเหล่านี้แล้ว การนำเสนอจะสามารถโหลดและแสดงผลโดยใช้แบบอักษรที่ระบุได้.

## **ใช้แบบอักษรเริ่มต้นสำหรับการแสดงผลการนำเสนอ**
Aspose.Slides ให้คุณตั้งค่าแบบอักษรเริ่มต้นสำหรับการแสดงผลการนำเสนอเป็น PDF, XPS หรือภาพย่อ บทความนี้แสดงวิธีกำหนด DefaultRegular Font และ DefaultAsian Font เพื่อใช้เป็นแบบอักษรเริ่มต้น โปรดทำตามขั้นตอนด้านล่างเพื่อโหลดแบบอักษรจากไดเรกทอรีภายนอกโดยใช้ Aspose.Slides for C++ API:

1. สร้างอินสแตนซ์ของ LoadOptions.
1. ตั้งค่า DefaultRegularFont เป็นแบบอักษรที่คุณต้องการ ในตัวอย่างต่อไปนี้ ฉันใช้ Wingdings.
1. ตั้งค่า DefaultAsianFont เป็นแบบอักษรที่คุณต้องการ ฉันใช้ Wingdings ในตัวอย่างต่อไปนี้.
1. โหลดการนำเสนอโดยใช้ Presentation พร้อมตั้งค่าตัวเลือกการโหลด.
1. ตอนนี้ สร้างภาพย่อของสไลด์, PDF และ XPS เพื่อยืนยันผลลัพธ์.

การนำไปใช้ของข้างต้นมีดังต่อไปนี้.

```cpp
// ใช้ตัวเลือกการโหลดเพื่อกำหนดแบบอักษรปกติและแบบอักษรเอเชียเริ่มต้น
auto loadOptions = MakeObject<LoadOptions>(LoadFormat::Auto);
loadOptions->set_DefaultRegularFont(u"Wingdings");
loadOptions->set_DefaultAsianFont(u"Wingdings");

auto pptx = MakeObject<Presentation>(u"DefaultFonts.pptx", loadOptions);

auto image = pptx->get_Slide(0)->GetImage(1, 1);
image->Save(u"DefaultFonts_out.png", ImageFormat::Png);
image->Dispose();

pptx->Save(u"DefaultFonts_out.pdf", SaveFormat::Pdf);
pptx->Save(u"DefaultFonts_out.xps", SaveFormat::Xps);

pptx->Dispose();
```

## **คำถามที่พบบ่อย**

**DefaultRegularFont และ DefaultAsianFont มีผลอย่างไรบ้าง—เฉพาะการส่งออกหรือรวมถึงภาพย่อ, PDF, XPS, HTML, และ SVG ด้วยหรือไม่?**

พวกเขามีส่วนร่วมในขั้นตอนการแสดงผลสำหรับผลลัพธ์ที่รองรับทั้งหมด ซึ่งรวมถึงภาพย่อของสไลด์, [PDF](/slides/th/cpp/convert-powerpoint-to-pdf/), [XPS](/slides/th/cpp/convert-powerpoint-to-xps/), [raster images](/slides/th/cpp/convert-powerpoint-to-png/), [HTML](/slides/th/cpp/convert-powerpoint-to-html/), และ [SVG](/slides/th/cpp/render-a-slide-as-an-svg-image/), เนื่องจาก Aspose.Slides ใช้ตรรกะเลย์เอาต์และการแก้ไข glyph เดียวกันในเป้าหมายเหล่านี้.

**แบบอักษรเริ่มต้นจะถูกนำไปใช้เมื่อตัวไฟล์ PPTX ถูกอ่านและบันทึกอย่างเดียวโดยไม่มีการแสดงผลหรือไม่?**

ไม่. แบบอักษรเริ่มต้นมีผลเมื่อข้อความต้องถูกวัดและวาด การเปิด‑บันทึกแบบตรงของการนำเสนอจะไม่เปลี่ยนแปลงชุดแบบอักษรที่เก็บไว้หรือโครงสร้างของไฟล์ แบบอักษรเริ่มต้นจะทำงานในระหว่างการดำเนินการที่แสดงผลหรือจัดข้อความใหม่.

**หากฉันเพิ่มโฟลเดอร์แบบอักษรของตนเองหรือให้แบบอักษรจากหน่วยความจำ จะถูกพิจารณาเมื่อเลือกแบบอักษรเริ่มต้นหรือไม่?**

ใช่. [Custom font sources](/slides/th/cpp/custom-font/) ขยายแคตาล็อกของครอบครัวและ glyph ที่มีให้เครื่องยนต์ใช้งาน แบบอักษรเริ่มต้นและ [fallback rules](/slides/th/cpp/fallback-font/) จะตรวจสอบแหล่งเหล่านั้นก่อน ทำให้ครอบคลุมได้อย่างเชื่อถือได้มากขึ้นบนเซิร์ฟเวอร์และในคอนเทนเนอร์.

**แบบอักษรเริ่มต้นจะส่งผลต่อเมตริกของข้อความ (kerning, advances) และดังนั้นการตัดบรรทัดและการห่อข้อความหรือไม่?**

ใช่. การเปลี่ยนแบบอักษรจะเปลี่ยนเมตริกของ glyph และอาจทำให้การตัดบรรทัด, การห่อข้อความ, และการจัดหน้าเปลี่ยนแปลงระหว่างการแสดงผล เพื่อความคงที่ของเลย์เอาต์ ควร [embed the original fonts](/slides/th/cpp/embedded-font/) หรือเลือกครอบครัวแบบอักษรเริ่มต้นและ fallback ที่เข้ากันทางเมตริก.

**มีเหตุผลใดในการตั้งค่าแบบอักษรเริ่มต้นหากแบบอักษรทั้งหมดที่ใช้ในการนำเสนอถูกฝังไว้แล้วหรือไม่?**

โดยส่วนใหญ่ไม่จำเป็น เนื่องจาก [embedded fonts](/slides/th/cpp/embedded-font/) รับประกันลักษณะการแสดงผลที่สอดคล้องแล้ว แบบอักษรเริ่มต้นยังคงเป็นประโยชน์เป็นเครือข่ายความปลอดภัยสำหรับอักขระที่ไม่อยู่ในส่วนย่อยที่ฝังไว้หรือเมื่อไฟล์ผสมข้อความที่ฝังและไม่ฝัง.
