## **คำนำ**

ใน PowerPoint 2019, Microsoft ได้นำเสนอการตั้งค่า **Always Open Read-Only** เป็นหนึ่งในตัวเลือกที่ผู้ใช้สามารถใช้เพื่อปกป้องการนำเสนอของตน คุณอาจต้องการใช้การตั้งค่า Read-Only นี้เพื่อปกป้องการนำเสนอเมื่อ

- คุณต้องการป้องกันการแก้ไขโดยไม่ตั้งใจและทำให้เนื้อหาของการนำเสนอของคุณปลอดภัย
- คุณต้องการแจ้งให้ผู้คนทราบว่าการนำเสนอที่คุณให้เป็นเวอร์ชันสุดท้าย

หลังจากที่คุณเลือกตัวเลือก **Always Open Read-Only** สำหรับการนำเสนอ เมื่อผู้ใช้เปิดการนำเสนอ พวกเขาจะเห็นคำแนะนำ **Read-Only** และอาจเห็นข้อความในรูปแบบนี้: *เพื่อป้องกันการเปลี่ยนแปลงโดยไม่ตั้งใจ ผู้เขียนได้ตั้งค่าไฟล์นี้ให้เปิดในโหมดอ่านอย่างเดียว*

คำแนะนำ **Read-Only** เป็นวิธีเตือนที่เรียบง่ายแต่มีประสิทธิภาพซึ่งทำให้ผู้ใช้ไม่อยากแก้ไข เนื่องจากผู้ใช้ต้องทำขั้นตอนเพื่อถอดออกก่อนจึงสามารถแก้ไขการนำเสนอได้ หากคุณไม่ต้องการให้ผู้ใช้ทำการเปลี่ยนแปลงการนำเสนอและต้องการบอกพวกเขาอย่างสุภาพ คำแนะนำ **Read-Only** อาจเป็นตัวเลือกที่ดีสำหรับคุณ

> หากการนำเสนอที่มีการป้องกัน **Read-Only** ถูกเปิดในแอปพลิเคชัน Microsoft PowerPoint รุ่นเก่า—ซึ่งไม่รองรับฟังก์ชันที่เพิ่งแนะนำ—คำแนะนำ **Read-Only** จะถูกละเลย (การนำเสนอจะเปิดตามปกติ)

## **ใช้โหมดอ่านอย่างเดียว**

Aspose.Slides for C++ ให้คุณตั้งค่าการนำเสนอให้เป็น **Read-Only** ซึ่งหมายความว่าผู้ใช้ (หลังจากที่เปิดการนำเสนอ) จะเห็นคำแนะนำ **Read-Only** ตัวอย่างโค้ดนี้แสดงวิธีตั้งค่าการนำเสนอให้เป็น **Read-Only** ใน C++ ด้วย Aspose.Slides:

``` cpp
auto pres = System::MakeObject<Presentation>();
pres->get_ProtectionManager()->set_ReadOnlyRecommended(true);
pres->Save(u"ReadOnlyPresentation.pptx", SaveFormat::Pptx);
```

{{% alert color="primary" %}} 

**หมายเหตุ**: คำแนะนำ **Read-Only** มีไว้เพื่อทำให้ผู้ใช้ไม่ต้องการแก้ไขหรือหยุดไม่ให้ทำการเปลี่ยนแปลงโดยไม่ตั้งใจในการนำเสนอ PowerPoint หากบุคคลที่มีความตั้งใจ—ซึ่งรู้ว่ากำลังทำอะไร—ตัดสินใจแก้ไขการนำเสนอของคุณ พวกเขาสามารถลบการตั้งค่า Read-Only ได้อย่างง่ายดาย หากคุณต้องการป้องกันการแก้ไขโดยไม่ได้รับอนุญาตอย่างจริงจัง คุณควรใช้ [มาตรการป้องกันที่เข้มงวดขึ้นที่เกี่ยวข้องกับการเข้ารหัสและรหัสผ่าน](https://docs.aspose.com/slides/th/cpp/password-protected-presentation/)

## **FAQ**

**'Read-Only recommended' แตกต่างจากการป้องกันด้วยรหัสผ่านเต็มอย่างไร?**

'Read-Only recommended' จะแสดงเพียงคำแนะนำให้เปิดไฟล์ในโหมดอ่านอย่างเดียวและสามารถข้ามได้ง่าย [การป้องกันด้วยรหัสผ่าน](/slides/th/cpp/password-protected-presentation/) จะจำกัดการเปิดหรือแก้ไขจริงๆ และเหมาะสมเมื่อคุณต้องการการควบคุมความปลอดภัยที่แท้จริง

**'Read-Only recommended' สามารถรวมกับลายน้ำเพื่อยับยั้งการแก้ไขเพิ่มเติมได้หรือไม่?**

ใช่ คำแนะนำสามารถจับคู่กับ [ลายน้ำ](/slides/th/cpp/watermark/) เป็นการเตือนด้วยภาพ; ทั้งสองเป็นกลไกแยกกันและทำงานร่วมกันได้ดี

**แมโครหรือเครื่องมือภายนอกยังสามารถแก้ไขไฟล์ได้หรือไม่เมื่อคำแนะนำถูกเปิดใช้งาน?**

ใช่ คำแนะนำไม่ได้บล็อกการเปลี่ยนแปลงแบบโปรแกรมเมชัน เพื่อป้องกันการแก้ไขอัตโนมัติ ใช้ [รหัสผ่านและการเข้ารหัส](/slides/th/cpp/password-protected-presentation/)

**'Read-Only recommended' เกี่ยวข้องกับธง 'is encrypted' และ 'is write protected' อย่างไร?**

พวกมันเป็นสัญญาณที่แตกต่างกัน 'Read-Only recommended' เป็นพรอมต์ที่อ่อนนุ่มและเลือกได้; [get_IsWriteProtected](https://reference.aspose.com/slides/th/cpp/aspose.slides/protectionmanager/get_iswriteprotected/) และ [get_IsEncrypted](https://reference.aspose.com/slides/th/cpp/aspose.slides/protectionmanager/get_isencrypted/) บ่งบอกข้อจำกัดการเขียนหรือการอ่านจริงที่ขึ้นอยู่กับรหัสผ่านหรือการเข้ารหัส
