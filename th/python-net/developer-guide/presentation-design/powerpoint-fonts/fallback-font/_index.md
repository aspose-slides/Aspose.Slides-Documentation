## **บทนำ**

ฟอนต์สำรองจะถูกใช้เมื่อฟอนต์ที่กำหนดสำหรับข้อความมีในระบบแต่ไม่มีอักขระที่ต้องการ ในกรณีนี้ Aspose.Slides สามารถใช้ฟอนต์สำรองที่ระบุไว้เพื่อแทนที่อักขระที่ขาดหายได้

## **ฟอนต์สำรอง**

Aspose.Slides รองรับการสร้างฟอนต์สำรอง, เพิ่มฟอนต์สำรองลงในคอลเลกชันฟอนต์สำรอง, ตั้งค่าคอลเลกชันฟอนต์สำรองสำหรับการนำเสนอเฉพาะ, ลบฟอนต์สำรองออกจากการนำเสนอ, ระบุกฎเพื่อใช้ฟอนต์สำรองและอื่น ๆ

เพื่อทำความคุ้นเคยกับคุณลักษณะเหล่านี้, โปรดใช้ลิงก์ต่อไปนี้:

- [สร้างฟอนต์สำรอง](/slides/th/python-net/create-fallback-font)
- [สร้างคอลเลกชันฟอนต์สำรอง](/slides/th/python-net/create-fallback-fonts-collection)
- [เรนเดอร์การนำเสนอด้วยฟอนต์สำรอง](/slides/th/python-net/render-presentation-with-fallback-font)

## **คำถามที่พบบ่อย**

**ฟอนต์สำรองแตกต่างจากการทดแทนฟอนต์อย่างไร?**

ฟอนต์สำรองจะถูกนำมาใช้ต่ออักขระหรือช่วง Unicode ที่ฟอนต์หลักไม่มีอักขระที่ต้องการ; จะเติมเฉพาะอักขระที่ขาดหายเท่านั้น. [การทดแทนฟอนต์](/slides/th/python-net/font-substitution/) จะเปลี่ยนฟอนต์ที่หายไปหรือไม่พร้อมใช้สำหรับช่วงข้อความทั้งหมดหรือรันหนึ่งให้เป็นฟอนต์อื่น. สามารถใช้ร่วมกันได้, แต่ขอบเขตและตรรกะการเลือกต่างกัน

**การตั้งค่าฟอนต์สำรองจะถูกบันทึกไว้ในไฟล์การนำเสนอหรือไม่?**

ไม่. การกำหนดค่าฟอนต์สำรองทำงานในช่วงการประมวลผล/เรนเดอร์ในไลบรารีและไม่ถูกซีเรียลไลซ์เข้าไปในไฟล์ PPTX. การนำเสนอไม่ได้เก็บกฎฟอนต์สำรองของคุณ

**ฟอนต์สำรองส่งผลต่อองค์ประกอบที่สร้างโดยวัตถุ PowerPoint (SmartArt, แผนภูมิ, WordArt) หรือไม่?**

ใช่. ข้อความภายในวัตถุเหล่านี้จะผ่านไปยังขั้นตอนการเรนเดอร์เดียวกัน, ดังนั้นกฎฟอนต์สำรองจะถูกนำไปใช้กับข้อความเหล่านั้นเช่นเดียวกับข้อความทั่วไป
