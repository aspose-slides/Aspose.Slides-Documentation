## **บทนำ**

Aspose.Slides for PHP via Java ไม่ต้องการผลิตภัณฑ์ของบุคคลที่สาม เช่น Microsoft PowerPoint เพื่อติดตั้ง Aspose.Slides เองเป็นเอนจิ้นสำหรับการสร้าง, แก้ไข, แปลง, และเรนเดอร์เอกสารในรูปแบบต่าง ๆ รวมถึงรูปแบบการนำเสนอของ Microsoft PowerPoint

## **ระบบปฏิบัติการที่รองรับ**

Aspose.Slides for Java รองรับระบบปฏิบัติการ 32‑bit หรือ 64‑bit ใดก็ตามที่ทำงานบน Java runtime รวมถึงแต่ไม่จำกัดเพียง:

### **Windows**
- Microsoft Windows 2003 Server ( x64, x86)
- Microsoft Windows 2008 Server ( x64, x86)
- Microsoft Windows 2012 Server ( x64, x86)
- Microsoft Windows 2012 R2 Server ( x64, x86)
- Microsoft Windows 2016 Server ( x64, x86)
- Microsoft Windows 2019 Server ( x64, x86)
- Microsoft Windows Vista ( x64, x86)
- Microsoft Windows XP ( x64, x86)
- Microsoft Windows 7 ( x64, x86)
- Microsoft Windows 8, 8.1 ( x64, x86)
- Microsoft Windows 10 ( x64, x86)

### **Linux**
- Linux (Ubuntu, OpenSUSE, CentOS and others)

### **Mac**
- Mac OS X

## **คำถามที่พบบ่อย**

**ฉันต้องติดตั้ง Microsoft PowerPoint เพื่อทำการแปลงและแสดงผลหรือไม่?**

ไม่จำเป็นต้องใช้ PowerPoint; Aspose.Slides เป็นเอนจิ้นแบบสแตนด์อโลนสำหรับ [การสร้าง](/slides/th/php-java/create-presentation/), การแก้ไข, [การแปลง](/slides/th/php-java/convert-presentation/), และ [การเรนเดอร์](/slides/th/php-java/convert-powerpoint-to-png/) งานนำเสนอ

**ฟอนต์อะไรที่ต้องการสำหรับการแสดงผลที่ถูกต้อง?**

โดยทั่วไป ฟอนต์ที่ใช้ในงานนำเสนอหรือ [ตัวทดแทน](/slides/th/php-java/font-substitution/) ที่เหมาะสมต้องมีอยู่บนระบบ เพื่อให้การเรนเดอร์สอดคล้องกันใน Linux/macOS ควรติดตั้งแพคเกจฟอนต์ที่ทั่วไป

**ทำไมฟอนต์ที่กำหนดเองจึงแสดงเป็นฟอนต์สำรองหรือข้อความหายบน Linux?**

หากไฟล์ฟอนต์มีรายการตารางชื่อที่ไม่สอดคล้องหรือเสียหาย, stack การจับคู่ฟอนต์ของ Linux (FreeType/fontconfig) อาจเลือกบันทึกที่ไม่ถูกต้องทำให้ฟอนต์ไม่ถูกแก้ไขได้ การใช้เวอร์ชันฟอนต์ที่แก้ไขรายการตารางชื่อหรือการติดตั้งการทดแทนที่สอดคล้องจะช่วยแก้ปัญหา
