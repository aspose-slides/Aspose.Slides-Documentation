## **ภาพรวม**

บทความนี้อธิบายว่าเมื่อไหร่ที่นักพัฒนาอาจเลือกใช้ Open XML SDK หรือ Aspose.Slides สำหรับทำงานกับเอกสารงานนำเสนอ โดยอธิบายว่า Open XML SDK เป็นไลบรารีสำหรับจัดการแพคเกจ OOXML และองค์ประกอบ XML ภายใน ในขณะที่ Aspose.Slides นำเสนอเป็นไลบรารีการประมวลผลงานนำเสนอด้วยโมเดลวัตถุระดับสูงและรองรับงานที่เกี่ยวข้องกับ PowerPoint มากมาย  

บทความเปรียบเทียบทั้งสองตัวเลือกตามรูปแบบที่สนับสนุน, โมเดลการเขียนโปรแกรม, ความสามารถในการเรนเดอร์และพิมพ์, การสนับสนุนแพลตฟอร์ม, และกรณีการใช้งานทั่วไป อีกทั้งยังชี้แจงว่า Open XML SDK อาจเหมาะสำหรับการดำเนินการ PPTX พื้นฐานหรือการเข้าถึงองค์ประกอบ OOXML โดยตรง ในขณะที่ Aspose.Slides เหมาะกับงานนำเสนอที่ซับซ้อนเช่นการทำงานกับรูปแบบ PowerPoint หลายรูปแบบ, การคัดลอกหรือโคลนรูปร่าง, การแทนที่ข้อความ, การใช้งานแอนิเมชัน, และการแปลงงานนำเสนอเป็น PDF, TIFF หรือ XPS  

## **Open XML SDK คืออะไร?**
ตามข้อมูลจาก [ไลบรารี MSDN](https://docs.microsoft.com/en-us/office/open-xml/open-xml-sdk) Open XML SDK ได้รับการกำหนดว่า:

Open XML SDK 2.0 ทำให้การจัดการแพคเกจ Open XML และองค์ประกอบสคีมา Open XML ภายในแพคเกจเป็นเรื่องง่ายขึ้น Open XML SDK 2.0 รวมงานทั่วไปจำนวนมากที่นักพัฒนาทำกับแพคเกจ Open XML ไว้แล้ว เพื่อให้คุณสามารถทำงานที่ซับซ้อนได้ด้วยเพียงไม่กี่บรรทัดของโค้ด  

เอกสาร OOXML โดยพื้นฐานคือไฟล์ XML ที่บีบอัดเป็น Zip และ Open XML SDK คือชุดคลาสตที่อนุญาตให้คุณทำงานกับเนื้อหาเอกสาร OOXML ในรูปแบบที่มีชนิดข้อมูลที่ชัดเจน แทนการแตกไฟล์เพื่อดึง XML, โหลด XML นั้นเข้าสู่โครงสร้าง DOM และทำงานกับองค์ประกอบและแอตทริบิวต์ของ XML โดยตรง Open XML SDK จะให้คลาสสำหรับทำสิ่งเหล่านั้นให้คุณ  

## **Aspose.Slides คืออะไร?**
Aspose.Slides เป็นไลบรารีคลาสที่ทำให้แอปพลิเคชันของคุณสามารถทำงานการประมวลผลงานนำเสนอได้ดังต่อไปนี้:

- การเขียนโปรแกรมด้วยโมเดลวัตถุ **Presentation**  
- การแปลงคุณภาพสูงระหว่างรูปแบบงานนำเสนอ PowerPoint ที่นิยมสนับสนุนทั้งหมด รวมถึงการแปลงเป็น PDF, XPS และ TIFF  
- ความสามารถในการสร้างภาพย่อของสไลด์ในรูปแบบที่รู้จักกันดีเช่น PNG, JPEG และ BMP พร้อมกับการส่งออกสไลด์เป็น SVG  
- ความสามารถในการสร้างงานนำเสนอจากศูนย์หรือโดยการรวมจากเอกสารหนึ่งหรือหลายเอกสาร  
- การรองรับการเพิ่มแอนิเมชัน, Ole Frames, ตาราง, การสร้างและจัดการแผนภูมิ  
- การควบคุมอย่างละเอียดสำหรับการจัดรูปแบบข้อความบน TextFrames, Paragraphs และ Portions  

สำหรับรายละเอียดเพิ่มเติมเกี่ยวกับคุณสมบัติที่สนับสนุน กรุณาเยี่ยมชม [คุณสมบัติของ Aspose.Slides](/slides/th/php-java/product-overview/)

## **สรุป**
{{% alert color="primary" %}} 

Open XML SDK และ Aspose.Slides ไม่แข่งขันกันโดยตรงเพราะพวกมันมุ่งตอบสนองความต้องการและกลุ่มผู้ใช้ที่แตกต่างกัน Open XML SDK เป็นไลบรารีคลาสที่ให้วิธีการทำงานแบบมีชนิดข้อมูลที่ชัดเจนกับเอกสาร OOXML ส่วน Aspose.Slides เป็นไลบรารีการประมวลผลงานนำเสนอที่มีประโยชน์อย่างมากและให้การสนับสนุนที่ครอบคลุมเกือบทุกรูปแบบไฟล์ Microsoft PowerPoint  

หากคุณเพียงต้องการทำการดำเนินการโปรแกรมพื้นฐานบนเอกสาร PPTX Open XML SDK อาจเป็นตัวเลือกที่เหมาะสม ด้วย Open XML SDK คุณจะทำงานง่ายๆ เช่น การสร้างเอกสาร PPTX ธรรมดา, การลบคอมเมนต์, ส่วนหัว/ส่วนท้าย, การดึงรูปภาพหรืออื่นๆ ได้อย่างสบายใจ งานบางอย่างทำได้ด้วย Open XML SDK แต่ทำไม่ได้ด้วย Aspose.Slides ตัวอย่างเช่น หากคุณต้องการเข้าถึงองค์ประกอบ XML และแอตทริบิวต์ของเอกสาร OOXML โดยตรง คุณควรใช้ Open XML SDK อย่างไรก็ตาม หากคุณต้องการทำงานที่ซับซ้อนบนเอกสาร เช่น งานต่อไปนี้ การใช้ Aspose.Slides จะเป็นตัวเลือกที่ดีที่สุด:

- รองรับรูปแบบ PowerPoint เก่าเพิ่มเติมนอกเหนือจาก PPTX  
- คัดลอกหรือโคลนรูปร่างภายในสไลด์ในลักษณะที่รวมวัตถุ, สไตล์และการจัดรูปแบบอื่นๆ อย่างเหมาะสม  
- แทนที่ข้อความที่มีการจัดรูปแบบหรือไม่มีการจัดรูปแบบ  
- ใช้แอนิเมชันและการเชื่อมต่อกับรูปร่างที่ใช้  
- แปลงเอกสารเป็น PDF, TIFF หรือ XPS ให้แสดงผลเหมือนที่ Microsoft PowerPoint จะทำได้  
- พัฒนาแอปพลิเคชัน .NET หรือ Java ทั้งบนเดสก์ท็อปและเว็บ
