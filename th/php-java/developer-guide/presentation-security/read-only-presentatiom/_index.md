## **บทนำ**

ใน PowerPoint 2019, Microsoft ได้นำเสนอการตั้งค่า **Always Open Read-Only** ซึ่งเป็นหนึ่งในตัวเลือกที่ผู้ใช้สามารถใช้เพื่อปกป้องการนำเสนอของตน คุณอาจต้องการใช้การตั้งค่า Read-Only นี้เพื่อปกป้องการนำเสนอเมื่อ

- คุณต้องการป้องกันการแก้ไขโดยไม่ได้ตั้งใจและรักษาเนื้อหาของการนำเสนอให้ปลอดภัย. 
- คุณต้องการแจ้งให้ผู้คนทราบว่าการนำเสนอที่คุณให้เป็นเวอร์ชันสุดท้าย. 

หลังจากคุณเลือกตัวเลือก **Always Open Read-Only** สำหรับการนำเสนอ เมื่อผู้ใช้เปิดการนำเสนอ พวกเขาจะเห็นคำแนะนำ **Read-Only** และอาจเห็นข้อความในรูปแบบนี้: *เพื่อป้องกันการเปลี่ยนแปลงโดยไม่ได้ตั้งใจ ผู้เขียนได้ตั้งค่าไฟล์นี้ให้เปิดในโหมดอ่านอย่างเดียว.*

คำแนะนำ **Read-Only** เป็นการขัดขวางที่ง่ายแต่มีประสิทธิภาพที่ทำให้ผู้ใช้ไม่แก้ไข เนื่องจากผู้ใช้ต้องดำเนินการบางอย่างเพื่อเอาออกก่อนที่พวกเขาจะสามารถแก้ไขการนำเสนอได้ หากคุณไม่ต้องการให้ผู้ใช้ทำการเปลี่ยนแปลงการนำเสนอและต้องการแจ้งให้พวกเขาทราบอย่างสุภาพ คำแนะนำ **Read-Only** อาจเป็นตัวเลือกที่ดีสำหรับคุณ. 

> หากการนำเสนอที่มีการป้องกัน **Read-Only** ถูกเปิดในแอปพลิเคชัน Microsoft PowerPoint รุ่นเก่า—ซึ่งไม่รองรับฟังก์ชันที่เพิ่งแนะนำ—คำแนะนำ **Read-Only** จะถูกละเว้น (การนำเสนอจะถูกเปิดตามปกติ).

## **ใช้โหมด Read-Only**

Aspose.Slides for PHP via Java ให้คุณตั้งค่าการนำเสนอเป็น **Read-Only** ซึ่งหมายความว่าผู้ใช้ (หลังจากเปิดการนำเสนอ) จะเห็นคำแนะนำ **Read-Only** ตัวอย่างโค้ดนี้แสดงให้คุณทราบวิธีตั้งค่าการนำเสนอเป็น **Read-Only** ด้วย Aspose.Slides:

```php
  $pres = new Presentation();
  try {
    $pres->getProtectionManager()->setReadOnlyRecommended(true);
    $pres->save("ReadOnlyPresentation.pptx", SaveFormat::Pptx);
  } finally {
    if (!java_is_null($pres)) {
      $pres->dispose();
    }
  }
```

{{% alert color="primary" %}} 

**หมายเหตุ**: คำแนะนำ **Read-Only** มีจุดประสงค์เพื่อทำให้ผู้ใช้ไม่แก้ไขหรือหยุดการเปลี่ยนแปลงโดยไม่ได้ตั้งใจในงานนำเสนอ PowerPoint หากบุคคลที่มีความตั้งใจ—ซึ่งรู้ว่าตัวเองทำอะไร—ตัดสินใจแก้ไขการนำเสนอของคุณ พวกเขาสามารถลบการตั้งค่า Read-Only ได้ง่าย หากคุณต้องการป้องกันการแก้ไขโดยไม่ได้รับอนุญาตอย่างจริงจัง คุณควรใช้ [การป้องกันที่เข้มงวดยิ่งขึ้นที่เกี่ยวข้องกับการเข้ารหัสและรหัสผ่าน](https://docs.aspose.com/slides/th/php-java/password-protected-presentation/).

## **คำถามที่พบบ่อย**

**'Read-Only recommended' แตกต่างจากการป้องกันด้วยรหัสผ่านเต็มอย่างไร?**

'Read-Only recommended' เพียงแสดงคำแนะนำให้เปิดไฟล์ในโหมดอ่านอย่างเดียวและง่ายต่อการข้ามผ่าน. [การป้องกันด้วยรหัสผ่าน](/slides/th/php-java/password-protected-presentation/) จะจำกัดการเปิดหรือแก้ไขจริงและเหมาะเมื่อคุณต้องการการควบคุมความปลอดภัยที่แท้จริง.

**สามารถใช้ 'Read-Only recommended' ร่วมกับลายน้ำเพื่อทำให้การแก้ไขยิ่งน้อยลงได้หรือไม่?**

ได้. คำแนะนำสามารถจับคู่กับ [ลายน้ำ](/slides/th/php-java/watermark/) เพื่อเป็นการขัดขวางด้วยวิธีมองเห็น; ทั้งสองเป็นกลไกแยกต่างหากและทำงานร่วมกันได้ดี.

**แมโครหรือเครื่องมือภายนอกยังสามารถแก้ไขไฟล์ได้หรือไม่เมื่อตั้งค่าคำแนะนำนี้?**

ได้. คำแนะนำไม่บล็อกการเปลี่ยนแปลงโดยโปรแกรม. เพื่อป้องกันการแก้ไขอัตโนมัติ ใช้ [รหัสผ่านและการเข้ารหัส](/slides/th/php-java/password-protected-presentation/).

**'Read-Only recommended' มีความสัมพันธ์กับเมธอด 'isEncrypted' และ 'isWriteProtected' อย่างไร?**

พวกเขาเป็นสัญญาณที่ต่างกัน. 'Read-Only recommended' เป็นการแจ้งเตือนที่อ่อนโยนและเป็นตัวเลือก; [isWriteProtected](https://reference.aspose.com/slides/th/php-java/aspose.slides/protectionmanager/iswriteprotected/) และ [isEncrypted](https://reference.aspose.com/slides/th/php-java/aspose.slides/protectionmanager/isencrypted/) แสดงข้อจำกัดการเขียนหรือการอ่านจริงที่ขึ้นกับรหัสผ่านหรือการเข้ารหัส.
