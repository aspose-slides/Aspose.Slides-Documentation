## **ภาพรวม**

เมื่อพรีเซนเทชันถูกโหลด, เรนเดอร์ หรือแปลงเป็นรูปแบบอื่น Aspose.Slides จะตรวจสอบว่าแบบอักษรที่ใช้ในพรีเซนเทชันนั้นมีอยู่ในระบบปฏิบัติการหรือไม่ หากแบบอักษรที่ต้องการขาดหายไป Aspose.Slides จะเลือกแบบอักษรแทนที่ที่ใกล้เคียงที่สุดกับแบบอักษรที่ PowerPoint จะใช้  

Aspose.Slides จะค้นหาแบบอักษรที่เลือกในระบบปฏิบัติการเป็นอันดับแรก หากพบแบบอักษรนั้น จะใช้มัน หากไม่พบ จะใช้แบบอักษรแทนที่ที่เหมาะสม เมื่อมีการกำหนดกฎการทดแทนแบบอักษรผ่าน `FontSubstRule` กฎเหล่านั้นก็จะถูกพิจารณาเช่นกัน  

คุณยังสามารถเพิ่มแบบอักษรในระหว่างการทำงานของแอปพลิเคชัน, ใช้แบบอักษรฝังจากพรีเซนเทชัน, หรือโหลดแบบอักษรภายนอกสำหรับเอกสารผลลัพธ์เช่นไฟล์ PDF  

## **การเลือกแบบอักษร**

มีกฎบางอย่างที่ใช้กับแบบอักษรในพรีเซนเทชันเมื่อพรีเซนเทชันถูกโหลด, เรนเดอร์ หรือแปลงเป็นรูปแบบอื่น ตัวอย่างเช่น เมื่อคุณพยายามแปลงพรีเซนเทชัน (สไลด์) เป็นภาพ แบบอักษรของพรีเซนเทชันจะถูกตรวจสอบเพื่อยืนยันว่าแบบอักษรที่เลือกมีอยู่ในระบบปฏิบัติการหรือไม่ หากยืนยันว่าแบบอักษรหายไป จะมีการแทนที่—ดูที่ [**Font Replacement**](https://docs.aspose.com/slides/th/php-java/font-replacement/) และ [**Font Substitution**](https://docs.aspose.com/slides/th/php-java/font-substitution/)  

นี่คือกระบวนการที่ Aspose.Slides ปฏิบัติตามเมื่อจัดการกับแบบอักษร:

1. Aspose.Slides ค้นหาแบบอักษรในระบบปฏิบัติการเพื่อหาแบบอักษรที่ตรงกับแบบอักษรที่พรีเซนเทชันเลือก.  
2. หากพบแบบอักษรที่เลือก Aspose.Slides จะใช้แบบอักษรนั้น มิฉะนั้น Aspose.Slides จะใช้แบบอักษรแทนที่ที่ใกล้เคียงที่สุดกับแบบอักษรที่ PowerPoint จะใช้.  
3. หากมีกฎการแทนที่แบบอักษรที่ตั้งค่าผ่าน [FontSubstRule](https://reference.aspose.com/slides/th/php-java/aspose.slides/fontsubstrule/) จะมีการนำไปใช้.  

Aspose.Slides อนุญาตให้คุณเพิ่มแบบอักษรเข้าไปใน runtime ของ Aspose แล้วใช้แบบอักษรเหล่านั้น ดูที่ [**Custom fonts**](https://docs.aspose.com/slides/th/php-java/custom-font/).  

เมื่อแบบอักษรเพิ่มเติมถูกวางไว้ภายในพรีเซนเทชัน พวกมันจะถูกเรียกว่า [**Embedded fonts**](https://docs.aspose.com/slides/th/php-java/embedded-font/).  

Aspose.Slides อนุญาตให้คุณเพิ่มแบบอักษรที่ใช้เฉพาะกับเอกสารผลลัพธ์เท่านั้น ตัวอย่างเช่น หากพรีเซนเทชันที่คุณต้องการแปลงเป็น PDF มีแบบอักษรที่หายจากระบบและแบบอักษรฝังอยู่ คุณสามารถเพิ่มหรือโหลดแบบอักษรที่จำเป็นเป็น **External fonts**.  

## **คำถามที่พบบ่อย**

**ฉันจะกำหนดได้อย่างไรว่าแบบอักษรใดที่ใช้จริงในพรีเซนเทชันก่อนการแปลง?**

Aspose.Slides ให้คุณตรวจสอบแบบอักษรที่ใช้ผ่าน [font manager](https://reference.aspose.com/slides/th/php-java/aspose.slides/fontsmanager/) เพื่อให้คุณสามารถตัดสินใจว่าจะ [embed](/slides/th/php-java/embedded-font/), [replace](/slides/th/php-java/font-replacement/), หรือเพิ่ม [external sources](/slides/th/php-java/custom-font/) หรือไม่ สิ่งนี้ช่วยป้องกันการแทนที่ที่ไม่ต้องการระหว่างการเรนเดอร์และการส่งออก  

**ฉันสามารถเพิ่มไดเรกทอรีแบบอักษรเพิ่มเติมโดยไม่ต้องติดตั้งในระบบปฏิบัติการได้หรือไม่?**

ได้ คุณสามารถลงทะเบียน [external font sources](/slides/th/php-java/custom-font/) เช่น โฟลเดอร์หรือสตรีมในหน่วยความจำสำหรับการเรนเดอร์และการส่งออก สิ่งนี้จะลบการพึ่งพาแบบอักษรของระบบโฮสต์และทำให้การจัดวางคาดเดาได้  

**ฉันจะป้องกันการเปลี่ยนเป็นแบบอักษรที่ไม่เหมาะสมโดยอัตโนมัติเมื่อไม่มี glyph ได้อย่างไร?**

กำหนด [font replacement](/slides/th/php-java/font-replacement/) และกฎ [fallback font](/slides/th/php-java/fallback-font/) อย่างชัดเจนล่วงหน้า โดยการวิเคราะห์แบบอักษรที่ใช้และตั้งลำดับความสำคัญที่ควบคุมได้สำหรับแบบอักษรทดแทน คุณจะทำให้การจัดแบบอักษรสม่ำเสมอและหลีกเลี่ยงผลลัพธ์ที่ไม่คาดคิด
