## **개요**

This article explains how to get support for Aspose.Slides and how to report issues effectively. It describes the main support channel, provides recommendations for submitting a problem, and points to useful technical resources such as the product page, blog, downloads, support forum, documentation, and API Reference. The article also answers common questions about paid support, issue escalation, and support for older library versions.

## **기술 지원**

{{% alert color="primary" %}} 

Aspose는 모든 제품에 대해 무제한 무료 기술 지원을 제공합니다. 평가 패키지를 사용 중인 사용자를 포함한 모든 사용자에게 지원이 제공됩니다.

{{% /alert %}} 

When you need help with Aspose.Slides for Android via Java, consider the following:

- 지원의 주요 경로는 Aspose.Forums입니다. Aspose.Slides 포럼에 질문을 게시하면 몇 시간 이내에 답변을 받을 수 있습니다. Aspose.Slides 지원 팀이 포럼에 게시된 질문에 직접 답변합니다.
- Aspose는 전화로 기술 지원을 제공하지 않음을 유념하십시오. 전화 지원은 판매 및 구매 문의에만 제공됩니다.
- 포럼에서 답변을 기대할 때는 인내심을 가지고 시차를 고려해 평가해 주시기 바랍니다.

When you experience an issue with Aspose.Slides for Android via Java, follow these guidelines to resolve it in the most efficient manner:

- 문제를 보고하기 전에 최신 Aspose.Slides for Android via Java 버전을 사용하고 있는지 확인하십시오. 경험한 문제가 이미 업데이트에서 해결되었을 수 있습니다.

  See Aspose.Slides download to get information on the latest version.

- 문제를 보고하기 전에 포럼, 이 문서, API 참조를 확인하십시오. 질문에 대한 답변이 이미 제공되었을 수 있습니다.

- 문제를 보고할 때 원본 문서와 (가능하다면) 해당 문제와 관련된 코드 조각을 포함하십시오. 여러 파일을 첨부해야 하는 경우 ZIP 패키지에 압축하십시오.

  You can safely attach your documents in Aspose.Forums because only you and Aspose developers have access to the attached files.

- 특정 문제에 대해 하나의 스레드를 생성하고 사용하십시오. 다른 문제가 발생하면 별도의 스레드를 만들어 보고해야 합니다.

## **자주 묻는 질문**

**사용 가능한 유료 지원 계층은 무엇이며 어떻게 다릅니까?**

[Developer, Business, and Enterprise](https://helpdesk.aspose.com/kb/faq/1-Introduction-to-Paid-Support-Services). 포함된 인시던트 수, 응답 우선순위, 이용 가능한 커뮤니케이션 채널에 따라 차이가 있습니다.

**버그를 직접 엔지니어링 팀에 에스컬레이트할 수 있나요?**

예. [Paid support](https://helpdesk.aspose.com/)은 문제가 더 깊은 조사가 필요할 때 개발자에게 에스컬레이션을 허용합니다.

**이전 라이브러리 버전도 지원되며, 해당 버전에 대한 수정이 제공되나요?**

이전 버전을 사용할 수 있지만, 수정 사항은 현재 릴리스에만 제공되며 레거시 버전에 대한 패치는 제공되지 않습니다.
