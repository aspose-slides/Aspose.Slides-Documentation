## **Конвертация презентации в PDF**
### **О PDF**
[Portable Document Format](https://en.wikipedia.org/wiki/PDF) — это файловый формат, созданный компанией Adobe System для обмена документами между различными организациями. Цель этого формата — обеспечить возможность отображения содержимого документов одинаково независимо от платформы, на которой они просматриваются.
### **PDF в Aspose.Slides for C++**
[PDF](https://docs.fileformat.com/pdf/) — это файловый формат, созданный компанией Adobe System для обмена документами между различными организациями. Цель этого формата — обеспечить возможность отображения визуального вида содержимого документов независимо от платформы, на которой они просматриваются.  

Любой документ презентации, который может быть загружен в Aspose.Slides for C++, может быть конвертирован в PDF. Вы можете экспортировать документы презентаций в PDF напрямую, используя только компонент Aspose.Slides for C++. Для этой цели не требуются сторонние библиотеки или компонент Aspose.PDF.

## **Конвертация презентации в XPS**
### **О XPS**
[XML Parser Specification](https://en.wikipedia.org/wiki/Open_XML_Paper_Specification) — это язык описания страниц и формат фиксированных документов, первоначально разработанный компанией Microsoft. Как и PDF, XPS — это формат фиксированного макета, предназначенный для сохранения точности документов и обеспечения независимого от устройства отображения.
### **Конвертация презентации в XPS с Aspose.Slides**
Любой документ презентации, который может быть загружен Aspose.Slides for C++, может быть конвертирован в формат XPS. Aspose.Slides for C++ использует высокоточный механизм разметки страниц и рендеринга для создания вывода в фиксированном формате XPS. Стоит отметить, что Aspose.Slides for C++ напрямую генерирует XPS без зависимости от классов Windows Presentation Foundation (WPF), упакованных в C++ Framework 3.5, что позволяет Aspose.Slides for C++ создавать документы XPS на машинах с версиями C++ Framework старше 3.5. Вы можете узнать о экспорте документов презентаций в документы XPS через Aspose.Slides for C++ в [this topic](/slides/ru/cpp/convert-powerpoint-to-xps/).

## **Конвертация презентации в TIFF**
### **О TIFF**
[Tagged Image File Format](https://en.wikipedia.org/wiki/TIFF) — известный формат для хранения нескольких изображений в одном документе, изначально созданный компанией Aldus. Этот формат широко поддерживается сканерами, факсами и другими приложениями для обработки изображений.
### **Конвертация презентации в TIFF с Aspose.Slides**
Любой документ, который может быть загружен в Aspose.Slides for C++, также может быть непосредственно конвертирован в документ TIFF с помощью Aspose.Slides for C++, без необходимости в сторонних компонентах. При желании можно задать размер изображений в результирующем документе TIFF. Информацию об экспорте документов презентаций в документы TIFF через Aspose.Slides for C++ вы найдёте в [this topic](/slides/ru/cpp/convert-powerpoint-to-tiff/).

## **FAQ**

**Можно ли включать скрытые слайды при экспорте в PDF/XPS?**

Да. Экспорт поддерживает включение скрытых слайдов через соответствующую опцию в настройках [PDF](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfoptions/set_showhiddenslides/)/[XPS](https://reference.aspose.com/slides/cpp/aspose.slides.export/xpsoptions/set_showhiddenslides/) .

**Поддерживается ли сохранение в формат PDF/A (для архивного хранения)?**

Да, уровни совместимости PDF/A [доступны](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfcompliance/) (включая A-2a/A-2b/A-2u и A-3a/A-3b) во время экспорта.

**Что происходит со шрифтами при конвертации: они встраиваются или заменяются?**

Есть гибкие варианты: вы можете [встраивать все глифы или только используемые подмножества](/slides/ru/cpp/embedded-font/), указать [резервный шрифт](/slides/ru/cpp/fallback-font/) и [управлять поведением](/slides/ru/cpp/font-substitution/) когда у шрифта нет определённых стилей.

**Как можно контролировать качество и размер получаемого PDF?**

Доступны параметры для [качества JPEG](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfoptions/set_jpegquality/), [сжатия текста](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfoptions/set_textcompression/), а также порог [достаточного разрешения](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfoptions/set_sufficientresolution/) для изображений, плюс режим, выбирающий [наилучшее сжатие для картинок](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfoptions/set_bestimagescompressionratio/) .

**Можно ли экспортировать только диапазон слайдов (например, 5–12)?**

Да, экспорт поддерживает выбор подмножества слайдов.

**Поддерживается ли многопоточная обработка нескольких файлов одновременно?**

Допустимо обрабатывать разные презентации параллельно в отдельных процессах. Важно: один и тот же объект [presentation](https://reference.aspose.com/slides/cpp/aspose.slides/presentation/) не должен быть загружен или сохранён из [multiple threads](/slides/ru/cpp/multithreading/) .

**Есть ли риски при применении лицензии из разных потоков?**

Да, вызовы [license-setting](/slides/ru/cpp/licensing/) не являются потокобезопасными и требуют синхронизации.
