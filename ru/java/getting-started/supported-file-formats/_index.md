## **Часто задаваемые вопросы**

**Можно ли сохранять презентации в PDF, соответствующем архивным и доступностным стандартам (PDF/A и PDF/UA)?**

Yes. Aspose.Slides поддерживает экспорт в PDF с уровнями соответствия, такими как PDF/A-2a, PDF/A-2b, PDF/A-2u, PDF/A-3a, PDF/A-3b, а также PDF/UA через настройку [compliance](https://reference.aspose.com/slides/java/com.aspose.slides/pdfoptions/#setCompliance-int-) в [PDF export options](https://reference.aspose.com/slides/java/com.aspose.slides/pdfoptions/).

**Поддерживает ли библиотека встраивание шрифтов при экспорте в PDF с детальным контролем того, какие шрифты встраиваются?**

Yes. Вы можете контролировать, будут ли шрифты полностью встроены или включены только используемые глифы, задавать правила обработки системных шрифтов и настраивать поведение для ASCII‑текста через [PDF export options](https://reference.aspose.com/slides/java/com.aspose.slides/pdfoptions/).

**Можно ли определить, защищён ли файл паролем, прежде чем полностью загружать его?**

Yes. С помощью [factory-based inspection API](https://reference.aspose.com/slides/java/com.aspose.slides/presentationfactory/) можно запросить информацию о презентации и определить, защищён ли она паролем, не открывая файл полностью.

**Существуют ли механизмы резервного шрифта и поддержка пользовательских шрифтов?**

Yes. Библиотека поддерживает [loading](/slides/ru/java/custom-font/) и [embedding](/slides/ru/java/embedded-font/) пользовательских шрифтов и предоставляет правила [fallback](/slides/ru/java/fallback-font/) для предотвращения отсутствия глифов при рендеринге и конвертации.

**Можно ли экспортировать слайды в XPS и есть ли параметры для настройки вывода XPS?**

Yes. [Export to XPS](/slides/ru/java/convert-powerpoint-to-xps/) поддерживается, и вы можете корректировать соответствующие [save options](https://reference.aspose.com/slides/java/com.aspose.slides/xpsoptions/) для управления качеством и содержимым XPS‑документа.
