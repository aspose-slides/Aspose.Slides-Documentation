## **Применить режим только чтения**

В PowerPoint 2019 Microsoft представила настройку **Always Open Read-Only** как одну из опций, которые пользователи могут использовать для защиты своих презентаций. Вам может понадобиться использовать эту настройку только чтения, чтобы защитить презентацию, когда:

- Вы хотите предотвратить случайные изменения и сохранить содержание презентации в безопасности. 
- Вы хотите предупредить людей, что предоставленная вами презентация является финальной версией. 

После того как вы выберете опцию **Always Open Read-Only** для презентации, при её открытии пользователи увидят рекомендацию **Read-Only** и могут увидеть сообщение в следующем виде: *Чтобы избежать случайных изменений, автор установил открытие файла только для чтения.*

Рекомендация **Read-Only** — это простой, но эффективный способ отпугнуть от редактирования, поскольку пользователям необходимо выполнить действие, чтобы снять её, прежде чем им будет разрешено редактировать презентацию. Если вы не хотите, чтобы пользователи вносили изменения в презентацию, и хотите вежливо об этом сообщить, рекомендация **Read-Only** может стать хорошим вариантом.

> Если презентация с защитой **Read-Only** открывается в более старой версии Microsoft PowerPoint, которая не поддерживает недавно введённую функцию, рекомендация **Read-Only** игнорируется (презентация открывается обычным способом).

Aspose.Slides for Node.js via Java позволяет установить презентацию в режим **Read-Only**, что означает, что пользователи (после открытия презентации) увидят рекомендацию **Read-Only**. Этот пример кода показывает, как установить презентацию в режим **Read-Only** на JavaScript с помощью Aspose.Slides:
```javascript
var pres = new aspose.slides.Presentation();
try {
    pres.getProtectionManager().setReadOnlyRecommended(true);
    pres.save("ReadOnlyPresentation.pptx", aspose.slides.SaveFormat.Pptx);
} finally {
    if (pres != null) {
        pres.dispose();
    }
}
```

**Примечание**: рекомендация **Read-Only** предназначена лишь для того, чтобы отговорить от редактирования или предотвратить случайные изменения PowerPoint‑презентации. Если мотивированный пользователь, который знает, что делает, решит отредактировать вашу презентацию, он сможет легко отключить настройку только чтения. Если вам действительно нужно предотвратить несанкционированное редактирование, лучше использовать [более строгие защиты, включающие шифрование и пароли](https://docs.aspose.com/slides/nodejs-java/password-protected-presentation/).

## **Часто задаваемые вопросы**

**Чем «Read-Only recommended» отличается от полной защиты паролем?**

«Read-Only recommended» только отображает предложение открыть файл в режиме только чтения и легко обходится. [Защита паролем](/slides/ru/nodejs-java/password-protected-presentation/) действительно ограничивает открытие или редактирование и подходит, когда нужны настоящие меры безопасности.

**Можно ли совместить «Read-Only recommended» с водяными знаками для дополнительного отпугивания редактирования?**

Да. Рекомендацию можно сочетать с [водяными знаками](/slides/ru/nodejs-java/watermark/) как визуальным средством отвращения; они работают независимо и хорошо дополняют друг друга.

**Может ли макрос или внешнее приложение всё‑равно изменить файл, когда включена рекомендация?**

Да. Рекомендация не блокирует программные изменения. Чтобы предотвратить автоматическое редактирование, используйте [пароли и шифрование](/slides/ru/nodejs-java/password-protected-presentation/).

**Как «Read-Only recommended» соотносится с флагами «IsEncrypted» и «IsWriteProtected»?**

Это разные сигналы. «Read-Only recommended» — мягкий, необязательный запрос; [isWriteProtected](https://reference.aspose.com/slides/nodejs-java/aspose.slides/protectionmanager/iswriteprotected/) и [isEncrypted](https://reference.aspose.com/slides/nodejs-java/aspose.slides/protectionmanager/isencrypted/) указывают на реальные ограничения записи или чтения, зависящие от паролей или шифрования.
