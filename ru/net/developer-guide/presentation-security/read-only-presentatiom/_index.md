## **Применить режим только для чтения**

В PowerPoint 2019 Microsoft представила настройку **Always Open Read-Only** как одну из возможностей защиты презентаций. Вы можете воспользоваться этой настройкой только для чтения, чтобы защитить презентацию, когда

- Вы хотите предотвратить случайные изменения и сохранить содержимое презентации в безопасности. 
- Вы хотите указать, что предоставленная вами презентация является финальной версией. 

После того как вы выберете опцию **Always Open Read-Only** для презентации, при её открытии пользователи видят рекомендацию **Read-Only** и могут увидеть сообщение в следующем виде: *To prevent accidental changes, the author has set this file to open as read-only.*

Рекомендация **Read-Only** представляет собой простое, но эффективное средство, отталкивающее от редактирования, поскольку пользователи должны выполнить действие, чтобы снять её, прежде чем им будет разрешено редактировать презентацию. Если вы не хотите, чтобы пользователи вносили изменения в презентацию, и желаете сообщить об этом вежливым способом, рекомендация **Read-Only** может быть хорошим вариантом.

> Если презентация с защитой **Read-Only** открывается в более старой версии Microsoft PowerPoint, которая не поддерживает недавно введённую функцию, рекомендация **Read-Only** игнорируется (презентация открывается обычным образом).

Aspose.Slides for .NET позволяет установить презентацию в режим **Read-Only**, что означает, что пользователи (после открытия презентации) видят рекомендацию **Read-Only**. Этот пример кода показывает, как установить презентацию в режим **Read-Only** на C# с помощью Aspose.Slides:
```c#
using (Presentation pres = new Presentation())
{
    pres.ProtectionManager.ReadOnlyRecommended = true;
    pres.Save("ReadOnlyPresentation.pptx", SaveFormat.Pptx);
}
```


{{% alert color="primary" %}} 

**Примечание**: Рекомендация **Read-Only** предназначена лишь для того, чтобы отпугнуть от редактирования или предотвратить случайные изменения PowerPoint‑презентации. Если мотивированный пользователь, знающий, что делает, решит отредактировать вашу презентацию, он легко может удалить настройку только для чтения. Если вам действительно нужно предотвратить несанкционированное редактирование, лучше использовать [более строгие защиты, включающие шифрование и пароли](https://docs.aspose.com/slides/net/password-protected-presentation/).

## **FAQ**

**Чем отличается «Read-Only recommended» от полной защиты паролем?**

«Read-Only recommended» лишь отображает предложение открыть файл в режиме только для чтения и легко обходится. [Password protection](/slides/ru/net/password-protected-presentation/) действительно ограничивает открытие или редактирование и подходит, когда необходимы реальные средства безопасности.

**Можно ли комбинировать «Read-Only recommended» с водяными знаками для дополнительного отпугивания правок?**

Да. Рекомендацию можно сочетать с [watermarks](/slides/ru/net/watermark/) как визуальным средством отталкивания; они являются отдельными механизмами и хорошо работают вместе.

**Может ли макрос или внешнее приложение всё ещё изменить файл, когда включена рекомендация?**

Да. Рекомендация не блокирует программные изменения. Чтобы предотвратить автоматические правки, используйте [passwords and encryption](/slides/ru/net/password-protected-presentation/).

**Как «Read-Only recommended» соотносится с флагами «IsEncrypted» и «IsWriteProtected»?**

Это разные сигналы. «Read-Only recommended» — мягкое, необязательное предложение; [IsWriteProtected](https://reference.aspose.com/slides/net/aspose.slides/protectionmanager/iswriteprotected/) и [IsEncrypted](https://reference.aspose.com/slides/net/aspose.slides/protectionmanager/isencrypted/) указывают на реальные ограничения записи или чтения, зависящие от паролей или шифрования.
