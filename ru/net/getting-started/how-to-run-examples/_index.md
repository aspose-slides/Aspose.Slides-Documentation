## **Требования к программному обеспечению**
Прежде чем скачивать и запускать примеры, пожалуйста, проверьте и убедитесь, что ваша среда соответствует этим требованиям: 

- Visual Studio 2010 или новее.
- NuGet Package Manager, установленный в Visual Studio. Убедитесь, что в Visual Studio установлена последняя версия API NuGet. 

Для получения инструкций по установке менеджера пакетов NuGet перейдите на страницу: https://docs.microsoft.com/en-us/nuget/install-nuget-client-tools

1. Перейдите в **Tools** > **Options** > **NuGet Package Manager**.

1. Разверните **NuGet Package Manager** (двойным щелчком) и затем выберите **Package Sources**. 

1. Убедитесь, что выбран параметр nuget.org. 

   Проект примера использует функцию автоматического восстановления пакетов NuGet, поэтому вам необходимо активное подключение к интернету. 

   Если у вас нет активного подключения к интернету на машине, где вы планируете выполнять примеры, проверьте [Installation](https://docs.aspose.com/slides/net/installation/) и (вручную) добавьте ссылку на Aspose.Slides.dll в проект примера.
## **Скачать Aspose.Slides с GitHub**
Все примеры Aspose.Slides для .NET размещены на [GitHub](https://github.com/aspose-slides/Aspose.Slides-for-.NET).

Вы можете либо клонировать репозиторий с помощью вашего любимого клиента GitHub, либо скачать ZIP‑файл [здесь](https://github.com/aspose-slides/Aspose.Slides-for-.NET/archive/master.zip).

1. Если вы скачали ZIP‑файл, вам необходимо извлечь его содержимое в папку на вашем компьютере. 

Все примеры находятся в папке **Examples**.

Существует файл решения C# для Visual Studio. Проекты созданы в Visual Studio 2013, но файлы решения совместимы с Visual Studio 2010 SP1 и новее.

2. Откройте файл решения в Visual Studio и соберите проект.

   При первом запуске зависимости автоматически загружаются через NuGet.

Папка **Data** в корневой директории **Examples** содержит входные файлы, используемые в C#‑примерах. Вам нужно скачать папку **Data** вместе с проектом примеров.

3. Откройте файл RunExamples.cs. Все примеры вызываются отсюда.

4. Раскомментируйте примеры, которые вы хотите запустить в проекте.

Если у вас возникнут проблемы с настройкой или запуском примеров, обращайтесь к нам через наши форумы.
## **Внести вклад**
Вы можете внести свой вклад в проект, добавив или улучшив пример. Все примеры и демонстрационные проекты в репозитории являются открытым исходным кодом, поэтому вы (и другие пользователи) можете свободно использовать их в приложениях.

Чтобы внести вклад, вы можете создать форк репозитория, отредактировать исходный код и создать pull request. Мы рассмотрим изменения. Если они окажутся полезными, мы добавим их в репозиторий.
