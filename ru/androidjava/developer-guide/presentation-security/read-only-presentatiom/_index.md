## **Применить режим только для чтения**

В PowerPoint 2019 компания Microsoft представила параметр **Always Open Read-Only**, как одну из возможностей, которые пользователи могут использовать для защиты своих презентаций. Вы можете захотеть использовать эту настройку «Только для чтения», чтобы защитить презентацию, когда

- Вы хотите предотвратить случайные изменения и сохранить содержание вашей презентации в безопасности.  
- Вы хотите сообщить людям, что предоставленная вами презентация является окончательной версией.  

После того как вы выберете параметр **Always Open Read-Only** для презентации, при открытии пользователями они увидят рекомендацию **Read-Only** и могут увидеть сообщение следующего вида: *Чтобы предотвратить случайные изменения, автор установил открытие этого файла только для чтения.*

Рекомендация «Только для чтения» — простое, но эффективное средство, которое препятствует редактированию, так как пользователям необходимо выполнить действие, чтобы удалить её, прежде чем им будет разрешено редактировать презентацию. Если вы не хотите, чтобы пользователи вносили изменения в презентацию, и хотите вежливо об этом сообщить, то рекомендация «Только для чтения» может быть хорошим вариантом для вас. 

> Если презентация с защитой **Read-Only** открывается в более старой версии Microsoft PowerPoint, которая не поддерживает недавно введённую функцию, рекомендация **Read-Only** игнорируется (презентация открывается обычным способом).

Aspose.Slides для Android через Java позволяет установить презентацию в режим **Read-Only**, что означает, что пользователи (после открытия презентации) видят рекомендацию **Read-Only**. В этом примере кода показано, как установить презентацию в режим **Read-Only** на Java с использованием Aspose.Slides:
```java
Presentation pres = new Presentation();
try {
    pres.getProtectionManager().setReadOnlyRecommended(true);
    pres.save("ReadOnlyPresentation.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```


{{% alert color="primary" %}} 

**Примечание**: Рекомендация **Read-Only** предназначена просто для того, чтобы отговорить от редактирования или предотвратить случайные изменения PowerPoint‑презентации. Если мотивированный пользователь — знающий, что делает — решит отредактировать вашу презентацию, он легко может снять настройку Read-Only. Если вам действительно нужно предотвратить неавторизованное редактирование, лучше использовать [более строгие защиты, включающие шифрование и пароли](https://docs.aspose.com/slides/androidjava/password-protected-presentation/).

## **FAQ**

**Чем «Read-Only recommended» отличается от полной защиты паролем?**

«Read-Only recommended» только отображает предложение открыть файл в режиме только для чтения и легко обходится. [Password protection](/slides/ru/androidjava/password-protected-presentation/) действительно ограничивает открытие или редактирование и подходит, когда нужны реальные средства безопасности.

**Можно ли комбинировать «Read-Only recommended» с водяными знаками для дополнительного сдерживания правок?**

Да. Рекомендацию можно сочетать с [watermarks](/slides/ru/androidjava/watermark/) в качестве визуального сдерживающего фактора; они являются отдельными механизмами и хорошо работают вместе.

**Может ли макрос или внешнее средство всё же изменить файл, когда рекомендация включена?**

Да. Рекомендация не блокирует программные изменения. Чтобы предотвратить автоматизированные правки, используйте [пароли и шифрование](/slides/ru/androidjava/password-protected-presentation/).

**Как «Read-Only recommended» соотносится с методами «isEncrypted» и «isWriteProtected»?**

Они являются разными сигналами. «Read-Only recommended» — мягкое, необязательное предложение; [isWriteProtected](https://reference.aspose.com/slides/androidjava/com.aspose.slides/protectionmanager/#isWriteProtected--) и [isEncrypted](https://reference.aspose.com/slides/androidjava/com.aspose.slides/protectionmanager/#isEncrypted--) указывают на реальные ограничения записи или чтения, зависящие от паролей или шифрования.
