## **Sicurezza in Aspose.Slides**

Aspose applica le migliori pratiche nello sviluppo dei propri prodotti. 

* Aspose.Slides per .NET viene utilizzato per manipolare e trasformare presentazioni in file di altri formati. Non esegue script nelle presentazioni. Aspose.Slides analizza semplicemente la struttura della presentazione e consente al codice degli utenti finali di manipolare il modello di oggetti in modo pratico. 
* Aspose.Slides funziona come una libreria nel contesto di analisi e interpretazione dei documenti senza l'esecuzione di codice remoto. Tutti i prodotti Aspose vengono eseguiti sui vostri computer. Non trasmettono alcun dato ad Aspose. L'unica eccezione è quando è coinvolta una [metered license](https://purchase.aspose.com/faqs/licensing/metered), e anche in quel caso (se si utilizza una licenza a consumo), vengono elaborati solo le informazioni sull'utilizzo dell'API.

* I componenti Aspose vengono eseguiti nello stesso contesto utente delle applicazioni tradizionali. Pertanto, i componenti Aspose non rappresentano un rischio per le risorse vitali del sistema. Inoltre, quando un documento viene aperto da un componente Aspose, le macro non vengono eseguite automaticamente. 

* I rischi intrinseci o associati al pacchetto Microsoft Office non si applicano ai componenti Aspose, quindi i prodotti Aspose sono molto sicuri.

## **FAQ**

**Quali sistemi vengono utilizzati per monitorare le vulnerabilità nel codice Aspose?**

Eseguiamo un'analisi statica del codice per ogni rilascio di Aspose.Slides. Possiamo fornire report di sicurezza che dimostrano che il codice di Aspose.Slides supera la top 10 di OWASP. 

**Quali sistemi utilizzate per monitorare le vulnerabilità nei pacchetti esterni?**

Non utilizziamo sistemi per verificare le vulnerabilità nei pacchetti esterni perché Aspose.Slides non contiene né utilizza pacchetti esterni. Aspose.Slides dipende SOLO dal framework .NET stesso.
