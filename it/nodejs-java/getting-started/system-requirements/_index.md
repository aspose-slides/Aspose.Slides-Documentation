## **Introduzione**

Aspose.Slides per Node.js tramite Java non richiede alcun prodotto di terze parti, come Microsoft PowerPoint, installato. Aspose.Slides stesso è un motore per la creazione, la modifica, la conversione e il rendering di documenti in vari formati, inclusi i formati di presentazione di Microsoft PowerPoint.

## **Sistemi Operativi Supportati**

Aspose.Slides per Node.js tramite Java supporta qualsiasi sistema operativo a 32-bit o 64-bit che esegue il runtime Java, inclusi ma non limitati a:

## **FAQ**

**Devo avere Microsoft PowerPoint installato per le conversioni e il rendering?**

No, PowerPoint non è necessario; Aspose.Slides è un motore autonomo per [creare](/slides/it/nodejs-java/create-presentation/), modificare, [convertire](/slides/it/nodejs-java/convert-presentation/), e [renderizzare](/slides/it/nodejs-java/convert-powerpoint-to-png/) le presentazioni.

**Quali font sono necessari per un rendering corretto?**

In pratica, i font utilizzati nella presentazione o i [sostituti](/slides/it/nodejs-java/font-substitution/) appropriati devono essere disponibili. Per garantire un rendering coerente su Linux/macOS, è consigliabile installare pacchetti di font comuni.

**Perché un font personalizzato viene visualizzato come fallback o testo mancante su Linux?**

Se il file del font contiene voci della tabella dei nomi incoerenti o corrotte, lo stack di corrispondenza dei font di Linux (FreeType/fontconfig) può selezionare un record non valido, causando la mancata risoluzione del font. L'utilizzo di una versione del font con le voci della tabella dei nomi corrette o l'installazione di una sostituzione coerente risolve il problema.
