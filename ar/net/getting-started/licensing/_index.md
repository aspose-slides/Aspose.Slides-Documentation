## **الترخيص في Aspose.Slides**
* تتحول نسخة التقييم إلى مرخصة بعد شراء ترخيص وإضافة بضع أسطر من الشفرة لتطبيق الترخيص.
* الترخيص هو ملف XML نصي عادي يحتوي على تفاصيل مثل اسم المنتج، عدد المطورين المرخص لهم، تاريخ انتهاء الاشتراك، وما إلى ذلك. 
* ملف الترخيص موقع رقمياً، لذا لا يجوز تعديل الملف. حتى إضافة سطر فارغ غير مقصودة إلى محتوى الملف سيجعل الترخيص غير صالح.
* عادةً ما يبحث Aspose.Slides for .NET عن الترخيص في المواقع التالية:
  * مسار صريح
  * المجلد الذي يحتوي على ملف DLL للمكون (متضمن في Aspose.Slides)
  * المجلد الذي يحتوي على التجميع الذي استدعى ملف DLL للمكون (متضمن في Aspose.Slides)
  * المجلد الذي يحتوي على التجميع الرئيسي (ملف .exe الخاص بك)
  * مورد مدمج في التجميع الذي استدعى ملف DLL للمكون (متضمن في Aspose.Slides).
* لتجنب القيود المرتبطة بنسخة التقييم، يجب تعيين الترخيص قبل استخدام Aspose.Slides. يكفي تعيين الترخيص مرة واحدة لكل تطبيق أو عملية.

{{% alert color="primary" %}} 

قد ترغب في الاطلاع على [الترخيص القائم على الاستهلاك](https://docs.aspose.com/slides/net/metered-licensing/).

{{% /alert %}} 


## **تطبيق الترخيص**
يمكن تحميل الترخيص من **ملف**، **تيار**، أو **مورد مضمن**. 

{{% alert color="primary" %}}

توفر Aspose.Slides فئة [License](https://reference.aspose.com/slides/net/aspose.slides/license) لعمليات الترخيص.

{{% /alert %}} 

{{% alert color="warning" %}} 

يمكن للتراخيص الجديدة تفعيل Aspose.Slides فقط مع الإصدار 21.4 أو لاحق. الإصدارات السابقة تستخدم نظام ترخيص مختلف ولن تتعرف على هذه التراخيص.

{{% /alert %}}

### **ملف**
أسهل طريقة لتعيين الترخيص هي وضع ملف الترخيص في نفس المجلد الذي يحتوي على ملف DLL للمكوّن (متضمن في Aspose.Slides) وتحديد اسم الملف فقط دون مساره.

يعرض هذا الكود C# كيفية تعيين ملف ترخيص:
``` csharp
// إنشاء كائن الفئة License
Aspose.Slides.License license = new Aspose.Slides.License();

// ضبط مسار ملف الترخيص
license.SetLicense("Aspose.Slides.lic");
```


{{% alert color="warning" %}} 

إذا وضعت ملف الترخيص في دليل مختلف، عند استدعاء طريقة [SetLicense](https://reference.aspose.com/slides/net/aspose.slides/license/setlicense/#setlicense_1)، يجب أن يكون اسم ملف الترخيص في النهاية صريحًا مطابقًا لاسم ملف الترخيص لديك.

على سبيل المثال، يمكنك تغيير اسم ملف الترخيص إلى *Aspose.Slides.lic.xml*. ثم، في الشفرة الخاصة بك، يتوجب تمرير المسار إلى الملف (المنتهي بـ *Aspose.Slides.lic.xml*) إلى طريقة [SetLicense](https://reference.aspose.com/slides/net/aspose.slides/license/setlicense/#setlicense_1).

{{% /alert %}}

### **تيار**
يمكنك تحميل الترخيص من تيار. يوضح هذا الكود C# كيفية تطبيق الترخيص من تيار:
``` csharp
// إنشاء كائن الفئة License 
Aspose.Slides.License license = new Aspose.Slides.License();

// تعيين الترخيص عبر تيار
license.SetLicense(myStream);
```


### **مورد مضمن**
يمكنك حزم الترخيص مع تطبيقك (لتجنب فقدانه) بإضافة الترخيص كمورد مدمج في أحد التجميعات التي تستدعي ملف DLL للمكوّن (متضمن في Aspose.Slides). 

إليك طريقة إضافة ملف الترخيص كمورد مدمج:

1. في Visual Studio، أضف ملف الترخيص (.lic) إلى المشروع بهذه الطريقة: عبر **File** > **Add Existing Item** > **Add**. 
2. اختر الملف في **Solution Explorer**.
3. في نافذة **Properties**، عيّن **Build Action** إلى **Embedded Resource**.
4. للوصول إلى الترخيص المدمج في التجميع، أضف ملف الترخيص كمورد مدمج إلى المشروع، ثم مرّر اسم ملف الترخيص إلى طريقة `SetLicense`. 

تبحث فئة `License` تلقائيًا عن ملف الترخيص في الموارد المدمجة. لا تحتاج إلى استدعاء طريقتي `GetExecutingAssembly` و `GetManifestResourceStream` من فئة `System.Reflection.Assembly` في إطار عمل Microsoft .NET.

يعرض هذا الكود C# كيفية تعيين الترخيص كمورد مدمج:
``` csharp
// إنشاء كائن الفئة License
Aspose.Slides.License license = new Aspose.Slides.License();

// تمرير اسم ملف الترخيص المدمج في التجميع
license.SetLicense("Aspose.Slides.lic");
```


## **التحقق من صحة الترخيص**

للتحقق مما إذا تم تعيين الترخيص بشكل صحيح، يمكنك التحقق منه. يوضح هذا الكود C# كيفية التحقق من الترخيص:
```c#
Aspose.Slides.License license = new Aspose.Slides.License();

license.SetLicense("Aspose.Slides.lic");

if (license.IsLicensed())
{
    Console.WriteLine("License is good!");
    Console.Read();
}
```


## **سلامة الخيوط**

{{% alert title="Note" color="warning" %}} 

طريقة [license.SetLicense](https://reference.aspose.com/slides/net/aspose.slides/license/setlicense/) غير آمنة في بيئة متعددة الخيوط. إذا كان لا بد من استدعاء هذه الطريقة من عدة خيوط في آنٍ واحد، قد ترغب في استخدام آليات التزامن (مثل القفل) لتجنب المشكلات. 

{{% /alert %}}

## **الأسئلة الشائعة**

**هل يمكنني تطبيق الترخيص في بيئة غير متصلة بالإنترنت تمامًا (بدون اتصال بالإنترنت)؟**

نعم. يتم التحقق من صحة الترخيص محليًا باستخدام ملف الترخيص؛ لا يلزم الاتصال بالإنترنت.

**ماذا يحدث بعد انتهاء الاشتراك السنوي؟ هل سيتوقف المكتبة عن العمل؟**

لا. الترخيص دائم: يمكنك الاستمرار في استخدام الإصدارات الصادرة قبل تاريخ انتهاء اشتراكك؛ فقط لن تكون مؤهلاً لاستخدام الإصدارات الأحدث دون التجديد.
