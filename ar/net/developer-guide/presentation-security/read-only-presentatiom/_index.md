في PowerPoint 2019، قدمت Microsoft إعداد **فتح دائمًا للقراءة فقط** كأحد الخيارات التي يمكن للمستخدمين استخدامها لحماية عروضهم التقديمية. قد ترغب في استخدام هذا الإعداد للقراءة فقط لحماية عرض تقديمي عندما

- تريد منع التعديلات العارضة والحفاظ على محتوى عرضك التقديمي بأمان.
- تريد تنبيه الأشخاص بأن العرض الذي قدمته هو النسخة النهائية.

بعد اختيارك خيار **فتح دائمًا للقراءة فقط** لعرض تقديمي، عندما يفتح المستخدمون العرض، يرون توصية **للقراءة فقط** وقد يرون رسالة بهذا الشكل: *لمنع التغييرات العارضة، قام المؤلف بتعيين هذا الملف ليفتح كعرض للقراءة فقط.*

تعتبر توصية القراءة فقط رادعًا بسيطًا ولكنه فعال، إذ discourages editing لأن المستخدمين يجب عليهم إتمام مهمة لإزالتها قبل السماح لهم بتحرير عرض تقديمي. إذا كنت لا تريد من المستخدمين إجراء تغييرات على عرض تقديمي وتريد إخبارهم بذلك بطريقة مهذبة، فإن توصية القراءة فقط قد تكون خيارًا جيدًا بالنسبة لك.

> إذا تم فتح عرض تقديمي محمي بـ **القراءة فقط** في إصدار أقدم من تطبيق Microsoft PowerPoint — الذي لا يدعم الوظيفة التي تم تقديمها مؤخرًا — يتم تجاهل توصية **القراءة فقط** (يتم فتح العرض بشكل طبيعي).

تتيح لك Aspose.Slides لـ .NET تعيين عرض تقديمي لـ **القراءة فقط**، مما يعني أن المستخدمين (بعد فتح العرض) يرون توصية **للقراءة فقط**. يُظهر لك هذا الكود النموذجي كيفية تعيين عرض تقديمي لـ **القراءة فقط** في C# باستخدام Aspose.Slides:

```c#
using (Presentation pres = new Presentation())
{
    pres.ProtectionManager.ReadOnlyRecommended = true;
    pres.Save("ReadOnlyPresentation.pptx", SaveFormat.Pptx);
}
```

{{% alert color="primary" %}}

**ملاحظة**: إن توصية **القراءة فقط** تهدف ببساطة إلى تثبيط التحرير أو منع المستخدمين من إجراء تغييرات عارضة على عرض PowerPoint. إذا قرر شخص مدفوع — يعرف ما يفعله — أن يعدل عرضك، يمكنه بسهولة إزالة إعداد القراءة فقط. إذا كنت بحاجة جديًا لمنع التحرير غير المصرح به، فمن الأفضل استخدام [حمايات أكثر صرامة تشمل التشفير وكلمات المرور](https://docs.aspose.com/slides/net/password-protected-presentation/).
