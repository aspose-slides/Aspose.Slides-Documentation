## **الأسئلة المتكررة**

**هل يمكنني حفظ العروض التقديمية إلى PDF تتوافق مع معايير الأرشفة وإمكانية الوصول (PDF/A و PDF/UA)؟**

نعم. يدعم Aspose.Slides التصدير إلى PDF بمستويات توافق مثل PDF/A-2a و PDF/A-2b و PDF/A-2u و PDF/A-3a و PDF/A-3b، بالإضافة إلى PDF/UA عبر إعداد [compliance](https://reference.aspose.com/slides/androidjava/com.aspose.slides/pdfoptions/#setCompliance-int-) في [PDF export options](https://reference.aspose.com/slides/androidjava/com.aspose.slides/pdfoptions/).

**هل تدعم المكتبة تضمين الخطوط عند التصدير إلى PDF مع تحكم دقيق فيما يتم تضمينه؟**

نعم. يمكنك التحكم فيما إذا كانت الخطوط مضمَّنة بالكامل أو جزئية (الأحرف المستخدمة فقط)، وتحديد كيفية التعامل مع الخطوط النظامية الشائعة، وتكوين سلوك النص ASCII من خلال [PDF export options](https://reference.aspose.com/slides/androidjava/com.aspose.slides/pdfoptions/).

**هل يمكنني اكتشاف ما إذا كان الملف محميًا بكلمة مرور قبل تحميله فعليًا؟**

نعم. باستخدام [factory-based inspection API](https://reference.aspose.com/slides/androidjava/com.aspose.slides/presentationfactory/)، يمكنك فحص ملف العرض لتحديد ما إذا كان محميًا بكلمة مرور دون فتحه بالكامل.

**هل توجد آليات احتياطية للخطوط ودعم للخطوط المخصصة؟**

نعم. تدعم المكتبة [loading](/slides/ar/androidjava/custom-font/) و[embedding](/slides/ar/androidjava/embedded-font/) الخطوط المخصصة وتوفر قواعد [fallback](/slides/ar/androidjava/fallback-font/) للخطوط لمنع فقدان الأحرف أثناء العرض والتحويل.

**هل يمكنني تصدير الشرائح إلى XPS، وهل هناك خيارات لضبط مخرجات XPS؟**

نعم. يتم دعم [Export to XPS](/slides/ar/androidjava/convert-powerpoint-to-xps/)، ويمكنك تعديل [save options](https://reference.aspose.com/slides/androidjava/com.aspose.slides/xpsoptions/) ذات الصلة للتحكم في جودة ومحتوى مستند XPS.
