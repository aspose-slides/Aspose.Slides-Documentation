في PowerPoint 2019، قدمت Microsoft الإعداد **دائمًا افتح كقراءة فقط** كواحد من الخيارات التي يمكن للمستخدمين استخدامها لحماية عروضهم التقديمية. قد ترغب في استخدام إعداد القراءة فقط لحماية عرض تقديمي عندما

- تريد منع التعديلات غير المقصودة والحفاظ على محتوى عرضك التقديمي آمنًا.
- تريد تنبيه الناس أن العرض الذي قدمته هو النسخة النهائية.

بعد أن تختار خيار **دائمًا افتح كقراءة فقط** لعرض تقديمي، عندما يفتح المستخدمون العرض، سيرون توصية **قراءة فقط** وقد يرون رسالة بهذا الشكل: *لمنع التغييرات غير المقصودة، قام المؤلف بتعيين هذا الملف ليفتح كقراءة فقط.*

تعد توصية القراءة فقط رادعًا بسيطًا ولكنه فعال يثني عن التعديل لأن المستخدمين يجب عليهم أداء مهمة لإزالته قبل السماح لهم بتحرير العرض التقديمي. إذا كنت لا تريد من المستخدمين إجراء تغييرات على عرض تقديمي وتريد إبلاغهم بذلك بطريقة مهذبة، فإن توصية القراءة فقط قد تكون خيارًا جيدًا بالنسبة لك.

> إذا تم فتح عرض تقديمي مع حماية **قراءة فقط** في إصدار قديم من Microsoft PowerPoint—لا يدعم الوظيفة التي تم تقديمها مؤخرًا—فسيتم تجاهل توصية **قراءة فقط** (سيتم فتح العرض بشكل طبيعي).

Aspose.Slides لنظام Android عبر Java تتيح لك تعيين عرض تقديمي كـ **قراءة فقط**، مما يعني أنه بعد أن يفتح المستخدمون العرض، سيرون توصية **قراءة فقط**. توضح لك هذه الشفرة النموذجية كيفية تعيين عرض تقديمي كـ **قراءة فقط** في Java باستخدام Aspose.Slides:

```java
Presentation pres = new Presentation();
try {
    pres.getProtectionManager().setReadOnlyRecommended(true);
    pres.save("ReadOnlyPresentation.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```

{{% alert color="primary" %}} 

**ملاحظة**: توصية **قراءة فقط** تهدف ببساطة إلى تثبيط التحرير أو منع المستخدمين من إجراء تغييرات غير مقصودة على عرض PowerPoint. إذا قرر شخص متحمس—يعرف ما يفعله—تحرير عرضك التقديمي، فيمكنه بسهولة إزالة إعداد القراءة فقط. إذا كنت بحاجة بجدية إلى منع التحرير غير المصرح به، فمن الأفضل لك استخدام [حمايات أكثر صرامة تتضمن تشفيرات وكلمات مرور](https://docs.aspose.com/slides/androidjava/password-protected-presentation/).
