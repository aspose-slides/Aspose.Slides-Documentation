## **FAQ**

**هل يمكنني حفظ العروض التقديمية كملفات PDF تتوافق مع معايير الأرشفة وإمكانية الوصول (PDF/A و PDF/UA)؟**

نعم. تدعم Aspose.Slides التصدير إلى PDF بمستويات امتثال مثل PDF/A-2a و PDF/A-2b و PDF/A-2u و PDF/A-3a و PDF/A-3b، بالإضافة إلى PDF/UA عبر إعداد [compliance](https://reference.aspose.com/slides/java/com.aspose.slides/pdfoptions/#setCompliance-int-) في [PDF export options](https://reference.aspose.com/slides/java/com.aspose.slides/pdfoptions/).

**هل تدعم المكتبة تضمين الخطوط عند التصدير إلى PDF مع تحكم دقيق في ما يتم تضمينه؟**

نعم. يمكنك التحكم ما إذا كانت الخطوط مضمَّنة بالكامل أو جزئياً (فقط الأحرف المستخدمة)، وتحديد كيفية معالجة الخطوط النظامية الشائعة، وتكوين سلوك النص ASCII عبر [PDF export options](https://reference.aspose.com/slides/java/com.aspose.slides/pdfoptions/).

**هل يمكنني اكتشاف ما إذا كان الملف محميًا بكلمة مرور قبل تحميله فعليًا؟**

نعم. باستخدام [factory-based inspection API](https://reference.aspose.com/slides/java/com.aspose.slides/presentationfactory/)، يمكنك الاستعلام عن ملف العرض لتحديد ما إذا كان محميًا بكلمة مرور دون فتحه بالكامل.

**هل هناك آليات احتياطي للخطوط ودعم للخطوط المخصصة؟**

نعم. تدعم المكتبة [loading](/slides/ar/java/custom-font/) و [embedding](/slides/ar/java/embedded-font/) الخطوط المخصصة وتوفر قواعد [fallback](/slides/ar/java/fallback-font/) للخطوط لمنع فقدان الأحرف أثناء العرض والتحويل.

**هل يمكنني تصدير الشرائح إلى XPS، وهل هناك خيارات لضبط إخراج XPS؟**

نعم. يُدعم [Export to XPS](/slides/ar/java/convert-powerpoint-to-xps/)، ويمكنك تعديل [save options](https://reference.aspose.com/slides/java/com.aspose.slides/xpsoptions/) ذات الصلة للتحكم في جودة ومحتوى ملف XPS.
