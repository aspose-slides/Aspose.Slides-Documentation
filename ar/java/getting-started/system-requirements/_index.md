## **نظرة عامة**
لا تتطلب Aspose.Slides for Java تثبيت Microsoft PowerPoint، حيث إن Aspose.Slides نفسه هو محرك لإنشاء وثائق Microsoft PowerPoint، والتحويل، وتنسيق الصفحات، وعرضها.

## **أنظمة التشغيل المدعومة**
تدعم Aspose.Slides for Java أي نظام تشغيل 32‑بت أو 64‑بت يشغل بيئة تشغيل Java، بما في ذلك، ولكن ليس حصرًا على:

### **ويندوز**
- Microsoft Windows 2003 Server ( x64, x86)
- Microsoft Windows 2008 Server ( x64, x86)
- Microsoft Windows 2012 Server ( x64, x86)
- Microsoft Windows 2012 R2 Server ( x64, x86)
- Microsoft Windows 2016 Server ( x64, x86)
- Microsoft Windows 2019 Server ( x64, x86)
- Microsoft Windows Vista ( x64, x86)
- Microsoft Windows XP ( x64, x86)
- Microsoft Windows 7 ( x64, x86)
- Microsoft Windows 8, 8.1 ( x64, x86)
- Microsoft Windows 10 ( x64, x86)


### **لينكس**
- Linux (Ubuntu, OpenSUSE, CentOS and others)

### **ماك**
- Mac OS X

## **إصدارات Java المدعومة**
تدعم Aspose.Slides for Java J2SE 6.0 (Java 1.6) أو أعلى.

## **الأسئلة المتكررة**

**هل أحتاج إلى تثبيت Microsoft PowerPoint للتحويلات والعرض؟**

لا، لا يلزم PowerPoint؛ Aspose.Slides هو محرك مستقل لـ [إنشاء](/slides/ar/java/create-presentation/)، وتعديل، و[تحويل](/slides/ar/java/convert-presentation/)، و[عرض](/slides/ar/java/convert-powerpoint-to-png/) العروض التقديمية.

**ما هي الخطوط المطلوبة للعرض الصحيح؟**

عمليًا، يجب أن تكون الخطوط المستخدمة في العرض أو [البدائل](/slides/ar/java/font-substitution/) المناسبة متوفرة. لضمان عرض متسق على Linux/macOS، يُنصَح بتثبيت حزم الخطوط الشائعة.

**لماذا يتم عرض خط مخصص كبديل أو كنص مفقود على Linux؟**

إذا كان ملف الخط يحتوي على سجلات جدول أسماء غير متسقة أو تالفة، قد يختار نظام مطابقة الخطوط في Linux (FreeType/fontconfig) سجلًا غير صالح، مما يؤدي إلى عدم التعرف على الخط. يؤدي استخدام نسخة من الخط مع سجلات جدول أسماء مصححة أو تثبيت بديل متسق إلى حل هذه المشكلة.
