## **تقييم Aspose.Slides**
يمكنك بسهولة تنزيل Aspose.Slides للتقييم. حزمة التقييم هي نفسها حزمة الشراء. يصبح إصدار التقييم مرخصًا بمجرد إضافة بضع أسطر من الشيفرة لتطبيق الترخيص. 

## **قيود نسخة التقييم**
إصدار التقييم من Aspose.Slides (بدون ترخيص محدد) يوفر كامل وظائف المنتج، لكنه يُدرج علامة مائية للتقييم في أعلى المستند عند الفتح والحفظ. كما أنك مقيد بشريحة واحدة عند استخراج النصوص من شرائح العرض.

{{% alert color="primary" %}} 
إذا كنت ترغب في اختبار Aspose.Slides دون قيود نسخة التقييم، يمكنك طلب **رخصة مؤقتة لمدة 30 يومًا**. يرجى الرجوع إلى [كيف تحصل على رخصة مؤقتة؟](https://purchase.aspose.com/temporary-license) لمزيد من المعلومات.
{{% /alert %}} 

## **حول الترخيص**
يمكنك بسهولة تنزيل نسخة تقييم من Aspose.Slides لـ Node.js عبر Java من صفحتها [للتنزيل](https://releases.aspose.com/slides/nodejs-java/). توفر نسخة التقييم **نفس الإمكانات** تمامًا كما النسخة المرخصة من Aspose.Slides. علاوة على ذلك، يصبح إصدار التقييم مرخصًا بمجرد شراء ترخيص وإضافة بضع أسطر من الشيفرة لتطبيق الترخيص.

الترخيص هو ملف XML نصي بسيط يحتوي على تفاصيل مثل اسم المنتج، عدد المطورين المرخص لهم، تاريخ انتهاء الاشتراك، وما إلى ذلك. الملف موقع رقميًا، لذلك لا تقم بتعديل الملف. حتى الإضافة غير المقصودة لسطر جديد إلى محتويات الملف سيجعل الترخيص غير صالح.

لتجنب القيود المرتبطة بنسخة التقييم، تحتاج إلى تعيين ترخيص قبل استخدام **Aspose.Slides**. يُطلب منك تعيين الترخيص مرة واحدة فقط لكل تطبيق أو عملية.

{{% alert color="primary" %}} 
قد ترغب في الاطلاع على [الترخيص القائم على العدّ](https://docs.aspose.com/slides/nodejs-java/metered-licensing/).
{{% /alert %}}

### **تعيين ترخيص في Aspose.Slides لـ Node.js عبر Java**

يمكن تطبيق التراخيص من المواقع التالية:

* مسار صريح
* تدفق
* كترخيص قائم على العدّ – آلية ترخيص جديدة

{{% alert color="primary" %}}
استخدم طريقة **setLicense** لترخيص مكون.

في حين أن استدعاءات متعددة لـ **setLicense** ليست ضارة، إلا أنها إهدار للموارد (المعالج).
{{% /alert %}}

{{% alert color="warning" %}}
التراخيص الجديدة يمكنها تفعيل Aspose.Slides فقط مع الإصدار 21.4 أو أحدث. الإصدارات السابقة تستخدم نظام ترخيص مختلف ولن تتعرف على هذه التراخيص.
{{% /alert %}}

#### **تطبيق ترخيص باستخدام ملف**
يتم استخدام مقتطف الشيفرة التالي لتعيين ملف ترخيص:
**Node.js**
```javascript
var aspose = aspose || {};

aspose.slides = require("aspose.slides.via.java");

var license = new aspose.slides.License();
license.setLicense("Aspose.Slides.lic");
```


عند استدعاء طريقة setLicense، يجب أن يكون اسم الترخيص هو نفسه اسم ملف الترخيص الخاص بك. على سبيل المثال، يمكنك تغيير اسم ملف الترخيص إلى "Aspose.Slides.lic.xml". ثم، في الشيفرة الخاصة بك، يجب تمرير اسم الترخيص الجديد (Aspose.Slides.lic.xml) إلى طريقة setLicense.

#### **تطبيق ترخيص من تدفق**
يتم استخدام مقتطف الشيفرة التالي لتطبيق ترخيص من تدفق:
**Node.js**
```javascript
var aspose = aspose || {};

aspose.slides = require("aspose.slides.via.java");

var license = new aspose.slides.License();

var fs = require("fs");

var readStream = fs.createReadStream("Aspose.Slides.lic");

license.setLicense(readStream, function(err, list) {
    if(err) { 
        console.error(err); return; 
    }});
```


## **الأسئلة المتكررة**

**هل يمكنني تطبيق الترخيص في بيئة غير متصلة تمامًا بالإنترنت (بدون اتصال بالإنترنت)؟**
نعم. يتم التحقق من صحة الترخيص محليًا باستخدام ملف الترخيص؛ لا يلزم اتصال بالإنترنت.

**ماذا يحدث بعد انتهاء الاشتراك لمدة سنة واحدة؟ هل سيتوقف المكتبة عن العمل؟**
لا. الترخيص دائم: يمكنك الاستمرار في استخدام الإصدارات التي تم إصدارها قبل تاريخ انتهاء اشتراكك؛ فقط لن تكون مؤهلاً لاستخدام الإصدارات الأحدث دون التجديد.
