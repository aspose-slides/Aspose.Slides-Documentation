## **الأسئلة الشائعة**

**هل يمكنني حفظ العروض التقديمية إلى PDF وفقًا لمعايير الأرشفة وإمكانية الوصول (PDF/A و PDF/UA)؟**

نعم. يدعم Aspose.Slides تصدير إلى PDF بمستويات التوافق مثل PDF/A-2a، PDF/A-2b، PDF/A-2u، PDF/A-3a، PDF/A-3b، بالإضافة إلى PDF/UA عبر إعداد [compliance](https://reference.aspose.com/slides/nodejs-java/aspose.slides/pdfoptions/setcompliance/) في [PDF export options](https://reference.aspose.com/slides/nodejs-java/aspose.slides/pdfoptions/).

**هل تدعم المكتبة تضمين الخطوط عند تصدير إلى PDF، مع التحكم التفصيلي في ما يتم تضمينه؟**

نعم. يمكنك التحكم فيما إذا كانت الخطوط مضمّنة بالكامل أو كجزء جزئي (الأحرف المستخدمة فقط)، وتحديد طريقة التعامل مع الخطوط النظامية الشائعة، وتكوين سلوك النص ASCII عبر [PDF export options](https://reference.aspose.com/slides/nodejs-java/aspose.slides/pdfoptions/).

**هل يمكنني اكتشاف ما إذا كان الملف محميًا بكلمة مرور قبل تحميله فعليًا؟**

نعم. باستخدام [factory-based inspection API](https://reference.aspose.com/slides/nodejs-java/aspose.slides/presentationfactory/)، يمكنك الاستعلام عن ملف العرض لتحديد ما إذا كان محميًا بكلمة مرور دون فتحه بالكامل.

**هل هناك آليات احتياطية للخطوط ودعم للخطوط المخصصة؟**

نعم. تدعم المكتبة [loading](/slides/ar/nodejs-java/custom-font/) و[embedding](/slides/ar/nodejs-java/embedded-font/) الخطوط المخصصة وتوفر [fallback rules](/slides/ar/nodejs-java/fallback-font/) لمنع فقد الأحرف أثناء العرض والتحويل.

**هل يمكنني تصدير الشرائح إلى XPS، وهل هناك خيارات لضبط مخرجات XPS؟**

نعم. [Export to XPS](/slides/ar/nodejs-java/convert-powerpoint-to-xps/) مدعوم، ويمكنك تعديل [save options](https://reference.aspose.com/slides/nodejs-java/aspose.slides/xpsoptions/) ذات الصلة للتحكم في جودة المخرج ومحتوى وثيقة XPS.
