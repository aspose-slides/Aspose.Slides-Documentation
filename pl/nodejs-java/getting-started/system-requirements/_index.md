## **Wprowadzenie**

Aspose.Slides for Node.js via Java nie wymaga instalacji żadnego produktu firm trzecich, takiego jak Microsoft PowerPoint. Sam Aspose.Slides jest silnikiem do tworzenia, modyfikowania, konwertowania i renderowania dokumentów w różnych formatach, w tym formatów prezentacji Microsoft PowerPoint.

## **Obsługiwane systemy operacyjne**

Aspose.Slides for Node.js via Java obsługuje każdy 32‑bitowy lub 64‑bitowy system operacyjny, na którym działa środowisko uruchomieniowe Java, w tym m.in.:

### **Windows**
- Microsoft Windows Server 2003 (x64, x86)
- Microsoft Windows Server 2008 (x64, x86)
- Microsoft Windows Server 2012 (x64, x86)
- Microsoft Windows Server 2012 R2 (x64, x86)
- Microsoft Windows Server 2016 (x64, x86)
- Microsoft Windows Server 2019 (x64, x86)
- Microsoft Windows Vista (x64, x86)
- Microsoft Windows XP (x64, x86)
- Microsoft Windows 7 (x64, x86)
- Microsoft Windows 8, 8.1 (x64, x86)
- Microsoft Windows 10 (x64, x86)

### **Linux**
- Linux (Ubuntu, CentOS i inne)

### **Mac**
- Mac OS X

## **FAQ**

**Czy potrzebuję zainstalowanego Microsoft PowerPoint do konwersji i renderowania?**

Nie, PowerPoint nie jest wymagany; Aspose.Slides jest samodzielnym silnikiem do [tworzenia](/slides/pl/nodejs-java/create-presentation/), modyfikowania, [konwertowania](/slides/pl/nodejs-java/convert-presentation/) i [renderowania](/slides/pl/nodejs-java/convert-powerpoint-to-png/) prezentacji.

**Jakie czcionki są potrzebne do poprawnego renderowania?**

W praktyce czcionki użyte w prezentacji lub odpowiednie [zastępniki](/slides/pl/nodejs-java/font-substitution/) muszą być dostępne. Aby zapewnić spójne renderowanie na Linux/macOS, zaleca się zainstalowanie popularnych pakietów czcionek.

**Dlaczego niestandardowa czcionka jest renderowana jako zastępstwo lub brakujący tekst w systemie Linux?**

Jeśli plik czcionki zawiera niezgodne lub uszkodzone wpisy w tabeli nazw, stos dopasowywania czcionek w Linux (FreeType/fontconfig) może wybrać nieprawidłowy rekord, co powoduje, że czcionka nie zostanie rozpoznana. Użycie wersji czcionki z poprawionymi rekordami tabeli nazw lub zainstalowanie spójnego zamiennika rozwiązuje problem.
