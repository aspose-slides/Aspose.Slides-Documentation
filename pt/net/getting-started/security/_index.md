## **Segurança no Aspose.Slides**

A Aspose aplica as melhores práticas ao desenvolver seus produtos. 

* Aspose.Slides para .NET é usado para manipular e transformar apresentações em arquivos em outros formatos. Ele não executa scripts em apresentações. Aspose.Slides simplesmente analisa a estrutura da apresentação e permite que o código dos usuários finais manipule o modelo de objetos de maneira conveniente. 
* Aspose.Slides funciona como uma biblioteca no contexto de análise e interpretação de documentos sem a execução de código remoto. Todos os produtos Aspose são executados em sua máquina. Eles não transmitem nenhum dado para a Aspose. A única exceção é quando uma [licença por medição](https://purchase.aspose.com/faqs/licensing/metered) está envolvida, e mesmo assim (se você usar uma licença por medição), apenas as informações de uso da sua API são processadas.

* Os componentes Aspose são executados no mesmo contexto de usuário que aplicativos regulares. Portanto, os componentes Aspose não representam risco para recursos críticos do sistema. Além disso, quando um documento é aberto por um componente Aspose, macros não são executadas automaticamente. 

* Os riscos inerentes ou associados ao pacote Microsoft Office não se aplicam aos componentes Aspose, portanto os produtos Aspose são muito seguros.

## **Perguntas Frequentes**

**Quais sistemas são usados para monitorar vulnerabilidades no código da Aspose?**

Executamos uma análise estática de código para cada versão do Aspose.Slides. Podemos fornecer relatórios de segurança que comprovam que o código do Aspose.Slides atende ao OWASP Top 10. 

**Quais sistemas vocês utilizam para monitorar vulnerabilidades em pacotes externos?**

Não utilizamos sistemas para verificar vulnerabilidades em pacotes externos porque o Aspose.Slides não contém ou usa nenhum pacote externo. O Aspose.Slides depende APENAS da própria estrutura .NET.
