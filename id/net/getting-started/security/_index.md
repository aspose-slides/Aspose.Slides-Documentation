## **Keamanan di Aspose.Slides**

Aspose menerapkan praktik terbaik saat mengembangkan produknya. 

* Aspose.Slides untuk .NET digunakan untuk memanipulasi dan mengubah presentasi dalam file dengan format lain. Ia tidak menjalankan skrip dalam presentasi. Aspose.Slides hanya mem‑parsing struktur presentasi dan memungkinkan kode pengguna akhir memanipulasi model objek dengan cara yang nyaman. 
* Aspose.Slides berfungsi sebagai perpustakaan dalam konteks mem‑parsing dan menginterpretasikan dokumen tanpa mengeksekusi kode jarak jauh. Semua produk Aspose dijalankan di mesin Anda. Mereka tidak mengirimkan data apa pun ke Aspose. Satu‑satunya pengecualian adalah ketika [lisensi bermeteran](https://purchase.aspose.com/faqs/licensing/metered) terlibat, dan bahkan dalam kasus itu (jika Anda menggunakan lisensi bermeteran), hanya informasi penggunaan API Anda yang diproses.
* Komponen Aspose berjalan dalam konteks pengguna yang sama dengan aplikasi biasa. Oleh karena itu, komponen Aspose tidak menimbulkan risiko terhadap sumber daya sistem yang penting. Lebih lanjut, ketika dokumen dibuka oleh komponen Aspose, makro tidak dijalankan secara otomatis. 
* Risiko yang melekat atau terkait dengan paket Microsoft Office tidak berlaku pada komponen Aspose, sehingga produk Aspose sangat aman.

## **FAQ**

**Sistem apa yang digunakan untuk memantau kerentanan dalam kode Aspose?**

Kami menjalankan analisis kode statis untuk setiap rilis Aspose.Slides. Kami dapat menyediakan laporan keamanan yang membuktikan kode Aspose.Slides memenuhi OWASP Top 10. 

**Sistem apa yang Anda gunakan untuk memantau kerentanan pada paket eksternal?**

Kami tidak menggunakan sistem untuk memeriksa kerentanan pada paket eksternal karena Aspose.Slides tidak mengandung atau menggunakan paket eksternal apa pun. Aspose.Slides hanya bergantung pada .NET framework itu sendiri.
