## **परिचय**

Aspose.Slides PowerPoint प्रेजेंटेशन में स्लाइड आकार और पहलू अनुपात को समायोजित करने के लिए व्यापक उपकरण प्रदान करता है, जो प्रिंटिंग और स्क्रीन पर प्रदर्शित करने दोनों के लिए महत्वपूर्ण है।

प्रसिद्ध स्लाइड आकार और अनुपात:

- **Standard (4:3 Aspect Ratio)**: पुराने स्क्रीन और डिवाइस के लिए आदर्श।
- **Widescreen (16:9 Aspect Ratio)**: आधुनिक प्रोजेक्टर और डिस्प्ले के लिए अनुशंसित।

सुनिश्चित करें कि आपके प्रेजेंटेशन के सभी स्लाइड में एक ही स्लाइड आकार और पहलू अनुपात लागू हो, जिससे सुसंगतता बनी रहे। सर्वोत्तम परिणाम के लिए, जटिलताओं से बचने हेतु प्रेजेंटेशन निर्माण प्रक्रिया की शुरुआत में ही अपनी स्लाइड आयाम निर्धारित करें।

{{% alert color="primary" %}} 
डिफ़ॉल्ट रूप से, Aspose.Slides द्वारा निर्मित प्रेजेंटेशन मानक 4:3 पहलू अनुपात का उपयोग करते हैं।
{{% /alert %}}

## **प्रेजेंटेशन में स्लाइड आकार बदलें**

यह नमूना कोड दिखाता है कि Aspose.Slides का उपयोग करते हुए जावा में प्रेजेंटेशन में स्लाइड आकार कैसे बदलें:

```java
Presentation pres = new Presentation("pres-4x3-aspect-ratio.pptx");
try {
    pres.getSlideSize().setSize(SlideSizeType.OnScreen16x9, SlideSizeScaleType.DoNotScale);
    pres.save("pres-4x3-aspect-ratio.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```

## **प्रेजेंटेशन में कस्टम स्लाइड आकार निर्दिष्ट करें**

यदि आप सामान्य स्लाइड आकार (4:3 और 16:9) को अपने काम के लिए अनुपयुक्त पाते हैं, तो आप किसी विशिष्ट या अद्वितीय स्लाइड आकार का उपयोग करने का निर्णय ले सकते हैं। उदाहरण के लिए, यदि आप अपने प्रेजेंटेशन से कस्टम पेज लेआउट पर पूर्ण आकार की स्लाइड प्रिंट करने की योजना बनाते हैं या यदि आप प्रेजेंटेशन को कुछ विशेष स्क्रीन प्रकारों पर प्रदर्शित करने की इच्छा रखते हैं, तो कस्टम आकार सेटिंग का उपयोग करने से आपको लाभ मिलेगा।

यह नमूना कोड दिखाता है कि जावा के माध्यम से Aspose.Slides for Android का उपयोग करके जावा में प्रेजेंटेशन के लिए कस्टम स्लाइड आकार कैसे निर्दिष्ट करें:

```java
Presentation pres = new Presentation("pres.pptx");
try {
    pres.getSlideSize().setSize(780, 540, SlideSizeScaleType.DoNotScale); // A4 कागज आकार
    pres.save("pres-a4-slide-size.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```

## **आकार बदलने के बाद स्लाइड सामग्री संभालें**

प्रेजेंटेशन के स्लाइड आकार को बदलने के बाद, स्लाइड की सामग्री (उदाहरण के लिए छवियां या वस्तुएँ) विकृत हो सकती है। डिफ़ॉल्ट रूप से, वस्तुओं को नए स्लाइड आकार के अनुसार स्वचालित रूप से री-साइज़ किया जाता है। हालांकि, जब आप प्रेजेंटेशन के स्लाइड आकार को बदलते हैं, तो आप एक सेटिंग निर्धारित कर सकते हैं जो यह तय करता है कि Aspose.Slides स्लाइड की सामग्री को कैसे संभालेगा।

आप अपनी आवश्यकता के अनुसार इन सेटिंग्स में से किसी भी का उपयोग कर सकते हैं:

- `DoNotScale`

  यदि आप स्लाइड पर वस्तुओं को री-साइज़ नहीं करना चाहते हैं, तो इस सेटिंग का उपयोग करें।

- `EnsureFit`

  यदि आप छोटे स्लाइड आकार पर स्केल करना चाहते हैं और चाहते हैं कि Aspose.Slides स्लाइड की वस्तुओं को छोटे आकार में स्केल डाउन करे ताकि सभी वस्तुएँ स्लाइड में फिट हो जाएँ (इस प्रकार आप सामग्री की हानि से बचते हैं), तो इस सेटिंग का उपयोग करें।

- `Maximize`

  यदि आप बड़े स्लाइड आकार पर स्केल करना चाहते हैं और चाहते हैं कि Aspose.Slides स्लाइड की वस्तुओं को बड़े आकार के अनुसार बढ़ाए ताकि वे नए स्लाइड आकार के अनुपात में हों, तो इस सेटिंग का उपयोग करें।

यह नमूना कोड दिखाता है कि प्रेजेंटेशन की स्लाइड का आकार बदलते समय `Maximize` सेटिंग का उपयोग कैसे किया जाए:

```java
Presentation pres = new Presentation("pres.pptx");
try {
    pres.getSlideSize().setSize(SlideSizeType.Ledger, SlideSizeScaleType.Maximize);
} finally {
    if (pres != null) pres.dispose();
}
```

## **अक्सर पूछे जाने वाले प्रश्न**

**क्या मैं इंच के अलावा अन्य इकाइयों (जैसे पॉइंट या मिलीमीटर) का उपयोग करके कस्टम स्लाइड आकार सेट कर सकता हूँ?**

हाँ। Aspose.Slides आंतरिक रूप से पॉइंट का उपयोग करता है, जहाँ 1 पॉइंट 1/72 इंच के बराबर होता है। आप किसी भी इकाई (जैसे मिलीमीटर या सेंटीमीटर) को पॉइंट में परिवर्तित कर सकते हैं और परिवर्तित मानों का उपयोग स्लाइड की चौड़ाई और ऊँचाई निर्धारित करने के लिए कर सकते हैं।

**क्या बहुत बड़ा कस्टम स्लाइड आकार रेंडरिंग के दौरान प्रदर्शन और मेमोरी उपयोग को प्रभावित करेगा?**

हाँ। बड़े स्लाइड आयाम (पॉइंट में) और उच्च रेंडरिंग स्केल मिलकर मेमोरी खपत को बढ़ाते हैं और प्रसंस्करण समय को लंबा करते हैं। व्यावहारिक स्लाइड आकार का लक्ष्य रखें और रेंडरिंग स्केल को केवल आवश्यकतानुसार समायोजित करें ताकि वांछित आउटपुट गुणवत्ता प्राप्त हो सके।

**क्या मैं एक गैर-मानक स्लाइड आकार परिभाषित कर सकता हूँ और फिर विभिन्न आकारों वाले प्रेजेंटेशन से स्लाइड्स को मर्ज कर सकता हूँ?**

आप विभिन्न स्लाइड आकारों वाले प्रेजेंटेशन को [merge presentations](/slides/hi/androidjava/merge-presentation/) नहीं कर सकते — पहले, एक प्रेजेंटेशन का आकार दूसरे के समान कर लें। स्लाइड आकार बदलते समय आप मौजूदा सामग्री को कैसे संभालना है, यह चुन सकते हैं [SlideSizeScaleType](https://reference.aspose.com/slides/hi/androidjava/com.aspose.slides/slidesizescaletype/) विकल्प के माध्यम से। आकारों को मिलाने के बाद आप फॉर्मेटिंग को संरक्षित रखते हुए स्लाइड्स को मर्ज कर सकते हैं।

**क्या मैं स्लाइड के व्यक्तिगत आकारों या विशिष्ट क्षेत्रों के लिए थंबनेल बना सकता हूँ, और क्या वे नए स्लाइड आकार का सम्मान करेंगे?**

हाँ। Aspose.Slides [entire slides](https://reference.aspose.com/slides/hi/androidjava/com.aspose.slides/slide/#getImage-com.aspose.slides.IRenderingOptions-float-float-) और [selected shapes](https://reference.aspose.com/slides/hi/androidjava/com.aspose.slides/shape/#getImage-int-float-float-) दोनों के लिए थंबनेल रेंडर कर सकता है। परिणामी छवियां वर्तमान स्लाइड आकार और पहलू अनुपात को दर्शाती हैं, जिससे स्थिर फ्रेमिंग और ज्यामिति सुनिश्चित होती है।
