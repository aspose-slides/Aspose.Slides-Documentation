## **अवलोकन**

प्रेजेंटेशन पहुँच सुनिश्चित करती है कि सहायक तकनीकों—जैसे स्क्रीन रीडर, ब्रेल डिस्प्ले, या केवल‑कीबोर्ड नेविगेशन—का उपयोग करने वाले लोग आपके स्लाइड्स को उसी प्रभावी ढंग से समझ और नेविगेट कर सकें जैसा देखे‑देखाए, माउस‑उपयोगकर्ता दर्शक करते हैं। सर्वोत्तम प्रथा में स्पष्ट पठन क्रम, सूचनात्मक दृश्यात्मकता के लिए सार्थक वैकल्पिक टेक्स्ट, पर्याप्त रंग कंट्रास्ट, पठनीय टाइपोग्राफी, वर्णनात्मक लिंक टेक्स्ट, और केवल रंग या स्थिति द्वारा अर्थ व्यक्त न करने पर ध्यान दिया जाता है। जब पहुँच को शुरुआत से ही योजना में शामिल किया जाता है, तो परिणामस्वरूप साफ़ संरचना, अधिक सुसंगत दृश्य और ऐसी सामग्री मिलती है जो प्रत्येक दर्शक तक बिना किसी वर्कअराउंड के पहुँचती है।

## **Mark as Decorative**

Mark as decorative फ्लैग केवल अलंकारिक दृश्यों को चिह्नित करता है ताकि स्क्रीन रीडर उन्हें छोड़ दें, शोर कम हो और ध्यान सार्थक सामग्री पर रहे। इसे बैकग्राउंड, सजावटी तत्व और स्पेसर पर लागू करें—कभी भी चार्ट, आइकन, या ऐसी छवियों पर नहीं जो जानकारी प्रदान करती हों। Aspose.Slides इस फ़्लैग को पहचान और सत्यापन के लिए उपलब्ध कराता है, जिससे स्वचालित पहुँच जांच और सफ़ाई संभव हो सके।

![Mark as Decorative](mark_as_decorative.png)

निम्नलिखित कोड नमूना दर्शाता है कि कैसे यह निर्धारित करें कि कोई आकार Mark as decorative के रूप में चिह्नित है।

```java
Presentation presentation = new Presentation("sample.pptx");
try {
    IShape shape = presentation.getSlides().get_Item(0).getShapes().get_Item(0);
    System.out.println("Is shape decorative: " + shape.isDecorative());
} finally {
    presentation.dispose();
}
```
