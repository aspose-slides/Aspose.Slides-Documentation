## **अवलोकन**

यह FAQ Aspose.Slides के सामान्य प्रश्नों के उत्तर प्रदान करती है। इसमें समर्थित फ़ाइल स्वरूप, बड़े प्रस्तुतीकरण के साथ काम करते समय अपवादों को संभालना, स्लाइड आकार बदलना, स्लाइड का पूर्वावलोकन, प्रस्तुतीकरण से पाठ निकालना, तालिका सीमाओं का स्वरूपण, चित्र लगाने, और PDF या छवि में रूपांतरित करते समय फ़ॉन्ट से जुड़ी समस्याओं का समाधान शामिल है।

## **समर्थित फ़ाइल स्वरूप**

**प्रश्न:** Aspose.Slides for Android via Java कौन‑से फ़ाइल स्वरूपों का समर्थन करता है?

**उत्तर:** Aspose.Slides for Android via Java द्वारा समर्थित फ़ाइल स्वरूपों की सूची के लिए देखें [समर्थित फ़ाइल स्वरूप](/slides/hi/androidjava/supported-file-formats/)।

## **अपवाद**

**प्रश्न:** बड़े PPT फ़ाइल को छवियों के साथ लोड करते समय मुझे मेमोरी अपवाद मिल रहा है। क्या Aspose.Slides में फ़ाइल आकार की कोई सीमा है?

**उत्तर:** Aspose.Slides द्वारा समर्थित प्रस्तुतीकरण आकार की कोई विशिष्ट गणना नहीं है। पूरे प्रस्तुतीकरण की संरचना और छवियों को मेमोरी में रखने के लिये पर्याप्त स्थान होना चाहिए। सामान्यतः, मेमोरी में छवि हार्ड डिस्क की तुलना में अधिक स्थान लेती हैं, विशेषकर जब छवियों पर अतिरिक्त प्रभाव हों।

सामान्य तौर पर, Aspose.Slides for Android via Java 4 GB RAM वाले सर्वर पर लगभग 300 MB आकार की प्रस्तुतीकरण फ़ाइलों को आसानी से संभाल सकता है।

## **स्लाइड्स के साथ काम करना**

**प्रश्न:** क्या मैं प्रस्तुतीकरण में स्लाइड्स का आकार बदल सकता हूँ?

**उत्तर:** आप प्रस्तुतीकरण में स्लाइड्स का आकार निर्धारित करने के लिये [Presentation](https://reference.aspose.com/slides/hi/androidjava/com.aspose.slides/presentation/) क्लास द्वारा प्रदत्त `getSlideSize` मेथड का उपयोग कर सकते हैं।

**प्रश्न:** क्या प्रस्तुतीकरण में अलग‑अलग आकार की स्लाइड्स निर्धारित करना संभव है?

**उत्तर:** माइक्रोसॉफ़्ट PowerPoint दस्तावेज़ों में स्लाइड आकार प्रस्तुतीकरण स्तर पर निर्धारित होता है, इसलिए यह संभव नहीं है।

**प्रश्न:** क्या Aspose.Slides for Android via Java स्लाइड को सहेजने से पहले उसका पूर्वावलोकन करने का समर्थन करता है?

**उत्तर:** आप प्रस्तुतीकरण स्लाइड्स को छवियों में रेंडर कर सकते हैं और इन छवियों का उपयोग स्लाइड का पूर्वावलोकन करने के लिये कर सकते हैं।

## **पाठ के साथ काम करना**

**प्रश्न:** क्या सम्पूर्ण पाठ को प्रस्तुतीकरण से प्राप्त करना संभव है?

**उत्तर:** Aspose.Slides for Android via Java [SlideUtil](https://reference.aspose.com/slides/hi/androidjava/com.aspose.slides/slideutil/) क्लास प्रदान करता है, जो प्रस्तुतीकरण से पूरे पाठ को प्राप्त करने के विभिन्न मेथड्स देता है।

**प्रश्न:** पीसी और एंड्रॉइड पर पैराग्राफ आकार में अंतर क्यों होता है?

**उत्तर:** पैराग्राफ आकार की गणना उस पैराग्राफ के पाठ आकार पर आधारित होती है। पाठ आकार की गणना PowerPoint प्रस्तुतीकरण में निर्दिष्ट फ़ॉन्ट के मीट्रिक पर निर्भर करती है। यदि निर्दिष्ट फ़ॉन्ट अनुपलब्ध है, तो उसे सबसे समान फ़ॉन्ट से बदल दिया जाता है, जिसके मीट्रिक मूल फ़ॉन्ट से अलग होते हैं। परिणामस्वरूप, विभिन्न सिस्टमों में स्थापित फ़ॉन्ट सेट के आधार पर पैराग्राफ आकार की गणना में अंतर आता है। विभिन्न ऑपरेटिंग सिस्टमों पर समान परिणाम प्राप्त करने के लिये, आपको समान फ़ॉन्ट स्थापित करने होंगे या उन्हें रन‑टाइम पर [बाहरी फ़ॉन्ट](/slides/hi/androidjava/custom-font/) के रूप में लोड करना होगा।

## **स्वरूपण और छवियां**

**प्रश्न:** तालिका सीमा का रंग कैसे सेट कर सकता हूँ?

**उत्तर:** आप सभी तालिका सीमाओं या पूरी तालिका के चारों ओर की सीमा का रंग बदल सकते हैं। सभी सीमाओं को बदलने के लिये, कृपया [ICell](https://reference.aspose.com/slides/hi/androidjava/com.aspose.slides/icell/) इंटरफ़ेस के `getCellFormat` मेथड का उपयोग करें। पूरी तालिका की सीमा के लिये, आपको कोशिकाओं को पार करके बाहरी सीमाओं का रंग बदलना होगा।

**प्रश्न:** Aspose.Slides for Android via Java चित्र रखने हेतु कौन‑सी माप इकाई उपयोग करता है?

**उत्तर:** स्लाइड पर सभी आकारों के निर्देशांक और आकार पॉइंट (72 dpi) में मापे जाते हैं।

## **फ़ॉन्ट के साथ काम करना**

**प्रश्न:** PPT को PDF या छवियों में बदलते समय आउटपुट दस्तावेज़ों में फ़ॉन्ट अलग क्यों होते हैं?

**उत्तर:** यह समस्या संकेत देती है कि प्रस्तुतीकरण में प्रयुक्त फ़ॉन्ट उस ऑपरेटिंग सिस्टम में उपलब्ध नहीं हैं जहाँ कोड निष्पादित हुआ था। आपको ऑपरेटिंग सिस्टम पर फ़ॉन्ट स्थापित करने चाहिए या उन्हें नीचे दिखाए अनुसार [FontsLoader](https://reference.aspose.com/slides/hi/androidjava/com.aspose.slides/fontsloader/) क्लास का उपयोग कर बाहरी फ़ॉन्ट के रूप में लोड करना चाहिए:
```java
String[] folders = new String[] { "path_to_a_folder_with_fonts" };
FontsLoader.loadExternalFonts(folders);
```
