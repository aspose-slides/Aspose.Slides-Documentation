सभी Aspose Java घटकों को Full Trust अनुमति सेट की आवश्यकता होती है। कारण यह है कि Aspose Java घटकों को कुछ कार्यों जैसे फ़ॉन्ट पार्सिंग आदि के लिए रजिस्ट्री सेटिंग्स, वर्चुअल डायरेक्टरी के अलावा सिस्टम फ़ाइलों तक पहुंच की आवश्यकता होती है। इसके अलावा, Aspose Java घटक कोर Java सिस्टम क्लासेज़ पर आधारित हैं जो कई मामलों में Full Trust अनुमति सेट की आवश्यकता रखते हैं। 

{{% /alert %}} 

इंटरनेट सेवा प्रदाता जो विभिन्न कंपनियों के कई अनुप्रयोगों की होस्टिंग करते हैं, आमतौर पर Medium Trust सुरक्षा स्तर लागू करते हैं: 

- OleDbPermission उपलब्ध नहीं है। इसका अर्थ है कि आप डेटाबेस तक पहुंचने के लिए ADO.NET मैनेज्ड OLE DB डेटा प्रदाता का उपयोग नहीं कर सकते।
- EventLogPermission उपलब्ध नहीं है। इसका अर्थ है कि आप Windows इवेंट लॉग तक पहुंच नहीं सकते।
- ReflectionPermission उपलब्ध नहीं है। इसका अर्थ है कि आप रिफ्लेक्शन का उपयोग नहीं कर सकते।
- RegistryPermission उपलब्ध नहीं है। इसका अर्थ है कि आप रजिस्ट्री तक पहुंच नहीं सकते।
- WebPermission प्रतिबंधित है। इसका अर्थ है कि आपका एप्लिकेशन केवल उन पते या पते की रेंज के साथ संचार कर सकता है जिन्हें आप <trust> तत्व में परिभाषित करते हैं।
- FileIOPermission प्रतिबंधित है। इसका अर्थ है कि आप केवल अपने एप्लिकेशन की वर्चुअल डायरेक्टरी पदानुक्रम में फ़ाइलों तक पहुंच सकते हैं।

{{% alert color="primary" %}} 

ऊपर उल्लिखित कारणों के कारण, Aspose Java घटकों का उपयोग उन सर्वरों पर नहीं किया जा सकता जो Full Trust के अलावा कोई अन्य अनुमति सेट प्रदान करते हैं।
