## **अवलोकन**
Aspose.Slides for Java को Microsoft PowerPoint इंस्टॉल करने की आवश्यकता नहीं है, क्योंकि Aspose.Slides स्वयं Microsoft PowerPoint दस्तावेज़ निर्माण, परिवर्तन, पृष्ठ लेआउट और रेंडरिंग इंजन है।

## **समर्थित ऑपरेटिंग सिस्टम**
Aspose.Slides for Java किसी भी 32‑bit या 64‑bit ऑपरेटिंग सिस्टम को समर्थन देता है जो Java runtime चलाता है, जिसमें, लेकिन इन तक सीमित नहीं:
### **विंडोज**
- Microsoft Windows 2003 Server ( x64, x86)
- Microsoft Windows 2008 Server ( x64, x86)
- Microsoft Windows 2012 Server ( x64, x86)
- Microsoft Windows 2012 R2 Server ( x64, x86)
- Microsoft Windows 2016 Server ( x64, x86)
- Microsoft Windows 2019 Server ( x64, x86)
- Microsoft Windows Vista ( x64, x86)
- Microsoft Windows XP ( x64, x86)
- Microsoft Windows 7 ( x64, x86)
- Microsoft Windows 8, 8.1 ( x64, x86)
- Microsoft Windows 10 ( x64, x86)


### **लिनक्स**
- Linux (Ubuntu, OpenSUSE, CentOS और अन्य)

### **मैकोज़**
- Mac OS X

## **समर्थित जावा संस्करण**
Aspose.Slides for Java J2SE 6.0 (Java 1.6) और उसके बाद के संस्करणों का समर्थन करता है।

## **FAQ**

**क्या परिवर्तनों और रेंडरिंग के लिए मुझे Microsoft PowerPoint इंस्टॉल करने की आवश्यकता है?**

नहीं, PowerPoint आवश्यक नहीं है; Aspose.Slides एक स्वतंत्र इंजन है जो प्रस्तुतियों को बनाने(banane) के लिए, संशोधित करने के लिए, परिवर्तित करने(converting) के लिए, और रेंडर करने(rendering) के लिए उपयोग किया जाता है।

**सही रेंडरिंग के लिए कौन से फ़ॉन्ट आवश्यक हैं?**

वास्तविक रूप से, प्रस्तुति में प्रयुक्त फ़ॉन्ट या उपयुक्त विकल्प(substitutes) उपलब्ध होने चाहिए। Linux/macOS पर निरंतर रेंडरिंग सुनिश्चित करने के लिए सामान्य फ़ॉन्ट पैकेज इंस्टॉल करने की सलाह दी जाती है।

**Linux पर कस्टम फ़ॉन्ट फ़ॉलबैक या गायब टेक्स्ट क्यों दिखाता है?**

यदि फ़ॉन्ट फ़ाइल में नाम‑टेबल एंट्री असंगत या भ्रष्ट हैं, तो Linux फ़ॉन्ट‑मैचिंग स्टैक (FreeType/fontconfig) अवैध रिकॉर्ड चुन सकता है, जिससे फ़ॉन्ट अनसुलझा रह जाता है। सुधारे हुए नाम‑टेबल रिकॉर्ड वाला फ़ॉन्ट संस्करण उपयोग करना या संगत प्रतिस्थापन इंस्टॉल करना इस समस्या को हल करता है।
