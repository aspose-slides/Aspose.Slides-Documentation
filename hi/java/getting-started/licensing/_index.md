## **परिचय**

Aspose.Slides को मूल्यांकन मोड या वैध लाइसेंस के साथ उपयोग किया जा सकता है। मूल्यांकन संस्करण लाइसेंस प्राप्त संस्करण के समान कार्यक्षमता प्रदान करता है, लेकिन प्रस्तुति खोलने या सहेजने पर एक मूल्यांकन वॉटरमार्क जोड़ता है और पाठ निकालने को एक स्लाइड तक सीमित करता है।

यह लेख बताता है कि Aspose.Slides में लाइसेंसिंग कैसे काम करती है और लाइब्रेरी का उपयोग करने से पहले लाइसेंस कैसे लागू किया जाता है। लाइसेंस को `License` क्लास का उपयोग करके फ़ाइल, स्ट्रीम, या एम्बेडेड रिसोर्स से लोड किया जा सकता है। लेख यह भी दिखाता है कि लाइसेंस सही तरीके से लागू हुआ है या नहीं, इसे कैसे मान्य किया जाए।

## **Aspose.Slides में लाइसेंसिंग**

* मूल्यांकन संस्करण लाइसेंस प्राप्त हो जाता है जब आप एक लाइसेंस खरीदते हैं और उसमें कुछ कोड की पंक्तियाँ जोड़ते हैं (लाइसेंस लागू करने के लिए)।
* लाइसेंस एक साधारण‑पाठ XML फ़ाइल है जिसमें उत्पाद का नाम, लाइसेंस प्राप्त डेवलपर्स की संख्या, सब्सक्रिप्शन समाप्ति तिथि आदि विवरण होते हैं। 
* लाइसेंस फ़ाइल डिजिटल रूप से हस्ताक्षरित होती है, इसलिए आपको फ़ाइल को संशोधित नहीं करना चाहिए। फ़ाइल की सामग्री में अनजाने में एक अतिरिक्त लाइन ब्रेक जोड़ने से भी लाइसेंस अमान्य हो जाएगा।
* Aspose.Slides for Java सामान्यतः लाइसेंस को इन स्थानों पर खोजता है:
  * एक स्पष्ट पथ
  * वह फ़ोल्डर जिसमें Aspose.Slides.jar स्थित है
* मूल्यांकन संस्करण से जुड़ी सीमाओं से बचने के लिए, आपको **Aspose.Slides** का उपयोग करने से पहले लाइसेंस सेट करना आवश्यक है। आपको प्रत्येक एप्लिकेशन या प्रक्रिया के लिए केवल एक बार लाइसेंस सेट करना पड़ता है।

## **लाइसेंस लागू करना**

एक लाइसेंस को **फ़ाइल** या **स्ट्रीम** से लोड किया जा सकता है।

{{% alert color="primary" %}}

Aspose.Slides लाइसेंसिंग कार्यों के लिए [License](https://reference.aspose.com/slides/hi/java/com.aspose.slides/License) क्लास प्रदान करता है।

{{% /alert %}} 

{{% alert color="warning" %}}

नए लाइसेंस केवल संस्करण 21.4 या इसके बाद के साथ ही Aspose.Slides को सक्रिय कर सकते हैं। पहले के संस्करण अलग लाइसेंसिंग सिस्टम का उपयोग करते हैं और इन लाइसेंसों को पहचान नहीं पाएँगे।

{{% /alert %}}

### **फ़ाइल**

लाइसेंस सेट करने की सबसे आसान विधि यह है कि आप लाइसेंस फ़ाइल को Aspose.Slides.jar वाले फ़ोल्डर या अपने एप्लिकेशन की jar में रखें।

यह Java कोड दिखाता है कि लाइसेंस फ़ाइल कैसे सेट की जाती है:

``` java
// लाइसेंस क्लास का उदाहरण बनाता है
com.aspose.slides.License license = new com.aspose.slides.License();

// लाइसेंस फ़ाइल पथ सेट करता है
license.setLicense("Aspose.Slides.Java.lic");
```

{{% alert color="warning" %}} 

यदि आप लाइसेंस फ़ाइल को किसी अलग डायरेक्टरी में रखते हैं, तो जब आप [SetLicense](https://reference.aspose.com/slides/hi/java/com.aspose.slides/License#setLicense-java.lang.String-) मेथड को कॉल करते हैं, तो निर्दिष्ट स्पष्ट पथ के अंत में लाइसेंस फ़ाइल का नाम आपके लाइसेंस फ़ाइल के समान होना चाहिए।

उदाहरण के लिए, आप लाइसेंस फ़ाइल का नाम *Aspose.Slides.Java.lic.xml* बदल सकते हैं। फिर अपने कोड में आपको फ़ाइल का पथ (जिसका अंत *Aspose.Slides.Java.lic.xml* हो) को [SetLicense](https://reference.aspose.com/slides/hi/java/com.aspose.slides/License#setLicense-java.lang.String-) मेथड को पास करना होगा।

{{% /alert %}}

### **स्ट्रीम**

आप लाइसेंस को स्ट्रीम से लोड कर सकते हैं। यह Java कोड दिखाता है कि स्ट्रीम से लाइसेंस कैसे लागू किया जाता है:

``` java
// License क्लास का उदाहरण बनाता है
com.aspose.slides.License license = new com.aspose.slides.License();

// स्ट्रीम के माध्यम से लाइसेंस सेट करता है
license.setLicense(new java.io.FileInputStream("Aspose.Slides.Java.lic"));
```

### **PHP/Java Bridge**

यदि आप Java के ज़रिए PHP के लिए Aspose.Slides का उपयोग करते हैं, तो आप PHP/Java ब्रिज के माध्यम से लाइसेंस सेट कर सकते हैं। यह ब्रिज आपको PHP सिंटैक्स में Java क्लासेज़ का उपयोग करने देता है। अधिक जानकारी के लिए देखें [License in PHP](/slides/hi/php-java/licensing/)।

## **लाइसेंस का सत्यापन**

यह जांचने के लिए कि लाइसेंस सही ढंग से सेट हुआ है या नहीं, आप इसे सत्यापित कर सकते हैं। यह Java कोड दिखाता है कि लाइसेंस को कैसे वैलिडेट किया जाता है:

```java
License license = new License();
license.setLicense("Aspose.Slides.Java.lic");

if (License.isLicensed()) 
{
    System.out.println("License is good!");
}
```

## **FAQ**

**क्या मैं लाइसेंस को पूरी तरह ऑफ़लाइन वातावरण (बिना इंटरनेट एक्सेस) में लागू कर सकता हूँ?**

हाँ। लाइसेंस वैधता लाइसेंस फ़ाइल का उपयोग करके स्थानीय रूप से की जाती है; इंटरनेट कनेक्शन की आवश्यकता नहीं होती।

**एक‑वर्ष की सब्सक्रिप्शन समाप्त होने के बाद क्या होता है? क्या लाइब्रेरी काम करना बंद कर देगी?**

नहीं। लाइसेंस स्थायी है: आप अपनी सब्सक्रिप्शन समाप्ति तिथि से पहले जारी किए गए संस्करणों का उपयोग जारी रख सकते हैं; हालांकि बिना नवीनीकरण के आप नए रिलीज़ का उपयोग करने के लिए पात्र नहीं रहेंगे।
