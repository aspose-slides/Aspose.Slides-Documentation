## **Overview**

फ़ॉन्ट प्रतिस्थापन Aspose.Slides को रेंडरिंग या रूपांतरण के दौरान मूल प्रस्तुति फ़ॉन्ट उपलब्ध नहीं होने पर किसी अन्य फ़ॉन्ट का उपयोग करने की अनुमति देता है। आप `IFontsManager` इंटरफ़ेस की `getSubstitutions` मेथड का उपयोग करके यह देख सकते हैं कि कौन से फ़ॉन्ट प्रतिस्थापित किए गए थे।

Aspose.Slides आपको फ़ॉन्ट प्रतिस्थापन नियम परिभाषित करने की भी सुविधा देता है। उदाहरण के लिए, आप यह निर्दिष्ट कर सकते हैं कि एक अनुपलब्ध फ़ॉन्ट को किसी उपलब्ध फ़ॉन्ट से बदल दिया जाए और फिर इन नियमों को प्रस्तुति के फ़ॉन्ट मैनेजर के माध्यम से लागू कर सकते हैं।

## **Set Font Substitution Rules**

Aspose.Slides आपको फ़ॉन्ट के लिए नियम निर्धारित करने की अनुमति देता है जो कुछ स्थितियों में क्या करना है (उदाहरण के लिए, जब फ़ॉन्ट तक पहुंच नहीं हो सकी) इसे इस प्रकार निर्धारित करता है:

1. संबंधित प्रस्तुति लोड करें।
2. वह फ़ॉन्ट लोड करें जिसे बदलना है।
3. नया फ़ॉन्ट लोड करें।
4. प्रतिस्थापन के लिए एक नियम जोड़ें।
5. नियम को प्रस्तुति फ़ॉन्ट प्रतिस्थापन नियम संग्रह में जोड़ें।
6. प्रभाव देखने के लिए स्लाइड छवि जनरेट करें।

यह Java कोड फ़ॉन्ट प्रतिस्थापन प्रक्रिया को दर्शाता है:

```java
// एक प्रस्तुति लोड करता है
Presentation pres = new Presentation("Fonts.pptx");
try {
    // वह स्रोत फ़ॉन्ट लोड करता है जिसे बदला जाएगा
    IFontData sourceFont = new FontData("SomeRareFont");
    
    // नया फ़ॉन्ट लोड करता है
    IFontData destFont = new FontData("Arial");
    
    // फ़ॉन्ट प्रतिस्थापन के लिए एक फ़ॉन्ट नियम जोड़ता है
    IFontSubstRule fontSubstRule = new FontSubstRule(sourceFont, destFont, FontSubstCondition.WhenInaccessible);
    
    // नियम को फ़ॉन्ट प्रतिस्थापन नियम संग्रह में जोड़ता है
    IFontSubstRuleCollection fontSubstRuleCollection = new FontSubstRuleCollection();
    fontSubstRuleCollection.add(fontSubstRule);
    
    // नियम सूची में एक फ़ॉन्ट नियम संग्रह जोड़ता है
    pres.getFontsManager().setFontSubstRuleList(fontSubstRuleCollection);
    
    // जब SomeRareFont पहुँच से बाहर हो तो Arial फ़ॉन्ट इसके स्थान पर उपयोग किया जाएगा
    IImage slideImage = pres.getSlides().get_Item(0).getImage(1f, 1f);
    
    // छवि को JPEG स्वरूप में डिस्क पर सहेजता है
    try {
          slideImage.save("Thumbnail_out.jpg", ImageFormat.Jpeg);
    } finally {
         if (slideImage != null) slideImage.dispose();
    }
} finally {
    if (pres != null) pres.dispose();
}
```

## **Limitations for Math Equation Fonts**

फ़ॉन्ट प्रतिस्थापन नियम रेंडरिंग और रूपांतरण के दौरान उपयोग की जाने वाली मानक फ़ॉन्ट चयन प्रक्रिया में भाग लेते हैं। वे सामान्य टेक्स्ट परिदृश्यों के लिए उपयुक्त हैं जहाँ Aspose.Slides कॉन्फ़िगर किए गए नियम के अनुसार एक अनुपलब्ध फ़ॉन्ट को किसी उपलब्ध फ़ॉन्ट से बदल सकता है।

हालांकि, Office गणितीय समीकरणों में एक महत्वपूर्ण सीमा है। यदि कोई समीकरण **Cambria Math** के साथ बनाया गया था, तो Aspose.Slides को समीकरण लेआउट की सही गणना और रेंडरिंग के लिए मूल **Cambria Math** फ़ॉन्ट अभी भी आवश्यक हो सकता है। इस कारण, **Cambria Math** को किसी अन्य गणित फ़ॉन्ट, जैसे **STIX Two Math**, से प्रतिस्थापित करना समीकरण रेंडरिंग के लिए समर्थित नहीं है और यह अभी भी एक अपवाद उत्पन्न कर सकता है जो दर्शाता है कि **Cambria Math** आवश्यक है।

ऐसी प्रस्तुतियों को सफलतापूर्वक रूपांतरित करने के लिए सुनिश्चित करें कि **Cambria Math** रनटाइम पर Aspose.Slides के लिए उपलब्ध है। आप फ़ॉन्ट को ऑपरेटिंग सिस्टम में स्थापित कर सकते हैं या इसे एक [external font](/slides/hi/java/custom-font/) के रूप में प्रदान कर सकते हैं ताकि यह रेंडरिंग और रूपांतरण के दौरान सामान्य फ़ॉन्ट चयन प्रक्रिया में भाग ले सके।

यह सीमा केवल समीकरण रेंडरिंग के लिए विशिष्ट है। ऊपर वर्णित मानक फ़ॉन्ट प्रतिस्थापन नियम अभी भी तब लागू होते हैं जब मूल फ़ॉन्ट उपलब्ध नहीं है और प्रस्तुति के सामान्य टेक्स्ट पर लागू होते हैं।

## **FAQ**

**फ़ॉन्ट प्रतिस्थापन और फ़ॉन्ट प्रतिस्थापन में क्या अंतर है?**

[Replacement](/slides/hi/java/font-replacement/) पूरी प्रस्तुति में एक फ़ॉन्ट को दूसरे से बलपूर्वक बदलना है। प्रतिस्थापन एक नियम है जो विशेष स्थिति में सक्रिय होता है, जैसे जब मूल फ़ॉन्ट अनुपलब्ध हो, और तब एक निर्दिष्ट बैकअप फ़ॉन्ट का उपयोग किया जाता है।

**प्रतिस्थापन नियम ठीक कब लागू होते हैं?**

ये नियम मानक [font selection](/slides/hi/java/font-selection-sequence/) क्रम में भाग लेते हैं जिसे लोडिंग, रेंडरिंग और रूपांतरण के दौरान मूल्यांकन किया जाता है; यदि चुना गया फ़ॉन्ट उपलब्ध नहीं है, तो प्रतिस्थापन या प्रतिस्थापन लागू किया जाता है।

**यदि न तो प्रतिस्थापन और न ही प्रतिस्थापन कॉन्फ़िगर किया गया हो और सिस्टम पर फ़ॉन्ट अनुपलब्ध हो, तो डिफ़ॉल्ट व्यवहार क्या है?**

लाइब्रेरी सबसे निकटतम उपलब्ध सिस्टम फ़ॉन्ट को चुनने की कोशिश करेगी, जैसा कि PowerPoint करता है।

**क्या मैं रनटाइम पर कस्टम एक्सटर्नल फ़ॉन्ट संलग्न कर सकता हूँ ताकि प्रतिस्थापन से बचा जा सके?**

हाँ। आप रनटाइम पर [add external fonts](/slides/hi/java/custom-font/) जोड़ सकते हैं ताकि लाइब्रेरी उन्हें चयन और रेंडरिंग, जिसमें बाद के रूपांतरण भी शामिल हैं, के लिए विचार करे।

**क्या Aspose लाइब्रेरी के साथ कोई फ़ॉन्ट वितरित करता है?**

नहीं। Aspose कोई पेड या फ्री फ़ॉन्ट वितरित नहीं करता; आप फ़ॉन्ट स्वयं अपने विवेक और जिम्मेदारी से जोड़ते और उपयोग करते हैं।

**Windows, Linux और macOS पर प्रतिस्थापन व्यवहार में क्या अंतर हैं?**

हाँ। फ़ॉन्ट खोज ऑपरेटिंग सिस्टम के फ़ॉन्ट डायरेक्टरी से शुरू होती है। डिफ़ॉल्ट उपलब्ध फ़ॉन्ट का सेट और खोज पथ प्लेटफ़ॉर्म के अनुसार भिन्न होते हैं, जिससे उपलब्धता और प्रतिस्थापन की आवश्यकता प्रभावित होती है।

**बैच रूपांतरण के दौरान अप्रत्याशित प्रतिस्थापन को न्यूनतम करने के लिए वातावरण कैसे तैयार करें?**

मशीनों या कंटेनरों में फ़ॉन्ट सेट को समक्रमित करें, आउटपुट दस्तावेज़ों के लिए आवश्यक [add the external fonts](/slides/hi/java/custom-font/) जोड़ें, और जहाँ संभव हो प्रस्तुति में [embed fonts](/slides/hi/java/embedded-font/) एम्बेड करें ताकि चयनित फ़ॉन्ट रेंडरिंग के दौरान उपलब्ध रहें।
