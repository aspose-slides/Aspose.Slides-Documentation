## **अवलोकन**

Aspose.Slides आपको एक प्रस्तुति में एक फ़ॉन्ट को दूसरे फ़ॉन्ट के साथ बदलने की अनुमति देता है। जब फ़ॉन्ट बदला जाता है, तो मूल फ़ॉन्ट के सभी उदाहरण नए फ़ॉन्ट में बदल दिए जाते हैं।

फ़ॉन्ट प्रतिस्थापन करने के लिए, प्रस्तुति लोड करें, स्रोत फ़ॉन्ट और प्रतिस्थापन फ़ॉन्ट निर्धारित करें, फ़ॉन्ट प्रतिस्थापन मेथड को कॉल करें, और संशोधित प्रस्तुति को PPTX फ़ाइल के रूप में सहेजें। यह तरीका तब उपयोगी होता है जब आप जानबूझकर पूरे प्रस्तुति में एक फ़ॉन्ट परिवार से दूसरे परिवार में स्विच करना चाहते हैं।

## **फ़ॉन्ट बदलें**

यदि आप फ़ॉन्ट उपयोग करने के बारे में अपना विचार बदलते हैं, तो आप उस फ़ॉन्ट को किसी अन्य फ़ॉन्ट से बदल सकते हैं। पुराने फ़ॉन्ट के सभी उदाहरण नए फ़ॉन्ट द्वारा प्रतिस्थापित कर दिए जाएंगे।

Aspose.Slides आपको फ़ॉन्ट इस प्रकार बदलने की सुविधा देता है:

1. संबंधित प्रस्तुति लोड करें। 
2. वह फ़ॉन्ट लोड करें जो बदला जाएगा। 
3. नया फ़ॉन्ट लोड करें। 
4. फ़ॉन्ट को बदलें। 
5. संशोधित प्रस्तुति को PPTX फ़ाइल के रूप में लिखें।

यह Java कोड फ़ॉन्ट प्रतिस्थापन को दर्शाता है:

```java
// एक प्रस्तुति लोड करता है
Presentation pres = new Presentation("Fonts.pptx");
try {
    // वह स्रोत फ़ॉन्ट लोड करता है जिसे बदला जाएगा
    IFontData sourceFont = new FontData("Arial");
    
    // नया फ़ॉन्ट लोड करता है
    IFontData destFont = new FontData("Times New Roman");
    
    // फ़ॉन्ट्स को बदलता है
    pres.getFontsManager().replaceFont(sourceFont, destFont);
    
    // प्रस्तुति को सहेजता है
    pres.save("UpdatedFont_out.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```

{{% alert title="Note" color="warning" %}} 
कुछ शर्तों में क्या होता है (उदाहरण के लिए, यदि फ़ॉन्ट तक पहुंच नहीं हो पाती) इसे निर्धारित करने के नियम सेट करने हेतु देखें [**Font Substitution**](/slides/hi/java/font-substitution/)। 
{{% /alert %}}

## **अक्सर पूछे जाने वाले प्रश्न**

**"फ़ॉन्ट प्रतिस्थापन", "फ़ॉन्ट प्रतिस्थापन" और "फ़ॉलबैक फ़ॉन्ट्स" में क्या अंतर है?**

प्रतिस्थापन पूरे दस्तावेज़ में एक परिवार से दूसरे परिवार में जानबूझकर स्विच है। [Substitution](/slides/hi/java/font-substitution/) एक नियम है जैसे "यदि फ़ॉन्ट उपलब्ध नहीं है, तो X उपयोग करें।" [Fallback](/slides/hi/java/fallback-font/) को व्यक्तिगत अनुपलब्ध ग्लिफ़ के लिए सर्जिकल रूप से लागू किया जाता है जब बेस फ़ॉन्ट स्थापित हो लेकिन आवश्यक अक्षर न हों।

**क्या प्रतिस्थापन मास्टर स्लाइड्स, लेआउट्स, नोट्स और टिप्पणियों पर भी लागू होता है?**

हां। प्रतिस्थापन सभी प्रस्तुति ऑब्जेक्ट्स को प्रभावित करता है जो मूल फ़ॉन्ट का उपयोग करते हैं, जिसमें मास्टर स्लाइड्स और नोट्स शामिल हैं; टिप्पणियां भी दस्तावेज़ का हिस्सा हैं और फ़ॉन्ट इंजन द्वारा ध्यान में ली जाती हैं।

**क्या एम्बेडेड OLE ऑब्जेक्ट्स (जैसे Excel) के भीतर फ़ॉन्ट बदलेगा?**

नहीं। [OLE content](/slides/hi/java/manage-ole/) अपने स्वयं के एप्लिकेशन द्वारा नियंत्रित होता है। प्रस्तुति में किए गए प्रतिस्थापन से आंतरिक OLE डेटा का फॉर्मेट नहीं बदलता; यह छवि के रूप में या बाहरी रूप से संपादन योग्य सामग्री के रूप में दिखाया जा सकता है।

**क्या मैं प्रस्तुति के केवल कुछ भाग (स्लाइड्स या क्षेत्रों) में फ़ॉन्ट बदल सकता हूं?**

लक्षित प्रतिस्थापन संभव है यदि आप आवश्यक ऑब्जेक्ट्स/रेंजेज़ स्तर पर फ़ॉन्ट बदलते हैं, न कि पूरे दस्तावेज़ पर वैश्विक प्रतिस्थापन लागू करते हैं। रेंडरिंग के दौरान कुल फ़ॉन्ट चयन तर्क वही रहता है।

**मैं पहले से कैसे पता कर सकता हूँ कि प्रस्तुति कौन से फ़ॉन्ट्स का उपयोग करती है?**

प्रस्तुति के [font manager](https://reference.aspose.com/slides/hi/java/com.aspose.slides/fontsmanager/) का उपयोग करें: यह [उपयोग में रहने वाले परिवारों](https://reference.aspose.com/slides/hi/java/com.aspose.slides/fontsmanager/#getFonts--) की सूची और [substitutions/"unknown" फ़ॉन्ट्स](https://reference.aspose.com/slides/hi/java/com.aspose.slides/fontsmanager/#getSubstitutions--) के बारे में जानकारी प्रदान करता है, जिससे प्रतिस्थापन की योजना बनाना आसान होता है।

**क्या फ़ॉन्ट प्रतिस्थापन PDF/छवियों में रूपांतरण पर काम करता है?**

हां। निर्यात के दौरान, Aspose.Slides वही [फ़ॉन्ट चयन/प्रतिस्थापन क्रम](/slides/hi/java/font-selection-sequence/) लागू करता है, इसलिए पहले किया गया प्रतिस्थापन रूपांतरण के दौरान सम्मानित किया जाएगा।

**क्या मुझे लक्ष्य फ़ॉन्ट सिस्टम में स्थापित करने की आवश्यकता है, या मैं फ़ॉन्ट्स फ़ोल्डर संलग्न कर सकता हूं?**

स्थापना आवश्यक नहीं है: लाइब्रेरी उपयोगकर्ता फ़ोल्डर से [बाहरी फ़ॉन्ट लोड करने](/slides/hi/java/custom-font/) की अनुमति देती है ताकि [रेंडरिंग और निर्यात](/slides/hi/java/convert-powerpoint/) के दौरान उपयोग किया जा सके।

**क्या प्रतिस्थापन "तोफू" (वर्ग) को अक्षर के बजाय ठीक कर देगा?**

केवल तभी जब लक्ष्य फ़ॉन्ट वास्तव में आवश्यक ग्लिफ़ रखता हो। यदि नहीं, तो गायब अक्षरों को कवर करने के लिए [fallback कॉन्फ़िगर](/slides/hi/java/fallback-font/) करें।
