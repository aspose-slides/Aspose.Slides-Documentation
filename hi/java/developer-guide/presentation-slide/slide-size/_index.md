## **परिचय**

Aspose.Slides व्यापक टूल्स प्रदान करता है जिससे PowerPoint प्रस्तुतियों में स्लाइड आकार और आस्पेक्ट रेशियो को समायोजित किया जा सके, जो प्रिंटिंग और स्क्रीन पर दिखाने दोनों के लिए महत्वपूर्ण है।

लोकप्रिय स्लाइड आकार और अनुपात:

- **Standard (4:3 Aspect Ratio)**: पुराने स्क्रीन और उपकरणों के लिए आदर्श।
- **Widescreen (16:9 Aspect Ratio)**: आधुनिक प्रोजेक्टर और डिस्प्ले के लिए अनुशंसित।

अपनी प्रस्तुति में निरंतरता सुनिश्चित करें क्योंकि सभी स्लाइड्स पर एक ही स्लाइड आकार और आस्पेक्ट रेशियो लागू होता है। सर्वोत्तम परिणामों के लिए, प्रस्तुति बनाने की प्रक्रिया की शुरुआत में ही स्लाइड आयाम निर्धारित करें ताकि बाद में जटिलताओं से बचा जा सके।

{{% alert color="primary" %}} 
डिफ़ॉल्ट रूप से, Aspose.Slides से निर्मित प्रस्तुतियों में मानक 4:3 आस्पेक्ट रेशियो उपयोग किया जाता है।
{{% /alert %}}

## **प्रस्तुति में स्लाइड आकार बदलें**

यह उदाहरण कोड दिखाता है कि Java में Aspose.Slides का उपयोग करके प्रस्तुति में स्लाइड आकार कैसे बदलें:

```java
Presentation pres = new Presentation("pres-4x3-aspect-ratio.pptx");
try {
    pres.getSlideSize().setSize(SlideSizeType.OnScreen16x9, SlideSizeScaleType.DoNotScale);
    pres.save("pres-4x3-aspect-ratio.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```

## **प्रस्तुति में अनुकूलित स्लाइड आकार निर्दिष्ट करें**

यदि आप सामान्य स्लाइड आकार (4:3 और 16:9) को अपने काम के लिए उपयुक्त नहीं पाते हैं, तो आप एक विशिष्ट या अनोखा स्लाइड आकार उपयोग कर सकते हैं। उदाहरण के लिए, यदि आप अपनी प्रस्तुतीकरण से कस्टम पेज लेआउट पर पूर्ण‑आकार स्लाइड प्रिंट करना चाहते हैं या कुछ स्क्रीन प्रकारों पर प्रस्तुत करना चाहते हैं, तो अनुकूलित आकार सेट करने से आपको लाभ होगा।

यह उदाहरण कोड दिखाता है कि Java के लिए Aspose.Slides का उपयोग करके प्रस्तुति में कस्टम स्लाइड आकार कैसे निर्दिष्ट करें:

```java
Presentation pres = new Presentation("pres.pptx");
try {
    pres.getSlideSize().setSize(780, 540, SlideSizeScaleType.DoNotScale); // A4 काग़ज आकार
    pres.save("pres-a4-slide-size.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```

## **आकार बदलने के बाद स्लाइड सामग्री को संभालें**

जब आप प्रस्तुति के स्लाइड आकार को बदलते हैं, तो स्लाइड की सामग्री (उदाहरण के लिए छवियां या ऑब्जेक्ट) विकृत हो सकती है। डिफ़ॉल्ट रूप से, ऑब्जेक्ट स्वचालित रूप से नए स्लाइड आकार में फिट होने के लिए पुनः आकार दिया जाता है। हालांकि, स्लाइड आकार बदलते समय आप एक सेटिंग निर्दिष्ट कर सकते हैं जो निर्धारित करती है कि Aspose.Slides स्लाइड्स की सामग्री को कैसे संभालता है।

आपकी आवश्यकता के अनुसार आप निम्नलिखित सेटिंग्स में से कोई भी उपयोग कर सकते हैं:

- `DoNotScale`

  यदि आप स्लाइड पर मौजूद ऑब्जेक्ट को पुनः आकार नहीं देना चाहते, तो इस सेटिंग का उपयोग करें।

- `EnsureFit`

  यदि आप छोटे स्लाइड आकार पर स्केल करना चाहते हैं और चाहते हैं कि Aspose.Slides स्लाइड के सभी ऑब्जेक्ट को छोटा करके फिट करे (इससे सामग्री खोने से बचा जा सके), तो इस सेटिंग का उपयोग करें।

- `Maximize`

  यदि आप बड़े स्लाइड आकार पर स्केल करना चाहते हैं और चाहते हैं कि Aspose.Slides स्लाइड के ऑब्जेक्ट को बड़ा करके नए स्लाइड आकार के अनुपात में लाए, तो इस सेटिंग का उपयोग करें।

यह उदाहरण कोड दिखाता है कि प्रस्तुति के स्लाइड के आकार को बदलते समय `Maximize` सेटिंग को कैसे उपयोग किया जाए:

```java
Presentation pres = new Presentation("pres.pptx");
try {
    pres.getSlideSize().setSize(SlideSizeType.Ledger, SlideSizeScaleType.Maximize);
} finally {
    if (pres != null) pres.dispose();
}
```

## **सामान्य प्रश्न**

**क्या मैं इंच के अलावा अन्य इकाइयों (जैसे पॉइंट या मिलीमीटर) का उपयोग करके कस्टम स्लाइड आकार सेट कर सकता हूँ?**

हाँ। Aspose.Slides आंतरिक रूप से पॉइंट का उपयोग करता है, जहाँ 1 पॉइंट 1/72 इंच के बराबर होता है। आप किसी भी इकाई (जैसे मिलीमीटर या सेंटीमीटर) को पॉइंट में बदल सकते हैं और परिवर्तित मानों का उपयोग करके स्लाइड की चौड़ाई और ऊँचाई निर्धारित कर सकते हैं।

**क्या बहुत बड़ा कस्टम स्लाइड आकार रेंडरिंग के दौरान प्रदर्शन और मेमोरी उपयोग को प्रभावित करता है?**

हाँ। बड़े स्लाइड आयाम (पॉइंट में) और उच्च रेंडरिंग स्केल मिलकर मेमोरी खपत बढ़ाते हैं और प्रोसेसिंग समय को लंबा करते हैं। व्यावहारिक स्लाइड आकार चुनें और आवश्यकतानुसार रेंडरिंग स्केल को समायोजित करें ताकि वांछित आउटपुट क्वालिटी प्राप्त हो।

**क्या मैं एक गैर‑मानक स्लाइड आकार निर्धारित कर सकता हूँ और फिर विभिन्न आकार वाली प्रस्तुतियों से स्लाइड्स को मर्ज कर सकता हूँ?**

आप विभिन्न स्लाइड आकार वाली प्रस्तुतियों को [merge presentations](/slides/hi/java/merge-presentation/) नहीं कर सकते — पहले किसी एक प्रस्तुति को दूसरे के आकार से मेल खाने के लिए पुनः आकार दें। स्लाइड आकार बदलते समय आप मौजूदा सामग्री के संचालन को [SlideSizeScaleType](https://reference.aspose.com/slides/hi/java/com.aspose.slides/slidesizescaletype/) विकल्प के माध्यम से चुन सकते हैं। आकार का मिलान करने के बाद आप फ़ॉर्मेटिंग को बरकरार रखते हुए स्लाइड्स को मर्ज कर सकते हैं।

**क्या मैं स्लाइड के व्यक्तिगत आकारों या विशिष्ट क्षेत्रों के थंबनेल उत्पन्न कर सकता हूँ, और क्या वे नए स्लाइड आकार का सम्मान करेंगे?**

हाँ। Aspose.Slides [entire slides](https://reference.aspose.com/slides/hi/java/com.aspose.slides/slide/#getImage-com.aspose.slides.IRenderingOptions-float-float-) और [selected shapes](https://reference.aspose.com/slides/hi/java/com.aspose.slides/shape/#getImage-int-float-float-) दोनों के लिए थंबनेल रेंडर कर सकता है। उत्पन्न छवियां वर्तमान स्लाइड आकार और आस्पेक्ट रेशियो को प्रतिबिंबित करती हैं, जिससे फ्रेमिंग और ज्यामिति में स्थिरता बनी रहती है।
