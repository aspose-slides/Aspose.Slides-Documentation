## **परिचय**

प्रेज़ेंटेशन एक्सेसिबिलिटी यह सुनिश्चित करती है कि सहायक प्रौद्योगिकियों—जैसे स्क्रीन रीडर, ब्रेल डिस्प्ले, या केवल‑कीबोर्ड नेविगेशन—का उपयोग करने वाले लोग आपके स्लाइड्स को उतनी ही प्रभावी रूप से समझ और नेविगेट कर सकें जितने दृष्टि‑सम्पन्न, माउस‑उपयोग करने वाले दर्शक। अच्छी प्रथा स्पष्ट पढ़ने के क्रम, सूचनात्मक दृश्यों के लिये सार्थक वैकल्पिक पाठ, पर्याप्त रंग कंट्रास्ट, पढ़ने योग्य टाइपोग्राफी, वर्णनात्मक लिंक टेक्स्ट, और केवल रंग या स्थिति द्वारा अर्थ न पहुँचाने पर केंद्रित होती है। जब एक्सेसिबिलिटी को शुरू से ही योजना में शामिल किया जाता है, तो परिणामस्वरूप एक साफ़ संरचना, अधिक सुसंगत दृश्यों, और ऐसी सामग्री मिलती है जो हर दर्शक तक बिना किसी वर्कअराउंड के पहुँचा सके।

## **सजावटी के रूप में चिह्नित करें**

सजावटी के रूप में चिह्नित करना केवल अलंकारिक दृश्यों को चिन्हित करता है ताकि स्क्रीन रीडर उन्हें छोड़ दें, शोर को कम किया जा सके और सार्थक सामग्री पर ध्यान केंद्रित रहे। इसे बैकग्राउंड, फ़्लॉरिश, और स्पेसर पर लागू करें—कभी भी चार्ट, आइकन, या सूचना देने वाली छवियों पर नहीं। Aspose.Slides इस फ़्लैग को पहचान और सत्यापन के लिये प्रदान करता है, जिससे स्वचालित एक्सेसिबिलिटी जांच और सफ़ाई संभव होती है।

![सजावटी के रूप में चिह्नित](mark_as_decorative.png)

निम्न कोड उदाहरण दर्शाता है कि कैसे निर्धारित करें कि कोई आकार सजावटी के रूप में चिह्नित है या नहीं।

```java
Presentation presentation = new Presentation("sample.pptx");
try {
    IShape shape = presentation.getSlides().get_Item(0).getShapes().get_Item(0);
    System.out.println("Is shape decorative: " + shape.isDecorative());
} finally {
    presentation.dispose();
}
```
