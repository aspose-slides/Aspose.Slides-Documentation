## **अवलोकन**

यह लेख Aspose.Slides का उपयोग करके प्रस्तुति स्लाइड को SVG छवियों के रूप में रेंडर करने के तरीके को समझाता है। यह SVG फ़ॉर्मेट और इसके लाभों का वर्णन करता है, जिसमें स्केलेबिलिटी, एक्सेसेबिलिटी, और वेब विकास के लिए उपयुक्तता शामिल हैं।

आप सीखेंगे कि प्रस्तुति फ़ाइल को कैसे लोड करें, उसकी स्लाइड्स पर इटरेट करें, और प्रत्येक स्लाइड को एक अलग SVG फ़ाइल के रूप में कैसे सहेजें। लेख PowerPoint और OpenDocument प्रस्तुति फ़ॉर्मेट्स, जैसे PPT, PPTX, ODP, और PPS को कवर करता है, तथा `Presentation` क्लास और `WriteAsSvg` मेथड का उपयोग करके प्रोग्रामेटिक रूप से रूपांतरण कैसे किया जाए दिखाता है।

## **SVG फ़ॉर्मेट**

SVG—Scalable Vector Graphics का संक्षिप्त रूप—एक मानक ग्राफ़िक प्रकार या फ़ॉर्मेट है जिसे द्वि‑आयामी छवियों को रेंडर करने के लिए उपयोग किया जाता है। SVG छवियों को XML में वेक्टर के रूप में संग्रहीत करता है जिसमें उनके व्यवहार या स्वरूप को परिभाषित करने वाले विवरण होते हैं।

SVG कुछ ही छवि फ़ॉर्मेट्स में से एक है जो इन शर्तों में बहुत उच्च मानकों को पूरा करता है: स्केलेबिलिटी, इंटरैक्टिविटी, प्रदर्शन, एक्सेसेबिलिटी, प्रोग्रामेबिलिटी, आदि। इन कारणों से यह वेब विकास में व्यापक रूप से उपयोग किया जाता है।

आप SVG फ़ाइलों का उपयोग तब करना चाह सकते हैं जब आपको

- **अपने प्रस्तुति को *बहुत बड़े फ़ॉर्मेट* में प्रिंट करें।** SVG छवियों को किसी भी रिज़ॉल्यूशन या स्तर तक स्केल किया जा सकता है। आप आवश्यकतानुसार SVG छवियों को आकार बदल सकते हैं बिना गुणवत्ता खोए।
- **अपनी स्लाइड्स से चार्ट और ग्राफ़ को *विभिन्न माध्यमों या प्लेटफ़ॉर्म* में उपयोग करें।** अधिकांश रीडर्स SVG फ़ाइलों को समझ सकते हैं।
- **छवियों का *सबसे छोटा संभव आकार* उपयोग करें।** SVG फ़ाइलें आम तौर पर अन्य फ़ॉर्मेट्स में समान उच्च‑रिज़ॉल्यूशन संस्करणों की तुलना में छोटी होती हैं, विशेष रूप से बिटमैप (JPEG या PNG) आधारित फ़ॉर्मेट्स के।

## **एक स्लाइड को SVG छवि के रूप में रेंडर करें**

Aspose.Slides for C++ आपको अपनी प्रस्तुतियों की स्लाइड्स को SVG छवियों के रूप में निर्यात करने की अनुमति देता है। SVG छवियों को उत्पन्न करने के लिए निम्न चरणों का पालन करें:

1. Presentation क्लास का एक उदाहरण बनाएं।
2. प्रस्तुति में सभी स्लाइड्स पर इटरेट करें।
3. प्रत्येक स्लाइड को उसके स्वयं के SVG फ़ाइल में FileStream के माध्यम से लिखें।

{{% alert color="primary" %}} 
आप हमारे [नि:शुल्क वेब एप्लिकेशन](https://products.aspose.app/slides/hi/conversion/ppt-to-svg) को आज़माना चाह सकते हैं, जिसमें हमने Aspose.Slides for C++ से PPT को SVG में परिवर्तित करने का कार्य लागू किया है। 
{{% /alert %}} 

यह C++ कोड उदाहरण दिखाता है कि कैसे Aspose.Slides का उपयोग करके PPT को SVG में बदलें:

``` cpp
auto pres = System::MakeObject<Presentation>(u"pres.pptx");
        
for (int32_t index = 0; index < pres->get_Slides()->get_Count(); index++)
{
    auto fileName = String::Format(u"slide-{0}.svg", index);
    auto fileStream = System::MakeObject<FileStream>(fileName, FileMode::Create, FileAccess::Write);

    auto slide = pres->get_Slides()->idx_get(index);
    slide->WriteAsSvg(fileStream);
}
```

## **FAQ**

**परिणामी SVG विभिन्न ब्राउज़र में अलग‑अलग क्यों दिखता है?**  
ब्राउज़र इंजन द्वारा विशिष्ट SVG सुविधाओं के समर्थन में अंतर होता है। [SVGOptions](https://reference.aspose.com/slides/hi/cpp/aspose.slides.export/svgoptions/) पैरामीटर असंगतियों को कम करने में मदद करते हैं।

**क्या सिर्फ स्लाइड्स ही नहीं, बल्कि व्यक्तिगत शैप को भी SVG के रूप में निर्यात करना संभव है?**  
हाँ। कोई भी [आकार एक अलग SVG के रूप में सहेजा जा सकता है](https://reference.aspose.com/slides/hi/cpp/aspose.slides/shape/writeassvg/), जो आइकॉन, पिक्टोग्राम, और ग्राफ़िक्स को पुनः उपयोग करने के लिए सुविधाजनक है।

**क्या कई स्लाइड्स को एकल SVG (स्ट्रिप/डॉक्यूमेंट) में संयोजित किया जा सकता है?**  
मानक परिदृश्य एक स्लाइड → एक SVG है। कई स्लाइड्स को एकल SVG कैनवास में संयोजित करना एक पोस्ट‑प्रोसेसिंग कदम है जिसे एप्लिकेशन स्तर पर किया जाता है।
