सभी Aspose Java कॉम्पोनेन्ट्स को Full Trust अनुमति सेट की आवश्यकता होती है। कारण यह है कि Aspose Java कॉम्पोनेन्ट्स को रजिस्ट्री सेटिंग्स, वर्चुअल डायरेक्टरी के अलावा सिस्टम फ़ाइलों तक पहुंचने की आवश्यकता होती है, कुछ संचालन जैसे फ़ॉन्ट्स को पार्स करना आदि के लिए। इसके अलावा, Aspose Java कॉम्पोनेन्ट्स कोर Java सिस्टम क्लासेज़ पर आधारित होते हैं जो कई मामलों में Full Trust अनुमति सेट की भी आवश्यकता रखते हैं। 

{{% /alert %}} 

विभिन्न कंपनियों के कई एप्लिकेशन होस्ट करने वाले इंटरनेट सर्विस प्रोवाइडर्स आमतौर पर Medium Trust सुरक्षा स्तर लागू करते हैं: 

- OleDbPermission उपलब्ध नहीं है। इसका मतलब है कि आप डेटाबेस तक पहुंचने के लिए ADO.NET प्रबंधित OLE DB डेटा प्रोवाइडर का उपयोग नहीं कर सकते।
- EventLogPermission उपलब्ध नहीं है। इसका मतलब है कि आप Windows इवेंट लॉग तक पहुंच नहीं सकते।
- ReflectionPermission उपलब्ध नहीं है। इसका मतलब है कि आप रिफ्लेक्शन का उपयोग नहीं कर सकते।
- RegistryPermission उपलब्ध नहीं है। इसका मतलब है कि आप रजिस्ट्री तक पहुंच नहीं सकते।
- WebPermission प्रतिबंधित है। इसका मतलब है कि आपका एप्लिकेशन केवल उन पतों या पतों की रेंज के साथ संचार कर सकता है जिन्हें आप <trust> एलिमेंट में परिभाषित करते हैं।
- FileIOPermission प्रतिबंधित है। इसका मतलब है कि आप केवल अपने एप्लिकेशन की वर्चुअल डायरेक्टरी श्रेणी में फ़ाइलों तक पहुंच सकते हैं।

{{% alert color="primary" %}} 

उपरोक्त उल्लिखित कारणों के कारण, Aspose Java कॉम्पोनेन्ट्स को Full Trust के अलावा किसी भी अनुमति सेट वाले सर्वरों पर उपयोग नहीं किया जा सकता।
