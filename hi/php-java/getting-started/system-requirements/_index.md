## **परिचय**

Aspose.Slides for PHP via Java को किसी भी तृतीय पक्ष उत्पाद जैसे Microsoft PowerPoint को स्थापित करने की आवश्यकता नहीं होती। Aspose.Slides स्वयं विभिन्न स्वरूपों में दस्तावेज़ों को बनाने, संशोधित करने, परिवर्तित करने और रेंडर करने के लिए एक इंजन है, जिसमें Microsoft PowerPoint प्रेज़ेंटेशन स्वरूप भी शामिल हैं।

## **अक्सर पूछे जाने वाले प्रश्न**

**क्या परिवर्तनों और रेंडरिंग के लिए मेरे पास Microsoft PowerPoint स्थापित होना आवश्यक है?**

नहीं, PowerPoint आवश्यक नहीं है; Aspose.Slides एक स्वतंत्र इंजन है जो प्रेज़ेंटेशन को [बनाने](/slides/hi/php-java/create-presentation/), संशोधित करने, [परिवर्तित](/slides/hi/php-java/convert-presentation/), और [रेंडर](/slides/hi/php-java/convert-powerpoint-to-png/) करने के लिए उपयोग किया जाता है।

**सही रेंडरिंग के लिए कौन से फ़ॉन्ट आवश्यक हैं?**

व्यावहारिक रूप से, प्रेज़ेंटेशन में उपयोग किए गए फ़ॉन्ट या उचित [प्रतिस्थापन](/slides/hi/php-java/font-substitution/) उपलब्ध होने चाहिए। Linux/macOS पर निरंतर रेंडरिंग सुनिश्चित करने के लिए सामान्य फ़ॉन्ट पैकेज स्थापित करना सलाहकार है।

**Linux पर कस्टम फ़ॉन्ट फॉलबैक या गायब टेक्स्ट के रूप में क्यों रेंडर होता है?**

यदि फ़ॉन्ट फ़ाइल में असंगत या भ्रष्ट नाम-टेबल प्रविष्टियां होती हैं, तो Linux फ़ॉन्ट‑मैचिंग स्टैक (FreeType/fontconfig) एक अमान्य रिकॉर्ड चुन सकता है, जिससे फ़ॉन्ट हल नहीं हो पाता। सुधारे हुए नाम-टेबल रिकॉर्ड वाले फ़ॉन्ट संस्करण का उपयोग करके या एक सुसंगत प्रतिस्थापन स्थापित करके समस्या का समाधान किया जा सकता है।
