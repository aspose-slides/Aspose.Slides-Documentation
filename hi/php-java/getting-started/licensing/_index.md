## **Aspose.Slides का मूल्यांकन करें**

आप आसानी से मूल्यांकन के लिए Aspose.Slides डाउनलोड कर सकते हैं। मूल्यांकन पैकेज खरीदे गए पैकेज के समान है। लाइसेंस लागू करने के लिए कुछ कोड लाइनों को जोड़ने पर मूल्यांकन संस्करण सरलता से लाइसेंस्ड हो जाता है।

## **मूल्यांकन संस्करण की सीमाएं**

Aspose.Slides का मूल्यांकन संस्करण (बिना किसी लाइसेंस के) पूर्ण उत्पाद कार्यक्षमता प्रदान करता है, लेकिन यह खोलने और सेव करने पर दस्तावेज़ के शीर्ष पर एक मूल्यांकन वॉटरमार्क डालता है। प्रस्तुति स्लाइड्स से पाठ निकालते समय आप केवल एक स्लाइड तक सीमित रहते हैं।

{{% alert color="primary" %}} 
यदि आप Aspose.Slides को मूल्यांकन संस्करण की सीमाओं के बिना परीक्षण करना चाहते हैं, तो आप **30 दिन का अस्थायी लाइसेंस** का अनुरोध कर सकते हैं। अधिक जानकारी के लिए कृपया [अस्थायी लाइसेंस कैसे प्राप्त करें?](https://purchase.aspose.com/temporary-license) देखें।
{{% /alert %}} 

## **लाइसेंस के बारे में**

आप आसानी से Aspose.Slides का PHP के लिए Java के माध्यम से [डाउनलोड पेज](https://packagist.org/packages/aspose/slides) से मूल्यांकन संस्करण डाउनलोड कर सकते हैं। मूल्यांकन संस्करण Aspose.Slides के लाइसेंस्ड संस्करण के समान **सभी क्षमताएं** प्रदान करता है। इसके अतिरिक्त, लाइसेंस खरीदने और लाइसेंस लागू करने के लिए कुछ कोड लाइनों को जोड़ने पर मूल्यांकन संस्करण सरलता से लाइसेंस्ड हो जाता है।

लाइसेंस एक साधारण-टेक्स्ट XML फ़ाइल है जिसमें उत्पाद नाम, लाइसेंस प्राप्त डेवलपर्स की संख्या, सब्सक्रिप्शन समाप्ति तिथि आदि विवरण होते हैं। फ़ाइल डिजिटल रूप से हस्ताक्षरित होती है, इसलिए इसे संशोधित न करें। फ़ाइल की सामग्री में अनजाने में अतिरिक्त लाइन ब्रेक जोड़ने से भी यह अमान्य हो जाएगी।

मूल्यांकन संस्करण से जुड़ी सीमाओं से बचने के लिए, **Aspose.Slides** का उपयोग करने से पहले आपको लाइसेंस सेट करना होगा। आपको प्रत्येक एप्लिकेशन या प्रक्रिया में केवल एक बार लाइसेंस सेट करना आवश्यक है।

### **Aspose.Slides के लिए PHP via Java में लाइसेंस सेट करें**

लाइसेंस इन स्थानों से लागू किए जा सकते हैं:

* स्पष्ट पथ
* स्ट्रीम
* मे़टर्ड लाइसेंस के रूप में – एक नया लाइसेंसिंग तंत्र

{{% alert color="primary" %}}
किसी घटक को लाइसेंस करने के लिए **setLicense** मेथड का उपयोग करें।

हालाँकि **setLicense** को कई बार कॉल करना हानिकारक नहीं है, लेकिन यह संसाधनों (प्रोसेसर) की बर्बादी है।
{{% /alert %}}

{{% alert color="warning" %}}
नए लाइसेंस केवल संस्करण 21.4 या बाद के साथ Aspose.Slides को सक्रिय कर सकते हैं। पहले के संस्करण एक अलग लाइसेंसिंग सिस्टम का उपयोग करते हैं और इन लाइसेंसों को पहचान नहीं पाएंगे।
{{% /alert %}}

#### **फ़ाइल का उपयोग करके लाइसेंस लागू करें**

इस कोड स्निपेट का उपयोग लाइसेंस फ़ाइल सेट करने के लिए किया जाता है:

**PHP**

```php
<?php
require_once("http://localhost:8080/JavaBridge/java/Java.inc");
require_once("lib/aspose.slides.php");

use aspose\slides\License;

$license = new License();
$license->setLicense("Aspose.Slides.lic");
?>
```

जब setLicense मेथड को कॉल किया जाता है, तो लाइसेंस नाम आपके लाइसेंस फ़ाइल के समान होना चाहिए। उदाहरण के लिए, आप लाइसेंस फ़ाइल का नाम "Aspose.Slides.lic.xml" बदल सकते हैं। फिर, अपने कोड में आपको नया लाइसेंस नाम (Aspose.Slides.lic.xml) setLicense मेथड को पास करना होगा।

#### **स्ट्रीम से लाइसेंस लागू करें**

इस कोड स्निपेट का उपयोग स्ट्रीम से लाइसेंस लागू करने के लिए किया जाता है:

```php
<?php
require_once("http://localhost:8080/JavaBridge/java/Java.inc");
require_once("lib/aspose.slides.php");

use aspose\slides\License;
```

## **अक्सर पूछे जाने वाले प्रश्न**

**क्या मैं पूरी तरह ऑफ़लाइन वातावरण (कोई इंटरनेट एक्सेस नहीं) में लाइसेंस लागू कर सकता हूँ?**

हाँ। लाइसेंस वैधता स्थानीय रूप से लाइसेंस फ़ाइल का उपयोग करके की जाती है; इंटरनेट कनेक्शन की आवश्यकता नहीं है।

**एक साल की सब्सक्रिप्शन समाप्त होने के बाद क्या होता है? क्या लाइब्रेरी काम करना बंद कर देगी?**

नहीं। लाइसेंस शाश्वत है: आप अपनी सब्सक्रिप्शन समाप्ति तिथि से पहले जारी किए गए संस्करणों का उपयोग जारी रख सकते हैं; केवल बिना नवीनीकरण के नए रिलीज़ का उपयोग नहीं कर पाएँगे।
