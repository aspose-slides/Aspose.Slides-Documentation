## **परिचय**

Aspose.Slides पावरपॉइंट प्रस्तुतियों में स्लाइड आकार और अनुपात को समायोजित करने के लिए व्यापक टूल प्रदान करता है, जो प्रिंटिंग और स्क्रीन पर प्रदर्शित करने दोनों के लिए महत्वपूर्ण है।  

लोकप्रिय स्लाइड आकार और अनुपात:

- **मानक (4:3 अनुपात)**: पुराने स्क्रीन और उपकरणों के लिए आदर्श।  
- **वाइडस्क्रीन (16:9 अनुपात)**: आधुनिक प्रोजेक्टर और डिस्प्ले के लिए अनुशंसित।  

सुनिश्चित करें कि आपके प्रस्तुतिकरण में स्थिरता बनी रहे, क्योंकि सभी स्लाइड्स पर एक ही स्लाइड आकार और अनुपात लागू होता है। इष्टतम परिणामों के लिए, अपने प्रस्तुति निर्माण प्रक्रिया की शुरुआत में स्लाइड आयाम निर्धारित करें ताकि जटिलताओं से बचा जा सके।  

{{% alert color="primary" %}} 
डिफ़ॉल्ट रूप से, Aspose.Slides से बनाई गई प्रस्तुतियों में मानक 4:3 अनुपात उपयोग किया जाता है।  
{{% /alert %}}

## **प्रस्तुतियों में स्लाइड आकार बदलें**

यह नमूना कोड आपको Aspose.Slides का उपयोग करके प्रस्तुति में स्लाइड आकार बदलने का तरीका दिखाता है:

```php
  $pres = new Presentation("pres-4x3-aspect-ratio.pptx");
  try {
    $pres->getSlideSize()->setSize(SlideSizeType::OnScreen16x9, SlideSizeScaleType::DoNotScale);
    $pres->save("pres-4x3-aspect-ratio.pptx", SaveFormat::Pptx);
  } finally {
    if (!java_is_null($pres)) {
      $pres->dispose();
    }
  }
```

## **प्रस्तुतियों में कस्टम स्लाइड आकार निर्दिष्ट करें**

यदि आप पाते हैं कि सामान्य स्लाइड आकार (4:3 और 16:9) आपके कार्य के लिए उपयुक्त नहीं हैं, तो आप एक विशिष्ट या अनन्य स्लाइड आकार उपयोग करने का निर्णय ले सकते हैं। उदाहरण के तौर पर, यदि आप अपनी प्रस्तुति से कस्टम पेज लेआउट पर पूर्ण आकार की स्लाइड प्रिंट करने की योजना बनाते हैं या यदि आप अपनी प्रस्तुति को कुछ विशिष्ट स्क्रीन प्रकारों पर प्रदर्शित करने चाहते हैं, तो कस्टम आकार सेटिंग उपयोग करने से आपको लाभ हो सकता है।  

यह नमूना कोड आपको जावा के माध्यम से PHP के लिए Aspose.Slides का उपयोग करके प्रस्तुति के लिए कस्टम स्लाइड आकार निर्दिष्ट करने का तरीका दिखाता है:

```php
  $pres = new Presentation("pres.pptx");
  try {
    $pres->getSlideSize()->setSize(780, 540, SlideSizeScaleType::DoNotScale);// A4 कागज आकार

    $pres->save("pres-a4-slide-size.pptx", SaveFormat::Pptx);
  } finally {
    if (!java_is_null($pres)) {
      $pres->dispose();
    }
  }
```

## **आकार बदलने के बाद स्लाइड सामग्री संभालें**

प्रस्तुति के लिए स्लाइड आकार बदलने के बाद, स्लाइड की सामग्री (जैसे चित्र या वस्तुएँ) विकृत हो सकती है। डिफ़ॉल्ट रूप से, वस्तुएँ नए स्लाइड आकार में फिट होने के लिए स्वतः री-साइज़ हो जाती हैं। हालांकि, प्रस्तुति के स्लाइड आकार को बदलते समय, आप एक सेटिंग निर्दिष्ट कर सकते हैं जो तय करती है कि Aspose.Slides स्लाइडों की सामग्री को कैसे संभालेगा।  

आपके उद्देश्य या लक्ष्य के आधार पर, आप इन सेटिंग्स में से कोई भी उपयोग कर सकते हैं:

- `DoNotScale`

  यदि आप स्लाइडों पर वस्तुओं को री-साइज़ नहीं चाहते हैं, तो इस सेटिंग का उपयोग करें।  

- `EnsureFit`

  यदि आप छोटे स्लाइड आकार में स्केल करना चाहते हैं और Aspose.Slides को स्लाइडों की वस्तुओं को नीचे स्केल करके सभी वस्तुओं को स्लाइडों में फिट करने देना चाहते हैं (इस प्रकार आप सामग्री खोने से बचते हैं), तो इस सेटिंग का उपयोग करें।  

- `Maximize`

  यदि आप बड़े स्लाइड आकार में स्केल करना चाहते हैं और Aspose.Slides को स्लाइडों की वस्तुओं को बड़े स्लाइड आकार के अनुपात में बड़ा करने देना चाहते हैं, तो इस सेटिंग का उपयोग करें।  

यह नमूना कोड आपको प्रस्तुति की स्लाइड का आकार बदलते समय `Maximize` सेटिंग का उपयोग कैसे करें, दिखाता है:

```php
  $pres = new Presentation("pres.pptx");
  try {
    $pres->getSlideSize()->setSize(SlideSizeType::Ledger, SlideSizeScaleType::Maximize);
  } finally {
    if (!java_is_null($pres)) {
      $pres->dispose();
    }
  }
```

## **अक्सर पूछे जाने वाले प्रश्न**

**क्या मैं इंच के अलावा अन्य इकाइयों (जैसे पॉइंट या मिलीमीटर) का उपयोग करके कस्टम स्लाइड आकार सेट कर सकता हूँ?**

हां। Aspose.Slides आंतरिक रूप से पॉइंट्स का उपयोग करता है, जहाँ 1 पॉइंट = 1/72 इंच के बराबर होता है। आप किसी भी इकाई (जैसे मिलीमीटर या सेंटीमीटर) को पॉइंट्स में परिवर्तित कर सकते हैं और परिवर्तनित मानों का प्रयोग स्लाइड की चौड़ाई और ऊँचाई परिभाषित करने के लिए कर सकते हैं।  

**क्या बहुत बड़े कस्टम स्लाइड आकार रेंडरिंग के दौरान प्रदर्शन और मेमोरी उपयोग को प्रभावित करेंगे?**

हां। बड़े स्लाइड आयाम (पॉइंट्स में) और उच्च रेंडरिंग स्केल मिलकर मेमोरी उपयोग को बढ़ाते हैं और प्रोसेसिंग समय को लंबा करते हैं। व्यावहारिक स्लाइड आकार का लक्ष्य रखें और आवश्यकतानुसार आउटपुट गुणवत्ता प्राप्त करने के लिए रेंडरिंग स्केल को ही समायोजित करें।  

**क्या मैं एक गैर-मानक स्लाइड आकार परिभाषित कर सकता हूँ और फिर विभिन्न आकार की प्रस्तुतियों से स्लाइड्स को मर्ज कर सकता हूँ?**

आप [प्रस्तुतियों को मिलाएँ](/slides/hi/php-java/merge-presentation/) नहीं कर सकते — पहले, एक प्रस्तुति का आकार दूसरे के अनुरूप बदलें। स्लाइड आकार बदलते समय, आप मौजूद सामग्री को कैसे संभालना है यह [SlideSizeScaleType](https://reference.aspose.com/slides/hi/php-java/aspose.slides/slidesizescaletype/) विकल्प के माध्यम से चुन सकते हैं। आकार संरेखित करने के बाद, आप फॉर्मेटिंग को संरक्षित रखते हुए स्लाइड्स को मर्ज कर सकते हैं।  

**क्या मैं एकल आकृतियों या स्लाइड के विशिष्ट क्षेत्रों के लिए थंबनेल जेनरेट कर सकता हूँ, और क्या वे नई स्लाइड आकार का सम्मान करेंगे?**

हां। Aspose.Slides [पूरी स्लाइडें](https://reference.aspose.com/slides/hi/php-java/aspose.slides/slide/#getImage) और [चुनी हुई आकृतियों](https://reference.aspose.com/slides/hi/php-java/aspose.slides/shape/#getImage) दोनों के लिए थंबनेल रेंडर कर सकता है। परिणामी छवियाँ वर्तमान स्लाइड आकार और अनुपात को दर्शाती हैं, जिससे निरंतर फ्रेमिंग और ज्यामिति सुनिश्चित होती है।
