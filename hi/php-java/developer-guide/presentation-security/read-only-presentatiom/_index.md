## **परिचय**

PowerPoint 2019 में, Microsoft ने **Always Open Read-Only** सेटिंग को पेश किया जो उपयोगकर्ता अपनी प्रस्तुतियों की सुरक्षा के लिए उपयोग कर सकते हैं। आप इस Read-Only सेटिंग का उपयोग तब करना चाह सकते हैं जब

- आप आकस्मिक संपादन को रोकना चाहते हैं और अपनी प्रस्तुति की सामग्री को सुरक्षित रखना चाहते हैं। 
- आप लोगों को यह बताना चाहते हैं कि आप द्वारा प्रदान किया गया प्रस्तुति अंतिम संस्करण है। 

जब आप किसी प्रस्तुति के लिए **Always Open Read-Only** विकल्प चुनते हैं, तो उपयोगकर्ता प्रस्तुति खोलते समय **Read-Only** अनुशंसा देखते हैं और ऐसा संदेश देख सकते हैं: *आकस्मिक परिवर्तन को रोकने के लिए, लेखक ने इस फ़ाइल को पढ़ने-के-लिए खोलने के रूप में सेट किया है।*

Read-Only अनुशंसा एक सरल लेकिन प्रभावी निरोधक है जो संपादन को हतोत्साहित करता है क्योंकि उपयोगकर्ताओं को इसे हटाने के लिए कार्य करना पड़ता है इससे पहले कि उन्हें प्रस्तुति संपादित करने की अनुमति मिले। यदि आप चाहते हैं कि उपयोगकर्ता प्रस्तुति में परिवर्तन न करें और यह बात विनम्रता से बताना चाहते हैं, तो Read-Only अनुशंसा आपके लिए एक अच्छा विकल्प हो सकता है। 

> यदि **Read-Only** सुरक्षा वाली प्रस्तुति को पुरानी Microsoft PowerPoint एप्लिकेशन में खोला जाता है—जो हाल ही में प्रस्तुत फ़ंक्शन का समर्थन नहीं करता—तो **Read-Only** अनुशंसा को नजरअंदाज कर दिया जाता है (प्रस्तुति सामान्य रूप से खुलती है)।

## **Read-Only मोड लागू करें**

Aspose.Slides for PHP via Java आपको प्रस्तुति को **Read-Only** सेट करने की अनुमति देता है, जिसका अर्थ है कि उपयोगकर्ता (प्रेज़ेंटेशन खोलने के बाद) **Read-Only** अनुशंसा देखते हैं। यह नमूना कोड दिखाता है कि Aspose.Slides का उपयोग करके प्रस्तुति को **Read-Only** कैसे सेट किया जाए:

```php
  $pres = new Presentation();
  try {
    $pres->getProtectionManager()->setReadOnlyRecommended(true);
    $pres->save("ReadOnlyPresentation.pptx", SaveFormat::Pptx);
  } finally {
    if (!java_is_null($pres)) {
      $pres->dispose();
    }
  }
```

{{% alert color="primary" %}} 

**नोट**: **Read-Only** अनुशंसा बस यह सुझाव देने के लिए है कि संपादन को हतोत्साहित किया जाए या उपयोगकर्ताओं को PowerPoint प्रस्तुति में आकस्मिक परिवर्तन करने से रोका जाए। यदि कोई प्रेरित व्यक्ति—जो जानता है वह क्या कर रहा है—आपकी प्रस्तुति को संपादित करने का निर्णय लेता है, तो वह आसानी से Read-Only सेटिंग को हटा सकता है। यदि आपको अनधिकृत संपादन को गंभीरता से रोकने की आवश्यकता है, तो आप [और अधिक कड़ी सुरक्षा जो एन्क्रिप्शन और पासवर्ड शामिल करती है](https://docs.aspose.com/slides/hi/php-java/password-protected-presentation/) का उपयोग करना बेहतर होगा।

{{% /alert %}} 

## **अक्सर पूछे जाने वाले प्रश्न**

**'Read-Only recommended' पूर्ण पासवर्ड संरक्षण से कैसे अलग है?**

'Read-Only recommended' केवल फ़ाइल को केवल-रीड मोड में खोलने का सुझाव देता है और इसे आसानी से बायपास किया जा सकता है। [Password protection](/slides/hi/php-java/password-protected-presentation/) वास्तव में खोलने या संपादन को प्रतिबंधित करता है और यह वास्तविक सुरक्षा नियंत्रण की आवश्यकता होने पर उपयुक्त है।

**क्या 'Read-Only recommended' को वॉटरमार्क के साथ मिलाकर और अधिक संपादन को हतोत्साहित किया जा सकता है?**

हां। अनुशंसा को [watermarks](/slides/hi/php-java/watermark/) के साथ एक दृश्य निरोधक के रूप में जोड़ा जा सकता है; वे अलग तंत्र हैं और एक साथ अच्छी तरह काम करते हैं।

**क्या अनुशंसा सक्षम होने पर भी मैक्रो या बाहरी टूल फ़ाइल को संशोधित कर सकता है?**

हां। अनुशंसा प्रोग्रामेटिक बदलावों को ब्लॉक नहीं करती है। स्वचालित संपादन को रोकने के लिए, [passwords and encryption](/slides/hi/php-java/password-protected-presentation/) का उपयोग करें।

**'Read-Only recommended' 'isEncrypted' और 'isWriteProtected' विधियों से कैसे संबंधित है?**

वे विभिन्न संकेत हैं। 'Read-Only recommended' एक कोमल, वैकल्पिक संकेत है; [isWriteProtected](https://reference.aspose.com/slides/hi/php-java/aspose.slides/protectionmanager/iswriteprotected/) और [isEncrypted](https://reference.aspose.com/slides/hi/php-java/aspose.slides/protectionmanager/isencrypted/) वास्तविक लिखने या पढ़ने प्रतिबंध दर्शाते हैं जो पासवर्ड या एन्क्रिप्शन पर निर्भर करते हैं।
