## **परिचय**

जब आप किसी प्रस्तुति को पासवर्ड से सुरक्षित करते हैं, तो इसका अर्थ है कि आप एक पासवर्ड सेट कर रहे हैं जो प्रस्तुति पर कुछ प्रतिबंध लागू करता है। प्रतिबंध हटाने के लिए पासवर्ड दर्ज करना आवश्यक है। पासवर्ड‑सुरक्षित प्रस्तुति को लॉक्ड प्रस्तुति माना जाता है।

आमतौर पर आप प्रस्तुति पर इन प्रतिबंधों को लागू करने के लिए पासवर्ड सेट कर सकते हैं:

- **संशोधन**

  यदि आप चाहते हैं कि केवल कुछ उपयोगकर्ता आपकी प्रस्तुति को संशोधित कर सकें, तो आप संशोधन प्रतिबंध सेट कर सकते हैं। यह प्रतिबंध लोगों को आपकी प्रस्तुति में संशोधन, परिवर्तन या कॉपी करने से रोकता है (जब तक वे पासवर्ड प्रदान न करें)।

  हालांकि, इस स्थिति में पासवर्ड के बिना भी उपयोगकर्ता आपका दस्तावेज़ खोल सकता है। इस केवल‑पढ़ने मोड में, उपयोगकर्ता आपके प्रस्तुति की सामग्री—हाइपरलिंक्स, एनीमेशन, इफ़ेक्ट्स आदि—को देख सकता है, लेकिन वह आइटम कॉपी या प्रस्तुति को सहेज नहीं सकता।

- **खोलना**

  यदि आप चाहते हैं कि केवल कुछ उपयोगकर्ता आपकी प्रस्तुति खोल सकें, तो आप खोलने का प्रतिबंध सेट कर सकते हैं। यह प्रतिबंध लोगों को आपकी प्रस्तुति की सामग्री देखने से रोकता है (जब तक वे पासवर्ड प्रदान न करें)।

  तकनीकी रूप से, खोलने का प्रतिबंध उपयोगकर्ताओं को आपकी प्रस्तुतियों को संशोधित करने से भी रोकता है: जब लोग प्रस्तुति नहीं खोल सकते, तो वे उसे संशोधित या बदल नहीं सकते।

  **ध्यान दें** कि जब आप प्रस्तुति को खोलने से रोकने के लिए पासवर्ड से सुरक्षित करते हैं, तो प्रस्तुति फ़ाइल एन्क्रिप्ट हो जाती है।

## **Aspose.Slides में प्रस्तुतियों के लिए पासवर्ड सुरक्षा**
**समर्थित फ़ॉर्मेट**

Aspose.Slides इन फ़ॉर्मेटों में प्रस्तुतियों के लिए पासवर्ड सुरक्षा, एन्क्रिप्शन और समान ऑपरेशन्स का समर्थन करता है:

- PPTX और PPT - Microsoft PowerPoint प्रस्तुति
- ODP - OpenDocument प्रस्तुति
- OTP - OpenDocument प्रस्तुति टेम्पलेट

**समर्थित ऑपरेशन्स**

Aspose.Slides आपको निम्न तरीकों से प्रस्तुतियों पर पासवर्ड सुरक्षा का उपयोग करके संशोधनों को रोकने की अनुमति देता है:

- प्रस्तुति को एन्क्रिप्ट करना
- प्रस्तुति पर लिखने की सुरक्षा सेट करना

**अन्य ऑपरेशन्स**

Aspose.Slides आपको निम्न तरीकों से पासवर्ड सुरक्षा और एन्क्रिप्शन से जुड़े अन्य कार्य करने देता है:

- प्रस्तुति को डिक्रिप्ट करना; एन्क्रिप्टेड प्रस्तुति खोलना
- एन्क्रिप्शन हटाना; पासवर्ड सुरक्षा निष्क्रिय करना
- प्रस्तुति से लिखने की सुरक्षा हटाना
- एन्क्रिप्टेड प्रस्तुति की प्रॉपर्टीज़ प्राप्त करना
- जांचना कि प्रस्तुति एन्क्रिप्टेड है या नहीं
- जांचना कि प्रस्तुति पासवर्ड‑सुरक्षित है या नहीं।

## **प्रस्तुति को एन्क्रिप्ट करें**

आप पासवर्ड सेट करके प्रस्तुति को एन्क्रिप्ट कर सकते हैं। तब, लॉक्ड प्रस्तुति को संशोधित करने के लिए उपयोगकर्ता को पासवर्ड प्रदान करना होगा।

प्रेज़ेंटेशन को एन्क्रिप्ट या पासवर्ड‑सुरक्षित करने के लिए आपको [ProtectionManager](https://reference.aspose.com/slides/hi/php-java/aspose.slides/protectionmanager/) की encrypt मेथड का उपयोग करके प्रस्तुति के लिए पासवर्ड सेट करना होगा। आप पासवर्ड को encrypt मेथड में पास करते हैं और अब एन्क्रिप्टेड प्रस्तुति को सहेजने के लिए save मेथड का उपयोग करते हैं।

यह सैंपल कोड आपको दिखाता है कि प्रस्तुति को कैसे एन्क्रिप्ट किया जाए:

```php
  $presentation = new Presentation("pres.pptx");
  try {
    $presentation->getProtectionManager()->encrypt("123123");
    $presentation->save("encrypted-pres.pptx", SaveFormat::Pptx);
  } finally {
    if (!java_is_null($presentation)) {
      $presentation->dispose();
    }
  }
```

## **प्रेज़ेंटेशन में लिखने की सुरक्षा सेट करें**

आप प्रस्तुति में “Do not modify” जैसा संदेश जोड़ सकते हैं। इस तरह आप उपयोगकर्ताओं को बता सकते हैं कि आप चाहते हैं कि वे प्रस्तुति में बदलाव न करें।

**ध्यान दें** कि लिखने की सुरक्षा प्रक्रिया प्रस्तुति को एन्क्रिप्ट नहीं करती। इसलिए, उपयोगकर्ता—यदि चाहें—प्रस्तुति को संशोधित कर सकते हैं, लेकिन बदलावों को सहेजने के लिए उन्हें अलग नाम के साथ नई प्रस्तुति बनानी होगी।

लिखने की सुरक्षा सेट करने के लिए आपको [setWriteProtection](https://reference.aspose.com/slides/hi/php-java/aspose.slides/protectionmanager/#setWriteProtection) मेथड का उपयोग करना होगा। यह सैंपल कोड आपको दिखाता है कि प्रस्तुति में लिखने की सुरक्षा कैसे सेट की जाए:

```php
  $presentation = new Presentation("pres.pptx");
  try {
    $presentation->getProtectionManager()->setWriteProtection("123123");
    $presentation->save("write-protected-pres.pptx", SaveFormat::Pptx);
  } finally {
    if (!java_is_null($presentation)) {
      $presentation->dispose();
    }
  }
```

## **एन्क्रिप्टेड प्रस्तुति लोड करें**

Aspose.Slides आपको उसका पासवर्ड पास करके एन्क्रिप्टेड फ़ाइल लोड करने देता है। प्रस्तुति को डिक्रिप्ट करने के लिए आपको [removeEncryption](https://reference.aspose.com/slides/hi/php-java/aspose.slides/protectionmanager/#removeEncryption) मेथड को बिना पैरामीटर के कॉल करना होगा। फिर सही पासवर्ड दर्ज करके आप प्रस्तुति लोड कर सकते हैं।

यह सैंपल कोड आपको दिखाता है कि प्रस्तुति को कैसे डिक्रिप्ट किया जाए:

```php
  $loadOptions = new LoadOptions();
  $loadOptions->setPassword("123123");
  $presentation = new Presentation("pres.pptx", $loadOptions);
  try {
    # डिक्रिप्टेड प्रस्तुति के साथ काम करें
  } finally {
    if (!java_is_null($presentation)) {
      $presentation->dispose();
    }
  }
```

## **प्रेज़ेंटेशन से एन्क्रिप्शन हटाएँ**

आप प्रस्तुति से एन्क्रिप्शन या पासवर्ड सुरक्षा हटा सकते हैं। इस तरह उपयोगकर्ता बिना किसी प्रतिबंध के प्रस्तुति तक पहुँच या उसे संशोधित कर सकते हैं।

एन्क्रिप्शन या पासवर्ड सुरक्षा हटाने के लिए आपको [removeEncryption](https://reference.aspose.com/slides/hi/php-java/aspose.slides/protectionmanager/#removeEncryption) मेथड को कॉल करना होगा। यह सैंपल कोड आपको दिखाता है कि प्रस्तुति से एन्क्रिप्शन कैसे हटाया जाए:

```php
  $loadOptions = new LoadOptions();
  $loadOptions->setPassword("123123");
  $presentation = new Presentation("pres.pptx", $loadOptions);
  try {
    $presentation->getProtectionManager()->removeEncryption();
    $presentation->save("encryption-removed.pptx", SaveFormat::Pptx);
  } finally {
    if (!java_is_null($presentation)) {
      $presentation->dispose();
    }
  }
```

## **प्रेज़ेंटेशन से लिखने की सुरक्षा हटाएँ**

आप Aspose.Slides का उपयोग करके प्रस्तुति फ़ाइल से लिखने की सुरक्षा हटा सकते हैं। इस तरह उपयोगकर्ता अपनी इच्छा से संशोधन कर सकते हैं—और उन्हें ऐसी कार्रवाई करने पर कोई चेतावनी नहीं मिलेगी।

आप [removeWriteProtection](https://reference.aspose.com/slides/hi/php-java/aspose.slides/protectionmanager/#removeWriteProtection) मेथड का उपयोग करके प्रस्तुति से लिखने की सुरक्षा हटा सकते हैं। यह सैंपल कोड आपको दिखाता है कि लिखने की सुरक्षा कैसे हटाई जाती है:

```php
  $presentation = new Presentation("pres.pptx");
  try {
    $presentation->getProtectionManager()->removeWriteProtection();
    $presentation->save("write-protection-removed.pptx", SaveFormat::Pptx);
  } finally {
    if (!java_is_null($presentation)) {
      $presentation->dispose();
    }
  }
```

## **एन्क्रिप्टेड प्रस्तुति की प्रॉपर्टीज़ प्राप्त करें**

आम तौर पर उपयोगकर्ता एन्क्रिप्टेड या पासवर्ड‑सुरक्षित प्रस्तुति की डॉक्यूमेंट प्रॉपर्टीज़ प्राप्त करने में दिक्कत महसूस करते हैं। Aspose.Slides, हालांकि, एक ऐसा तंत्र प्रदान करता है जिससे आप प्रस्तुति को पासवर्ड‑सुरक्षित करते हुए उपयोगकर्ताओं को उसकी प्रॉपर्टीज़ तक पहुँच की सुविधा भी दे सकते हैं।

**ध्यान दें** कि जब Aspose.Slides प्रस्तुति को एन्क्रिप्ट करता है, तो प्रस्तुति की डॉक्यूमेंट प्रॉपर्टीज़ भी स्वचालित रूप से पासवर्ड‑सुरक्षित हो जाती हैं। लेकिन यदि आप चाहते हैं कि प्रस्तुति एन्क्रिप्ट होने के बाद भी उसकी प्रॉपर्टीज़ पहुंच योग्य रहें, तो Aspose.Slides आपको यही करने की अनुमति देता है।

यदि आप चाहते हैं कि उपयोगकर्ता आपके एन्क्रिप्टेड प्रस्तुति की प्रॉपर्टीज़ तक पहुँच बनाए रखें, तो आप [encryptDocumentProperties](https://reference.aspose.com/slides/hi/php-java/aspose.slides/protectionmanager/#getEncryptDocumentProperties) मेथड को `true` मान के साथ उपयोग कर सकते हैं। यह सैंपल कोड दिखाता है कि प्रस्तुति को एन्क्रिप्ट करते हुए उपयोगकर्ताओं को उसकी डॉक्यूमेंट प्रॉपर्टीज़ तक पहुँच कैसे प्रदान की जाए:

```php
  $presentation = new Presentation("pres.pptx");
  try {
    $presentation->getProtectionManager()->setEncryptDocumentProperties(true);
    $presentation->getProtectionManager()->encrypt("123123");
  } finally {
    if (!java_is_null($presentation)) {
      $presentation->dispose();
    }
  }
```

## **जाँचें कि प्रस्तुति पासवर्ड‑सुरक्षित है या नहीं**

प्रेज़ेंटेशन लोड करने से पहले आप यह जांचना चाह सकते हैं कि वह पासवर्ड से सुरक्षित है या नहीं। इस तरह आप उन त्रुटियों और समान मुद्दों से बच सकते हैं जो पासवर्ड‑सुरक्षित प्रस्तुति को बिना पासवर्ड के लोड करने पर उत्पन्न होते हैं।

यह PHP कोड आपको दिखाता है कि लोड किए बिना यह कैसे जाँचें कि प्रस्तुति पासवर्ड‑सुरक्षित है या नहीं:

```php
  $presentationInfo = PresentationFactory->getInstance()->getPresentationInfo("example.pptx");
  echo("The presentation is password protected: " . $presentationInfo->isPasswordProtected());

```

## **जाँचें कि प्रस्तुति एन्क्रिप्टेड है या नहीं**

Aspose.Slides आपको यह जाँचने देता है कि प्रस्तुति एन्क्रिप्टेड है या नहीं। इस कार्य के लिए आप [isEncrypted](https://reference.aspose.com/slides/hi/php-java/aspose.slides/protectionmanager/#isEncrypted) मेथड का उपयोग कर सकते हैं, जो प्रस्तुति एन्क्रिप्टेड होने पर `true` और नहीं होने पर `false` लौटाता है।

यह सैंपल कोड आपको दिखाता है कि प्रस्तुति एन्क्रिप्टेड है या नहीं कैसे जाँचें:

```php
  $presentation = new Presentation("pres.pptx");
  try {
    $isEncrypted = $presentation->getProtectionManager()->isEncrypted();
  } finally {
    if (!java_is_null($presentation)) {
      $presentation->dispose();
    }
  }
```

## **जाँचें कि प्रस्तुति लिखने‑सुरक्षित है या नहीं**

Aspose.Slides आपको यह जाँचने देता है कि प्रस्तुति लिखने‑सुरक्षित है या नहीं। इस कार्य के लिए आप [isWriteProtected](https://reference.aspose.com/slides/hi/php-java/aspose.slides/protectionmanager/#isWriteProtected) मेथड का उपयोग कर सकते हैं, जो प्रस्तुति लिखने‑सुरक्षित होने पर `true` और नहीं होने पर `false` लौटाता है।

यह सैंपल कोड आपको दिखाता है कि लिखने‑सुरक्षित प्रस्तुति को कैसे जाँचें:

```php
  $presentation = new Presentation("pres.pptx");
  try {
    $isEncrypted = $presentation->getProtectionManager()->isWriteProtected();
  } finally {
    if (!java_is_null($presentation)) {
      $presentation->dispose();
    }
  }
```

## **पुष्टि करें कि कोई विशिष्ट पासवर्ड उपयोग किया गया है या नहीं**

आप यह जांचना और पुष्टि करना चाह सकते हैं कि प्रस्तुति दस्तावेज़ को सुरक्षित करने के लिए कोई विशिष्ट पासवर्ड उपयोग किया गया है या नहीं। Aspose.Slides आपको पासवर्ड वैधता जाँचने का तरीका प्रदान करता है।

यह सैंपल कोड आपको दिखाता है कि पासवर्ड कैसे वैध किया जाए:

```php
  $presentation = new Presentation("pres.pptx");
  try {
    # जाँचें कि "pass" मेल खाता है
    $isWriteProtected = $presentation->getProtectionManager()->checkWriteProtection("my_password");
  } finally {
    if (!java_is_null($presentation)) {
      $presentation->dispose();
    }
  }
```

यह `true` लौटाता है यदि प्रस्तुति निर्दिष्ट पासवर्ड से एन्क्रिप्ट की गई है। अन्यथा यह `false` लौटाता है।

## **FAQ**

**Aspose.Slides द्वारा कौन‑से एन्क्रिप्शन मेथड्स समर्थित हैं?**

Aspose.Slides आधुनिक एन्क्रिप्शन मेथड्स का समर्थन करता है, जिनमें AES‑आधारित एल्गोरिद्म शामिल हैं, जिससे आपकी प्रस्तुतियों के डेटा की सुरक्षा उच्च स्तर की बनी रहती है।

**यदि प्रस्तुति खोलते समय गलत पासवर्ड दर्ज किया जाए तो क्या होता है?**

यदि गलत पासवर्ड उपयोग किया जाता है तो एक एक्सेप्शन उत्पन्न होता है, जिससे पता चलता है कि प्रस्तुति तक पहुँच अस्वीकृत है। यह अनधिकृत पहुँच को रोकने और प्रस्तुति सामग्री की सुरक्षा में मदद करता है।

**क्या पासवर्ड‑सुरक्षित प्रस्तुतियों के साथ काम करने पर प्रदर्शन पर कोई असर पड़ता है?**

एन्क्रिप्शन और डिक्रिप्शन प्रक्रिया खोलने और सहेजने के कार्यों के दौरान थोड़ी अतिरिक्त समय ले सकती है। अधिकांश मामलों में यह प्रदर्शन प्रभाव न्यूनतम होता है और आपके कार्यों की कुल प्रोसेसिंग समय को उल्लेखनीय रूप से प्रभावित नहीं करता।
