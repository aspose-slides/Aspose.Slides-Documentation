## **अवलोकन**

यह लेख .NET में Open XML SDK का उपयोग करके प्रस्तुति फाइलों से पाठ निकालने के तरीके को समझाता है। यह PPTX फ़ाइलों के लिए सीधे XML पहुंच पर केंद्रित है, जहाँ संरचित स्लाइड तत्वों से स्लाइडों को रेंडर किए बिना या Microsoft PowerPoint की आवश्यकता के बिना पाठ प्राप्त किया जा सकता है। लेख में तेज़ प्रोसेसिंग और कम मेमोरी उपयोग जैसी प्रदर्शन लाभों का भी उल्लेख किया गया है।

PPT और ODP फाइलों के लिए, लेख बताता है कि Open XML SDK के साथ पाठ को सीधे निकालना संभव नहीं है। इसके बजाय, इन फॉर्मेट्स को पहले PPTX में परिवर्तित करना आवश्यक है, जिसके बाद परिणामी फ़ाइल से पाठ निकाला जा सकता है।

## **Open XML SDK**

**Open XML SDK** एक अत्यधिक संरचित और कुशल विधि प्रदान करता है जिससे प्रस्तुति फ़ाइलों—विशेषकर **PPTX**—से पाठ निकाला जा सकता है, जो Open XML मानक का पालन करता है। मूल XML तक सीधे पहुँच प्रदान करके, यह SDK पारंपरिक तरीकों की तुलना में स्लाइड सामग्री को तेज़ और अधिक लचीले ढंग से संभालने में सक्षम बनाता है।

## **सीधे XML पहुँच**

- **टेक्स्ट का सीधे विश्लेषण**: Open XML SDK आपको स्लाइडों को रेंडर किए बिना XML भागों से टेक्स्ट निकालने की अनुमति देता है।
- **संरचित तत्व**: क्योंकि टेक्स्ट सुव्यवस्थित XML टैग में संग्रहीत होता है, इसे प्राप्त करना और प्रोसेस करना आसान होता है।

### **उदाहरण: स्लाइड XML सामग्री से सीधे टेक्स्ट निकालना**

```csharp
using (PresentationDocument presentation = PresentationDocument.Open("presentation.pptx", false))
{
    var slidePart = presentation.PresentationPart.SlideParts.FirstOrDefault();
    if (slidePart != null)
    {
        var textElements = slidePart.Slide.Descendants<DocumentFormat.OpenXml.Drawing.Text>();
        foreach (var text in textElements)
        {
            Console.WriteLine(text.Text);
        }
    }
}
```

## **प्रदर्शन लाभ**

- **तेज़ निष्कर्षण**: PowerPoint या अन्य उच्च-स्तरीय API को खोलने के ओवरहेड को बाईपास करता है।
- **कम मेमोरी उपयोग**: केवल आवश्यक XML भागों तक पहुँच की जाती है, जिससे संसाधन खपत घटती है।
- **Microsoft PowerPoint की आवश्यकता नहीं**: अतिरिक्त इंस्टालेशन आवश्यकताओं से आपका बोझ हल्का करता है।

### **उदाहरण: पूरी प्रस्तुति लोड किए बिना कुशलतापूर्वक टेक्स्ट निकालना**

```csharp
using (PresentationDocument presentation = PresentationDocument.Open("presentation.pptx", false))
{
    foreach (var slidePart in presentation.PresentationPart.SlideParts)
    {
        var texts = slidePart.Slide.Descendants<DocumentFormat.OpenXml.Drawing.Text>().Select(t => t.Text);
        Console.WriteLine(string.Join(" ", texts));
    }
}
```

## **टेक्स्ट तत्वों की पहचान**

### **प्रस्तुति से टेक्स्ट निकालने के विशिष्ट पहलू**

प्रस्तुति से टेक्स्ट निकालते समय, इन कारकों पर विचार करें:

- **टेक्स्ट विभिन्न सेक्शन में हो सकता है**: सामान्य स्लाइडें, मास्टर स्लाइडें, लेआउट या स्पीकर नोट्स।
- **डिफ़ॉल्ट प्लेसहोल्डर**: मास्टर स्लाइड और लेआउट में प्लेसहोल्डर हो सकते हैं (जैसे, “Click to edit Master title style”) जो वास्तविक प्रस्तुति सामग्री नहीं होते।
- **खाली या छिपे हुए टेक्स्ट को फ़िल्टर करना**: कुछ तत्व खाली हो सकते हैं या प्रदर्शित करने के लिए नहीं बनाए गए हैं।

### **टेक्स्ट वाले टैग**

एक **PPTX** फ़ाइल में, टेक्स्ट आमतौर पर इस रूप में संग्रहीत होता है:

- `<a:t>` तत्व `<a:p>` (पैराग्राफ) के अंदर
- `<a:r>` तत्व (पैराग्राफ के भीतर टेक्स्ट सेगमेंट)

### **उदाहरण: स्लाइड से सभी टेक्स्ट तत्व निकालना**

```csharp
var textElements = slidePart.Slide.Descendants<DocumentFormat.OpenXml.Drawing.Text>();
foreach (var text in textElements)
{
    Console.WriteLine(text.Text);
}
```

## **ODP और PPT**

### **सिधा टेक्स्ट निकालने में असमर्थता**

- **PPTX** के विपरीत, **PPT** (बाइनरी फॉर्मेट) और **ODP** (OpenDocument प्रस्तुति) **Open XML SDK द्वारा समर्थित नहीं हैं**।
- **PPT** सामग्री को बंद बाइनरी फॉर्मेट में संग्रहीत करता है, जिससे टेक्स्ट निकालना जटिल हो जाता है।
- **ODP** **OpenDocument XML** पर निर्भर करता है, जो संरचनात्मक रूप से PPTX से अलग है।

### **विकल्प: PPTX में परिवर्तित करना**

**PPT** या **ODP** से टेक्स्ट निकालने के लिए, अनुशंसित तरीका यह है:

1. **PPT → PPTX** को PowerPoint या किसी थर्ड‑पार्टी टूल का उपयोग करके परिवर्तित करें।
2. **ODP → PPTX** को LibreOffice या PowerPoint के माध्यम से परिवर्तित करें।
3. नई PPTX से Open XML SDK का उपयोग करके **टेक्स्ट निकालें**।

### **उदाहरण: LibreOffice कमांड लाइन के माध्यम से ODP को PPTX में बदलना**

```sh
soffice --headless --convert-to pptx presentation.odp
```

## **समर्थित प्लेटफ़ॉर्म और फ्रेमवर्क**

- **Windows**: .NET Framework 4.6.1 और ऊपर, .NET Core 2.1+, .NET 5/6/7।
- **Linux/macOS**: .NET Core 2.1+, .NET 5/6/7।
- **क्लाउड वातावरण**: Microsoft Azure Functions, AWS Lambda (.NET Core), Docker कंटेनर।
- **ऑफ़िस एप्लिकेशन्स के साथ संगतता**: Microsoft Office की इंस्टालेशन आवश्यक नहीं।
- **समर्थित प्रोग्रामिंग भाषाएँ**: Open XML SDK को **C#**, **VB.NET**, **F#**, और अन्य .NET‑समर्थित भाषाओं के साथ उपयोग किया जा सकता है।

## **निष्कर्ष**

**Open XML SDK** का उपयोग करके **PPTX टेक्स्ट निष्कर्षण** दक्षता और स्पष्टता दोनों प्रदान करता है, जबकि **PPT और ODP** के लिए सुगम प्रोसेसिंग के लिये प्रारम्भिक परिवर्तन कदम आवश्यक होता है। इस दृष्टिकोण को अपनाने से **उच्च प्रदर्शन**, **लचीलापन**, और आधुनिक .NET अनुप्रयोगों के साथ **व्यापक संगतता** सुनिश्चित होती है।
