## **Overview**

Aspose.Slides को मूल्यांकन मोड में या वैध लाइसेंस के साथ इस्तेमाल किया जा सकता है। मूल्यांकन संस्करण लाइसेंस प्राप्त संस्करण के समान कार्यक्षमता प्रदान करता है, लेकिन प्रस्तुतियों को खोलने या सहेजने पर एक मूल्यांकन वॉटरमार्क जोड़ता है और टेक्स्ट निष्कर्षण को केवल एक स्लाइड तक सीमित करता है।

यह लेख Aspose.Slides में लाइसेंसिंग कैसे काम करती है और लाइब्रेरी का उपयोग करने से पहले लाइसेंस कैसे लागू किया जाता है, इसे समझाता है। लाइसेंस को `License` क्लास का उपयोग करके फ़ाइल, स्ट्रीम या एंबेडेड रिसोर्स से लोड किया जा सकता है। लेख यह भी दिखाता है कि लाइसेंस सही तरीके से लागू हुआ है या नहीं, इसे कैसे मान्य किया जाए।

## **Licensing in Aspose.Slides**
* एक मूल्यांकन संस्करण लाइसेंस प्राप्त हो जाता है जब आप लाइसेंस खरीदते हैं और उसमें कुछ पंक्तियों का कोड जोड़ते हैं (लाइसेंस लागू करने के लिए)।
* लाइसेंस एक साधारण टेक्स्ट XML फ़ाइल होती है जिसमें उत्पाद नाम, लाइसेंस प्राप्त डेवलपर्स की संख्या, सब्सक्रिप्शन समाप्ति तिथि आदि विवरण होते हैं। 
* लाइसेंस फ़ाइल डिजिटल रूप से साइन की गई होती है, इसलिए आपको फ़ाइल को संशोधित नहीं करना चाहिए। फ़ाइल की सामग्री में अनजाने में अतिरिक्त लाइन ब्रेक जोड़ने से भी लाइसेंस अमान्य हो जाएगा।
* Aspose.Slides for .NET सामान्यतः लाइसेंस को इन स्थानों पर खोजता है:
  * स्पष्ट पथ
  * घटक की DLL वाले फ़ोल्डर (Aspose.Slides में शामिल)
  * उस असेंबली का फ़ोल्डर जिसने घटक की DLL को कॉल किया (Aspose.Slides में शामिल)
  * एंट्री असेंबली का फ़ोल्डर (आपका .exe)
  * उस असेंबली में एंबेडेड रिसोर्स जहाँ घटक की DLL को कॉल किया गया (Aspose.Slides में शामिल)।
* मूल्यांकन संस्करण से जुड़ी सीमाओं से बचने के लिए, आपको Aspose.Slides का उपयोग करने से पहले लाइसेंस सेट करना होगा। आपको केवल एक बार प्रति एप्लिकेशन या प्रक्रिया लाइसेंस सेट करना होता है।

## **Apply a License**
लाइसेंस को **फ़ाइल**, **स्ट्रीम**, या **एंबेडेड रिसोर्स** से लोड किया जा सकता है। 

{{% alert color="primary" %}}

Aspose.Slides लाइसेंसिंग ऑपरेशनों के लिए [License](https://reference.aspose.com/slides/hi/net/aspose.slides/license) क्लास प्रदान करता है।

{{% /alert %}} 

{{% alert color="warning" %}} 

नए लाइसेंस केवल संस्करण 21.4 या उसके बाद के साथ Aspose.Slides को सक्रिय कर सकते हैं। पहले के संस्करण अलग लाइसेंसिंग सिस्टम का उपयोग करते हैं और इन लाइसेंसों को पहचान नहीं पाएंगे।

{{% /alert %}}

### **File**
लाइसेंस सेट करने की सबसे आसान विधि यह है कि आप लाइसेंस फ़ाइल को उसी फ़ोल्डर में रखें जहाँ घटक की DLL (Aspose.Slides में शामिल) स्थित है और केवल फ़ाइल नाम बिना पथ के निर्दिष्ट करें।

यह C# कोड दिखाता है कि लाइसेंस फ़ाइल कैसे सेट की जाए:

``` csharp
// लाइसेंस क्लास का उदाहरण बनाता है 
Aspose.Slides.License license = new Aspose.Slides.License();

// लाइसेंस फ़ाइल पथ सेट करता है
license.SetLicense("Aspose.Slides.lic");
```

{{% alert color="warning" %}} 

यदि आप लाइसेंस फ़ाइल को किसी अन्य निर्देशिका में रखते हैं, तो जब आप [SetLicense](https://reference.aspose.com/slides/hi/net/aspose.slides/license/setlicense/#setlicense_1) मेथड को कॉल करते हैं, तो निर्दिष्ट स्पष्ट पथ के अंत में लाइसेंस फ़ाइल का नाम आपके लाइसेंस फ़ाइल के समान होना चाहिए।

उदाहरण के लिए, आप लाइसेंस फ़ाइल का नाम *Aspose.Slides.lic.xml* बदल सकते हैं। फिर अपने कोड में आपको फ़ाइल का पथ (जिसके अंत में *Aspose.Slides.lic.xml* हो) को [SetLicense](https://reference.aspose.com/slides/hi/net/aspose.slides/license/setlicense/#setlicense_1) मेथड में पास करना होगा।

{{% /alert %}}

### **Stream**
आप लाइसेंस को एक स्ट्रीम से लोड कर सकते हैं। यह C# कोड दिखाता है कि स्ट्रीम से लाइसेंस कैसे लागू किया जाए:

``` csharp
// लाइसेंस क्लास का उदाहरण बनाता है 
Aspose.Slides.License license = new Aspose.Slides.License();

// स्ट्रीम के माध्यम से लाइसेंस सेट करता है
license.SetLicense(myStream);
```

### **Embedded Resource**
आप लाइसेंस को अपनी एप्लिकेशन के साथ पैकेज कर सकते हैं (ताकि यह खो न जाए) इसे घटक की DLL को कॉल करने वाली किसी असेंबली में एंबेडेड रिसोर्स के रूप में जोड़कर (Aspose.Slides में शामिल)।

यह है कि लाइसेंस फ़ाइल को एंबेडेड रिसोर्स के रूप में कैसे जोड़ें:

1. Visual Studio में, लाइसेंस (.lic) फ़ाइल को प्रोजेक्ट में इस प्रकार जोड़ें: **File** > **Add Existing Item** > **Add**। 
2. **Solution Explorer** में फ़ाइल चुनें।
3. **Properties** विंडो में, **Build Action** को **Embedded Resource** सेट करें।
4. असेंबली में एंबेडेड लाइसेंस तक पहुँचने के लिए, प्रोजेक्ट में लाइसेंस फ़ाइल को एंबेडेड रिसोर्स के रूप में जोड़ें, और फिर `SetLicense` मेथड को लाइसेंस फ़ाइल का नाम पास करें। 

`License` क्लास एंबेडेड रिसोर्स में लाइसेंस फ़ाइल को स्वचालित रूप से खोज लेती है। आपको Microsoft .NET Framework में `System.Reflection.Assembly` क्लास के `GetExecutingAssembly` और `GetManifestResourceStream` मेथड को कॉल करने की आवश्यकता नहीं है।

यह C# कोड दिखाता है कि लाइसेंस को एंबेडेड रिसोर्स के रूप में कैसे सेट किया जाए:

``` csharp
// लाइसेंस क्लास का उदाहरण बनाता है
Aspose.Slides.License license = new Aspose.Slides.License();

// असेंबली में एंबेडेड लाइसेंस फ़ाइल नाम पास करता है
license.SetLicense("Aspose.Slides.lic");
```

## **Validate a License**

यह जांचने के लिए कि लाइसेंस सही ढंग से सेट हुआ है या नहीं, आप इसे मान्य कर सकते हैं। यह C# कोड लाइसेंस को वैलिडेट करने का तरीका दिखाता है:

```c#
Aspose.Slides.License license = new Aspose.Slides.License();

license.SetLicense("Aspose.Slides.lic");

if (license.IsLicensed())
{
    Console.WriteLine("License is good!");
    Console.Read();
}
```

## **FAQ**

**क्या मैं लाइसेंस को पूरी तरह ऑफ़लाइन पर्यावरण (बिना इंटरनेट एक्सेस) में लागू कर सकता हूँ?**

हाँ। लाइसेंस वैलिडेशन स्थानीय रूप से लाइसेंस फ़ाइल का उपयोग करके किया जाता है; इंटरनेट कनेक्शन आवश्यक नहीं है।

**एक‑साल की सब्सक्रिप्शन समाप्त होने के बाद क्या होता है? क्या लाइब्रेरी काम करना बंद कर देगी?**

नहीं। लाइसेंस स्थायी है: आप अपनी सब्सक्रिप्शन समाप्ति तिथि से पहले जारी किए गए संस्करणों को उपयोग जारी रख सकते हैं; लेकिन नए रीलिज़ का उपयोग करने के लिए नवीनीकरण आवश्यक होगा।
