## **परिचय**

जब आप किसी प्रस्तुति को पासवर्ड‑प्रोटेक्ट करते हैं, इसका मतलब है कि आप एक पासवर्ड सेट कर रहे हैं जो प्रस्तुति पर कुछ प्रतिबंध लगाता है। इन प्रतिबंधों को हटाने के लिए पासवर्ड दर्ज करना आवश्यक है। पासवर्ड‑सुरक्षित प्रस्तुति को लॉक्ड प्रस्तुति माना जाता है।

आमतौर पर, आप इन प्रतिबंधों को लागू करने के लिए प्रस्तुति पर एक पासवर्ड सेट कर सकते हैं:

- **संशोधन**

यदि आप चाहते हैं कि केवल कुछ उपयोगकर्ता ही आपकी प्रस्तुति को संशोधित कर सकें, तो आप एक संशोधन प्रतिबंध सेट कर सकते हैं। यह प्रतिबंध लोगों को आपके प्रस्तुति के तत्वों को संशोधित, बदलने या कॉपी करने से रोकता है, जब तक कि वे पासवर्ड न दें।  

हालाँकि, पासवर्ड के बिना भी, उपयोगकर्ता अभी भी आपके दस्तावेज़ को एक्सेस और खोल सकेगा। इस केवल‑पढ़ने मोड में, उपयोगकर्ता आपकी प्रस्तुति की सामग्री—जिसमें हाइपरलिंक, एनीमेशन, इफ़ेक्ट और अन्य तत्व शामिल हैं—देख सकता है, लेकिन वह आइटम कॉपी या प्रस्तुति को सहेज नहीं सकेगा।

- **खोलना**

यदि आप चाहते हैं कि केवल कुछ उपयोगकर्ता ही आपकी प्रस्तुति खोल सकें, तो आप एक खोलने का प्रतिबंध सेट कर सकते हैं। यह प्रतिबंध लोगों को आपके प्रस्तुति की सामग्री को देखने से भी रोकता है, जब तक कि वे पासवर्ड न दें।  

तकनीकी रूप से, खोलने का प्रतिबंध उपयोगकर्ताओं को आपकी प्रस्तुतियों को संशोधित करने से भी रोकता है—यदि लोग प्रस्तुति नहीं खोल सकते, तो वे उसे संशोधित या बदल नहीं सकते।

**नोट:** जब आप किसी प्रस्तुति को खोलने से रोकने के लिए पासवर्ड‑प्रोटेक्ट करते हैं, तो प्रस्तुति फ़ाइल एन्क्रिप्ट हो जाती है।

## **Aspose.Slides में पासवर्ड संरक्षण**

**समर्थित फ़ॉर्मेट**

Aspose.Slides इन फ़ॉर्मेट की प्रस्तुतियों के लिए पासवर्ड संरक्षण, एन्क्रिप्शन और समान ऑपरेशन को सपोर्ट करता है:

- PPTX और PPT – Microsoft PowerPoint प्रस्तुतियाँ
- ODP – OpenDocument प्रस्तुतियाँ
- OTP – OpenDocument प्रस्तुति टेम्पलेट्स

**समर्थित ऑपरेशन**

Aspose.Slides आपको प्रस्तुतियों पर पासवर्ड संरक्षण का उपयोग करके निम्नलिखित तरीकों से संशोधनों को रोकने की अनुमति देता है:

- प्रस्तुति को एन्क्रिप्ट करना
- प्रस्तुति पर लिखने की सुरक्षा स्थापित करना

**अन्य ऑपरेशन**

Aspose.Slides आपको अतिरिक्त कार्य करने देता है जो पासवर्ड संरक्षण और एन्क्रिप्शन से संबंधित हैं, इस प्रकार:

- प्रस्तुति को डिक्रिप्ट करना; एन्क्रिप्टेड प्रस्तुति खोलना
- एन्क्रिप्शन हटाना; पासवर्ड संरक्षण निष्क्रिय करना
- प्रस्तुति से लिखने की सुरक्षा हटाना
- एन्क्रिप्टेड प्रस्तुति की प्रॉपर्टीज़ प्राप्त करना
- लोड करने से पहले जांचना कि प्रस्तुति पासवर्ड‑प्रोटेक्टेड है या नहीं
- यह जांचना कि प्रस्तुति एन्क्रिप्टेड है या नहीं
- यह जांचना कि प्रस्तुति पासवर्ड‑प्रोटेक्टेड है या नहीं

## **पासवर्ड के साथ प्रस्तुति सुरक्षित करें**

आप पासवर्ड सेट करके प्रस्तुति को एन्क्रिप्ट कर सकते हैं। फिर, लॉक्ड प्रस्तुति को संशोधित करने के लिए उपयोगकर्ता को पासवर्ड प्रदान करना होगा।

किसी प्रस्तुति को एन्क्रिप्ट (या पासवर्ड‑प्रोटेक्ट) करने के लिए, पासवर्ड सेट करने हेतु `Encrypt` मेथड का उपयोग करें, जो आप [ProtectionManager](https://reference.aspose.com/slides/hi/net/aspose.slides/protectionmanager) से प्राप्त कर सकते हैं। पासवर्ड को `Encrypt` मेथड में पास करें, फिर `Save` मेथड का उपयोग करके अब एन्क्रिप्टेड प्रस्तुति को सहेजें।

यह नमूना कोड दिखाता है कि प्रस्तुति को कैसे एन्क्रिप्ट किया जाए:

```c#
using (Presentation presentation = new Presentation("pres.pptx"))
{
    presentation.ProtectionManager.Encrypt("123123");
    presentation.Save("encrypted-pres.pptx", SaveFormat.Pptx);
}
```

## **प्रस्तुति पर लिखने की सुरक्षा सेट करें** 

आप प्रस्तुति में "Do not modify" (संशोधित न करें) का निशान जोड़ सकते हैं। यह उपयोगकर्ताओं को सूचित करता है कि आप नहीं चाहते कि वे प्रस्तुति में परिवर्तन करें।

**नोट:** लिखने की सुरक्षा प्रक्रिया प्रस्तुति को एन्क्रिप्ट नहीं करती। इसलिए, उपयोगकर्ता—यदि वे चाहें—प्रस्तुति को संशोधित कर सकते हैं, लेकिन बदलाव सहेजने के लिए उन्हें इसे किसी अलग नाम से सहेजना होगा।

लिखने की सुरक्षा सेट करने के लिए, `SetWriteProtection` मेथड का उपयोग करें। यह नमूना कोड दिखाता है कि प्रस्तुति पर लिखने की सुरक्षा कैसे सेट की जाए:

```c#
using (Presentation presentation = new Presentation("pres.pptx"))
{
    presentation.ProtectionManager.SetWriteProtection("123123");
    presentation.Save("write-protected-pres.pptx", SaveFormat.Pptx);
}
```

## **एन्क्रिप्टेड प्रस्तुति लोड करें**

Aspose.Slides आपको सही पासवर्ड पास करके एन्क्रिप्टेड प्रस्तुति लोड करने की अनुमति देता है। यह नमूना कोड दिखाता है कि एन्क्रिप्टेड प्रस्तुति को कैसे लोड किया जाए:

```c#
LoadOptions loadOptions = new LoadOptions { Password = "123123" };
using (Presentation presentation = new Presentation("pres.pptx", loadOptions))
{
    // डिक्रिप्टेड प्रस्तुति के साथ काम करें।
}
```

## **प्रस्तुति से एन्क्रिप्शन हटाएँ**

आप प्रस्तुति से एन्क्रिप्शन या पासवर्ड संरक्षण को हटा सकते हैं, जिससे उपयोगकर्ता बिना किसी प्रतिबंध के इसे एक्सेस या संशोधित कर सकें।

एन्क्रिप्शन या पासवर्ड संरक्षण हटाने के लिए, [RemoveEncryption](https://reference.aspose.com/slides/hi/net/aspose.slides/protectionmanager/methods/removeencryption) मेथड को कॉल करें। यह नमूना कोड दिखाता है कि प्रस्तुति से एन्क्रिप्शन कैसे हटाया जाए:

```c#
LoadOptions loadOptions = new LoadOptions { Password = "123123" };
using (Presentation presentation = new Presentation("pres.pptx", loadOptions))
{
    presentation.ProtectionManager.RemoveEncryption();
    presentation.Save("encryption-removed.pptx", SaveFormat.Pptx);
}
```

## **प्रस्तुति से लिखने की सुरक्षा हटाएँ**

आप Aspose.Slides का उपयोग करके प्रस्तुति फ़ाइल से लिखने की सुरक्षा हटा सकते हैं। इस तरह, उपयोगकर्ता इसे अपनी इच्छानुसार संशोधित कर सकते हैं—और ऐसे कार्य करते समय उन्हें कोई चेतावनी नहीं मिलेगी।

आप [RemoveWriteProtection](https://reference.aspose.com/slides/hi/net/aspose.slides/protectionmanager/methods/removewriteprotection) मेथड का उपयोग करके लिखने की सुरक्षा हटा सकते हैं। यह नमूना कोड दिखाता है कि प्रस्तुति से लिखने की सुरक्षा कैसे हटाई जाए:

```c#
using (Presentation presentation = new Presentation("pres.pptx"))
{
    presentation.ProtectionManager.RemoveWriteProtection();
    presentation.Save("write-protection-removed.pptx", SaveFormat.Pptx);
}
```

## **एन्क्रिप्टेड प्रस्तुति की प्रॉपर्टीज़ प्राप्त करें**

आमतौर पर, उपयोगकर्ताओं को एन्क्रिप्टेड या पासवर्ड‑प्रोटेक्टेड प्रस्तुति के दस्तावेज़ प्रॉपर्टीज़ प्राप्त करने में कठिनाई होती है। हालांकि, Aspose.Slides एक ऐसी विधि प्रदान करता है जो प्रस्तुति को पासवर्ड‑प्रोटेक्ट करने के साथ‑साथ उपयोगकर्ताओं को उसकी प्रॉपर्टीज़ तक पहुँच करने की क्षमता भी रखती है।

**नोट:** डिफ़ॉल्ट रूप से, जब Aspose.Slides प्रस्तुति को एन्क्रिप्ट करता है, तो प्रस्तुति की दस्तावेज़ प्रॉपर्टीज़ भी पासवर्ड‑प्रोटेक्टेड हो जाती हैं। यदि आपको एन्क्रिप्शन के बाद भी दस्तावेज़ प्रॉपर्टीज़ को एक्सेस करना हो, तो Aspose.Slides आपको यह करने की अनुमति देता है।

यदि आप चाहते हैं कि उपयोगकर्ता एन्क्रिप्टेड प्रस्तुति की प्रॉपर्टीज़ तक पहुँच बना रखें, तो आप [EncryptDocumentProperties](https://reference.aspose.com/slides/hi/net/aspose.slides/protectionmanager/properties/encryptdocumentproperties) प्रॉपर्टी को `true` सेट कर सकते हैं। यह नमूना कोड दिखाता है कि प्रस्तुति को एन्क्रिप्ट करते हुए भी उपयोगकर्ता को उसकी दस्तावेज़ प्रॉपर्टीज़ तक पहुँच कैसे प्रदान की जाए:

```c#
using (Presentation presentation = new Presentation("pres.pptx"))
{
    presentation.ProtectionManager.EncryptDocumentProperties = true;
    presentation.ProtectionManager.Encrypt("123123");
}
```

## **जांचें कि क्या प्रस्तुति पासवर्ड‑सुरक्षित है**

प्रस्तुति लोड करने से पहले, आप यह जांचना चाह सकते हैं कि क्या वह पासवर्ड से सुरक्षित नहीं है। यह आपको त्रुटियों और उन समस्याओं से बचाता है जो तब उत्पन्न होती हैं जब सही पासवर्ड के बिना पासवर्ड‑प्रोटेक्टेड प्रस्तुति लोड की जाती है।

यह C# कोड दिखाता है कि प्रस्तुति को वास्तव में लोड किए बिना यह कैसे जांचा जाए कि वह पासवर्ड‑प्रोटेक्टेड है या नहीं:

```c#
var presentationInfo = PresentationFactory.Instance.GetPresentationInfo("example.pptx");
Console.WriteLine("The presentation is password protected: " + presentationInfo.IsPasswordProtected);
```

## **जांचें कि क्या प्रस्तुति एन्क्रिप्टेड है**

Aspose.Slides आपको यह जांचने की अनुमति देता है कि प्रस्तुति एन्क्रिप्टेड है या नहीं। इस कार्य के लिए, आप [IsEncrypted](https://reference.aspose.com/slides/hi/net/aspose.slides/protectionmanager/properties/isencrypted) प्रॉपर्टी का उपयोग कर सकते हैं, जो `true` लौटाती है यदि प्रस्तुति एन्क्रिप्टेड है, अन्यथा `false`।

यह नमूना कोड दिखाता है कि प्रस्तुति एन्क्रिप्टेड है या नहीं, यह कैसे जांचें:

```c#
using (Presentation presentation = new Presentation("pres.pptx"))
{
    bool isEncrypted = presentation.ProtectionManager.IsEncrypted;
}
```

## **जांचें कि क्या प्रस्तुति लिखने की सुरक्षा रखती है**

Aspose.Slides आपको यह जांचने की अनुमति देता है कि प्रस्तुति लिखने‑प्रोटेक्टेड है या नहीं। इस कार्य के लिए, आप [IsWriteProtected](https://reference.aspose.com/slides/hi/net/aspose.slides/protectionmanager/properties/iswriteprotected) प्रॉपर्टी का उपयोग कर सकते हैं, जो `true` लौटाती है यदि प्रस्तुति लिखने‑प्रोटेक्टेड है, अन्यथा `false`।

यह नमूना कोड दिखाता है कि प्रस्तुति लिखने‑प्रोटेक्टेड है या नहीं, यह कैसे जांचें:

```c#
using (Presentation presentation = new Presentation("pres.pptx"))
{
    bool isEncrypted = presentation.ProtectionManager.IsWriteProtected;
}
```

## **प्रस्तुति पासवर्ड उपयोग की पुष्टि करें**

आप यह जांच और पुष्टि करना चाह सकते हैं कि किसी विशिष्ट पासवर्ड का उपयोग प्रस्तुति दस्तावेज़ को सुरक्षित करने के लिए किया गया है। Aspose.Slides आपको पासवर्ड वैधता जांचने का साधन प्रदान करता है।

यह नमूना कोड दिखाता है कि पासवर्ड को कैसे वैध किया जाए:

```c#
using (IPresentation presentation = new Presentation("pres.pptx"))
{
    // जांचें कि पासवर्ड मेल खाता है या नहीं।
    bool isWriteProtected = presentation.ProtectionManager.CheckWriteProtection("my_password");
}
```

यह `true` लौटाता है यदि प्रस्तुति निर्दिष्ट पासवर्ड से एन्क्रिप्ट की गई है; अन्यथा यह `false` लौटाता है।

## **ऑनलाइन प्रस्तुति को पासवर्ड‑प्रोटेक्ट करें**

## **अक्सर पूछे जाने वाले प्रश्न**

**Aspose.Slides द्वारा कौन‑से एन्क्रिप्शन विधियों का समर्थन किया जाता है?**  
Aspose.Slides आधुनिक एन्क्रिप्शन विधियों का समर्थन करता है, जिसमें AES‑आधारित एल्गोरिदम शामिल हैं, जिससे आपकी प्रस्तुतियों के डेटा की उच्च सुरक्षा सुनिश्चित होती है।

**यदि प्रस्तुति खोलने का प्रयास करते समय गलत पासवर्ड दर्ज किया जाता है तो क्या होता है?**  
यदि गलत पासवर्ड उपयोग किया जाता है तो एक अपवाद (exception) उत्पन्न होता है, जो आपको बताता है कि प्रस्तुति तक पहुँच अस्वीकार कर दी गई है। यह अनधिकृत पहुँच को रोकने में मदद करता है और प्रस्तुति की सामग्री की रक्षा करता है।

**क्या पासवर्ड‑प्रोटेक्टेड प्रस्तुतियों के साथ काम करने पर प्रदर्शन संबंधी प्रभाव पड़ते हैं?**  
एन्क्रिप्शन और डिक्रिप्शन प्रक्रिया खोलने और सहेजने के कार्यों के दौरान थोड़ा अतिरिक्त ओवरहेड जोड़ सकती है। अधिकांश मामलों में, यह प्रदर्शन प्रभाव न्यूनतम होता है और आपके प्रस्तुति कार्यों के समग्र प्रोसेसिंग समय को महत्वपूर्ण रूप से प्रभावित नहीं करता।
