## **अवलोकन**

Aspose.Slides आपको डिफ़ॉल्ट फ़ॉन्ट निर्दिष्ट करने की अनुमति देता है जो प्रस्तुति के रेंडर होने पर उपयोग किए जाते हैं। यह स्लाइड थंबनेल जनरेट करने या प्रस्तुति को PDF और XPS जैसे फ़ॉर्मेट में एक्सपोर्ट करने के समय उपयोगी है। डिफ़ॉल्ट फ़ॉन्ट `LoadOptions` के माध्यम से कॉन्फ़िगर किए जाते हैं, इससे पहले कि प्रस्तुति लोड हो।

`DefaultRegularFont` प्रॉपर्टी नियमित टेक्स्ट के लिए डिफ़ॉल्ट फ़ॉन्ट परिभाषित करती है, जबकि `DefaultAsianFont` एशियन टेक्स्ट के लिए डिफ़ॉल्ट फ़ॉन्ट परिभाषित करती है। इन विकल्पों को सेट करने के बाद, प्रस्तुति को लोड किया जा सकता है और निर्दिष्ट फ़ॉन्ट्स का उपयोग करके रेंडर किया जा सकता है।

## **प्रस्तुति को रेंडर करने के लिए डिफ़ॉल्ट फ़ॉन्ट का उपयोग**

Aspose.Slides आपको प्रस्तुति को PDF, XPS या थंबनेल में रेंडर करने के लिए डिफ़ॉल्ट फ़ॉन्ट सेट करने देता है। यह लेख दिखाता है कि DefaultRegularFont और DefaultAsianFont को डिफ़ॉल्ट फ़ॉन्ट के रूप में कैसे परिभाषित किया जाए। कृपया नीचे दिए गए चरणों का पालन करें ताकि Aspose.Slides for .NET API का उपयोग करके बाहरी डायरेक्टरीज़ से फ़ॉन्ट लोड किए जा सकें:

1. LoadOptions का एक उदाहरण बनाएं।  
2. DefaultRegularFont को अपनी इच्छित फ़ॉन्ट पर सेट करें। निम्न उदाहरण में, मैंने **Wingdings** का उपयोग किया है।  
3. DefaultAsianFont को अपनी इच्छित फ़ॉन्ट पर सेट करें। मैंने नीचे के उदाहरण में **Wingdings** का उपयोग किया है।  
4. Presentation का उपयोग करके प्रस्तुति लोड करें और लोड विकल्प सेट करें।  
5. अब, स्लाइड थंबनेल, PDF और XPS उत्पन्न करें ताकि परिणामों की पुष्टि हो सके।

ऊपर दिया गया कार्यान्वयन नीचे दिया गया है।

```c#
// लोड विकल्पों का उपयोग करके डिफ़ॉल्ट रेगुलर और एशियन फ़ॉन्ट निर्दिष्ट करें
LoadOptions loadOptions = new LoadOptions(LoadFormat.Auto);
loadOptions.DefaultRegularFont = "Wingdings";
loadOptions.DefaultAsianFont = "Wingdings";

using (Presentation pptx = new Presentation("DefaultFonts.pptx", loadOptions))
{
    using (IImage image = pptx.Slides[0].GetImage(1, 1))
    {
        image.Save("DefaultFonts_out.png", ImageFormat.Png);
    }

    pptx.Save("DefaultFonts_out.pdf", SaveFormat.Pdf);
    pptx.Save("DefaultFonts_out.xps", SaveFormat.Xps);
}
```

## **अक्सर पूछे जाने वाले प्रश्न**

**DefaultRegularFont और DefaultAsianFont वास्तव में क्या प्रभावित करते हैं—केवल एक्सपोर्ट, या थंबनेल, PDF, XPS, HTML, और SVG भी?**  
वे सभी समर्थित आउटपुट्स के रेंडरिंग पाइपलाइन में भाग लेते हैं। इसमें स्लाइड थंबनेल, [PDF](/slides/hi/net/convert-powerpoint-to-pdf/), [XPS](/slides/hi/net/convert-powerpoint-to-xps/), [raster images](/slides/hi/net/convert-powerpoint-to-png/), [HTML](/slides/hi/net/convert-powerpoint-to-html/), और [SVG](/slides/hi/net/render-a-slide-as-an-svg-image/) शामिल हैं, क्योंकि Aspose.Slides इन लक्ष्यों के बीच समान लेआउट और ग्लिफ़ रेज़ॉल्यूशन लॉजिक का उपयोग करता है।

**क्या डिफ़ॉल्ट फ़ॉन्ट केवल PPTX को पढ़ने और सहेजने पर लगाए जाते हैं, बिना किसी रेंडरिंग के?**  
नहीं। डिफ़ॉल्ट फ़ॉन्ट तब मायने रखते हैं जब टेक्स्ट को मापना और ड्रॉ करना आवश्यक हो। प्रस्तुति को केवल खोल कर सहेजने से संग्रहीत फ़ॉन्ट रन या फ़ाइल की संरचना नहीं बदलती। डिफ़ॉल्ट फ़ॉन्ट उन ऑपरेशनों में उपयोग होते हैं जो टेक्स्ट को रेंडर या रीफ़्लो करते हैं।

**यदि मैं अपने स्वयं के फ़ॉन्ट फोल्डर जोड़ूँ या मेमोरी से फ़ॉन्ट प्रदान करूँ, क्या वे डिफ़ॉल्ट फ़ॉन्ट चुनते समय berücksichtigt किए जाएंगे?**  
हां। [Custom font sources](/slides/hi/net/custom-font/) उपलब्ध फ़ॉन्ट परिवार और ग्लिफ़ की सूची का विस्तार करते हैं जिन्हें इंजन उपयोग कर सकता है। डिफ़ॉल्ट फ़ॉन्ट और कोई भी [fallback rules](/slides/hi/net/fallback-font/) पहले इन स्रोतों के विरुद्ध हल होते हैं, जिससे सर्वर और कंटेनर में अधिक विश्वसनीय कवरेज मिलता है।

**क्या डिफ़ॉल्ट फ़ॉन्ट टेक्स्ट मेट्रिक्स (केरनिंग, एडवांस) को प्रभावित करेंगे और इस प्रकार लाइन ब्रेक और रैपिंग को?**  
हां। फ़ॉन्ट बदलने से ग्लिफ़ मेट्रिक्स बदलते हैं और रेंडरिंग के दौरान लाइन ब्रेक, रैपिंग और पेजिनेशन प्रभावित हो सकते हैं। लेआउट स्थिरता के लिए, [embed the original fonts](/slides/hi/net/embedded-font/) या मेट्रिकली संगत डिफ़ॉल्ट एवं फॉलबैक फ़ॉन्ट परिवार चुनें।

**यदि प्रस्तुति में उपयोग किए सभी फ़ॉन्ट एम्बेडेड हैं, तो डिफ़ॉल्ट फ़ॉन्ट सेट करने का कोई मतलब है?**  
अक्सर आवश्यक नहीं होता, क्योंकि [embedded fonts](/slides/hi/net/embedded-font/) पहले से ही सुसंगत रूप सुनिश्चित करते हैं। डिफ़ॉल्ट फ़ॉन्ट अभी भी उन कैरैक्टरों के लिए सुरक्षा जाल प्रदान करते हैं जो एम्बेडेड उपसमुच्चय में शामिल नहीं हैं, या जब फ़ाइल में एम्बेडेड और गैर-एंबेडेड टेक्स्ट दोनों मिश्रित हों।
