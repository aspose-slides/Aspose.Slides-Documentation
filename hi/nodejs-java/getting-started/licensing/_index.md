## **Aspose.Slides का मूल्यांकन करें**
आप आसानी से Aspose.Slides को मूल्यांकन हेतु डाउनलोड कर सकते हैं। मूल्यांकन पैकेज खरीदे गए पैकेज के समान है। केवल कुछ कोड लाइनों को जोड़कर लाइसेंस लागू करने के बाद मूल्यांकन संस्करण स्वचालित रूप से लाइसेंस्ड बन जाता है। 

## **मूल्यांकन संस्करण की सीमा**
Aspose.Slides का मूल्यांकन संस्करण (बिना किसी लाइसेंस के) पूरी उत्पाद कार्यक्षमता प्रदान करता है, लेकिन यह खोलने और सहेजने पर दस्तावेज़ के शीर्ष पर एक मूल्यांकन वॉटरमार्क डालता है। प्रस्तुति स्लाइड्स से टेक्स्ट निकालते समय आप केवल एक स्लाइड तक सीमित रहते हैं।

## **लाइसेंस के बारे में**
आप आसानी से Aspose.Slides का मूल्यांकन संस्करण Node.js के लिए Java के माध्यम से उसके [download page](https://releases.aspose.com/slides/hi/nodejs-java/) से डाउनलोड कर सकते हैं। मूल्यांकन संस्करण Aspose.Slides के लाइसेंस्ड संस्करण के समान **वही क्षमताएँ** प्रदान करता है। आगे, लाइसेंस खरीदने और कुछ कोड लाइनों को जोड़ने के बाद यह स्वचालित रूप से लाइसेंस्ड बन जाता है।

लाइसेंस एक साधारण-टेक्स्ट XML फ़ाइल है जिसमें उत्पाद नाम, लाइसेंस प्राप्त डेवलपर्स की संख्या, सब्सक्रिप्शन समाप्ति तिथि आदि विवरण शामिल होते हैं। फ़ाइल डिजिटल रूप से हस्ताक्षरित है, इसलिए इसे संशोधित न करें। फ़ाइल की सामग्री में अनजाने में एक अतिरिक्त लाइन ब्रेक जोड़ने से भी यह अवैध हो जाएगा।

मूल्यांकन संस्करण की सीमाओं से बचने के लिए, आपको **Aspose.Slides** का उपयोग करने से पहले लाइसेंस सेट करना होगा। आपको प्रति एप्लिकेशन या प्रक्रिया केवल एक बार लाइसेंस सेट करना आवश्यक है।

### **Aspose.Slides के लिए Node.js के माध्यम से Java में लाइसेंस सेट करना**

लाइसेंस इन स्थानों से लागू किए जा सकते हैं:

* स्पष्ट पथ
* स्ट्रीम
* Metered License के रूप में – एक नया लाइसेंसिंग तंत्र

{{% alert color="primary" %}}
**setLicense** मेथड का उपयोग करके किसी कॉम्पोनेंट को लाइसेंस किया जाता है।

जबकि कई बार **setLicense** को कॉल करना हानिकारक नहीं है, यह संसाधनों (प्रोसेसर) की बर्बादी है।
{{% /alert %}}

{{% alert color="warning" %}}
नए लाइसेंस केवल संस्करण 21.4 या बाद के साथ Aspose.Slides को सक्रिय कर सकते हैं। पहले के संस्करण अलग लाइसेंसिंग सिस्टम का उपयोग करते हैं और इन लाइसेंसों को पहचान नहीं पाएँगे।
{{% /alert %}}

#### **फ़ाइल का उपयोग करके लाइसेंस लागू करना**

यह कोड स्निपेट लाइसेंस फ़ाइल सेट करने के लिए उपयोग किया जाता है:

**Node.js**

```javascript
var aspose = aspose || {};

aspose.slides = require("aspose.slides.via.java");

var license = new aspose.slides.License();
license.setLicense("Aspose.Slides.lic");
```

setLicense मेथड को कॉल करते समय, लाइसेंस का नाम आपके लाइसेंस फ़ाइल के नाम के समान होना चाहिए। उदाहरण के लिए, आप लाइसेंस फ़ाइल का नाम "Aspose.Slides.lic.xml" बदल सकते हैं। तब अपने कोड में आपको नया लाइसेंस नाम (Aspose.Slides.lic.xml) setLicense मेथड को पास करना होगा।

#### **स्ट्रीम से लाइसेंस लागू करना**

यह कोड स्निपेट स्ट्रीम से लाइसेंस लागू करने के लिए उपयोग किया जाता है:

**Node.js**

```javascript
var aspose = aspose || {};

aspose.slides = require("aspose.slides.via.java");

var license = new aspose.slides.License();

var fs = require("fs");

var readStream = fs.createReadStream("Aspose.Slides.lic");

license.setLicense(readStream, function(err, list) {
    if(err) { 
        console.error(err); return; 
    }});
```

## **अक्सर पूछे जाने वाले प्रश्न**

**क्या मैं लाइसेंस को पूरी तरह ऑफ़लाइन वातावरण (कोई इंटरनेट कनेक्शन नहीं) में लागू कर सकता हूँ?**

हाँ। लाइसेंस सत्यापन स्थानीय रूप से लाइसेंस फ़ाइल का उपयोग करके किया जाता है; इंटरनेट कनेक्शन की आवश्यकता नहीं है।

**एक‑साल की सब्सक्रिप्शन समाप्त होने के बाद क्या होता है? क्या लाइब्रेरी काम करना बंद कर देगी?**

नहीं। लाइसेंस स्थायी है: आप अपनी सब्सक्रिप्शन समाप्ति तिथि से पहले जारी संस्करणों का उपयोग जारी रख सकते हैं; केवल नए रिलीज़ के लिए नवीनीकरण आवश्यक होगा।
