## **परिचय**

Aspose.Slides PowerPoint प्रस्तुतियों में स्लाइड आकार और आस्पेक्ट अनुपात को समायोजित करने के लिए व्यापक उपकरण प्रदान करता है, जो प्रिंटिंग और ऑन‑स्क्रीन प्रदर्शन दोनों के लिए महत्वपूर्ण है।

लोकप्रिय स्लाइड आकार और अनुपात:

- **मानक (4:3 आस्पेक्ट अनुपात)**: पुराने स्क्रीन और डिवाइसों के लिए आदर्श।
- **वाइडस्क्रीन (16:9 आस्पेक्ट अनुपात)**: आधुनिक प्रोजेक्टर्स और डिस्प्ले के लिए अनुशंसित।

सुनिश्चित करें कि आपके पूरे प्रेजेंटेशन में एक ही स्लाइड आकार और आस्पेक्ट अनुपात लागू हो। सर्वोत्तम परिणामों के लिए, प्रस्तुति निर्माण प्रक्रिया की शुरुआत में ही स्लाइड आयाम सेट करें ताकि बाद में जटिलताओं से बचा जा सके।

{{% alert color="primary" %}} 
डिफ़ॉल्ट रूप से, Aspose.Slides के साथ बनाई गई प्रस्तुतियों में मानक 4:3 आस्पेक्ट अनुपात उपयोग किया जाता है।
{{% /alert %}}

## **प्रेजेंटेशन में स्लाइड आकार बदलना**

यह नमूना कोड आपको दिखाता है कि JavaScript में Aspose.Slides का उपयोग करके प्रस्तुति में स्लाइड आकार कैसे बदलें:

```javascript
var pres = new aspose.slides.Presentation("pres-4x3-aspect-ratio.pptx");
try {
    pres.getSlideSize().setSize(aspose.slides.SlideSizeType.OnScreen16x9, aspose.slides.SlideSizeScaleType.DoNotScale);
    pres.save("pres-4x3-aspect-ratio.pptx", aspose.slides.SaveFormat.Pptx);
} finally {
    if (pres != null) {
        pres.dispose();
    }
}
```

## **प्रेजेंटेशन में कस्टम स्लाइड आकार निर्दिष्ट करना**

यदि आपको सामान्य स्लाइड आकार (4:3 और 16:9) आपके कार्य के लिए उपयुक्त नहीं लगते, तो आप एक विशिष्ट या अनन्य स्लाइड आकार का उपयोग कर सकते हैं। उदाहरण के लिए, यदि आप अपने प्रेजेंटेशन से कस्टम पेज लेआउट पर पूर्ण आकार की स्लाइडें प्रिंट करना चाहते हैं या कुछ स्क्रीन प्रकारों पर अपनी प्रस्तुति प्रदर्शित करना चाहते हैं, तो कस्टम आकार सेटिंग आपके लिए लाभदायक होगी।

यह नमूना कोड आपको दिखाता है कि JavaScript में Node.js के लिए Aspose.Slides को Java के माध्यम से उपयोग करके कस्टम स्लाइड आकार कैसे निर्दिष्ट करें:

```javascript
var pres = new aspose.slides.Presentation("pres.pptx");
try {
    pres.getSlideSize().setSize(780, 540, aspose.slides.SlideSizeScaleType.DoNotScale);// A4 कागज का आकार
    pres.save("pres-a4-slide-size.pptx", aspose.slides.SaveFormat.Pptx);
} finally {
    if (pres != null) {
        pres.dispose();
    }
}
```

## **स्लाइड आकार बदलते समय उत्पन्न समस्याओं का समाधान**

स्लाइड आकार बदलने के बाद, स्लाइड की सामग्री (जैसे छवियां या ऑब्जेक्ट) विकृत हो सकती है। डिफ़ॉल्ट रूप से, ऑब्जेक्ट स्वचालित रूप से नई स्लाइड आकार में फिट होने के लिए री‑साइज़ हो जाते हैं। हालांकि, आप प्रस्तुति के स्लाइड आकार को बदलते समय ऐसा सेटिंग निर्धारित कर सकते हैं जो तय करता है कि Aspose.Slides स्लाइडों की सामग्री को कैसे संभालेगा।

आपके उद्देश्य के अनुसार आप निम्नलिखित सेटिंग्स में से कोई भी उपयोग कर सकते हैं:

- `DoNotScale`

  यदि आप स्लाइडों पर मौजूद ऑब्जेक्ट्स को री‑साइज़ नहीं करना चाहते, तो इस सेटिंग का उपयोग करें।

- `EnsureFit`

  यदि आप छोटे स्लाइड आकार में स्केल करना चाहते हैं और चाहते हैं कि Aspose.Slides स्लाइडों के ऑब्जेक्ट्स को इस तरह स्केल डाउन करे कि सभी ऑब्जेक्ट्स स्लाइड पर फिट हो जाएँ (इससे सामग्री खोने से बचा जा सके), तो इस सेटिंग को चुनें।

- `Maximize`

  यदि आप बड़े स्लाइड आकार में स्केल करना चाहते हैं और चाहते हैं कि Aspose.Slides स्लाइडों के ऑब्जेक्ट्स को नए स्लाइड आकार के अनुपात में बड़ा करे, तो इस सेटिंग को चुनें।

यह नमूना कोड दिखाता है कि प्रस्तुति के स्लाइड आकार को बदलते समय `Maximize` सेटिंग का उपयोग कैसे किया जाता है:

```javascript
var pres = new aspose.slides.Presentation("pres.pptx");
try {
    pres.getSlideSize().setSize(aspose.slides.SlideSizeType.Ledger, aspose.slides.SlideSizeScaleType.Maximize);
} finally {
    if (pres != null) {
        pres.dispose();
    }
}
```

## **FAQ**

**क्या मैं इंच के अलावा अन्य इकाइयों (जैसे पॉइंट या मिलीमीटर) का उपयोग करके कस्टम स्लाइड आकार सेट कर सकता हूँ?**

हां। Aspose.Slides आंतरिक रूप से पॉइंट्स का उपयोग करता है, जहाँ 1 पॉइंट 1/72 इंच के बराबर होता है। आप किसी भी इकाई (जैसे मिलीमीटर या सेंटीमीटर) को पॉइंट्स में बदलकर स्लाइड की चौड़ाई और ऊँचाई निर्धारित कर सकते हैं।

**क्या बहुत बड़ा कस्टम स्लाइड आकार रेंडरिंग के दौरान प्रदर्शन और मेमोरी उपयोग को प्रभावित करेगा?**

हां। बड़े स्लाइड आयाम (पॉइंट्स में) और उच्च रेंडरिंग स्केल मिलकर मेमोरी की खपत और प्रोसेसिंग समय को बढ़ा देते हैं। व्यावहारिक स्लाइड आकार चुनें और केवल आवश्यकतानुसार रेंडरिंग स्केल समायोजित करें ताकि वांछित आउटपुट गुणवत्ता प्राप्त हो सके।

**क्या मैं एक गैर‑मानक स्लाइड आकार परिभाषित करके फिर विभिन्न आकार वाली प्रस्तुतियों से स्लाइड्स को मर्ज कर सकता हूँ?**

आप विभिन्न स्लाइड आकार वाली प्रस्तुतियों को [merge presentations](/slides/hi/nodejs-java/merge-presentation/) नहीं कर सकते — पहले एक प्रस्तुति को दूसरे के समान आकार में री‑साइज़ करें। स्लाइड आकार बदलते समय आप मौजूदा सामग्री के संचालन को [SlideSizeScaleType](https://reference.aspose.com/slides/hi/nodejs-java/aspose.slides/slidesizescaletype/) विकल्प के माध्यम से निर्धारित कर सकते हैं। आकार को संरेखित करने के बाद, आप फ़ॉर्मेटिंग को संरक्षित रखते हुए स्लाइड्स को मर्ज कर सकते हैं।

**क्या मैं व्यक्तिगत आकृति या स्लाइड के विशिष्ट क्षेत्रों के लिए थंबनेल बना सकता हूँ, और क्या यह नए स्लाइड आकार का सम्मान करेगा?**

हां। Aspose.Slides [entire slides](https://reference.aspose.com/slides/hi/nodejs-java/aspose.slides/slide/#getImage) के साथ-साथ [selected shapes](https://reference.aspose.com/slides/hi/nodejs-java/aspose.slides/shape/#getImage) के लिए थंबनेल रेंडर कर सकता है। उत्पन्न छवियां वर्तमान स्लाइड आकार और आस्पेक्ट अनुपात को दर्शाती हैं, जिससे फ्रेमिंग और ज्यामिति में निरंतरता बनी रहती है।
