## **अवलोकन**

Aspose.Slides आपको प्रस्तुति में एक फ़ॉन्ट को दूसरे फ़ॉन्ट से बदलने की अनुमति देता है। जब फ़ॉन्ट बदल दिया जाता है, तो मूल फ़ॉन्ट के सभी उदाहरण नई फ़ॉन्ट में बदल दिए जाते हैं।

फ़ॉन्ट प्रतिस्थापन करने के लिए, प्रस्तुति लोड करें, स्रोत फ़ॉन्ट और प्रतिस्थापन फ़ॉन्ट निर्धारित करें, फ़ॉन्ट प्रतिस्थापन मेथड को कॉल करें, और संशोधित प्रस्तुति को PPTX फ़ाइल के रूप में सहेजें। यह विधि तभी उपयोगी होती है जब आप जानबूझ कर पूरी प्रस्तुति में एक फ़ॉन्ट परिवार से दूसरे फ़ॉन्ट परिवार में स्विच करना चाहते हैं।

## **फ़ॉन्ट बदलें**

यदि आप फ़ॉन्ट उपयोग के बारे में अपना विचार बदलते हैं, तो आप उस फ़ॉन्ट को किसी अन्य फ़ॉन्ट से बदल सकते हैं। पुराने फ़ॉन्ट के सभी उदाहरण नए फ़ॉन्ट द्वारा प्रतिस्थापित किए जाएंगे।

Aspose.Slides आपको इस प्रकार फ़ॉन्ट बदलने की अनुमति देता है:

1. संबंधित प्रस्तुति लोड करें।  
2. उस फ़ॉन्ट को लोड करें जिसे बदलना है।  
3. नया फ़ॉन्ट लोड करें।  
4. फ़ॉन्ट को बदलें।  
5. संशोधित प्रस्तुति को PPTX फ़ाइल के रूप में लिखें।

यह JavaScript कोड फ़ॉन्ट प्रतिस्थापन को दर्शाता है:

```javascript
// एक प्रस्तुति लोड करता है
var pres = new aspose.slides.Presentation("Fonts.pptx");
try {
    // वह स्रोत फ़ॉन्ट लोड करता है जिसे बदलना है
    var sourceFont = new aspose.slides.FontData("Arial");
    // नया फ़ॉन्ट लोड करता है
    var destFont = new aspose.slides.FontData("Times New Roman");
    // फ़ॉन्ट्स को बदलता है
    pres.getFontsManager().replaceFont(sourceFont, destFont);
    // प्रस्तुति को सहेजता है
    pres.save("UpdatedFont_out.pptx", aspose.slides.SaveFormat.Pptx);
} finally {
    if (pres != null) {
        pres.dispose();
    }
}
```

{{% alert title="Note" color="warning" %}} 
किसी विशेष स्थिति में क्या होना चाहिए (उदाहरण के लिए, यदि फ़ॉन्ट तक पहुँच नहीं सकी) यह निर्धारित करने के लिए नियम सेट करने हेतु देखें [**फ़ॉन्ट प्रतिस्थापन**](/slides/hi/nodejs-java/font-substitution/).
{{% /alert %}}

## **अक्सर पूछे जाने वाले प्रश्न**

**"फ़ॉन्ट प्रतिस्थापन", "फ़ॉन्ट प्रतिस्थापन" और "फ़ॉलबैक फ़ॉन्ट" में क्या अंतर है?**

प्रतिस्थापन दस्तावेज़ में पूरे फ़ॉन्ट परिवार को जानबूझ कर बदलना है। [प्रतिस्थापन](/slides/hi/nodejs-java/font-substitution/) वह नियम है जैसे "यदि फ़ॉन्ट उपलब्ध नहीं है तो X उपयोग करो"। [फ़ॉलबैक](/slides/hi/nodejs-java/fallback-font/) तब लागू होता है जब मूल फ़ॉन्ट स्थापित है पर आवश्यक अक्षर नहीं होते, और व्यक्तिगत गायब glyphs के लिए लागू किया जाता है।

**क्या प्रतिस्थापन मास्टर स्लाइड, लेआउट, नोट्स और कॉमेंट्स पर लागू होता है?**

हाँ। प्रतिस्थापन सभी प्रस्तुति वस्तुओं पर प्रभाव डालता है जो मूल फ़ॉन्ट का उपयोग करती हैं, जिसमें मास्टर स्लाइड और नोट्स शामिल हैं; कॉमेंट्स भी दस्तावेज़ का हिस्सा हैं और फ़ॉन्ट इंजन द्वारा ध्यान में रखे जाते हैं।

**क्या एम्बेडेड OLE ऑब्जेक्ट्स (जैसे Excel) के भीतर फ़ॉन्ट बदलेंगे?**

नहीं। [OLE सामग्री](/slides/hi/nodejs-java/manage-ole/) अपना स्वयं का अनुप्रयोग द्वारा नियंत्रित होती है। प्रस्तुति में किया गया प्रतिस्थापन आंतरिक OLE डेटा को पुनः स्वरूपित नहीं करता; यह चित्र के रूप में या बाहरी रूप से संपादन योग्य सामग्री के रूप में प्रदर्शित हो सकता है।

**क्या मैं प्रस्तुति के केवल कुछ भाग (स्लाइड या क्षेत्र) में फ़ॉन्ट बदल सकता हूँ?**

लक्षित प्रतिस्थापन संभव है यदि आप आवश्यक वस्तुओं/रेंजों के स्तर पर फ़ॉन्ट बदलते हैं, न कि पूरे दस्तावेज़ पर वैश्विक प्रतिस्थापन लागू करते हैं। रेंडरिंग के दौरान कुल फ़ॉन्ट चयन तर्क वही रहता है।

**मैं कैसे पूर्व में निर्धारित करूँ कि प्रस्तुति कौन‑से फ़ॉन्ट्स का उपयोग करती है?**

प्रस्तुति के [फ़ॉन्ट मैनेजर](https://reference.aspose.com/slides/hi/nodejs-java/aspose.slides/fontsmanager/) का उपयोग करें: यह उपयोग में रहने वाले [फ़ॉन्ट परिवारों]((https://reference.aspose.com/slides/hi/nodejs-java/aspose.slides/fontsmanager/getfonts/)) की सूची और [प्रतिस्थापन/"अज्ञात" फ़ॉन्ट]((https://reference.aspose.com/slides/hi/nodejs-java/aspose.slides/fontsmanager/getsubstitutions/)) की जानकारी प्रदान करता है, जो प्रतिस्थापन की योजना बनाने में मदद करता है।

**क्या फ़ॉन्ट प्रतिस्थापन PDF/इमेज में परिवर्तित करने पर कार्य करता है?**

हाँ। निर्यात के दौरान, Aspose.Slides वही [फ़ॉन्ट चयन/प्रतिस्थापन क्रम](/slides/hi/nodejs-java/font-selection-sequence/) लागू करता है, इसलिए पहले किया गया प्रतिस्थापन परिवर्तन के दौरान सम्मानित होता है।

**क्या मुझे लक्ष्य फ़ॉन्ट सिस्टम में इंस्टॉल करना आवश्यक है, या मैं फ़ॉन्ट फ़ोल्डर संलग्न कर सकता हूँ?**

इंस्टॉलेशन आवश्यकता नहीं है: लाइब्रेरी उपयोगकर्ता फ़ोल्डर से [बाहरी फ़ॉन्ट लोड करने](/slides/hi/nodejs-java/custom-font/) की अनुमति देती है, जिसे [रेंडरिंग और निर्यात](/slides/hi/nodejs-java/convert-powerpoint/) के दौरान उपयोग किया जाता है।

**क्या प्रतिस्थापन "टोफ़ू" (वर्ग) की समस्या को ठीक करेगा?**

केवल तभी जब लक्ष्य फ़ॉन्ट वास्तव में आवश्यक glyphs रखता हो। यदि नहीं, तो गायब अक्षरों को कवर करने हेतु [फ़ॉलबैक कॉन्फ़िगर](/slides/hi/nodejs-java/fallback-font/) करें।
