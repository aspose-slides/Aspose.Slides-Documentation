## **مقدمه**

Aspose.Slides نیازی به نصب Microsoft PowerPoint ندارد زیرا Aspose.Slides یک موتور مستقل برای ایجاد، تبدیل، چیدمان صفحه و رندر اسناد Microsoft PowerPoint است.

## **سیستم‌عامل‌های پشتیبانی‌شده**
Aspose.Slides for C++ یک کتابخانه بومی C++ است. Aspose.Slides for C++ از سیستم‌عامل‌ها و پلتفرم‌های 64‑بیتی و 32‑بیتی زیر پشتیبانی می‌کند:

### **Windows**
- Microsoft Windows Server 2008 (x64, x86)
- Microsoft Windows Server 2012 (x64, x86)
- Microsoft Windows Server 2012 R2 (x64, x86)
- Microsoft Windows Server 2016 (x64, x86)
- Microsoft Windows Server 2019 (x64, x86)
- Microsoft Windows XP (x64, x86)
- Microsoft Windows 7 (x64, x86)
- Microsoft Windows 8, 8.1 (x64, x86)
- Microsoft Windows 10 (x64, x86)

### **Linux**
- OS Ubuntu 16.04 یا بعد از آن.
- CentOS 8 یا بعد از آن.
- Fedora 24 یا بعد از آن.
- و سایر توزیع‌های Linux x86_64 با glibc 2.23 یا بعد از آن.

### **macOS**
- macOS Monterey 12.1 یا بعد از آن.

## **محیط‌های توسعه**
شما می‌توانید هنگام توسعه برنامه‌ها برای Windows، Linux یا macOS از Aspose.Slides for C++ استفاده کنید.

### **Windows**
- Microsoft Visual Studio 2017 یا بعد از آن.
- CMake 3.18 یا بعد از آن.

### **Linux**
- Clang 3.9 یا بعد از آن.
- GCC 6.1 یا بعد از آن.
- CMake 3.18 یا بعد از آن.

### **macOS**
- Xcode 13.4 یا بعد از آن.

## **سؤالات متداول**

**آیا برای تبدیل و رندر کردن نیاز به نصب Microsoft PowerPoint دارم؟**

خیر، PowerPoint لازم نیست؛ Aspose.Slides یک موتور مستقل برای [ایجاد](/slides/fa/cpp/create-presentation/)، اصلاح، [تبدیل](/slides/fa/cpp/convert-presentation/) و [رندر](/slides/fa/cpp/convert-powerpoint-to-png/) ارائه‌ها است.

**کدام فونت‌ها برای رندر صحیح لازم هستند؟**

در عمل، فونت‌های مورد استفاده در ارائه یا [جایگزین‌های](/slides/fa/cpp/font-substitution/) مناسب باید موجود باشند. برای اطمینان از رندر یکسان در Linux/macOS، نصب بسته‌های فونت رایج توصیه می‌شود.

**چرا یک فونت سفارشی در Linux به‌صورت جایگزین یا متن گمشده رندر می‌شود؟**

اگر فایل فونت دارای ورودی‌های نام‌جدول ناسازگار یا آسیب‌دیده باشد، پشتهٔ مطابقت فونت در Linux (FreeType/fontconfig) ممکن است رکورد نامعتبری را انتخاب کند که منجر به عدم شناسایی فونت می‌شود. استفاده از نسخه‌ای از فونت با رکوردهای نام‌جدول اصلاح‌شده یا نصب یک جایگزین سازگار این مشکل را حل می‌کند.
