## **نمای کلی**

Aspose.Slides می‌تواند در حالت ارزیابی یا با یک مجوز معتبر استفاده شود. نسخه ارزیابی همان عملکرد نسخه دارای مجوز را فراهم می‌کند، اما هنگام باز کردن یا ذخیره ارائه‌ها یک آبجاک ارزیابی اضافه می‌کند و استخراج متن را به یک اسلاید محدود می‌سازد.

این مقاله توضیح می‌دهد که مجوزدهی در Aspose.Slides چگونه کار می‌کند و چگونه قبل از استفاده از کتابخانه یک مجوز اعمال شود. یک مجوز می‌تواند از یک فایل، جریان یا منبع توکار با استفاده از کلاس `License` بارگذاری شود. مقاله همچنین نشان می‌دهد چگونه اعتبارسنجی شود که آیا مجوز به‌درستی اعمال شده است یا نه.

## **مجوزدهی در Aspose.Slides**

* پس از خرید یک مجوز و اعمال آن با افزودن چند خط کد، نسخه ارزیابی تبدیل به دارای مجوز می‌شود.
* مجوز یک فایل XML متنی ساده است که شامل جزئیاتی مانند نام محصول، تعداد توسعه‌دهندگانی که مجوز دارند، تاریخ انقضای اشتراک و موارد دیگر می‌باشد.
* فایل مجوز به‌صورت دیجیتالی امضا شده است، بنابراین نباید تغییر یابد. حتی یک تغییر تصادفی مانند افزودن شکست خط می‌تواند فایل را نامعتبر کند.
* Aspose.Slides for C++ معمولاً به دنبال فایل مجوز در مکان‌های زیر می‌گردد:
  * مسیر صریحی که در کد شما مشخص شده است
  * پوشه‌ای که DLL مؤلفه را شامل می‌شود (در Aspose.Slides گنجانده شده است)
  * پوشه‌ای که اسمبلی فراخوانی‌کننده DLL مؤلفه در آن قرار دارد
* برای جلوگیری از محدودیت‌های نسخه ارزیابی، باید قبل از استفاده از Aspose.Slides مجوز را تنظیم کنید. یک مجوز تنها یک بار برای هر برنامه یا فرآیند نیاز به تنظیم دارد.

## **اعمال یک مجوز**

یک مجوز می‌تواند از **فایل**، **جریان** یا **منبع توکار** بارگذاری شود.

{{% alert color="primary" %}}

Aspose.Slides کلاس [License](https://reference.aspose.com/slides/fa/cpp/class/aspose.slides.license/) را برای عملیات مجوزدهی فراهم می‌کند.

{{% /alert %}} 

{{% alert color="warning" %}}

مجوزهای جدید فقط می‌توانند Aspose.Slides را در نسخه 21.4 یا بالاتر فعال کنند. نسخه‌های قبلی از سیستم مجوزدهی متفاوتی استفاده می‌کنند و این مجوزها را شناسایی نمی‌کنند.

{{% /alert %}}

### **فایل**

ساده‌ترین راه برای تنظیم یک مجوز این است که فایل مجوز را در همان پوشه DLL مؤلفه (در Aspose.Slides گنجانده شده) قرار دهید و فقط نام فایل را بدون مسیر مشخص کنید.

کد C++ زیر نشان می‌دهد چگونه یک فایل مجوز تنظیم شود:

```c++
#include <Util/License.h>

using namespace Aspose::Slides;

int main()
{
    auto license = MakeObject<License>();
    license->SetLicense(u"Aspose.Slides.lic");

    return 0;
}
```

{{% alert color="warning" %}} 

اگر فایل مجوز را در دایرکتوری دیگری قرار دهید، هنگام فراخوانی متد [License::SetLicense](https://reference.aspose.com/slides/fa/cpp/aspose.slides/license/setlicense/) باید نام فایل در انتهای مسیر صریح مشخص شده دقیقاً با نام فایل مجوز شما مطابقت داشته باشد.

به‌عنوان مثال، اگر نام فایل مجوز خود را به *Aspose.Slides.lic.xml* تغییر دهید، باید مسیر کامل که با *Aspose.Slides.lic.xml* پایان می‌یابد را به متد [License::SetLicense](https://reference.aspose.com/slides/fa/cpp/aspose.slides/license/setlicense/) در کد خود پاس کنید.

{{% /alert %}}

### **جریان**

می‌توانید یک مجوز را از یک جریان بارگذاری کنید. کد C++ زیر نشان می‌دهد چگونه یک مجوز را از یک جریان اعمال کنید:

```c++
auto license = MakeObject<License>();

auto stream = File::OpenRead(u"Aspose.Slides.lic");

license->SetLicense(stream);
```

## **اعتبارسنجی یک مجوز**

برای بررسی اینکه آیا یک مجوز به‌درستی تنظیم شده است یا نه، می‌توانید آن را اعتبارسنجی کنید. کد C++ زیر نشان می‌دهد چگونه یک مجوز را اعتبارسنجی کنید:

```c++
auto license = MakeObject<License>();

license->SetLicense(u"Aspose.Slides.lic");

if (license->IsLicensed())
{
    Console::WriteLine(u"License is good!");
    Console::ReadKey();
}
```

## **ایمنی در نخ‌ها**

{{% alert title="Note" color="warning" %}} 

متد [License::SetLicense](https://reference.aspose.com/slides/fa/cpp/aspose.slides/license/setlicense/) **آمن در مقابل چندنخی نیست**. اگر نیاز دارید این متد را هم‌زمان از چندین نخ فراخوانی کنید، توصیه می‌شود از primitiveهای همگام‌سازی (مانند یک قفل) برای جلوگیری از مشکلات احتمالی استفاده کنید.

{{% /alert %}}

## **سؤالات متداول**

**آیا می‌توانم مجوز را در محیط کاملاً آفلاین (بدون دسترسی به اینترنت) اعمال کنم؟**

بله. اعتبارسنجی مجوز به‌صورت محلی با استفاده از فایل مجوز انجام می‌شود؛ نیازی به اتصال اینترنت نیست.

**بعد از انقضای اشتراک یک‌ساله چه می‌شود؟ آیا کتابخانه متوقف می‌شود؟**

خیر. مجوز دائمی است: می‌توانید از نسخه‌های منتشر شده قبل از تاریخ پایان اشتراک خود استفاده کنید؛ فقط برای استفاده از نسخه‌های جدیدتر بدون تمدید مجوز مجاز نخواهید بود.
