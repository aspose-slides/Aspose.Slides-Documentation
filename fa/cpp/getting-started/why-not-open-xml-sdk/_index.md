## **بررسی کلی**

این مقاله توضیح می‌دهد که چه زمانی توسعه‌دهندگان ممکن است برای کار با اسناد ارائه، Open XML SDK یا Aspose.Slides را انتخاب کنند. این مقاله Open XML SDK را به‌عنوان کتابخانه‌ای برای دست‌کاری بسته‌های OOXML و عناصر XML زیرین آن توصیف می‌کند، در حالی که Aspose.Slides به‌عنوان کتابخانه‌ای برای پردازش ارائه با مدل شیء سطح‑بالا و پشتیبانی از بسیاری از وظایف مرتبط با PowerPoint معرفی می‌شود.

مقاله هر دو گزینه را بر اساس فرمت‌های پشتیبانی‌شده، مدل برنامه‌نویسی، قابلیت‌های رندر و چاپ، پشتیبانی از پلتفرم‌ها و موارد استفاده رایج مقایسه می‌کند. همچنین روشن می‌کند که Open XML SDK ممکن است برای عملیات پایه‌ای PPTX یا دسترسی مستقیم به عناصر OOXML مناسب باشد، در حالی که Aspose.Slides برای وظایف پیچیده ارائه مانند کار با فرمت‌های متعدد PowerPoint، کپی یا کلون کردن اشکال، جایگزینی متن، اعمال انیمیشن‌ها و تبدیل ارائه‌ها به PDF، TIFF یا XPS مناسب‌تر است.

## **Open XML SDK چیست؟**
ما گاهی این سؤال را می‌شنویم: چرا باید به جای Open XML SDK رایگان، از محصولات Aspose استفاده کنیم؟ این سؤال به سادگی پاسخ می‌شود: ویژگی‌ها و عملکردها. بر اساس the[MSDN Library](https://docs.microsoft.com/en-us/office/open-xml/open-xml-sdk)، Open XML SDK به‌صورت زیر تعریف شده است: Open XML SDK 2.0 کار دست‌کاری بسته‌های Open XML و عناصر اسکیماهای زیرین درون یک بسته را ساده می‌کند. Open XML SDK 2.0 بسیاری از وظایف رایجی که توسعه‌دهندگان روی بسته‌های Open XML انجام می‌دهند را در خود گنجانده است، به‌طوری که می‌توانید عملیات پیچیده را با تنها چند خط کد انجام دهید. اسناد OOXML اساساً فایل‌های XML فشرده‌ شده هستند و Open XML SDK مجموعه‌ای از کلاس‌هاست که به شما اجازه می‌دهد محتویات اسناد OOXML را به صورت strongly‑typed اداره کنید. به عبارت دیگر به جای استخراج یک فایل برای استخراج XML، بارگذاری آن XML در یک درخت DOM و کار مستقیم با عناصر و ویژگی‌های XML، Open XML SDK کلاس‌هایی را فراهم می‌کند که این کار را انجام می‌دهند.

## **Aspose.Slides چیست؟**
Aspose.Slides یک کتابخانهٔ کلاس است که به برنامهٔ شما امکان انجام وظایف پردازش ارائه زیر را می‌دهد:

- برنامه‌نویسی با مدل شیء **Presentation**.
- تبدیل‌های با کیفیت بالا بین تمام فرمت‌های محبوب پشتیبانی‌شده از ارائه‌های PowerPoint، از جمله تبدیل به PDF و XPS.
- قابلیت تولید تصویر بندانگشتی اسلایدها در قالب‌های شناخته‌شده‌ای مانند PNG، JPEG و BMP به‌همراه صادرات اسلاید به SVG.
- قابلیت ساخت ارائه‌ها از صفر یا با ترکیب یک یا چند سند.
- پشتیبانی از افزودن انیمیشن‌ها، Ole Frames، جداول، ایجاد و مدیریت نمودارها.
- دسترس‌پذیری کنترل گسترده برای مدیریت قالب‌بندی متن در سطوح TextFrames، Paragraphs و Portions.

برای جزئیات بیشتر درباره ویژگی‌های پشتیبانی‌شده، لطفاً به [Aspose.Slides Features](/slides/fa/cpp/product-overview/) مراجعه کنید.

## **نتیجه‌گیری**
Open XML SDK و Aspose.Slides سرسختانه رقابت نمی‌کنند زیرا به نیازها و مخاطبان کاملاً متفاوتی می‌پردازند. Open XML SDK یک کتابخانهٔ کلاس است که روش strongly‑typed برای کار با اسناد OOXML فراهم می‌کند. Aspose.Slides یک کتابخانهٔ پردازش ارائه بسیار مفید است که پشتیبانی عالی برای تقریباً تمام قالب‌های فایل Microsoft PowerPoint ارائه می‌دهد. اگر تنها نیاز دارید یک عملیات برنامه‌نویسی نسبتاً پایه‌ای روی یک سند PPTX انجام دهید، Open XML SDK می‌تواند گزینهٔ مناسب باشد. با Open XML SDK می‌توانید کارهای ساده‌ای مانند تولید یک سند PPTX ساده یا حذف نظرات، سرصفحه/پاصفحه، استخراج تصاویر و غیره را به راحتی انجام دهید. برخی کارها می‌توانند با Open XML SDK انجام شوند، اما با Aspose.Slides قابل انجام نیستند. برای مثال، اگر نیاز به دسترسی مستقیم به عناصر و ویژگی‌های XML یک سند OOXML داشته باشید، باید از Open XML SDK استفاده کنید. اما اگر نیاز به انجام عملیات پیچیده روی اسناد داشته باشید، مانند برخی از کارهای زیر، استفاده از Aspose.Slides بهترین گزینهٔ شماست:

- پشتیبانی از فرمت‌های قدیمی PowerPoint علاوه بر PPTX.
- کپی یا کلون کردن اشکال درون اسلایدها به‌گونه‌ای که اشیاء، سبک‌ها و دیگر قالب‌بندی‌ها را به‌صورت مناسب ترکیب کند.
- جایگزینی متن قالب‌بندی‌شده یا بدون قالب‌بندی.
- اعمال انیمیشن‌ها و استفاده از کانکتورها با اشکال.
- تبدیل یک سند به PDF یا XPS به‌طوری که دقیقاً همان‌گونه‌ که Microsoft PowerPoint تبدیل می‌کرد نمایش داده شود.
- توسعه یک برنامه C++ در هر دو محیط دسکتاپ و کنسول.
