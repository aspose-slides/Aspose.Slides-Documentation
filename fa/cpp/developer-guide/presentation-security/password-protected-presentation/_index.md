## **مقدمه**

هنگامی که یک ارائه را با رمز عبور محافظت می‌کنید، به این معنی است که رمزی تنظیم می‌کنید که محدودیت‌های خاصی را بر روی ارائه اعمال می‌کند. برای حذف این محدودیت‌ها، باید رمز عبور وارد شود. یک ارائهٔ محافظت‌شده با رمز عبور به‌عنوان یک ارائهٔ قفل‌شده در نظر گرفته می‌شود.

به‌طور معمول، می‌توانید رمز عبوری تنظیم کنید تا این محدودیت‌ها بر روی یک ارائه اعمال شوند:

- **تغییر**

  اگر می‌خواهید فقط کاربران خاصی بتوانند ارائهٔ شما را تغییر دهند، می‌توانید محدودیت تغییر را تنظیم کنید. این محدودیت مانع از تغییر، ویرایش یا کپی کردن محتویات ارائه می‌شود (مگر این که رمز عبور ارائه شود).

  با این حال، در این حالت حتی بدون رمز عبور، کاربر می‌تواند به سند شما دسترسی پیدا کرده و آن را باز کند. در حالت فقط‑خواندنی، کاربر می‌تواند محتویات یا مواردی مانند پیوندها، انیمیشن‌ها، افکت‌ها و غیره را درون ارائه مشاهده کند، اما نمی‌تواند موارد را کپی یا ارائه را ذخیره کند.

- **باز کردن**

  اگر می‌خواهید فقط کاربران خاصی بتوانند ارائهٔ شما را باز کنند، می‌توانید محدودیت باز کردن را تنظیم کنید. این محدودیت مانع از حتی مشاهده محتویات ارائه می‌شود (مگر این که رمز عبور ارائه شود).

  از نظر فنی، محدودیت باز کردن همچنین از ویرایش ارائه توسط کاربران جلوگیری می‌کند: وقتی کاربران نمی‌توانند یک ارائه را باز کنند، نمی‌توانند آن را تغییر دهند یا اصلاح کنند.

  **توجه** داشته باشید که وقتی ارائه‌ای را برای جلوگیری از باز کردن با رمز عبور محافظت می‌کنید، فایل ارائه رمزگذاری می‌شود.

## **نحوهٔ محافظت با رمز عبور از یک ارائه به‌صورت آنلاین**

1. به صفحهٔ [**قفل Aspose.Slides**](https://products.aspose.app/slides/fa/lock) ما بروید.

   ![todo:image_alt_text](slides-lock.png)

2. روی **Drop or upload your files** کلیک کنید.

3. فایلی که می‌خواهید با رمز عبور محافظت کنید را از کامپیوتر خود انتخاب کنید.

4. رمز عبور دلخواه خود را برای محافظت از ویرایش وارد کنید؛ رمز عبور دلخواه خود را برای محافظت از مشاهده وارد کنید.

5. اگر می‌خواهید کاربران ارائهٔ شما را به‌عنوان نسخهٔ نهایی ببیند، گزینهٔ **Mark as final** را علامت بزنید.

6. روی **PROTECT NOW.** کلیک کنید.

7. روی **DOWNLOAD NOW.** کلیک کنید.

## **محافظت با رمز عبور برای ارائه‌ها در Aspose.Slides**
**قالب‌های پشتیبانی‌شده**

Aspose.Slides از محافظت با رمز عبور، رمزگذاری و عملیات مشابه برای ارائه‌ها در این قالب‌ها پشتیبانی می‌کند:

- PPTX و PPT - ارائهٔ Microsoft PowerPoint
- ODP - ارائهٔ OpenDocument
- OTP - قالب ارائهٔ OpenDocument

**عملیات پشتیبانی‌شده**

Aspose.Slides به شما اجازه می‌دهد تا با استفاده از محافظت با رمز عبور از ارائه‌ها، از تغییرات به روش‌های زیر جلوگیری کنید:

- رمزگذاری یک ارائه
- تنظیم محافظت نوشتن برای یک ارائه

**سایر عملیات**

Aspose.Slides به شما امکان می‌دهد تا سایر کارها مرتبط با محافظت با رمز عبور و رمزگذاری را به روش‌های زیر انجام دهید:

- رمزگشایی یک ارائه؛ باز کردن یک ارائهٔ رمزگذاری‌شده
- حذف رمزگذاری؛ غیرفعال‌سازی محافظت با رمز عبور
- حذف محافظت نوشتن از یک ارائه
- دریافت ویژگی‌های یک ارائهٔ رمزگذاری‌شده
- بررسی اینکه آیا یک ارائه رمزگذاری شده است یا خیر
- بررسی اینکه آیا یک ارائه با رمز عبور محافظت شده است یا خیر.

## **رمزگذاری یک ارائه**

می‌توانید یک ارائه را با تنظیم یک رمز عبور رمزگذاری کنید. سپس، برای تغییر ارائهٔ قفل‌شده، کاربر باید رمز عبور را ارائه دهد.

برای رمزگذاری یا محافظت با رمز عبور از یک ارائه، باید از متد encrypt (از [ProtectionManager](https://reference.aspose.com/slides/fa/cpp/class/aspose.slides.protection_manager)) استفاده کنید تا رمز عبور را برای ارائه تنظیم کنید. رمز عبور را به متد encrypt می‌دهید و با استفاده از متد save ارائهٔ اکنون رمزگذاری‌شده را ذخیره می‌کنید.

این کد نمونه نشان می‌دهد چگونه یک ارائه را رمزگذاری کنید:

``` cpp
auto presentation = System::MakeObject<Presentation>(u"pres.pptx");

presentation->get_ProtectionManager()->Encrypt(u"123123");
presentation->Save(u"encrypted-pres.pptx", SaveFormat::Pptx);
```

## **تنظیم محافظت نوشتن برای یک ارائه**

می‌توانید علامتی با متن «تغییر نکنید» به یک ارائه اضافه کنید. به این ترتیب، به کاربران می‌گویید که نمی‌خواهید آنها تغییراتی در ارائه ایجاد کنند.

**توجه** داشته باشید که فرآیند محافظت نوشتن ارائه را رمزگذاری نمی‌کند. بنابراین، کاربران—اگر بخواهند—می‌توانند ارائه را تغییر دهند، اما برای ذخیرهٔ تغییرات باید ارائه‌ای با نام دیگری ایجاد کنند.

برای تنظیم محافظت نوشتن، باید از متد setWriteProtection استفاده کنید. این کد نمونه نشان می‌دهد چگونه یک محافظت نوشتن برای یک ارائه تنظیم کنید:

``` cpp
auto presentation = System::MakeObject<Presentation>(u"pres.pptx");

presentation->get_ProtectionManager()->SetWriteProtection(u"123123");
presentation->Save(u"write-protected-pres.pptx", SaveFormat::Pptx);
```

## **بارگذاری یک ارائهٔ رمزگذاری‌شده**

Aspose.Slides به شما امکان می‌دهد فایل رمزگذاری‌شده را با وارد کردن رمز عبور آن بارگذاری کنید. برای رمزگشایی یک ارائه، باید متد [RemoveEncryption](https://reference.aspose.com/slides/fa/cpp/class/aspose.slides.protection_manager#a422059278b430a0493680252aa975d4d) را بدون پارامتر فراخوانی کنید. سپس باید رمز عبور صحیح را وارد کنید تا ارائه بارگذاری شود.

این کد نمونه نشان می‌دهد چگونه یک ارائه را رمزگشایی کنید:

``` cpp
auto loadOptions = System::MakeObject<LoadOptions>();
loadOptions->set_Password(u"123123");
    
System::SharedPtr<Presentation> presentation = System::MakeObject<Presentation>(u"pres.pptx", loadOptions);

// کار با ارائه رمزگشایی‌شده
```

## **حذف رمزگذاری از یک ارائه**

می‌توانید رمزگذاری یا محافظت با رمز عبور یک ارائه را حذف کنید. به این ترتیب، کاربران می‌توانند بدون محدودیت به ارائه دسترسی پیدا کنند یا آن را تغییر دهند.

برای حذف رمزگذاری یا محافظت با رمز عبور، باید متد [RemoveEncryption](https://reference.aspose.com/slides/fa/cpp/class/aspose.slides.protection_manager#a422059278b430a0493680252aa975d4d) را فراخوانی کنید. این کد نمونه نشان می‌دهد چگونه رمزگذاری از یک ارائه حذف شود:

``` cpp
auto loadOptions = System::MakeObject<LoadOptions>();
loadOptions->set_Password(u"123123");
    
auto presentation = System::MakeObject<Presentation>(u"pres.pptx", loadOptions);

presentation->get_ProtectionManager()->RemoveEncryption();
presentation->Save(u"encryption-removed.pptx", SaveFormat::Pptx);
```

## **حذف محافظت نوشتن از یک ارائه**

می‌توانید با استفاده از Aspose.Slides محافظت نوشتن اعمال‌شده بر روی یک فایل ارائه را حذف کنید. به این ترتیب، کاربران می‌توانند به‌هر دلخواه تغییر دهند و هنگام انجام این کار هیچ هشداری دریافت نمی‌کنند.

می‌توانید محافظت نوشتن را با استفاده از متد [RemoveWriteProtection](https://reference.aspose.com/slides/fa/cpp/class/aspose.slides.protection_manager#a9f9e6de5983965157dac0f270a0a9e50) حذف کنید. این کد نمونه نشان می‌دهد چگونه محافظت نوشتن از یک ارائه حذف شود:

``` cpp
auto presentation = System::MakeObject<Presentation>(u"pres.pptx");

presentation->get_ProtectionManager()->RemoveWriteProtection();
presentation->Save(u"write-protection-removed.pptx", SaveFormat::Pptx);
```

## **دریافت ویژگی‌های یک ارائهٔ رمزگذاری‌شده**

معمولاً کاربران در دریافت ویژگی‌های سند یک ارائهٔ رمزگذاری‌شده یا محافظت‌شده با رمز عبور مشکل دارند. Aspose.Slides اما مکانیزمی ارائه می‌دهد که به شما اجازه می‌دهد ارائه‌ای را با رمز عبور محافظت کنید و در عین حال امکان دسترسی کاربران به ویژگی‌های آن ارائه را حفظ کنید.

**توجه** داشته باشید که وقتی Aspose.Slides یک ارائه را رمزگذاری می‌کند، ویژگی‌های سند ارائه نیز به‌صورت پیش‌فرض با رمز عبور محافظت می‌شوند. اما اگر نیاز داشته باشید ویژگی‌های ارائه را حتی پس از رمزگذاری در دسترس کاربران قرار دهید، Aspose.Slides این امکان را به شما می‌دهد.

اگر می‌خواهید کاربران توانایی دسترسی به ویژگی‌های یک ارائهٔ رمزگذاری‌شده را حفظ کنند، می‌توانید مقدار `true` را به متد [set_EncryptDocumentProperties()](https://reference.aspose.com/slides/fa/cpp/class/aspose.slides.protection_manager#a67e041b432552969d106f72fa7fe5a1d) پاس دهید. این کد نمونه نشان می‌دهد چگونه یک ارائه را رمزگذاری کنید و همزمان امکان دسترسی کاربران به ویژگی‌های سند آن را فراهم کنید:

``` cpp
auto presentation = System::MakeObject<Presentation>(u"pres.pptx");

presentation->get_ProtectionManager()->set_EncryptDocumentProperties(true);
presentation->get_ProtectionManager()->Encrypt(u"123123");
```

## **بررسی اینکه آیا یک ارائه با رمز عبور محافظت شده است**

قبل از بارگذاری یک ارائه، ممکن است بخواهید بررسی کنید و تأیید کنید که ارائه با رمز عبور محافظت نشده است. به این ترتیب می‌توانید از خطاها و مشکلات مشابهی که هنگام بارگذاری یک ارائهٔ محافظت‌شده با رمز عبور بدون داشتن رمز عبور رخ می‌دهد، جلوگیری کنید.

این کد C++ نشان می‌دهد چگونه یک ارائه را بررسی کنید تا ببینید آیا با رمز عبور محافظت شده است (بدون بارگذاری خود ارائه):

```c++
auto presentationInfo = PresentationFactory::get_Instance()->GetPresentationInfo(u"example.pptx");
System::Console::WriteLine(System::String(u"The presentation is password protected: ") +
                           presentationInfo->get_IsPasswordProtected());
```

## **بررسی اینکه آیا یک ارائه رمزگذاری شده است**

Aspose.Slides به شما اجازه می‌دهد بررسی کنید که آیا یک ارائه رمزگذاری شده است یا خیر. برای انجام این کار می‌توانید از متد [get_IsEncrypted()](https://reference.aspose.com/slides/fa/cpp/class/aspose.slides.protection_manager#ad88b984e44b378f335317ded49b34e68) استفاده کنید که در صورت رمزگذاری ارائه مقدار `true` و در غیر این صورت مقدار `false` برمی‌گرداند.

این کد نمونه نشان می‌دهد چگونه بررسی کنید که آیا یک ارائه رمزگذاری شده است:

``` cpp
auto presentation = System::MakeObject<Presentation>(u"pres.pptx");

bool isEncrypted = presentation->get_ProtectionManager()->get_IsEncrypted();
```

## **بررسی اینکه آیا یک ارائه محافظت نوشتن شده است**

Aspose.Slides به شما اجازه می‌دهد بررسی کنید که آیا یک ارائه محافظت نوشتن شده است یا خیر. برای انجام این کار می‌توانید از متد [get_IsWriteProtected()](https://reference.aspose.com/slides/fa/cpp/class/aspose.slides.protection_manager#a0b4a82c0f7b3a32ca5762c5fcc8844a2) استفاده کنید که در صورت محافظت نوشتن مقدار `true` و در غیر این صورت مقدار `false` برمی‌گرداند.

این کد نمونه نشان می‌دهد چگونه بررسی کنید که آیا یک ارائه محافظت نوشتن شده است:

``` cpp
auto presentation = System::MakeObject<Presentation>(u"pres.pptx");

bool isEncrypted = presentation->get_ProtectionManager()->get_IsWriteProtected();
```

## **تصدیق استفاده از رمز عبور برای ارائه**

ممکن است بخواهید بررسی کنید و تأیید کنید که یک رمز عبور خاص برای محافظت از سند ارائه استفاده شده است. Aspose.Slides ابزارهایی برای اعتبارسنجی رمز عبور فراهم می‌کند.

این کد نمونه نشان می‌دهد چگونه یک رمز عبور را اعتبارسنجی کنید:

``` cpp
auto pres = System::MakeObject<Presentation>(u"pres.pptx");

// بررسی اینکه آیا "pass" مطابقت دارد
bool isWriteProtected = pres->get_ProtectionManager()->CheckWriteProtection(u"my_password");
```

این مقدار `true` بر می‌گرداند اگر ارائه با رمز عبور مشخص رمزگذاری شده باشد. در غیر این صورت مقدار `false` برمی‌گرداند.

## **سوالات متداول**

**روش‌های رمزگذاری پشتیبانی‌شده توسط Aspose.Slides چیست؟**

Aspose.Slides از روش‌های رمزگذاری مدرن شامل الگوریتم‌های مبتنی بر AES پشتیبانی می‌کند و سطح بالایی از امنیت داده‌ها را برای ارائه‌های شما تضمین می‌نماید.

**اگر هنگام تلاش برای باز کردن یک ارائه، رمز عبور نادرست وارد شود، چه اتفاقی می‌افتد؟**

یک استثنا پرتاب می‌شود اگر رمز عبور نادرست استفاده شود و به شما اطلاع می‌دهد که دسترسی به ارائه رد شده است. این امر به جلوگیری از دسترسی غیرمجاز و حفاظت از محتوای ارائه کمک می‌کند.

**آیا هنگام کار با ارائه‌های محافظت‌شده با رمز عبور، تأثیرات عملکردی وجود دارد؟**

فرآیند رمزگذاری و رمزگشایی ممکن است کمی سربار در طول عملیات باز کردن و ذخیره‌سازی ایجاد کند. در اکثر موارد این تأثیر عملکردی کم است و به‌طور قابل‌توجهی بر زمان کلی پردازش کارهای ارائه شما تأثیر نمی‌گذارد.
