## **نمای کلی**
Aspose.Slides for Java نیازی به نصب Microsoft PowerPoint ندارد، زیرا Aspose.Slides خود یک موتور ایجاد، تبدیل، چیدمان صفحه و رندر اسناد Microsoft PowerPoint است.

## **سیستم‌عامل‌های پشتیبانی شده**
Aspose.Slides for Java هر سیستم‌عامل 32 بیت یا 64 بیتی که محیط اجرای Java را اجرا می‌کند را پشتیبانی می‌کند، از جمله اما نه محدود به:

### **ویندوز**
- Microsoft Windows 2003 Server ( x64, x86)
- Microsoft Windows 2008 Server ( x64, x86)
- Microsoft Windows 2012 Server ( x64, x86)
- Microsoft Windows 2012 R2 Server ( x64, x86)
- Microsoft Windows 2016 Server ( x64, x86)
- Microsoft Windows 2019 Server ( x64, x86)
- Microsoft Windows Vista ( x64, x86)
- Microsoft Windows XP ( x64, x86)
- Microsoft Windows 7 ( x64, x86)
- Microsoft Windows 8, 8.1 ( x64, x86)
- Microsoft Windows 10 ( x64, x86)

### **لینوکس**
- Linux (Ubuntu, OpenSUSE, CentOS و دیگران)

### **مک**
- Mac OS X

## **نسخه‌های جاوا پشتیبانی شده**
Aspose.Slides for Java از J2SE 6.0 (Java 1.6) و بالاتر پشتیبانی می‌کند.

## **سوالات متداول**

**آیا برای تبدیل و رندرینگ نیاز به نصب Microsoft PowerPoint دارم؟**

خیر، PowerPoint لازم نیست؛ Aspose.Slides یک موتور مستقل برای [ایجاد](/slides/fa/java/create-presentation/)، تغییر، [تبدیل](/slides/fa/java/convert-presentation/)، و [رندرینگ](/slides/fa/java/convert-powerpoint-to-png/) ارائه‌ها است.

**کدام قلم‌ها برای رندرینگ صحیح لازم هستند؟**

در عمل، قلم‌های استفاده‌شده در ارائه یا [جایگزین‌های](/slides/fa/java/font-substitution/) مناسب باید در دسترس باشند. برای اطمینان از رندرینگ یکسان در لینوکس/macOS، نصب بسته‌های قلم عمومی توصیه می‌شود.

**چرا یک قلم سفارشی به‌عنوان جایگزین یا متن گمشده در لینوکس رندر می‌شود؟**

اگر فایل قلم دارای ورودی‌های جدول نام ناسازگار یا خراب باشد، پشته تطبیق قلم لینوکس (FreeType/fontconfig) ممکن است رکورد نامعتبر را انتخاب کند که باعث می‌شود قلم شناسایی نشود. استفاده از نسخه‌ای از قلم با رکوردهای جدول نام اصلاح‌شده یا نصب یک جایگزین سازگار این مشکل را حل می‌کند.
