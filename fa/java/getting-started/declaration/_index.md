{{% alert color="primary" %}} 

تمام کامپوننت‌های Aspose Java به مجموعهٔ مجوز Full Trust نیاز دارند. دلیل این است که کامپوننت‌های Aspose Java برای انجام برخی عملیات مانند تجزیهٔ فونت‌ها و غیره نیاز به دسترسی به تنظیمات رجیستری، فایل‌های سیستم خارج از دایرکتوری مجازی دارند. علاوه بر این، کامپوننت‌های Aspose Java بر پایهٔ کلاس‌های هسته‌ای سیستم Java ساخته شده‌اند که در بسیاری از موارد نیز به مجموعهٔ مجوز Full Trust نیاز دارند. 

{{% /alert %}} 

ارائه‌دهندگان خدمات اینترنتی (ISP) که برنامه‌های متعددی از شرکت‌های مختلف را میزبانی می‌کنند، عمدتاً سطح امنیتی Medium Trust را اعمال می‌نمایند: 

- OleDbPermission در دسترس نیست. این به این معنی است که شما نمی‌توانید از ارائه‌کننده دادهٔ مدیریت شدهٔ ADO.NET OLE DB برای دسترسی به پایگاه‌های داده استفاده کنید.
- EventLogPermission در دسترس نیست. این به این معنی است که شما نمی‌توانید به لاگ رویدادهای ویندوز دسترسی پیدا کنید.
- ReflectionPermission در دسترس نیست. این به این معنی است که شما نمی‌توانید از Reflection استفاده کنید.
- RegistryPermission در دسترس نیست. این به این معنی است که شما نمی‌توانید به رجیستری دسترسی پیدا کنید.
- WebPermission محدود شده است. این به این معنی است که برنامهٔ شما فقط می‌تواند با آدرسی یا محدوده‌ای از آدرس‌ها که در عنصر <trust> تعریف می‌کنید ارتباط برقرار کند.
- FileIOPermission محدود شده است. این به این معنی است که شما فقط می‌توانید به فایل‌های موجود در سلسله‌مراتبی دایرکتوری مجازی برنامهٔ خود دسترسی داشته باشید.

{{% alert color="primary" %}} 

به دلیل دلایل مذکور، کامپوننت‌های Aspose Java نمی‌توانند بر روی سرورهایی که مجموعهٔ مجوز دیگری به‌جز Full Trust ارائه می‌دهند، استفاده شوند. 

{{% /alert %}}
