## **بررسی کلی**

این مقاله توضیح می‌دهد که چه زمانی توسعه‌دهندگان ممکن است برای کار با اسناد ارائه، Open XML SDK یا Aspose.Slides را انتخاب کنند. Open XML SDK به عنوان کتابخانه‌ای برای دستکاری بسته‌های OOXML و عناصر XML پایهٔ آن توصیف شده است، در حالی که Aspose.Slides به عنوان کتابخانهٔ پردازش ارائه با یک مدل شیء سطح بالا و پشتیبانی از بسیاری از وظایف مرتبط با PowerPoint معرفی می‌شود.

مقاله هر دو گزینه را از نظر قالب‌های پشتیبانی شده، مدل برنامه‌نویسی، قابلیت‌های رندرینگ و چاپ، پشتیبانی از پلتفرم‌ها و موارد استفاده رایج مقایسه می‌کند. همچنین روشن می‌کند که Open XML SDK ممکن است برای عملیات‌های پایه‌ای PPTX یا دسترسی مستقیم به عناصر OOXML مناسب باشد، در حالی که Aspose.Slides برای وظایف پیشرفتهٔ ارائه مانند کار با چندین فرمت PowerPoint، کپی یا کلون‌کردن اشکال، جایگزینی متن، اعمال انیمیشن‌ها و تبدیل ارائه‌ها به PDF، TIFF یا XPS مناسب‌تر است.

## **Open XML SDK چیست؟**
بر اساس [MSDN Library](https://docs.microsoft.com/en-us/office/open-xml/open-xml-sdk)، Open XML SDK به‌صورت زیر تعریف می‌شود:

Open XML SDK 2.0 کار دستکاری بسته‌های Open XML و عناصر طرح‌وارهٔ Open XML زیرین را ساده می‌کند. Open XML SDK 2.0 بسیاری از کارهای رایج که توسعه‌دهندگان بر روی بسته‌های Open XML انجام می‌دهند را در خود جای می‌دهد، به‌طوری که بتوانید عملیات‌های پیچیده را تنها با چند خط کد انجام دهید.

اسناد OOXML در اصل فایل‌های XML فشرده هستند و Open XML SDK مجموعه‌ای از کلاس‌هاست که به شما امکان کار با محتوای اسناد OOXML را به‌صورت strongly‑typed می‌دهد. به‌عبارت دیگر، به‌جای استخراج XML از یک فایل فشرده، بارگذاری آن XML در یک درخت DOM و کار مستقیم با عناصر و ویژگی‌های XML، Open XML SDK کلاس‌هایی برای انجام این کار فراهم می‌کند.

## **Aspose.Slides چیست؟**
Aspose.Slides کتابخانهٔ کلاسی است که به برنامهٔ شما امکان انجام وظایف زیر را می‌دهد:

- برنامه‌نویسی با مدل شیء **Presentation**.
- تبدیل‌های با کیفیت بالا بین تمام فرمت‌های محبوب پشتیبانی‌شدهٔ PowerPoint، از جمله تبدیل به PDF، XPS و TIFF.
- قابلیت تولید تصویر کوچک (thumbnail) اسلایدها در فرمت‌های شناخته‌شده‌ای مانند PNG، JPEG و BMP به‌همراه خروجی اسلاید به SVG.
- قابلیت ساخت ارائه‌ها از ابتدا یا ترکیب آن‌ها از یک یا چند سند.
- پشتیبانی از افزودن انیمیشن‌ها، فریم‌های OLE، جداول، ایجاد و مدیریت نمودارها.
- در دسترس بودن کنترل گسترده برای مدیریت قالب‌بندی متن در سطوح TextFrames، Paragraphs و Portions.

برای جزئیات بیشتر دربارهٔ ویژگی‌های پشتیبانی‌شده، لطفاً به [Aspose.Slides Features](/slides/fa/java/product-overview/) مراجعه کنید.

## **نتیجه‌گیری**
{{% alert color="primary" %}} 

Open XML SDK و Aspose.Slides رقیب مستقیم یکدیگر نیستند زیرا نیازها و مخاطبان متفاوتی را هدف قرار می‌دهند. Open XML SDK کتابخانهٔ کلاسی برای ارائهٔ روشی strongly‑typed جهت کار با اسناد OOXML است. Aspose.Slides کتابخانهٔ پردازش ارائه بسیار مفیدی است که پشتیبانی عالی از تقریباً تمام فرمت‌های فایل Microsoft PowerPoint را فراهم می‌کند.

اگر تنها نیاز شما انجام یک عملیات برنامه‌نویسی نسبتاً پایه‌ای بر روی یک سند PPTX باشد، Open XML SDK می‌تواند گزینهٔ مناسب باشد. با Open XML SDK می‌توانید به‌راحتی کارهای ساده‌ای مانند تولید یک سند PPTX ساده یا حذف نظرات، سرصفحه/پاورقی‌ها، استخراج تصاویر و موارد مشابه را انجام دهید. برخی وظایف می‌توانند با Open XML SDK انجام شوند، اما با Aspose.Slides قابل انجام نیستند. برای مثال، اگر نیاز به دسترسی مستقیم به عناصر و ویژگی‌های XML یک سند OOXML دارید، باید از Open XML SDK استفاده کنید. اما اگر نیاز به انجام عملیات‌های پیچیده بر روی اسناد دارید، مانند برخی از وظایف زیر، استفاده از Aspose.Slides بهترین گزینه است:

- پشتیبانی از فرمت‌های قدیمی PowerPoint علاوه بر PPTX.
- کپی یا کلون‌کردن اشکال در اسلایدها به‌گونه‌ای که اشیا، سبک‌ها و قالب‌بندی‌های دیگر به‌صورت مناسب ترکیب شوند.
- جایگزینی متن قالب‌بندی‌شده یا بدون قالب‌بندی.
- اعمال انیمیشن‌ها و استفاده از اتصالات بین اشکال.
- تبدیل یک سند به PDF، TIFF یا XPS به‌گونه‌ای که دقیقاً مشابه خروجی Microsoft PowerPoint باشد.
- توسعهٔ برنامهٔ .NET یا Java در هر دو محیط دسکتاپ و وب.
