## **بررسی کلی**

Aspose.Slides می‌تواند در حالت ارزیابی یا با یک لایسنس معتبر استفاده شود. نسخه ارزیابی همان عملکرد نسخه لایسنس‌دار را دارد، اما هنگام باز یا ذخیره‌سازی ارائه‌ها یک واترمارک ارزیابی اضافه می‌کند و استخراج متن را به یک اسلاید محدود می‌سازد.

این مقاله توضیح می‌دهد که لایسنس‌گذاری در Aspose.Slides چگونه کار می‌کند و چگونه قبل از استفاده از کتابخانه یک لایسنس اعمال کنید. یک لایسنس می‌تواند از فایل، جریان (stream) یا منبع جاسازی‌شده با استفاده از کلاس `License` بارگذاری شود. مقاله همچنین نشان می‌دهد چطور صحت اعمال لایسنس را بررسی کنید.

## **لایسنس‌گذاری در Aspose.Slides**

* یک نسخه ارزیابی پس از خرید لایسنس و افزودن چند خط کد برای اعمال لایسنس، لایسنس‌دار می‌شود.
* لایسنس یک فایل XML متنی ساده است که جزئیاتی مانند نام محصول، تعداد توسعه‌دهندگانی که لایسنس به آن‌ها تعلق دارد، تاریخ انقضای اشتراک و غیره را شامل می‌شود. 
* فایل لایسنس به صورت دیجیتالی امضا شده است، بنابراین نباید آن را تغییر دهید. حتی افزودن یک خط جدید ناخواسته به محتوای فایل، لایسنس را نامعتبر می‌کند.
* Aspose.Slides for Java معمولاً سعی می‌کند لایسنس را در این مکان‌ها پیدا کند:
  * مسیر صریح
  * پوشه‌ای که Aspose.Slides.jar در آن قرار دارد
* برای جلوگیری از محدودیت‌های نسخه ارزیابی، باید قبل از استفاده از **Aspose.Slides** یک لایسنس تنظیم کنید. این کار فقط یک بار برای هر برنامه یا پردازش کافی است.

ممکن است بخواهید به [لایسنس‌گذاری متراژ دار](/slides/fa/java/metered-licensing/) مراجعه کنید.

## **اعمال لایسنس**

یک لایسنس می‌تواند از **فایل** یا **جریان** بارگذاری شود.

{{% alert color="primary" %}}

Aspose.Slides کلاس [License](https://reference.aspose.com/slides/fa/java/com.aspose.slides/License) را برای عملیات لایسنس‌گذاری فراهم می‌کند.

{{% /alert %}} 

{{% alert color="warning" %}}

لایسنس‌های جدید می‌توانند Aspose.Slides را فقط از نسخه 21.4 به بالا فعال کنند. نسخه‌های قبلی از سیستم لایسنس‌گذاری متفاوتی استفاده می‌کنند و این لایسنس‌ها را شناسایی نمی‌کنند.

{{% /alert %}}

### **فایل**

ساده‌ترین روش برای تنظیم لایسنس این است که فایل لایسنس را در پوشه‌ای که Aspose.Slides.jar یا jar برنامهٔ شما قرار دارد بگذارید.

این کد جاوا نشان می‌دهد چطور یک فایل لایسنس تنظیم کنید:

``` java
// نمونه‌سازی کلاس License
com.aspose.slides.License license = new com.aspose.slides.License();

// مسیر فایل لایسنس را تنظیم می‌کند
license.setLicense("Aspose.Slides.Java.lic");
```

{{% alert color="warning" %}} 

اگر فایل لایسنس را در مسیر دیگری قرار دهید، هنگام فراخوانی متد [SetLicense](https://reference.aspose.com/slides/fa/java/com.aspose.slides/License#setLicense-java.lang.String-)، نام فایل لایسنس در انتهای مسیر صریح باید همان نام فایل لایسنس شما باشد.

به عنوان مثال، می‌توانید نام فایل لایسنس را به *Aspose.Slides.Java.lic.xml* تغییر دهید. سپس در کد خود باید مسیر فایل (که با *Aspose.Slides.Java.lic.xml* پایان می‌یابد) را به متد [SetLicense](https://reference.aspose.com/slides/fa/java/com.aspose.slides/License#setLicense-java.lang.String-) پاس بدهید.

{{% /alert %}}

### **جریان**

می‌توانید یک لایسنس را از یک جریان بارگذاری کنید. این کد جاوا نشان می‌دهد چطور یک لایسنس را از جریان اعمال کنید:

``` java
// نمونه‌سازی کلاس License
com.aspose.slides.License license = new com.aspose.slides.License();

// تنظیم لایسنس از طریق یک جریان
license.setLicense(new java.io.FileInputStream("Aspose.Slides.Java.lic"));
```

### **پل PHP/Java**

اگر از Aspose.Slides برای PHP از طریق Java استفاده می‌کنید، می‌توانید لایسنس را از طریق یک پل PHP/Java تنظیم کنید. این پل به شما اجازه می‌دهد کلاس‌های Java را در قالب سینتکس PHP استفاده کنید. برای اطلاعات بیشتر، به [License in PHP](/slides/fa/php-java/licensing/) مراجعه کنید.

## **اعتبارسنجی لایسنس**

برای بررسی اینکه آیا لایسنس به‌درستی تنظیم شده است یا خیر، می‌توانید آن را اعتبارسنجی کنید. این کد جاوا نشان می‌دهد چطور یک لایسنس را اعتبارسنجی کنید:

```java
License license = new License();
license.setLicense("Aspose.Slides.Java.lic");

if (License.isLicensed()) 
{
    System.out.println("License is good!");
}
```

## **ایمنی در برابر تردها**

{{% alert title="Note" color="warning" %}} 

متد [SetLicense](https://reference.aspose.com/slides/fa/java/com.aspose.slides/License#setLicense-java.io.InputStream-) ایمن در برابر تردها نیست. اگر این متد باید همزمان از چندین ترد فراخوانی شود، ممکن است بخواهید از primitives همگام‌سازی (مانند یک قفل) استفاده کنید تا از بروز مشکلات جلوگیری شود. 

{{% /alert %}}

## **سوالات متداول**

**آیا می‌توانم لایسنس را در یک محیط کاملاً آفلاین (بدون دسترسی به اینترنت) اعمال کنم؟**

بله. اعتبارسنجی لایسنس به‌صورت محلی با استفاده از فایل لایسنس انجام می‌شود؛ نیازی به اتصال اینترنت نیست.

**پس از پایان دورهٔ اشتراک یک‌ساله چه اتفاقی می‌افتد؟ آیا کتابخانه متوقف می‌شود؟**

خیر. لایسنس دائمی است: می‌توانید از نسخه‌های منتشرشده قبل از تاریخ پایان اشتراک خود استفاده کنید؛ فقط بدون تمدید نمی‌توانید از نسخه‌های جدیدتر استفاده کنید.
