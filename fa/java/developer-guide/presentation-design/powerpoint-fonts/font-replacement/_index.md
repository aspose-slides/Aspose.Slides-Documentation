## **بررسی کلی**

Aspose.Slides به شما امکان می‌دهد یک قلم را در سراسر ارائه با قلم دیگری جایگزین کنید. هنگامی که یک قلم جایگزین می‌شود، تمام نمونه‌های قلم اصلی به قلم جدید تغییر می‌یابند.

برای انجام جایگزینی قلم، ارائه را بارگذاری کنید، قلم منبع و قلم جایگزین را تعریف کنید، متد جایگزینی قلم را فراخوانی کنید و ارائهٔ تغییر یافته را به صورت فایل PPTX ذخیره کنید. این روش زمانی مفید است که عمداً بخواهید از یک خانوادهٔ قلم به خانوادهٔ دیگری در تمام ارائه تغییر دهید.

## **جایگزینی قلم‌ها**

اگر در مورد استفاده از یک قلم نظر خود را تغییر دادید، می‌توانید آن قلم را با قلم دیگری جایگزین کنید. تمام نمونه‌های قلم قدیم با قلم جدید جایگزین خواهند شد.

Aspose.Slides به شما اجازه می‌دهد قلم را به این شکل جایگزین کنید:

1. ارائهٔ مربوطه را بارگذاری کنید.  
2. قلمی که قرار است جایگزین شود را بارگذاری کنید.  
3. قلم جدید را بارگذاری کنید.  
4. قلم را جایگزین کنید.  
5. ارائهٔ تغییر یافته را به صورت فایل PPTX بنویسید.

این کد جاوا نشان‌دهندهٔ جایگزینی قلم است:

```java
// یک ارائه را بارگذاری می‌کند
Presentation pres = new Presentation("Fonts.pptx");
try {
    // قلم منبع که قرار است جایگزین شود را بارگذاری می‌کند
    IFontData sourceFont = new FontData("Arial");
    
    // قلم جدید را بارگذاری می‌کند
    IFontData destFont = new FontData("Times New Roman");
    
    // قلم‌ها را جایگزین می‌کند
    pres.getFontsManager().replaceFont(sourceFont, destFont);
    
    // ارائه را ذخیره می‌کند
    pres.save("UpdatedFont_out.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```

## **سؤالات متداول**

**تفاوت «جایگزینی قلم»، «جایگزینی فونت» و «قلم‌های پیش‌فرض» چیست؟**

جایگزینی یک تغییر عمدی از یک خانواده به خانوادهٔ دیگر در کل سند است. [**جایگزینی فونت**](/slides/fa/java/font-substitution/) قانونی شبیه «اگر قلم در دسترس نیست، از X استفاده کن» است. [**قلم پیش‌فرض**](/slides/fa/java/fallback-font/) به صورت موضعی برای گلیف‌های گمشدهٔ فردی اعمال می‌شود وقتی قلم پایه نصب است اما کاراکترهای مورد نیاز را ندارد.

**آیا جایگزینی بر اسلایدهای اصلی، طرح‌بندی‌ها، یادداشت‌ها و نظرات اعمال می‌شود؟**

بله. جایگزینی بر تمام اشیای ارائه‌ای که از قلم اصلی استفاده می‌کنند، شامل اسلایدهای اصلی و یادداشت‌ها، تأثیر می‌گذارد؛ نظرات نیز بخشی از سند هستند و توسط موتور قلم در نظر گرفته می‌شوند.

**آیا قلم داخل اشیای OLE توکار (مثلاً Excel) تغییر می‌کند؟**

خیر. محتوای [OLE](/slides/fa/java/manage-ole/) توسط برنامهٔ خود کنترل می‌شود. جایگزینی در ارائه داده‌های داخلی OLE را بازفرمت نمی‌کند؛ ممکن است به‌صورت تصویر یا محتوای قابل ویرایش خارجی نمایش داده شود.

**آیا می‌توانم قلم را فقط در بخشی از ارائه (مثلاً بر اساس اسلایدها یا نواحی) جایگزین کنم؟**

جایگزینی هدفمند امکان‌پذیر است اگر قلم را در سطح اشیاء/بازه‌های موردنیاز تغییر دهید نه اینکه یک جایگزینی سراسری برای کل سند اعمال کنید. منطق کلی انتخاب قلم هنگام رندر همچنان یکسان می‌ماند.

**چگونه می‌توانم پیش از پیش تعیین کنم که ارائه از چه قلم‌هایی استفاده می‌کند؟**

از [مدیر قلم]‌(https://reference.aspose.com/slides/fa/java/com.aspose.slides/fontsmanager/) ارائه استفاده کنید: این ابزار فهرستی از [خانواده‌های مورد استفاده]‌(https://reference.aspose.com/slides/fa/java/com.aspose.slides/fontsmanager/#getFonts--) و اطلاعاتی درباره [جایگزینی‌ها/قلم‌های «ناشناخته»]‌(https://reference.aspose.com/slides/fa/java/com.aspose.slides/fontsmanager/#getSubstitutions--) ارائه می‌دهد که به برنامه‌ریزی جایگزینی کمک می‌کند.

**آیا جایگزینی قلم هنگام تبدیل به PDF/تصاویر کار می‌کند؟**

بله. در زمان خروجی‌گیری، Aspose.Slides همان [دنبالهٔ انتخاب/جایگزینی قلم](/slides/fa/java/font-selection-sequence/) را اعمال می‌کند، بنابراین جایگزینی انجام‌شده پیش از آن در حین تبدیل مورد توجه قرار می‌گیرد.

**آیا نیاز است قلم هدف را در سیستم نصب کنم یا می‌توانم پوشهٔ قلم‌ها را ضمیمه کنم؟**

نیازی به نصب نیست: کتابخانه امکان [بارگذاری قلم‌های خارجی](/slides/fa/java/custom-font/) از پوشه‌های کاربری را برای استفاده در هنگام [رندر و خروجی](/slides/fa/java/convert-powerpoint/) فراهم می‌کند.

**آیا جایگزینی مشکل «توفو» (مربعات به جای کاراکترها) را رفع می‌کند؟**

فقط در صورتی که قلم هدف واقعاً گلیف‌های مورد نیاز را داشته باشد. در غیر این صورت، برای پوشش کاراکترهای گمشده باید [قلم پیش‌فرض](/slides/fa/java/fallback-font/) را پیکربندی کنید.
