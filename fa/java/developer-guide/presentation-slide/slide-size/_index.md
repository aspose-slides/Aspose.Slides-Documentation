## **معرفی**

Aspose.Slides ابزارهای جامع برای تنظیم اندازه اسلاید و نسبت ابعاد در ارائه‌های PowerPoint فراهم می‌کند، که برای چاپ و نمایش روی صفحه‌نمایش مهم است.

اندازه‌ها و نسبت‌های پرطرفدار اسلاید:

- **استاندارد (نسبت ابعاد 4:3)**: مناسب برای صفحه‌نمایش‌ها و دستگاه‌های قدیمی.
- **پهن‌صفحه (نسبت ابعاد 16:9)**: برای پروژکتورها و نمایشگرهای مدرن توصیه می‌شود.

برای حفظ یکپارچگی در سراسر ارائه خود، یک اندازه اسلاید و نسبت ابعاد واحد بر تمام اسلایدها اعمال می‌شود. برای نتایج بهینه، ابعاد اسلاید را در ابتدای فرآیند ایجاد ارائه تنظیم کنید تا از بروز مشکلات جلوگیری شود.

{{% alert color="primary" %}} 
به طور پیش‌فرض، ارائه‌های ایجاد شده با Aspose.Slides از نسبت ابعاد استاندارد 4:3 استفاده می‌کنند.
{{% /alert %}}

## **تغییر اندازه اسلاید در ارائه‌ها**

این نمونه کد نشان می‌دهد چگونه با استفاده از Aspose.Slides در Java اندازه اسلاید یک ارائه را تغییر دهید:

```java
Presentation pres = new Presentation("pres-4x3-aspect-ratio.pptx");
try {
    pres.getSlideSize().setSize(SlideSizeType.OnScreen16x9, SlideSizeScaleType.DoNotScale);
    pres.save("pres-4x3-aspect-ratio.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```

## **مشخص کردن اندازه‌های سفارشی اسلاید در ارائه‌ها**

اگر اندازه‌های معمول اسلاید (4:3 و 16:9) برای کار شما مناسب نیستند، ممکن است تصمیم بگیرید از یک اندازه اسلاید خاص یا منحصر به فرد استفاده کنید. برای مثال، اگر قصد داشته باشید اسلایدهای کامل‌سایز را از ارائه خود بر روی یک قالب صفحه سفارشی چاپ کنید یا اگر می‌خواهید ارائه خود را روی برخی نوع‌های صفحه‌نمایش نمایش دهید، احتمالاً با استفاده از تنظیم اندازه سفارشی برای ارائه خود بهره‌مند خواهید شد.

این نمونه کد نشان می‌دهد چگونه از Aspose.Slides برای Java برای مشخص کردن یک اندازه اسلاید سفارشی برای یک ارائه در Java استفاده کنید:

```java
Presentation pres = new Presentation("pres.pptx");
try {
    pres.getSlideSize().setSize(780, 540, SlideSizeScaleType.DoNotScale); // اندازه کاغذ A4
    pres.save("pres-a4-slide-size.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```

## **مدیریت محتوای اسلاید پس از تغییر اندازه**

پس از تغییر اندازه اسلاید یک ارائه، محتوای اسلایدها (مانند تصاویر یا اشیا) ممکن است دچار اعوجاج شوند. به‌طور پیش‌فرض، اشیا به‌صورت خودکار برای منطبق شدن با اندازه جدید اسلاید تغییر اندازه می‌یابند. با این حال، هنگام تغییر اندازه اسلاید یک ارائه، می‌توانید تنظیماتی را مشخص کنید که تعیین می‌کند Aspose.Slides چگونه با محتوای اسلایدها برخورد می‌کند.

بسته به هدف یا انتظاری که دارید، می‌توانید از هر یک از این تنظیمات استفاده کنید:

- `DoNotScale`

  اگر نمی‌خواهید اشیا روی اسلایدها تغییر اندازه یابند، از این تنظیم استفاده کنید.

- `EnsureFit`

  اگر می‌خواهید به یک اندازه اسلاید کوچکتر مقیاس کنید و نیاز دارید Aspose.Slides اشیای اسلایدها را کاهش دهد تا همه روی اسلایدها جا بگیرند (به این ترتیب، از از دست رفتن محتوا جلوگیری می‌کنید)، از این تنظیم استفاده کنید.

- `Maximize`

  اگر می‌خواهید به یک اندازه اسلاید بزرگتر مقیاس کنید و نیاز دارید Aspose.Slides اشیای اسلایدها را بزرگتر کند تا نسبت به اندازه جدید اسلاید متناسب شوند، از این تنظیم استفاده کنید.

این نمونه کد نشان می‌دهد چگونه از تنظیم `Maximize` هنگام تغییر اندازه اسلاید یک ارائه استفاده کنید:

```java
Presentation pres = new Presentation("pres.pptx");
try {
    pres.getSlideSize().setSize(SlideSizeType.Ledger, SlideSizeScaleType.Maximize);
} finally {
    if (pres != null) pres.dispose();
}
```

## **پرسش‌های متداول**

**آیا می‌توانم اندازه اسلاید سفارشی را با واحدهای دیگر به‌جز اینچ (مثلاً پوینت یا میلی‌متر) تنظیم کنم؟**

بله. Aspose.Slides به‌صورت داخلی از پوینت استفاده می‌کند که 1 پوینت برابر 1/72 اینچ است. می‌توانید هر واحدی (مانند میلی‌متر یا سانتی‌متر) را به پوینت تبدیل کرده و از مقادیر تبدیل‌شده برای تعریف عرض و ارتفاع اسلاید استفاده کنید.

**آیا اندازه اسلاید سفارشی بسیار بزرگ بر عملکرد و مصرف حافظه در هنگام رندر کردن تأثیر می‌گذارد؟**

بله. ابعاد بزرگتر اسلاید (به پوینت) همراه با مقیاس رندر بالاتر منجر به افزایش مصرف حافظه و زمان پردازش طولانی‌تر می‌شود. سعی کنید اندازه اسلایدی عملی انتخاب کنید و مقیاس رندر را فقط در صورت نیاز برای دستیابی به کیفیت خروجی موردنظر تنظیم کنید.

**آیا می‌توانم یک اندازه اسلاید غیر استاندارد تعریف کنم و سپس اسلایدها را از ارائه‌هایی که اندازه‌های متفاوتی دارند ترکیب کنم؟**

شما نمی‌توانید [ادغام ارائه‌ها](/slides/fa/java/merge-presentation/) را انجام دهید در حالی که اندازه اسلایدهای آنها متفاوت است — ابتدا یک ارائه را به اندازه دیگر تغییر دهید. هنگام تغییر اندازه اسلاید، می‌توانید با استفاده از گزینه [SlideSizeScaleType](https://reference.aspose.com/slides/fa/java/com.aspose.slides/slidesizescaletype/) تعیین کنید محتویات موجود چگونه مدیریت شوند. پس از هم‌تراز کردن اندازه‌ها، می‌توانید اسلایدها را ترکیب کنید در حالی که قالب‌بندی حفظ می‌شود.

**آیا می‌توانم تصویرهای کوچک برای شکل‌های منفرد یا مناطق خاصی از اسلاید تولید کنم، و آیا آنها به اندازه جدید اسلاید احترام می‌گذارند؟**

بله. Aspose.Slides می‌تواند تصویرهای کوچک برای [تمام اسلایدها](https://reference.aspose.com/slides/fa/java/com.aspose.slides/slide/#getImage-com.aspose.slides.IRenderingOptions-float-float-) و همچنین برای [شکل‌های انتخاب‌شده](https://reference.aspose.com/slides/fa/java/com.aspose.slides/shape/#getImage-int-float-float-) تولید کند. تصاویر حاصل اندازه اسلاید فعلی و نسبت ابعاد را منعکس می‌کند و چارچوب و هندسهٔ سازگاری را تضمین می‌نماید.
