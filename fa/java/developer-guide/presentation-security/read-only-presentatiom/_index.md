## **مقدمه**

در PowerPoint 2019، مایکروسافت تنظیم **Always Open Read-Only** را به عنوان یکی از گزینه‌هایی که کاربران می‌توانند برای محافظت از ارائه‌های خود استفاده کنند، معرفی کرد. ممکن است بخواهید از این تنظیم Read-Only برای محافظت از یک ارائه استفاده کنید وقتی که

- می‌خواهید از ویرایش‌های تصادفی جلوگیری کنید و محتوای ارائه‌تان را ایمن نگه دارید.  
- می‌خواهید به افراد اطلاع دهید که ارائه‌ای که ارائه کرده‌اید نسخه نهایی است.  

پس از اینکه گزینه **Always Open Read-Only** را برای یک ارائه انتخاب کردید، وقتی کاربران ارائه را باز می‌کنند، توصیه **Read-Only** را می‌بینند و ممکن است پیامی به این شکل مشاهده کنند: *برای جلوگیری از تغییرات تصادفی، نویسنده این فایل را به‌صورت فقط‑خواندنی تنظیم کرده است.*

توصیه Read-Only یک بازدارنده ساده اما مؤثر است که ویرایش را دلسرد می‌کند، زیرا کاربران باید کاری انجام دهند تا آن را حذف کنند پیش از اینکه اجازه ویرایش ارائه را داشته باشند. اگر نمی‌خواهید کاربران تغییراتی در ارائه اعمال کنند و می‌خواهید به‌صورت مودبانه این موضوع را به آن‌ها اطلاع دهید، توصیه Read-Only می‌تواند گزینه خوبی برای شما باشد.

> اگر یک ارائه با محافظت **Read-Only** در یک نسخه قدیمی‌تر Microsoft PowerPoint باز شود—که از عملکرد تازه معرفی‌شده پشتیبانی نمی‌کند—توصیه **Read-Only** نادیده گرفته می‌شود (ارائه به‌صورت معمولی باز می‌شود).

## **اعمال حالت فقط‌خواندنی**

Aspose.Slides for Java به شما امکان می‌دهد یک ارائه را به حالت **Read-Only** تنظیم کنید، به این معنی که کاربران (پس از باز کردن ارائه) توصیه **Read-Only** را می‌بینند. این کد نمونه نشان می‌دهد چگونه یک ارائه را به حالت **Read-Only** در Java با استفاده از Aspose.Slides تنظیم کنید:

```java
Presentation pres = new Presentation();
try {
    pres.getProtectionManager().setReadOnlyRecommended(true);
    pres.save("ReadOnlyPresentation.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```

{{% alert color="primary" %}} 
**توجه**: توصیه **Read-Only** صرفاً برای دلسرد کردن ویرایش یا جلوگیری از تغییرات تصادفی در یک ارائه PowerPoint است. اگر شخصی با انگیزه—که می‌داند چه کاری انجام می‌دهد—تصمیم بگیرد ارائه شما را ویرایش کند، به راحتی می‌تواند تنظیم Read-Only را حذف کند. اگر به‌طور جدی نیاز به جلوگیری از ویرایش غیرمجاز دارید، بهتر است از [حفاظت‌های سخت‌گیرانه‌تر که شامل رمزنگاری‌ها و گذرواژه‌ها می‌شود](https://docs.aspose.com/slides/fa/java/password-protected-presentation/) استفاده کنید. 
{{% /alert %}} 

## **سؤالات متداول**

**'Read-Only recommended' چگونه با حفاظت کامل با گذرواژه متفاوت است؟**  
'Read-Only recommended' فقط یک پیشنهاد برای باز کردن فایل در حالت فقط‑خواندنی نمایش می‌دهد و به‌سادگی قابل عبور است. [حفاظت با گذرواژه](/slides/fa/java/password-protected-presentation/) در واقع باز کردن یا ویرایش را محدود می‌کند و زمانی مناسب است که به کنترل‌های امنیتی واقعی نیاز داشته باشید.

**آیا می‌توان 'Read-Only recommended' را با watermarks ترکیب کرد تا ویرایش‌ها بیشتر دلسرد شوند؟**  
بله. این توصیه می‌تواند همراه با [آثار تصویری](/slides/fa/java/watermark/) به عنوان یک بازدارنده بصری استفاده شود؛ آن‌ها مکانیزم‌های جداگانه‌ای هستند و با هم به‌خوبی کار می‌کنند.

**آیا ماکرو یا ابزار خارجی همچنان می‌تواند فایل را وقتی توصیه فعال باشد، تغییر دهد؟**  
بله. این توصیه برنامه‌نویسی تغییرات را مسدود نمی‌کند. برای جلوگیری از ویرایش‌های خودکار، از [گذرواژه‌ها و رمزنگاری](/slides/fa/java/password-protected-presentation/) استفاده کنید.

**توصیه 'Read-Only recommended' چگونه با متدهای 'isEncrypted' و 'isWriteProtected' مرتبط است؟**  
آن‌ها سیگنال‌های متفاوتی هستند. 'Read-Only recommended' یک اعلان نرم و اختیاری است؛ [isWriteProtected](https://reference.aspose.com/slides/fa/java/com.aspose.slides/protectionmanager/#isWriteProtected--) و [isEncrypted](https://reference.aspose.com/slides/fa/java/com.aspose.slides/protectionmanager/#isEncrypted--) محدودیت‌های واقعی نوشتن یا خواندن را نشان می‌دهند که به گذرواژه‌ها یا رمزنگاری وابسته‌اند.
