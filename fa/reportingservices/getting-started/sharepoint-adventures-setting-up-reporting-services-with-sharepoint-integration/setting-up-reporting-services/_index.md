## **حساب سرویس**
اطمینان حاصل کنید که حساب سرویسی که برای Reporting Services استفاده می‌کنید را درک کنید. اگر با مشکلاتی مواجه شدیم، ممکن است مربوط به حساب سرویس مورد استفاده باشد. مقدار پیش‌فرض Network Service است. هر زمان که نسخه‌های جدید را مستقر می‌کنم، همیشه از حساب‌های دامنه استفاده می‌کنم، زیرا احتمال بروز مشکلات در آنجا بیشتر است. برای این پیکربندی روی سرورم، از یک حساب دامنه به نام **RSService** استفاده کرده‌ام. 
## **آدرس وب سرویس**
ما باید آدرس وب سرویس را پیکربندی کنیم. این یک پوشهٔ مجازی **ReportServer** (vdir) است که سرویس‌های وب Reporting Services را میزبانی می‌کند و با آن SharePoint ارتباط برقرار می‌کند. مگر اینکه بخواهید ویژگی‌های vdir را سفارشی کنید (مثلاً SSL، پورت‌ها، سرآیندهای میزبان و غیره)، کافی است در اینجا روی Apply کلیک کنید و آماده باشید.

## **پایگاه داده**
ما نیاز به ایجاد پایگاه دادهٔ فهرست Reporting Services داریم. این می‌تواند بر روی هر موتور پایگاه داده SQL 2008 یا SQL 2008 R2 قرار گیرد. SQL11 نیز به‌خوبی کار می‌کند، اما هنوز در حالت BETA است. این عمل دو پایگاه داده **ReportServer** و **ReportServerTempDB** را به‌صورت پیش‌فرض ایجاد می‌کند. 
مرحله مهم دیگر این است که مطمئن شوید نوع پایگاه داده را به‌صورت SharePoint Integrated انتخاب کنید. پس از انتخاب این گزینه، امکان تغییر آن نیست. لطفاً برای مرجع به شکل‌های 5، 6 و 7 توجه کنید.

برای اعتبارها، این همان روشی است که Report Server با SQL Server ارتباط برقرار می‌کند. هر حسابی را که انتخاب کنید، حقوقی در داخل پایگاه داده فهرست و همچنین چند پایگاه داده سیستمی از طریق RSExecRole خواهد داشت. ## **آدرس Report Manager**
می‌توانیم آدرس Report Manager را نادیده بگیریم، زیرا در حالت SharePoint Integrated استفاده نمی‌شود. SharePoint رابط کاربری ماست. Report Manager کار نمی‌کند. 
## **کلیدهای رمزنگاری**
کلیدهای رمزنگاری خود را پشتیبان‌گیری کنید و مطمئن شوید مکان ذخیره‌سازی آن‌ها را می‌دانید. در صورتی که به وضعیت انتقال یا بازیابی پایگاه داده نیاز داشته باشید، به این کلیدها نیاز خواهید داشت. 

![todo:image_alt_text](setting-up-reporting-services_9.png)

این پایان کار با مدیر پیکربندی سرویس‌های گزارش‌دهی است. اگر به URL در زبانهٔ آدرس وب سرویس مرور کنید، باید چیزی شبیه به شکل زیر نمایش داده شود.

**شکل 12**: دسترسی به Report Server پس از نصب 

چه اتفاقی افتاد؟ SharePoint بر روی WFE من نصب شده است و من تنظیمات Reporting Services را به پایان رساندم. در این مثال، Reporting Services و SharePoint روی ماشین‌های متفاوتی هستند. اگر بر روی یک ماشین بودند، این خطا را نمی‌دیدید. از نظر فنی لازم است SharePoint را بر روی سرور RS نصب کنیم. این به این معنی است که IIS نیز فعال خواهد شد.
