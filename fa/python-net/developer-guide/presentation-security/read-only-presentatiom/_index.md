## **مقدمه**

در PowerPoint 2019، Microsoft تنظیم **Always Open Read-Only** را به عنوان یکی از گزینه‌هایی که کاربران می‌توانند برای محافظت از ارائه‌های خود استفاده کنند، معرفی کرد. ممکن است بخواهید از این تنظیم خواندنی‑فقط برای محافظت از یک ارائه استفاده کنید وقتی

- می‌خواهید از ویرایش‌های ناخواسته جلوگیری کنید و محتویات ارائه خود را ایمن نگه دارید.  
- می‌خواهید به افراد اطلاع دهید که ارائه‌ای که ارائه داده‌اید نسخه نهایی است.  

پس از انتخاب گزینه **Always Open Read-Only** برای یک ارائه، وقتی کاربران آن را باز می‌کنند، توصیه **Read-Only** را می‌بینند و ممکن است پیامی به این شکل نمایش داده شود: *برای جلوگیری از تغییرات ناخواسته، نویسنده این فایل را برای باز کردن به صورت فقط‑خواندنی تنظیم کرده است.*

توصیه Read-Only یک مانع ساده اما مؤثر است که از ویرایش جلوگیری می‌کند زیرا کاربران باید کاری انجام دهند تا قبل از امکان ویرایش، این توصیه را حذف کنند. اگر نمی‌خواهید کاربران به ارائه تغییر بدهند و می‌خواهید این موضوع را به‌صورت مؤدبانه اطلاع دهید، توصیه Read-Only می‌تواند گزینهٔ خوبی برای شما باشد.

> اگر یک ارائه با محافظت **Read-Only** در برنامهٔ Microsoft PowerPoint قدیمی‌تری که این عملکرد جدید را پشتیبانی نمی‌کند باز شود، توصیه **Read-Only** نادیده گرفته می‌شود (ارائه به‌صورت عادی باز می‌شود).

## **اعمال حالت Read-Only**

Aspose.Slides for Python via .NET به شما اجازه می‌دهد یک ارائه را به **Read-Only** تنظیم کنید، به این معنی که کاربران (پس از باز کردن ارائه) توصیه **Read-Only** را می‌بینند. این کد نمونه نشان می‌دهد چگونه یک ارائه را به **Read-Only** در Python با استفاده از Aspose.Slides تنظیم کنید:

```py
import aspose.slides as slides

with slides.Presentation() as pres:
    pres.protection_manager.read_only_recommended = True
    pres.save("ReadOnlyPresentation.pptx", slides.export.SaveFormat.PPTX)
```

{{% alert color="primary" %}} 

**نکته**: توصیه **Read-Only** صرفاً برای جلوگیری از ویرایش یا ممانعت از تغییرات ناخواسته در یک ارائهٔ PowerPoint است. اگر فردی با انگیزه و دانش کافی تصمیم بگیرد ارائهٔ شما را ویرایش کند، می‌تواند به راحتی تنظیم Read-Only را حذف کند. اگر به‌طور جدی نیاز به جلوگیری از ویرایش غیرمجاز دارید، بهتر است از [محافظت‌های سخت‌تری که شامل رمزگذاری و رمزهای عبور می‌شوند](https://docs.aspose.com/slides/fa/python-net/password-protected-presentation/) استفاده کنید.

## **پرسش‌های متداول**

**تفاوت «Read-Only recommended» با محافظت کامل با رمز عبور چیست؟**

«Read-Only recommended» تنها یک پیشنهاد برای باز کردن فایل به حالت فقط‑خواندنی نمایش می‌دهد و به‌راحتی قابل دور زدن است. [محافظت با رمز عبور](/slides/fa/python-net/password-protected-presentation/) در واقع باز کردن یا ویرایش را محدود می‌کند و زمانی که نیاز به کنترل‌های امنیتی واقعی دارید، مناسب است.

**آیا می‌توان «Read-Only recommended» را با واترمارک‌ها ترکیب کرد تا ویرایش‌ها بیشتر دلسرد شوند؟**

بله. این توصیه می‌تواند با [واترمارک‌ها](/slides/fa/python-net/watermark/) به‌عنوان یک مانع بصری همراه شود؛ آن‌ها مکانیزم‌های جداگانه‌ای هستند که به‌خوبی با هم کار می‌کنند.

**آیا یک ماکرو یا ابزار خارجی هنوز می‌تواند فایل را وقتی توصیه فعال است، تغییر دهد؟**

بله. این توصیه برنامه‌نویسی تغییرات را مسدود نمی‌کند. برای جلوگیری از ویرایش‌های خودکار، از [رمزهای عبور و رمزگذاری](/slides/fa/python-net/password-protected-presentation/) استفاده کنید.

**«Read-Only recommended» چگونه با پرچم‌های «is_encrypted» و «is_write_protected» مرتبط است؟**

این‌ها سیگنال‌های متفاوتی هستند. «Read-Only recommended» یک اعلان نرم و اختیاری است؛ [is_write_protected](https://reference.aspose.com/slides/fa/python-net/aspose.slides/protectionmanager/is_write_protected/) و [is_encrypted](https://reference.aspose.com/slides/fa/python-net/aspose.slides/protectionmanager/is_encrypted/) نشان‌دهندهٔ محدودیت‌های واقعی نوشتن یا خواندن هستند که بسته به رمز عبور یا رمزگذاری فعال می‌شوند.
