## **مقدمه**

مجوز متریکی یک مکانیزم مجوزدهی است که می‌تواند همراه با روش‌های مجوزدهی موجود استفاده شود. اگر می‌خواهید بر اساس استفاده‌تان از قابلیت‌های Aspose.Slides API هزینه‌گیری شوید، مجوز متریکی را انتخاب می‌کنید.

## **اعمال کلیدهای متریکی**

{{% alert color="primary" %}} 

مجوز متریکی یک مکانیزم جدید مجوزدهی است که می‌تواند همراه با روش‌های مجوزدهی موجود استفاده شود. اگر می‌خواهید بر اساس استفاده‌تان از قابلیت‌های Aspose.Slides API هزینه‌گیری شوید، مجوز متریکی را انتخاب می‌کنید.

هنگامی که یک مجوز متریکی خریداری می‌کنید، کلیدها (و نه یک فایل مجوز) دریافت می‌کنید. این کلید متریکی می‌تواند با استفاده از کلاس [Metered](https://reference.aspose.com/slides/fa/python-net/aspose.slides/metered/) که Aspose برای عملیات متریک ارائه داده است، اعمال شود. برای جزئیات بیشتر، به [Metered Licensing FAQ](https://purchase.aspose.com/faqs/licensing/metered) مراجعه کنید.

{{% /alert %}} 

1. یک نمونه از کلاس [Metered](https://reference.aspose.com/slides/fa/python-net/aspose.slides/metered/) بسازید.
1. کلیدهای عمومی و خصوصی خود را به متد [set_metered_key](https://reference.aspose.com/slides/fa/python-net/aspose.slides/metered/set_metered_key/#str-str) پاس دهید.
1. برخی پردازش‌ها را انجام دهید (کارها را انجام دهید).
1. متد [get_consumption_quantity](https://reference.aspose.com/slides/fa/python-net/aspose.slides/metered/get_consumption_quantity/#) از کلاس `Metered` را صدا بزنید.

باید مقدار/تعداد درخواست‌های API که تا کنون مصرف کرده‌اید را ببینید.

این کد نمونه نشان می‌دهد چگونه از مجوز متریکی استفاده کنید:

```python
import aspose.slides as slides

# یک نمونه از کلاس Metered را ایجاد می‌کند
metered = slides.Metered()

# کلیدهای عمومی و خصوصی را به شی Metered پاس می‌دهد
metered.set_metered_key("<valid public key>", "<valid private key>")

# مقدار مقدار مصرف شده را پیش از فراخوانی‌های API دریافت می‌کند
amount_before = slides.Metered.get_consumption_quantity()
print("Amount consumed before:", amount_before)

# در اینجا کاری با Aspose.Slides API انجام دهید
# ...

# مقدار مقدار مصرف شده را پس از فراخوانی‌های API دریافت می‌کند
amount_after = slides.Metered.get_consumption_quantity()
print("Amount consumed after:", amount_after)
```

{{% alert color="warning" title="NOTE"  %}} 

برای استفاده از مجوز متریکی، به یک اتصال اینترنتی پایدار نیاز دارید زیرا مکانیزم مجوزدهی از اینترنت برای تعامل مداوم با سرویس‌های ما و انجام محاسبات استفاده می‌کند.

{{% /alert %}} 

## **سوالات متداول**

**آیا می‌توانم یک مجوز متریکی را به همراه یک مجوز معمولی (دائمی یا موقت) در همان برنامه استفاده کنم؟**

بله. مجوز متریکی یک مکانیزم افزودنی مجوزدهی است که می‌تواند همراه با [روش‌های مجوزدهی](/slides/fa/python-net/licensing/) موجود استفاده شود. شما هنگام شروع برنامه تصمیم می‌گیرید کدام مکانیزم را اعمال کنید.

**مصرف به چه چیزی تحت یک مجوز متریکی شمرده می‌شود: عملیات‌ها یا فایل‌ها؟**

مصرف به استفاده از API محاسبه می‌شود، یعنی تعداد درخواست‌ها یا عملیات‌ها. می‌توانید مصرف فعلی را از طریق [روش‌های ردیابی مصرف](https://reference.aspose.com/slides/fa/python-net/aspose.slides/metered/) به دست آورید.

**آیا مجوز متریکی برای میکروسرویس‌ها و محیط‌های سرورلس که نمونه‌ها به‌طور مکرر راه‌اندازی می‌شوند مناسب است؟**

بله. از آنجا که حسابداری در سطح تماس API انجام می‌شود، سناریوهای با راه‌اندازی سرد مکرر سازگار هستند، مشروط بر این که دسترسی شبکه‌ای پایدار برای محاسبات متریکی وجود داشته باشد.

**آیا عملکرد کتابخانه هنگام استفاده از مجوز متریکی با مجوز دائمی متفاوت است؟**

خیر. این فقط مربوط به مکانیزم مجوزدهی و صورتحساب است؛ قابلیت‌های محصول همانند قبلی است.

**مجوز متریکی چه رابطه‌ای با نسخه آزمایشی و مجوز موقت دارد؟**

نسخه آزمایشی دارای محدودیت‌ها و واترمارک‌ها است، [مجوز موقت](https://purchase.aspose.com/temporary-license/) محدودیت‌ها را به مدت 30 روز حذف می‌کند، و مجوز متریکی محدودیت‌ها را حذف کرده و براساس استفاده واقعی هزینه می‌گیرد.

**آیا می‌توانم با ردعمل خودکار وقتی آستانه مصرف تجاوز می‌شود بودجه را کنترل کنم؟**

بله. یک روش معمول این است که به‌صورت دوره‌ای مصرف فعلی را از طریق [روش‌های ردیابی](https://reference.aspose.com/slides/fa/python-net/aspose.slides/metered/) بخوانید و محدودیت‌ها یا هشدارهای خود را در سطح برنامه یا نظارت پیاده‌سازی کنید.
