## **مرور کلی**

Aspose.Slides به شما امکان می‌دهد یک قلم را در تمام ارائه با قلم دیگری جایگزین کنید. وقتی قلم جایگزین می‌شود، تمام نمونه‌های قلم اصلی به قلم جدید تغییر می‌کنند.

برای انجام جایگزینی قلم، ارائه را بارگذاری کنید، قلم منبع و قلم جایگزین را تعریف کنید، متد جایگزینی قلم را فراخوانی کنید و ارائه‌ اصلاح‌شده را به‌صورت فایل PPTX ذخیره کنید. این روش زمانی مفید است که بخواهید به‌صورت عمدی یک خانواده قلم را در تمام ارائه با خانواده دیگری تعویض کنید.

## **جایگزینی قلم‌ها**

اگر نظر خود را درباره استفاده از یک قلم تغییر دهید، می‌توانید آن قلم را با قلم دیگری جایگزین کنید. تمام نمونه‌های قلم قدیمی توسط قلم جدید جایگزین خواهند شد.

Aspose.Slides به شما امکان می‌دهد قلم را به این شکل جایگزین کنید:

1. ارائه مربوطه را بارگذاری کنید.  
2. قلمی که قرار است جایگزین شود را بارگذاری کنید.  
3. قلم جدید را بارگذاری کنید.  
4. قلم را جایگزین کنید.  
5. ارائه اصلاح‌شده را به‌صورت فایل PPTX بنویسید.

این کد Java جایگزینی قلم را نشان می‌دهد:

```java
// ارائه‌ای را بارگذاری می‌کند
Presentation pres = new Presentation("Fonts.pptx");
try {
    // قلم منبعی که قرار است جایگزین شود را بارگذاری می‌کند
    IFontData sourceFont = new FontData("Arial");
    
    // قلم جدید را بارگذاری می‌کند
    IFontData destFont = new FontData("Times New Roman");
    
    // قلم‌ها را جایگزین می‌کند
    pres.getFontsManager().replaceFont(sourceFont, destFont);
    
    // ارائه را ذخیره می‌کند
    pres.save("UpdatedFont_out.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```

## **سوالات متداول**

**تفاوت بین «جایگزینی قلم»، «جایگزینی قلم» و «قلم‌های پیش‌فرض» چیست؟**

جایگزینی یک تغییر عمدی از یک خانواده قلم به خانواده دیگری در تمام سند است. [جایگزینی](/slides/fa/androidjava/font-substitution/) قانونی است مانند «اگر قلم در دسترس نباشد، از X استفاده شود». [قلم پیش‌فرض](/slides/fa/androidjava/fallback-font/) به‌صورت هدفمند برای گلیف‌های از دست رفتهٔ تک‌تک اعمال می‌شود وقتی قلم پایه نصب شده باشد اما کاراکترهای مورد نیاز را نداشته باشد.

**آیا جایگزینی بر اسلایدهای اصلی، طرح‌بندی‌ها، یادداشت‌ها و نظرات اعمال می‌شود؟**

بله. جایگزینی بر تمام اشیای ارائه که از قلم اصلی استفاده می‌کنند تأثیر می‌گذارد، از جمله اسلایدهای اصلی و یادداشت‌ها؛ نظرات نیز بخشی از سند هستند و توسط موتور قلم در نظر گرفته می‌شوند.

**آیا قلم داخل اشیای OLE جاسازی‌شده (مثلاً Excel) تغییر خواهد کرد؟**

خیر. [محتوای OLE](/slides/fa/androidjava/manage-ole/) توسط برنامهٔ خودش کنترل می‌شود. جایگزینی در ارائه ساختار داخلی داده‌های OLE را بازنویسی نمی‌کند؛ ممکن است به‌عنوان تصویر یا محتوای قابل ویرایش خارجی نمایش داده شود.

**آیا می‌توانم قلم را فقط در بخشی از ارائه (بر اساس اسلایدها یا نواحی) جایگزین کنم؟**

جایگزینی هدفمند امکان‌پذیر است اگر قلم را در سطح اشیاء/بازه‌های مورد نیاز تغییر دهید، نه اینکه یک جایگزینی سراسری بر کل سند اعمال کنید. منطق کلی انتخاب قلم در زمان رندر همان‌طور باقی می‌ماند.

**چگونه می‌توانم پیشاپیش بفهمم که ارائه از چه قلم‌هایی استفاده می‌کند؟**

از [مدیر قلم](https://reference.aspose.com/slides/fa/androidjava/com.aspose.slides/fontsmanager/) ارائه استفاده کنید: این ابزار فهرستی از [خانواده‌های در حال استفاده](https://reference.aspose.com/slides/fa/androidjava/com.aspose.slides/fontsmanager/#getFonts--) و اطلاعاتی درباره [جایگزینی‌ها/قلم‌های «نامشخص»](https://reference.aspose.com/slides/fa/androidjava/com.aspose.slides/fontsmanager/#getSubstitutions--) فراهم می‌کند که به برنامه‌ریزی جایگزینی کمک می‌کند.

**آیا جایگزینی قلم هنگام تبدیل به PDF/تصاویر کار می‌کند؟**

بله. هنگام خروجی‌گیری، Aspose.Slides همان [دنبالهٔ انتخاب/جایگزینی قلم](/slides/fa/androidjava/font-selection-sequence/) را اعمال می‌کند، بنابراین جایگزینی انجام‌شده قبلاً در زمان تبدیل رعایت می‌شود.

**آیا باید قلم هدف را در سیستم نصب کنم، یا می‌توانم پوشهٔ قلم‌ها را ضمیمه کنم؟**

نیازی به نصب نیست: کتابخانه امکان [بارگذاری قلم‌های خارجی](/slides/fa/androidjava/custom-font/) را از پوشه‌های کاربر برای استفاده در طول [رندر و خروجی‌گیری](/slides/fa/androidjava/convert-powerpoint/) فراهم می‌کند.

**آیا جایگزینی مشکل «توفو» (مربعات) به‌جای کاراکترها را رفع می‌کند؟**

فقط در صورتی که قلم هدف واقعاً گلیف‌های مورد نیاز را داشته باشد. در غیر این صورت، [پیکربندی fallback](/slides/fa/androidjava/fallback-font/) را برای پوشش کاراکترهای گمشده انجام دهید.
