## **مقدمه**

Aspose.Slides ابزارهای جامعی برای تنظیم اندازه اسلاید و نسبت تصویر در ارائه‌های PowerPoint فراهم می‌کند که برای چاپ و نمایش روی صفحه اهمیت دارند.

اندازه‌ها و نسبت‌های رایج اسلاید:

- **استاندارد (نسبت تصویر ۴:۳)**: ایده‌آل برای صفحه‌نمایش‌ها و دستگاه‌های قدیمی.
- **پهن‌صفحه (نسبت تصویر ۱۶:۹)**: برای پروژکتورها و نمایشگرهای مدرن توصیه می‌شود.

برای حفظ ثبات در طول ارائه خود، اطمینان حاصل کنید که یک اندازه اسلاید و نسبت تصویر برای تمام اسلایدها اعمال می‌شود. برای نتایج بهینه، ابعاد اسلاید خود را در ابتدای فرآیند ایجاد ارائه تنظیم کنید تا از بروز مشکلات جلوگیری شود.

{{% alert color="primary" %}} 
به‌طور پیش‌فرض، ارائه‌های ایجاد شده با Aspose.Slides از نسبت تصویر استاندارد ۴:۳ استفاده می‌کنند.
{{% /alert %}}

## **تغییر اندازه اسلید در ارائه‌ها**

این کد نمونه نشان می‌دهد چگونه اندازه اسلاید را در یک ارائه با استفاده از Aspose.Slides در جاوا تغییر دهید:

```java
Presentation pres = new Presentation("pres-4x3-aspect-ratio.pptx");
try {
    pres.getSlideSize().setSize(SlideSizeType.OnScreen16x9, SlideSizeScaleType.DoNotScale);
    pres.save("pres-4x3-aspect-ratio.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```

## **مشخص کردن اندازه سفارشی اسلاید در ارائه‌ها**

اگر اندازه‌های رایج اسلاید (۴:۳ و ۱۶:۹) برای کار شما مناسب نیستند، می‌توانید از یک اندازه اسلاید خاص یا منحصر به فرد استفاده کنید. برای مثال، اگر برنامه‌ریزی کنید اسلایدهای تمام‌اندازه از ارائه خود را بر روی قالب صفحه سفارشی چاپ کنید یا قصد داشته باشید ارائه‌تان را بر روی انواع خاصی از صفحه‌نمایش‌ها نمایش دهید، استفاده از تنظیم اندازه سفارشی برای ارائه‌تان می‌تواند مفید باشد.

این کد نمونه نشان می‌دهد چگونه از Aspose.Slides برای Android از طریق جاوا برای تعیین اندازه سفارشی اسلاید در یک ارائه استفاده کنید:

```java
Presentation pres = new Presentation("pres.pptx");
try {
    pres.getSlideSize().setSize(780, 540, SlideSizeScaleType.DoNotScale); // اندازه کاغذ A4
    pres.save("pres-a4-slide-size.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```

## **مدیریت محتوای اسلاید پس از تغییر اندازه**

پس از تغییر اندازه اسلاید برای یک ارائه، محتویات اسلایدها (مانند تصاویر یا اشیا) ممکن است دچار اعوجاج شوند. به‌طور پیش‌فرض، اشیا به‌طور خودکار برای متناسب شدن با اندازه جدید اسلاید تغییر اندازه می‌یابند. با این حال، هنگام تغییر اندازه اسلاید ارائه، می‌توانید تنظیمی را تعیین کنید که نحوه‌ٔ برخورد Aspose.Slides با محتویات اسلایدها را مشخص می‌کند.

بسته به هدف یا نیازی که دارید، می‌توانید از هر یک از این تنظیمات استفاده کنید:

- `DoNotScale`

  اگر نمی‌خواهید اشیا روی اسلایدها تغییر اندازه یابند، از این تنظیم استفاده کنید.

- `EnsureFit`

  اگر می‌خواهید به اندازه اسلاید کوچکتر مقیاس‌بندی کنید و نیاز دارید Aspose.Slides اشیا را کوچک‌تر کند تا همه آن‌ها در اسلایدها جا بگیرند (به این ترتیب از از دست رفتن محتوا جلوگیری می‌کنید)، از این تنظیم استفاده کنید.

- `Maximize`

  اگر می‌خواهید به اندازه اسلاید بزرگتر مقیاس‌بندی کنید و نیاز دارید Aspose.Slides اشیا را بزرگ‌تر کند تا نسبت به اندازه جدید اسلاید متناسب شوند، از این تنظیم استفاده کنید.

این کد نمونه نشان می‌دهد چگونه از تنظیم `Maximize` هنگام تغییر اندازه اسلاید یک ارائه استفاده کنید:

```java
Presentation pres = new Presentation("pres.pptx");
try {
    pres.getSlideSize().setSize(SlideSizeType.Ledger, SlideSizeScaleType.Maximize);
} finally {
    if (pres != null) pres.dispose();
}
```

## **سوالات متداول**

**آیا می‌توانم اندازه سفارشی اسلاید را با واحدهایی غیر از اینچ (مثلاً پوینت یا میلی‌متر) تنظیم کنم؟**

بله. Aspose.Slides به‌صورت داخلی از پوینت استفاده می‌کند، به‌طوری که ۱ پوینت برابر ۱/۷۲ اینچ است. می‌توانید هر واحدی (مانند میلی‌متر یا سانتی‌متر) را به پوینت تبدیل کرده و مقادیر تبدیل شده را برای تعیین عرض و ارتفاع اسلاید به‌کار ببرید.

**آیا اندازه سفارشی بسیار بزرگ اسلاید بر عملکرد و مصرف حافظه هنگام رندر تاثیر می‌گذارد؟**

بله. ابعاد بزرگ‌تر اسلاید (به پوینت) همراه با مقیاس رندر بالاتر منجر به افزایش مصرف حافظه و زمان پردازش طولانی‌تر می‌شود. سعی کنید اندازه اسلایدی عملی انتخاب کنید و مقیاس رندر را فقط در صورت نیاز برای دستیابی به کیفیت خروجی مطلوب تنظیم کنید.

**آیا می‌توانم یک اندازه اسلاید غیر استاندارد تعریف کنم و سپس اسلایدهای ارائه‌های با اندازه‌های متفاوت را ادغام کنم؟**

شما نمی‌توانید [ادغام ارائه‌ها](/slides/fa/androidjava/merge-presentation/) را انجام دهید در حالی که اندازه اسلایدها متفاوت است — ابتدا یک ارائه را به اندازه دیگری تغییر دهید. هنگام تغییر اندازه اسلاید، می‌توانید نحوه‌ٔ پردازش محتویات موجود را با گزینهٔ [SlideSizeScaleType](https://reference.aspose.com/slides/fa/androidjava/com.aspose.slides/slidesizescaletype/) انتخاب کنید. پس از هماهنگ‌سازی اندازه‌ها، می‌توانید اسلایدها را با حفظ قالب‌بندی ادغام کنید.

**آیا می‌توانم تصویر بندانگشتی برای اشکال منفرد یا نواحی خاص یک اسلاید ایجاد کنم و آیا این تصاویر به اندازه جدید اسلاید احترام می‌گذارند؟**

بله. Aspose.Slides می‌تواند تصویر بندانگشتی برای [کل اسلایدها](https://reference.aspose.com/slides/fa/androidjava/com.aspose.slides/slide/#getImage-com.aspose.slides.IRenderingOptions-float-float-) و همچنین برای [اشکال انتخابی](https://reference.aspose.com/slides/fa/androidjava/com.aspose.slides/shape/#getImage-int-float-float-) تولید کند. تصاویر خروجی اندازه و نسبت تصویر فعلی اسلاید را منعکس می‌کنند و چارچوب و هندسهٔ سازگاری را تضمین می‌کنند.
