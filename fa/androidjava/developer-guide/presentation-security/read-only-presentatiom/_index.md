## **معرفی**

در PowerPoint 2019، مایکروسافت تنظیم **Always Open Read-Only** را به عنوان یکی از گزینه‌هایی که کاربران می‌توانند برای محافظت از ارائه‌های خود استفاده کنند، معرفی کرد. ممکن است بخواهید از این تنظیم فقط‑خواندن برای محافظت از یک ارائه استفاده کنید وقتی که

- می‌خواهید از ویرایش‌های تصادفی جلوگیری کنید و محتوای ارائه خود را ایمن نگه دارید.  
- می‌خواهید به افراد اعلام کنید که ارائه‌ای که ارائه داده‌اید نسخه نهایی است.  

پس از انتخاب گزینه **Always Open Read-Only** برای یک ارائه، وقتی کاربران آن را باز می‌کنند، توصیه **Read-Only** را می‌بینند و ممکن است پیامی به شکل زیر مشاهده کنند: *برای جلوگیری از تغییرات تصادفی، نویسنده این فایل را برای باز شدن به صورت فقط‑خواندن تنظیم کرده است.*

توصیه Read-Only یک مانع ساده اما مؤثر است که با مجبور کردن کاربران به انجام یک کار برای حذف آن، از ویرایش جلوگیری می‌کند. اگر نمی‌خواهید کاربران تغییراتی در ارائه ایجاد کنند و می‌خواهید این را به شیوه‌ای مودبانه به آن‌ها اطلاع دهید، توصیه Read-Only می‌تواند گزینه مناسبی برای شما باشد.

> اگر یک ارائه با محافظت **Read-Only** در برنامه‌ی قدیمی‌تری از Microsoft PowerPoint باز شود — که از این عملکرد جدید پشتیبانی نمی‌کند — توصیه **Read-Only** نادیده گرفته می‌شود (ارائه به‌صورت معمولی باز می‌شود).

## **اعمال حالت فقط‑خواندن**

Aspose.Slides برای Android از طریق Java به شما امکان می‌دهد یک ارائه را به حالت **Read-Only** تنظیم کنید، به این معنی که کاربران (پس از باز کردن ارائه) توصیه **Read-Only** را می‌بینند. این کد نمونه نشان می‌دهد چگونه می‌توانید یک ارائه را به حالت **Read-Only** در Java با استفاده از Aspose.Slides تنظیم کنید:

```java
Presentation pres = new Presentation();
try {
    pres.getProtectionManager().setReadOnlyRecommended(true);
    pres.save("ReadOnlyPresentation.pptx", SaveFormat.Pptx);
} finally {
    if (pres != null) pres.dispose();
}
```

{{% alert color="primary" %}} 

**نکته**: توصیه **Read-Only** صرفاً برای دلسرد کردن ویرایش یا جلوگیری از تغییرات تصادفی کاربران در یک ارائه PowerPoint است. اگر شخصی که می‌داند چه کاری انجام می‌دهد تصمیم بگیرد ارائه شما را ویرایش کند، به راحتی می‌تواند تنظیم فقط‑خواندن را حذف کند. اگر به‌صورت جدی نیاز به جلوگیری از ویرایش غیرمجاز دارید، بهتر است از [محافظت‌های سخت‌تری که شامل رمزگذاری و گذرواژه می‌شوند](https://docs.aspose.com/slides/fa/androidjava/password-protected-presentation/) استفاده کنید.

{{% /alert %}} 

## **سوالات متداول**

**«توصیه فقط‑خواندن» با محافظت کامل با گذرواژه چه تفاوتی دارد؟**

«توصیه فقط‑خواندن» تنها یک پیشنهاد برای باز کردن فایل در حالت فقط‑خواندن نمایش می‌دهد و به راحتی می‌توان آن را دور زد. [محافظت با گذرواژه](/slides/fa/androidjava/password-protected-presentation/) واقعاً باز کردن یا ویرایش را محدود می‌کند و زمانی مناسب است که به کنترل‌های امنیتی واقعی نیاز دارید.

**آیا می‌توان «توصیه فقط‑خواندن» را با واترمارک ترکیب کرد تا از ویرایش بیشتر جلوگیری شود؟**

بله. این توصیه می‌تواند همراه با [واترمارک‌ها](/slides/fa/androidjava/watermark/) به‌عنوان یک مانع بصری استفاده شود؛ آن‌ها مکانیزم‌های جداگانه‌ای هستند و به‌خوبی با هم کار می‌کنند.

**آیا یک ماکرو یا ابزار خارجی همچنان می‌تواند فایل را وقتی توصیه فعال است تغییر دهد؟**

بله. این توصیه تغییرات برنامه‌ای را مسدود نمی‌کند. برای جلوگیری از ویرایش‌های خودکار، از [گذرواژه‌ها و رمزگذاری](/slides/fa/androidjava/password-protected-presentation/) استفاده کنید.

**«توصیه فقط‑خواندن» چگونه با روش‌های «isEncrypted» و «isWriteProtected» مرتبط است؟**

آن‌ها سیگنال‌های متفاوتی هستند. «توصیه فقط‑خواندن» یک اعلان نرم و اختیاری است؛ [isWriteProtected](https://reference.aspose.com/slides/fa/androidjava/com.aspose.slides/protectionmanager/#isWriteProtected--) و [isEncrypted](https://reference.aspose.com/slides/fa/androidjava/com.aspose.slides/protectionmanager/#isEncrypted--) محدودیت‌های واقعی نوشتن یا خواندن را که به گذرواژه یا رمزگذاری وابسته‌اند، نشان می‌دهند.
