## **بررسی کلی**

Aspose.Slides به شما امکان می‌دهد قلم‌های پیش‌فرضی که هنگام رندر ارائه استفاده می‌شوند را مشخص کنید. این ویژگی هنگام تولید تصویر بندانگشتی اسلاید یا خروجی گرفتن ارائه به فرمت‌هایی مانند PDF و XPS مفید است. قلم‌های پیش‌فرض از طریق `LoadOptions` قبل از بارگذاری ارائه پیکربندی می‌شوند.

ویژگی `DefaultRegularFont` قلم پیش‌فرض برای متن عادی را تعریف می‌کند، در حالی که `DefaultAsianFont` قلم پیش‌فرض برای متن‌های آسیایی را تعیین می‌سازد. پس از تنظیم این گزینه‌ها، می‌توانید ارائه را بارگذاری و با استفاده از قلم‌های مشخص شده رندر کنید.

## **استفاده از قلم‌های پیش‌فرض برای رندر یک ارائه**
Aspose.Slides به شما اجازه می‌دهد قلم پیش‌فرض برای رندر ارائه به PDF، XPS یا تصویر بندانگشتی را تنظیم کنید. این مقاله نشان می‌دهد چگونه `DefaultRegularFont` و `DefaultAsianFont` را به عنوان قلم‌های پیش‌فرض تعریف کنید. لطفاً مراحل زیر را برای بارگذاری قلم‌ها از پوشه‌های خارجی با استفاده از API Aspose.Slides for .NET دنبال کنید:

1. یک نمونه از `LoadOptions` ایجاد کنید.
1. `DefaultRegularFont` را به قلم دلخواه خود تنظیم کنید. در مثال زیر من از Wingdings استفاده کرده‌ام.
1. `DefaultAsianFont` را به قلم دلخواه خود تنظیم کنید. من در نمونه زیر از Wingdings استفاده کرده‌ام.
1. ارائه را با استفاده از `Presentation` و تنظیم گزینه‌های بارگذاری لود کنید.
1. حالا تصویر بندانگشتی اسلاید، PDF و XPS را تولید کنید تا نتایج را بررسی کنید.

پیاده‌سازی موارد فوق در زیر آورده شده است.

```c#
// استفاده از گزینه‌های بارگذاری برای تعیین قلم‌های پیش‌فرض منظم و آسیایی
LoadOptions loadOptions = new LoadOptions(LoadFormat.Auto);
loadOptions.DefaultRegularFont = "Wingdings";
loadOptions.DefaultAsianFont = "Wingdings";

using (Presentation pptx = new Presentation("DefaultFonts.pptx", loadOptions))
{
    using (IImage image = pptx.Slides[0].GetImage(1, 1))
    {
        image.Save("DefaultFonts_out.png", ImageFormat.Png);
    }

    pptx.Save("DefaultFonts_out.pdf", SaveFormat.Pdf);
    pptx.Save("DefaultFonts_out.xps", SaveFormat.Xps);
}
```

## **سوالات متداول**

**دقیقاً `DefaultRegularFont` و `DefaultAsianFont` بر چه چیزی تأثیر می‌گذارند—فقط خروجی یا همچنین تصویرهای بندانگشتی، PDF، XPS، HTML و SVG؟**

آنها در مسیر رندر برای تمام خروجی‌های پشتیبانی‌شده شرکت می‌کنند. این شامل تصویرهای بندانگشتی اسلاید، [PDF](/slides/fa/net/convert-powerpoint-to-pdf/)، [XPS](/slides/fa/net/convert-powerpoint-to-xps/)، [تصاویر رستر](/slides/fa/net/convert-powerpoint-to-png/)، [HTML](/slides/fa/net/convert-powerpoint-to-html/)، و [SVG](/slides/fa/net/render-a-slide-as-an-svg-image/) هستند، زیرا Aspose.Slides از همان منطق چیدمان و حل گلیف در این هدف‌ها استفاده می‌کند.

**آیا قلم‌های پیش‌فرض زمانی اعمال می‌شوند که فقط یک PPTX خوانده و ذخیره شود بدون هیچ رندری؟**

خیر. قلم‌های پیش‌فرض زمانی مهم می‌شوند که متن باید اندازه‌گیری و رسم شود. یک باز‑ذخیره سادهٔ ارائه ساختار قلم‌های ذخیره‌شده یا فایل را تغییر نمی‌دهد. قلم‌های پیش‌فرض در عملیات‌هایی که متن را رندر یا باز فرمت می‌کنند به کار می‌روند.

**اگر پوشه‌های قلم خودم را اضافه کنم یا قلم‌ها را از حافظه تأمین کنم، آیا در انتخاب قلم‌های پیش‌فرض در نظر گرفته می‌شوند؟**

بله. [منابع قلم سفارشی](/slides/fa/net/custom-font/) فهرست خانواده‌ها و گلیف‌های موجود را گسترش می‌دهند. قلم‌های پیش‌فرض و هر [قواعد بازگشتی](/slides/fa/net/fallback-font/) ابتدا در برابر این منابع حل می‌شوند و پوشش بهتری در سرورها و کانتینرها فراهم می‌آورند.

**آیا قلم‌های پیش‌فرض بر معیارهای متنی (کرنینگ، پیشرفت‌ها) و در نتیجه شکستن خطوط و پیچاندن متن تأثیر می‌گذارند؟**

بله. تغییر قلم معیارهای گلیف را تغییر می‌دهد و می‌تواند شکستن خطوط، پیچاندن و صفحه‌بندی را در حین رندر تحت تأثیر قرار دهد. برای پایداری چیدمان، [قلم‌های جاسازی‌شده](/slides/fa/net/embedded-font/) را بکار ببرید یا خانواده‌های پیش‌فرض و بازگشتی متریکاً سازگار را انتخاب کنید.

**آیا تنظیم قلم‌های پیش‌فرض وقتی تمام قلم‌های استفاده‌شده در ارائه جاسازی شده‌اند، لازمی است؟**

اغلب لازم نیست، زیرا [قلم‌های جاسازی‌شده](/slides/fa/net/embedded-font/) از ظاهر یکسان اطمینان می‌دهند. قلم‌های پیش‌فرض هنوز به عنوان یک شبکه ایمنی برای کاراکترهایی که توسط زیرمجموعهٔ جاسازی‌شده پوشش داده نمی‌شوند یا وقتی فایلی ترکیبی از متن‌های جاسازی‌شده و غیرجاسازی‌شده دارد، مفید هستند.
