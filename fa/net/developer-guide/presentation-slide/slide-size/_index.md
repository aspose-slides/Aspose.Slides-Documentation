## **مقدمه**

Aspose.Slides for .NET ابزارهای کاملی برای تنظیم اندازه اسلاید و نسبت تصویر در ارائه‌های PowerPoint فراهم می‌کند که برای چاپ و نمایش روی صفحه نمایش اهمیت دارند.

اندازه‌ها و نسبت‌های رایج اسلاید:

- **استاندارد (نسبت تصویر 4:3)**: برای نمایشگرها و دستگاه‌های قدیمی مناسب است.
- **پهن‌صفحه (نسبت تصویر 16:9)**: برای پروژکتورها و نمایشگرهای مدرن توصیه می‌شود.

اطمینان حاصل کنید که در تمام ارائه‌تان اندازه اسلاید و نسبت تصویر ثابت باشد؛ یک اندازه اسلاید و نسبت تصویر برای همه اسلایدها اعمال می‌شود. برای نتایج بهینه، ابعاد اسلاید را در ابتدای فرآیند ایجاد ارائه تنظیم کنید تا از بروز مشکلات جلوگیری شود.

{{% alert color="primary" %}} 
به طور پیش‌فرض، ارائه‌های ایجاد شده با Aspose.Slides از نسبت تصویر استاندارد 4:3 استفاده می‌کنند.
{{% /alert %}}

## **چگونه اندازه اسلاید را در یک ارائه تغییر دهیم**

این مثال نشان می‌دهد چگونه با استفاده از Aspose.Slides در C# اندازه اسلاید یک ارائه را تغییر دهیم:

```csharp
using (Presentation pres = new Presentation("presentation-4x3.pptx"))
{
    pres.SlideSize.SetSize(SlideSizeType.OnScreen16x9, SlideSizeScaleType.DoNotScale);
    pres.Save("presentation-16x9.pptx", SaveFormat.Pptx);
}
```

## **مشخص کردن اندازه‌های سفارشی اسلاید**

سفارشی‌سازی اندازه اسلاید متناسب با نیازهای خاص شما، مانند قالب‌بندی‌های ویژه کاغذ یا مشخصات صفحه نمایش، می‌تواند مفید باشد. در ادامه نحوه تنظیم اندازه سفارشی اسلاید با Aspose.Slides for .NET آورده شده است:

```csharp
using (Presentation pres = new Presentation("presentation.pptx"))
{
    pres.SlideSize.SetSize(780, 540, SlideSizeScaleType.DoNotScale); // اندازه کاغذ A4
    pres.Save("presentation-a4.pptx", SaveFormat.Pptx);
}
```

## **مدیریت محتوای اسلاید پس از تغییر اندازه**

پس از تغییر اندازه، ممکن است محتوای اسلاید خراب شود. می‌توانید نحوه مدیریت این تغییر اندازه توسط Aspose.Slides را کنترل کنید:

- **`DoNotScale`**: اشیا را با اندازه اصلی خود نگه می‌دارد تا مقیاس‌گذاری نشوند.
- **`EnsureFit`**: اشیا را برای مطابقت با اسلایدهای کوچکتر مقیاس می‌کند و از از دست رفتن محتوا جلوگیری می‌کند.
- **`Maximize`**: اشیا را برای اسلایدهای بزرگتر بزرگ می‌کند تا سازگاری ظاهری حفظ شود.

مثال استفاده از تنظیم `Maximize` برای تنظیم اندازه اسلاید:

```csharp
using (Presentation pres = new Presentation("presentation.pptx"))
{
   pres.SlideSize.SetSize(SlideSizeType.Ledger, SlideSizeScaleType.Maximize);
}
```

## **سؤالات متداول**

**آیا می‌توانم اندازه اسلاید سفارشی را با واحدهای دیگری غیر از اینچ (مانند پوینت یا میلی‌متر) تنظیم کنم؟**

بله. Aspose.Slides به‌صورت داخلی از پوینت استفاده می‌کند، که 1 پوینت برابر 1/72 اینچ است. می‌توانید هر واحدی (مانند میلی‌متر یا سانتی‌متر) را به پوینت تبدیل کنید و از مقادیر تبدیل‌شده برای تعریف عرض و ارتفاع اسلاید استفاده کنید.

**آیا اندازه سفارشی بسیار بزرگ اسلاید بر عملکرد و مصرف حافظه در زمان رندرینگ تاثیر می‌گذارد؟**

بله. ابعاد بزرگ‌تر اسلاید (به پوینت) همراه با مقیاس رندرینگ بالاتر باعث مصرف بیشتر حافظه و زمان پردازش طولانی‌تر می‌شود. سعی کنید اندازه اسلایدی عملی انتخاب کنید و مقیاس رندرینگ را فقط در صورت نیاز برای دستیابی به کیفیت خروجی مطلوب تنظیم کنید.

**آیا می‌توانم یک اندازه اسلاید غیر استاندارد تعریف کنم و سپس اسلایدها را از ارائه‌های با اندازه‌های مختلف ادغام کنم؟**

نمی‌توانید[merge presentations](/slides/fa/net/merge-presentation/) را زمانی که اندازه اسلایدها متفاوت است انجام دهید — ابتدا یک ارائه را برای تطابق با دیگری تغییر اندازه دهید. هنگام تغییر اندازه اسلاید می‌توانید نحوه تنظیم محتوا را از طریق گزینه[SlideSizeScaleType](https://reference.aspose.com/slides/fa/net/aspose.slides/slidesizescaletype/) انتخاب کنید. پس از هم‌ترازی اندازه‌ها، می‌توانید اسلایدها را ادغام کنید و قالب‌بندی را حفظ نمایید.

**آیا می‌توانم تصاویر بندانگشتی برای شکل‌های فردی یا مناطق خاصی از اسلاید تولید کنم و آیا این تصاویر به اندازه جدید اسلاید احترام می‌گذارند؟**

بله. Aspose.Slides می‌تواند تصویر بندانگشتی برای[entire slides](https://reference.aspose.com/slides/fa/net/aspose.slides/slide/getimage/) و همچنین برای[selected shapes](https://reference.aspose.com/slides/fa/net/aspose.slides/shape/getimage/) تولید کند. تصاویر حاصل اندازه و نسبت تصویر فعلی اسلاید را منعکس می‌کنند و چارچوب و هندسهٔ ثابت را تضمین می‌نمایند.
