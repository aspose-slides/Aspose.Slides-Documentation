## **بررسی کلی**

این مقاله توضیح می‌دهد که چه زمانی توسعه‌دهندگان ممکن است برای کار با اسناد ارائه، Open XML SDK یا Aspose.Slides را انتخاب کنند. Open XML SDK را به‌عنوان کتابخانه‌ای برای دستکاری بسته‌های OOXML و عناصر XML زیرساختی آن‌ها توصیف می‌کند، در حالی که Aspose.Slides به‌عنوان کتابخانه‌ای برای پردازش ارائه با مدل شیئ‌مستند سطح بالا و پشتیبانی از بسیاری از وظایف مرتبط با PowerPoint معرفی می‌شود.

مقاله هر دو گزینه را از نظر فرمت‌های پشتیبانی‌شده، مدل برنامه‌نویسی، قابلیت‌های رندرینگ و چاپ، پشتیبانی از پلتفرم‌ها و موارد استفاده رایج مقایسه می‌کند. همچنین روشن می‌سازد که Open XML SDK ممکن است برای عملیات پایه‌ای PPTX یا دسترسی مستقیم به عناصر OOXML مناسب باشد، در حالی که Aspose.Slides برای وظایف پیچیده‌تر مانند کار با فرمت‌های متعدد PowerPoint، کپی یا کلون کردن اشکال، جایگزینی متن، اعمال انیمیشن‌ها و تبدیل ارائه‌ها به PDF، TIFF یا XPS مناسب‌تر است.

## **Open XML SDK چیست؟**
گاهی این سؤال پیش می‌آید: *چرا باید به‌جای Open XML SDK رایگان، از محصولات Aspose استفاده کنیم؟* 

پاسخ به این سؤال به‌راحتی می‌تواند بر پایه ویژگی‌ها و قابلیت‌ها ارائه شود. 

بر اساس [کتابخانه MSDN](https://docs.microsoft.com/en-us/office/open-xml/open-xml-sdk) Open XML SDK به این صورت تعریف می‌شود: 

> "Open XML SDK 2.0 فرآیند دستکاری بسته‌های Open XML و عناصر طرح‌واره زیرین آن‌ها را ساده می‌کند. Open XML SDK 2.0 بسیاری از وظایف رایجی که توسعه‌دهندگان بر روی بسته‌های Open XML انجام می‌دهند را در بر می‌گیرد، به طوری که بتوانید عملیات پیچیده را با تنها چند خط کد انجام دهید. اسناد OOXML در واقع فایل‌های فشرده XML هستند و Open XML SDK مجموعه‌ای از کلاس‌هاست که به شما اجازه می‌دهد محتویات اسناد OOXML را به صورت strongly‑typed دستکاری کنید. به این معنی که به‌جای استخراج XML از فایل فشرده، بارگذاری آن در درخت DOM و کار مستقیم با عناصر و ویژگی‌های XML، Open XML SDK کلاس‌هایی فراهم می‌کند که این کار را برای شما انجام می‌دهند."

## **Aspose.Slides چیست؟**
Aspose.Slides کتابخانه‌ای کلاس‑محور است که به برنامه‌ها امکان انجام این وظایف پردازش ارائه را می‌دهد: 

- برنامه‌نویسی با مدل شی‌ء‑مستند ارائه.  
- تبدیل با کیفیت بالا شامل تمام فرمت‌های محبوب پشتیبانی‌شده PowerPoint، شامل تبدیل به PDF، XPS، TIFF و چاپ.  
- تولید تصویر کوچک اسلایدها در قالب‌های شناخته‌شده‌ای مانند PNG، JPEG و BMP به‌همراه استخراج اسلاید به SVG.  
- ساخت ارائه‌ها از ابتدا یا ترکیب عناصر از یک یا چند سند.  
- افزودن انیمیشن‌ها، فریم‌های OLE، جدول‌ها، ایجاد و مدیریت نمودارها.  
- کنترل (کنترل گسترده) و مدیریت قالب‌بندی متن در سطوح TextFrames، Paragraphs و Portions.  

برای جزئیات بیشتر درباره ویژگی‌های موجود، لطفاً صفحه [ویژگی‌های Aspose.Slides](/slides/fa/net/product-overview/) را مشاهده کنید.

| :- | :- | :- |
|فرمت‌های ارائه پشتیبانی‌شده|PPTX|PPT, POT, PPS, PPTX, POTX, PPSX, ODP|
|تبدیل از PPT به PPTX|No|Yes|
|<p>برنامه‌نویسی سطح بالا با مدل شیئ‌مستند Presentation Document Object Model (DOM):</p><p>- پیدا کردن و جایگزینی متن.</p><p>- ترکیب اسلایدها در ارائه‌ها.</p>|No|Yes|
|برنامه‌نویسی دقیق با مدل شیئ‌مستند؛ دسترسی به عناصر و قالب‌بندی‌های فردی مانند TextHolders، TextFrames، Paragraphs و Portions.|Yes|Yes|
|دسترسی مستقیم و کامل سطح پایین به عناصر و ویژگی‌های XML زیرساختی مانند شناسه‌های رابطه، شناسه‌های فهرست در یک سند OOXML.|Yes|No|
|<p>رندرینگ و چاپ:</p><p>- رندر کردن ارائه‌ها به PDF، PDF Notes، XPS، تصاویر TIFF.</p><p>- رندر تصویرهای کوچک اسلاید به PNG، JPEG، BMP، SVG و TIFF.</p><p>- تعیین وضوح تصویر، کیفیت، فشرده‌سازی و گزینه‌های دیگر.</p><p>- چاپ ارائه‌ها با استفاده از زیرساخت چاپ .NET. این مؤلفه دارای روش چاپ داخلی برای چاپ ارائه‌ها همان‌گونه که در پیش‌نمایش چاپ Microsoft PowerPoint نمایش داده می‌شود.</p>|No|Yes|
|پلتفرم‌های پشتیبانی‌شده|Windows, .NET|Windows, Linux, Java, .NET, Mono|

## **نتیجه‌گیری**
Open XML SDK و Aspose.Slides به‌طور مستقیم با یکدیگر رقابت نمی‌کنند زیرا نیازهای متفاوتی را برآورده می‌کنند و مخاطبان متفاوتی دارند. 

{{% alert color="primary" %}} 

Open XML SDK کتابخانه‌ای کلاس‑محور است که روشی strongly‑typed برای کار با اسناد OOXML فراهم می‌کند، در حالی که Aspose.Slides کتابخانه‌ای پردازش ارائه فوق‌العاده مفید است که پشتیبانی عالی از تقریباً تمام فرمت‌های فایل Microsoft PowerPoint را ارائه می‌دهد. 

{{% /alert %}} 

اگر جریان کاری شما عملی ساده برنامه‌نویسی بر روی یک سند PPTX باشد، Open XML SDK می‌تواند گزینه مناسبی باشد. با Open XML SDK باید بتوانید کارهای ساده‌ای مانند تولید یک سند PPTX ساده یا حذف نظرات، سرصفحه/پاورقی، استخراج تصاویر یا کارهای مشابه را به راحتی انجام دهید. برخی کارها می‌توانند با Open XML SDK انجام شوند اما با Aspose.Slides قابل انجام نیستند. برای مثال، اگر نیاز به دسترسی مستقیم به عناصر و ویژگی‌های XML یک سند OOXML داشته باشید، باید از Open XML SDK استفاده کنید. 

اگر نیاز به انجام کارهای پیچیده روی اسناد دارید — مانند موارد زیر — Aspose.Slides بهترین گزینه برای شماست. 

- عملیات مرتبط با فرمت‌های قدیمی PowerPoint (و PPTX نیز).  
- کپی یا کلون کردن اشکال در داخل اسلایدها به‑گونه‌ای که اشیاء، سبک‌ها و سایر عناصر قالب‌بندی را به‌طور مناسب ترکیب کند.  
- جایگزینی متن قالب‌بندی‌شده یا بدون قالب‌بندی.  
- اعمال انیمیشن‌ها و استفاده از کانکتورها با اشکال.  
- تبدیل یک سند به PDF، TIFF یا XPS به‌طوری که همانند تبدیل توسط Microsoft PowerPoint به‌نظر برسد.  
- توسعه برنامه‌های .NET یا Java در محیط‌های دسکتاپ و وب.
