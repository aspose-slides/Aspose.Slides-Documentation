## **نمای کلی**

Aspose.Slides می‌تواند در حالت ارزیابی یا با یک لایسنس معتبر استفاده شود. نسخه ارزیابی همان قابلیت‌های نسخه دارای لایسنس را فراهم می‌کند، اما هنگام باز یا ذخیره‌سازی ارائه‌ها یک واترمارک ارزیابی اضافه می‌کند و استخراج متن را به یک اسلاید محدود می‌نماید.

این مقاله توضیح می‌دهد که مجوزدهی در Aspose.Slides چگونه کار می‌کند و چگونه قبل از استفاده از کتابخانه یک لایسنس اعمال شود. یک لایسنس می‌تواند با استفاده از کلاس `License` از یک فایل، جریان یا منبع نهفته بارگذاری شود. مقاله همچنین نشان می‌دهد چگونه بررسی کنید که لایسنس به درستی اعمال شده است.

## **ارزیابی Aspose.Slides**
{{% alert color="primary" %}} 

می‌توانید نسخه ارزیابی **Aspose.Slides for NET** را از [صفحه دانلود NuGet آن](https://www.nuget.org/packages/Aspose.Slides.NET/) دریافت کنید. نسخه ارزیابی همان عملکردهای نسخه دارای لایسنس محصول را فراهم می‌کند. بسته ارزیابی همان بسته خریداری شده است. نسخه ارزیابی پس از افزودن چند خط کد (برای اعمال لایسنس) به‌سرعت تبدیل به نسخه دارای لایسنس می‌شود.

پس از رضایت از ارزیابی **Aspose.Slides** می‌توانید [یک لایسنس خریداری کنید](https://purchase.aspose.com/buy). توصیه می‌کنیم انواع مختلف اشتراک را بررسی کنید. اگر سؤال دارید، با تیم فروش Aspose تماس بگیرید.

هر لایسنس Aspose همراه با یک اشتراک یک‌ساله برای ارتقاء رایگان به نسخه‌های جدید یا رفع اشکالات منتشرشده در دوره اشتراک است. کاربران محصولات دارای لایسنس یا حتی نسخه‌های ارزیابی پشتیبانی فنی رایگان و نامحدود دریافت می‌کنند.

{{% /alert %}}

## **مجوزدهی در Aspose.Slides**
* یک نسخه ارزیابی پس از خرید لایسنس و افزودن چند خط کد (برای اعمال لایسنس) به‌سرعت تبدیل به نسخه دارای لایسنس می‌شود.
* لایسنس یک فایل XML متن ساده است که شامل جزئیاتی مانند نام محصول، تعداد توسعه‌دهندگان مجاز، تاریخ انقضای اشتراک و غیره می‌باشد. 
* فایل لایسنس به‌صورت دیجیتالی امضا شده است، بنابراین نباید آن را تغییر دهید. حتی افزودن ناخواسته یک خط جدید به محتوای فایل، اعتبار آن را از بین می‌برد.
* Aspose.Slides برای .NET معمولاً سعی می‌کند لایسنس را در این مکان‌ها پیدا کند:
  * مسیر صریح
  * پوشه‌ای که DLL مؤلفه در آن قرار دارد (شامل در Aspose.Slides)
  * پوشه‌ای که اسمبلی فراخوانی‌کننده DLL مؤلفه در آن قرار دارد (شامل در Aspose.Slides)
  * پوشه‌ای که اسمبلی ورودی (exe شما) در آن قرار دارد
  * منبع نهفته در اسمبلی فراخوانی‌کننده DLL مؤلفه (شامل در Aspose.Slides).
* برای جلوگیری از محدودیت‌های مربوط به نسخه ارزیابی، باید قبل از استفاده از Aspose.Slides یک لایسنس تنظیم کنید. تنها یک بار برای هر برنامه یا فرآیند نیاز به تنظیم لایسنس دارید.

{{% alert color="primary" %}} 

ممکن است بخواهید به [مجوزدهی بر پایه مترش](https://docs.aspose.com/slides/fa/net/metered-licensing/) نگاهی بیندازید.

{{% /alert %}} 


## **اعمال یک لایسنس**
یک لایسنس می‌تواند از **فایل**، **جریان** یا **منبع نهفته** بارگذاری شود. 

{{% alert color="primary" %}}

Aspose.Slides کلاس [License](https://reference.aspose.com/slides/fa/net/aspose.slides/license) را برای عملیات مجوزدهی فراهم می‌کند.

{{% /alert %}} 

{{% alert color="warning" %}} 

لایسنس‌های جدید می‌توانند Aspose.Slides را تنها از نسخه 21.4 به بعد فعال کنند. نسخه‌های قبلی از سیستم مجوزدهی متفاوتی استفاده می‌کنند و این لایسنس‌ها را شناسایی نمی‌کنند.

{{% /alert %}}

### **فایل**
ساده‌ترین روش تنظیم لایسنس این است که فایل لایسنس را در همان پوشه‌ای که DLL مؤلفه در آن قرار دارد (شامل در Aspose.Slides) قرار داده و فقط نام فایل بدون مسیر را مشخص کنید.

این کد C# نشان می‌دهد چگونه یک فایل لایسنس تنظیم شود:

``` csharp
// یک نمونه از کلاس License ایجاد می‌کند 
Aspose.Slides.License license = new Aspose.Slides.License();

// مسیر فایل لایسنس را تنظیم می‌کند
license.SetLicense("Aspose.Slides.lic");
```

{{% alert color="warning" %}} 

اگر فایل لایسنس را در پوشه‌ای متفاوت قرار دهید، هنگام فراخوانی متد [SetLicense](https://reference.aspose.com/slides/fa/net/aspose.slides/license/setlicense/#setlicense_1)، نام فایل لایسنس در انتهای مسیر صریح باید دقیقا همان نام فایل لایسنس شما باشد.

به‌عنوان مثال می‌توانید نام فایل لایسنس را به *Aspose.Slides.lic.xml* تغییر دهید. سپس در کد خود مسیر فایل (با انتهای *Aspose.Slides.lic.xml*) را به متد [SetLicense](https://reference.aspose.com/slides/fa/net/aspose.slides/license/setlicense/#setlicense_1) پاس دهید.

{{% /alert %}}

### **جریان**
می‌توانید لایسنس را از یک جریان بارگذاری کنید. این کد C# نشان می‌دهد چگونه یک لایسنس را از یک جریان اعمال کنید:

``` csharp
// یک نمونه از کلاس License ایجاد می‌کند 
Aspose.Slides.License license = new Aspose.Slides.License();

// مجوز را از طریق یک جریان تنظیم می‌کند
license.SetLicense(myStream);
```

### **منبع نهفته**
می‌توانید لایسنس را همراه با برنامه خود بسته‌بندی کنید (تا از گم شدن آن جلوگیری شود) با افزودن لایسنس به عنوان یک منبع نهفته در یکی از اسمبلی‌هایی که DLL مؤلفه را فرا می‌خوانند (شامل در Aspose.Slides). 

این روش افزودن فایل لایسنس به عنوان یک منبع نهفته است:

1. در Visual Studio، فایل لایسنس (.lic) را به پروژه اضافه کنید به این صورت: از **File** > **Add Existing Item** > **Add** عبور کنید. 
2. فایل را در **Solution Explorer** انتخاب کنید.
3. در پنجره **Properties**، **Build Action** را روی **Embedded Resource** تنظیم کنید.
4. برای دسترسی به لایسنس نهفته در اسمبلی، فایل لایسنس را به عنوان منبع نهفته به پروژه اضافه کنید و سپس نام فایل لایسنس را به متد `SetLicense` پاس دهید. 


کلاس `License` به‌صورت خودکار فایل لایسنس را در منابع نهفته پیدا می‌کند. نیازی به فراخوانی متدهای `GetExecutingAssembly` و `GetManifestResourceStream` از کلاس `System.Reflection.Assembly` در چارچوب Microsoft .NET ندارید.

این کد C# نشان می‌دهد چگونه یک لایسنس را به عنوان منبع نهفته تنظیم کنید:

``` csharp
// یک نمونه از کلاس License ایجاد می‌کند
Aspose.Slides.License license = new Aspose.Slides.License();

// نام فایل لایسنس موجود در اسمبلی را پاس می‌دهد
license.SetLicense("Aspose.Slides.lic");
```

## **اعتبارسنجی لایسنس**

برای بررسی این که آیا لایسنس به‌درستی تنظیم شده است یا نه، می‌توانید آن را اعتبارسنجی کنید. این کد C# نشان می‌دهد چگونه یک لایسنس را اعتبارسنجی کنید:

```c#
Aspose.Slides.License license = new Aspose.Slides.License();

license.SetLicense("Aspose.Slides.lic");

if (license.IsLicensed())
{
    Console.WriteLine("License is good!");
    Console.Read();
}
```

## **ایمنی چندنخی**

{{% alert title="Note" color="warning" %}} 

متد [license.SetLicense](https://reference.aspose.com/slides/fa/net/aspose.slides/license/setlicense/) ایمن برای چندنخی نیست. اگر این متد باید همزمان از چندین نخ فراخوانی شود، ممکن است بخواهید از primitives همگام‌سازی (مانند lock) برای جلوگیری از مشکلات استفاده کنید. 

{{% /alert %}}

## **سوالات متداول**

**آیا می‌توانم لایسنس را در محیط کاملاً آفلاین (بدون دسترسی به اینترنت) اعمال کنم؟**

بله. اعتبارسنجی لایسنس به‌صورت محلی با استفاده از فایل لایسنس انجام می‌شود؛ نیازی به اتصال به اینترنت نیست.

**بعد از انقضای یک‌سالهٔ اشتراک چه اتفاقی می‌فتد؟ آیا کتابخانه کار کردن متوقف می‌شود؟**

خیر. لایسنس دائمی است: می‌توانید از نسخه‌های منتشرشده پیش از تاریخ پایان اشتراک خود استفاده کنید؛ فقط بدون تمدید، حق استفاده از نسخه‌های جدیدتر را ندارید.
