## **معرفی**

مجوز مبتنی بر مصرف (Metered licensing) یک مکانیزم مجوزدهی است که می‌تواند همراه با روش‌های مجوزدهی موجود استفاده شود. اگر می‌خواهید بر اساس استفاده‌تان از ویژگی‌های API Aspose.Slides هزینه‌گیری شوید، مجوز مبتنی بر مصرف را انتخاب می‌کنید.

## **اعمال کلیدهای مصرفی**

وقتی یک مجوز مبتنی بر مصرف خریداری می‌کنید، کلیدها (و نه یک فایل مجوز) دریافت می‌کنید. این کلید مصرفی می‌تواند با استفاده از کلاس [Metered](https://reference.aspose.com/slides/fa/net/aspose.slides/metered/) که Aspose برای عملیات مصرفی فراهم کرده است، اعمال شود. برای جزئیات بیشتر، به [Metered Licensing FAQ](https://purchase.aspose.com/faqs/licensing/metered) مراجعه کنید.

1. یک نمونه از کلاس [Metered](https://reference.aspose.com/slides/fa/net/aspose.slides/metered/) ایجاد کنید.  
2. کلیدهای عمومی و خصوصی خود را به متد [SetMeteredKey](https://reference.aspose.com/slides/fa/net/aspose.slides/metered/setmeteredkey/) پاس بدهید.  
3. برخی پردازش‌ها را انجام دهید (کارهایی را اجرا کنید).  
4. متد [GetConsumptionQuantity](https://reference.aspose.com/slides/fa/net/aspose.slides/metered/getconsumptionquantity/) کلاس `Metered` را فراخوانی کنید.

در این مرحله باید مقدار/تعداد درخواست‌های API که تا کنون مصرف کرده‌اید را مشاهده کنید.

این قطعه کد نمونه نشان می‌دهد که چگونه از مجوز مبتنی بر مصرف استفاده کنید:

```cs
// یک نمونه از کلاس Metered ایجاد می‌کند
Aspose.Slides.Metered metered = new Aspose.Slides.Metered();

// کلیدهای عمومی و خصوصی را به شی Metered پاس می‌دهد
metered.SetMeteredKey("<valid public key>", "<valid private key>");

// مقدار داده‌های مصرفی را قبل از فراخوانی API دریافت می‌کند
decimal amountBefore = Aspose.Slides.Metered.GetConsumptionQuantity();
Console.WriteLine("Amount consumed before: " + amountBefore.ToString());

// در اینجا کاری با API Aspose.Slides انجام دهید
// ...
// مقدار داده‌های مصرفی را پس از فراخوانی API دریافت می‌کند
decimal amountAfter = Aspose.Slides.Metered.GetConsumptionQuantity();
Console.WriteLine("Amount consumed after: " + amountAfter.ToString());
```

{{% alert color="warning" title="NOTE" %}} 

برای استفاده از مجوز مبتنی بر مصرف، به یک اتصال اینترنتی پایدار نیاز دارید زیرا مکانیزم مجوزدهی برای تعامل مستمر با سرویس‌های ما و انجام محاسبات از اینترنت استفاده می‌کند.

{{% /alert %}} 

## **سوالات متداول**

**آیا می‌توانم یک مجوز مبتنی بر مصرف را همراه با یک مجوز معمولی (دائم یا موقت) در یک برنامه استفاده کنم؟**

بله. مجوز مبتنی بر مصرف یک مکانیزم اضافه‌ای است که می‌تواند همراه با [روش‌های مجوزدهی](/slides/fa/net/licensing/) موجود استفاده شود. شما می‌توانید هنگام شروع برنامه تصمیم بگیرید که کدام مکانیزم را اعمال کنید.

**دقیقاً چه چیزی تحت یک مجوز مبتنی بر مصرف به عنوان مصرف در نظر گرفته می‌شود: عملیات‌ها یا فایل‌ها؟**

مصرف به استفاده از API شمارش می‌شود، یعنی تعداد درخواست‌ها یا عملیات‌ها. می‌توانید مصرف فعلی را از طریق [روش‌های ردیابی مصرف](https://reference.aspose.com/slides/fa/net/aspose.slides/metered/) به دست آورید.

**آیا مجوز مبتنی بر مصرف برای میکروسرویس‌ها و محیط‌های بدون سرور که نمونه‌ها به‌طور مکرر Restart می‌شوند مناسب است؟**

بله. از آنجا که حساب‌رسی در سطح فراخوانی API انجام می‌شود، سناریوهای با راه‌اندازی سرد (cold start) مکرر سازگار هستند، به‌شرطی که دسترسی شبکه‌ای پایدار برای محاسبات مصرفی موجود باشد.

**آیا عملکرد کتابخانه هنگام استفاده از مجوز مبتنی بر مصرف نسبت به مجوز دائمی متفاوت است؟**

خیر. این صرفاً در مورد مکانیزم مجوزدهی و هزینه‌گیری است؛ قابلیت‌های محصول یکسان باقی می‌مانند.

**مجوز مبتنی بر مصرف چگونه با نسخه آزمایشی و مجوز موقت ارتباط دارد؟**

نسخه آزمایشی دارای محدودیت‌ها و واترمارک‌ها است، [مجوز موقت](https://purchase.aspose.com/temporary-license/) محدودیت‌ها را برای ۳۰ روز حذف می‌کند، و مجوز مبتنی بر مصرف نیز محدودیت‌ها را حذف کرده و بر اساس استفاده واقعی هزینه‌گیری می‌کند.

**آیا می‌توانم بودجه را با واکنش خودکار هنگام عبور از آستانه مصرف کنترل کنم؟**

بله. یک روش معمول این است که به‌صورت دوره‌ای مصرف فعلی را از طریق [روش‌های ردیابی](https://reference.aspose.com/slides/fa/net/aspose.slides/metered/) بخوانید و محدودیت‌ها یا هشدارهای خود را در سطح برنامه یا مانیتورینگ پیاده‌سازی کنید.
