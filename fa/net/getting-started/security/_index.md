## **امنیت در Aspose.Slides**

Aspose هنگام توسعه محصولات خود از بهترین شیوه‌ها پیروی می‌کند. 

* Aspose.Slides برای .NET برای دستکاری و تبدیل ارائه‌ها در فایل‌های با فرمت‌های دیگر استفاده می‌شود. این محصول اسکریپت‌ها را در ارائه‌ها اجرا نمی‌کند. Aspose.Slides صرفاً ساختار ارائه را تجزیه می‌کند و به کد کاربران نهایی امکان می‌دهد مدل شیء را به‑صورت راحتی دستکاری کنند. 
* Aspose.Slides به‌عنوان یک کتابخانه در زمینه تجزیه و تفسیر اسناد بدون اجرای کدهای دوردست عمل می‌کند. تمام محصولات Aspose روی دستگاه‌های شما اجرا می‌شوند. آن‌ها هیچ داده‌ای را به Aspose منتقل نمی‌کنند. تنها استثنا زمانی است که یک [metered license](https://purchase.aspose.com/faqs/licensing/metered) درگیر باشد، و حتی در آن صورت (اگر از لایسنس مترد استفاده کنید) فقط اطلاعات استفاده از API شما پردازش می‌شود.

* اجزای Aspose در همان زمینه کاربری همانند برنامه‌های معمولی اجرا می‌شوند. بنابراین، اجزای Aspose خطری برای منابع حیاتی سیستم ایجاد نمی‌کنند. علاوه بر این، زمانی که یک سند توسط یک مؤلفه Aspose باز می‌شود، ماکروها به‌صورت خودکار اجرا نمی‌شوند. 

* خطرات ذاتی یا مرتبط با بسته Microsoft Office بر اجزای Aspose اعمال نمی‌شود، بنابراین محصولات Aspose بسیار امن هستند.

## **سوالات متداول**

**کدام سیستم‌ها برای نظارت بر آسیب‌پذیری‌های کد Aspose استفاده می‌شوند؟**

ما برای هر نسخه از Aspose.Slides تجزیه‌وتحلیل ایستای کد را اجرا می‌کنیم. می‌توانیم گزارش‌های امنیتی ارائه دهیم که نشان می‌دهد کد Aspose.Slides استانداردهای OWASP Top 10 را پاس می‌کند. 

**کدام سیستم‌ها را برای نظارت بر آسیب‌پذیری‌های بسته‌های خارجی به کار می‌برید؟**

ما سیستمی برای بررسی آسیب‌پذیری‌های بسته‌های خارجی استفاده نمی‌کنیم زیرا Aspose.Slides هیچ بسته خارجی را شامل یا استفاده نمی‌کند. Aspose.Slides تنها بر خود چارچوب .NET وابسته است.
