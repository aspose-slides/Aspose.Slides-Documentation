## **معرفی**

Aspose.Slides for Node.js via Java نیازی به نصب هیچ محصول شخص ثالثی مانند Microsoft PowerPoint ندارد. Aspose.Slides خود یک موتور برای ایجاد، اصلاح، تبدیل و رندر اسناد در فرمت‌های مختلف است، از جمله فرمت‌های ارائه Microsoft PowerPoint.

## **سیستم‌عامل‌های پشتیبانی‌شده**

Aspose.Slides for Node.js via Java از هر سیستم‌عامل 32‑bit یا 64‑bit که محیط زمان اجراى Java را اجرا می‌کند، پشتیبانی می‌کند، از جمله ولی نه محدود به موارد زیر:

### **ویندوز**
- Microsoft Windows 2003 Server ( x64, x86)
- Microsoft Windows 2008 Server ( x64, x86)
- Microsoft Windows 2012 Server ( x64, x86)
- Microsoft Windows 2012 R2 Server ( x64, x86)
- Microsoft Windows 2016 Server ( x64, x86)
- Microsoft Windows 2019 Server ( x64, x86)
- Microsoft Windows Vista ( x64, x86)
- Microsoft Windows XP ( x64, x86)
- Microsoft Windows 7 ( x64, x86)
- Microsoft Windows 8, 8.1 ( x64, x86)
- Microsoft Windows 10 ( x64, x86)

### **لینوکس**
- Linux (Ubuntu, CentOS و دیگر توزیع‌ها)

### **مک**
- Mac OS X

## **سؤالات متداول**

**آیا برای تبدیل و رندر نیاز به نصب Microsoft PowerPoint دارم؟**

نه، PowerPoint ضروری نیست؛ Aspose.Slides یک موتور مستقل برای [ایجاد](/slides/fa/nodejs-java/create-presentation/)، اصلاح، [تبدیل](/slides/fa/nodejs-java/convert-presentation/)، و [رندر](/slides/fa/nodejs-java/convert-powerpoint-to-png/) ارائه‌ها است.

**کدام فونت‌ها برای رندر صحیح لازم هستند؟**

در عمل، فونت‌های استفاده‑شده در ارائه یا [جایگزین‌ها](/slides/fa/nodejs-java/font-substitution/) باید در دسترس باشند. برای اطمینان از رندر یکنواخت در Linux/macOS، توصیه می‌شود بسته‌های فونت رایج نصب شوند.

**چرا یک فونت سفارشی در لینوکس به عنوان متن جایگزین یا گمشده رندر می‌شود؟**

اگر فایل فونت دارای ورودی‌های نام‌جدول ناسازگار یا خراب باشد، پشته تطبیق فونت لینوکس (FreeType/fontconfig) ممکن است رکورد نامعتبری را انتخاب کند که منجر به عدم شناسایی فونت می‌شود. استفاده از نسخه‌ای از فونت با رکوردهای نام‌جدول اصلاح‌شده یا نصب یک جایگزین سازگار این مشکل را رفع می‌کند.
