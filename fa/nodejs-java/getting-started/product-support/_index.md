## **نمای کلی**

این مقاله توضیح می‌دهد چگونه می‌توانید پشتیبانی Aspose.Slides را دریافت کنید و نحوه گزارش مؤثر مشکلات را بیاموزید. این مقاله کانال اصلی پشتیبانی را توصیف می‌کند، توصیه‌هایی برای ارسال یک مشکل ارائه می‌دهد، و به منابع فنی مفیدی مانند صفحه محصول، وبلاگ، دانلودها، انجمن پشتیبانی، مستندات و مرجع API اشاره می‌کند. همچنین به سؤالات رایج در مورد پشتیبانی پولی، ارتقاء مشکل و پشتیبانی از نسخه‌های قدیمی کتابخانه پاسخ می‌دهد.

## **پشتیبانی فنی**
{{% alert color="primary" %}}

Aspose پشتیبانی فنی رایگان نامحدود برای همه محصولات خود فراهم می‌کند. این پشتیبانی برای همه کاربران (از جمله کاربران با بسته‌های ارزیابی) در دسترس است.

{{% /alert %}} 

زمانی که به کمک برای Aspose.Slides for Node.js via Java نیاز دارید، موارد زیر را در نظر بگیرید:

- مسیر اصلی پشتیبانی، [Aspose.Forums](https://forum.aspose.com/c/slides/fa/11) است. سؤال خود را در انجمن Aspose.Slides ارسال کنید—و در عرض چند ساعت پاسخ می‌گیرید. تیم پشتیبانی Aspose.Slides به‌طور مستقیم به سؤالات ارسال شده در انجمن پاسخ می‌دهد.
- لطفاً توجه داشته باشید که Aspose پشتیبانی فنی از طریق تلفن ارائه نمی‌دهد. پشتیبانی تلفنی فقط برای سؤالات فروش و خرید در دسترس است.
- هنگام انتظار برای دریافت پاسخ در انجمن‌ها، لطفاً صبور باشید و اختلافات منطقه زمانی را در نظر بگیرید.

زمانی که با مشکلی در Aspose.Slides for Node.js via Java مواجه می‌شوید، برای حل کارآمد آن از راهنمایی‌های زیر پیروی کنید:

- قبل از گزارش مشکل، بررسی و تأیید کنید که از آخرین نسخه Aspose.Slides for Node.js via Java استفاده می‌کنید. شاید مشکلی که تجربه کردید در به‌روزرسانی قبلی رفع شده باشد.

  برای اطلاعات درباره آخرین نسخه به [Aspose.Slides download](https://releases.aspose.com/slides/fa/nodejs-java/) مراجعه کنید.

- پیش از گزارش یک مشکل، می‌توانید انجمن‌های ما، این مستندات و مرجع API را بررسی کنید. شاید پاسخ سؤال شما قبلاً ارائه شده باشد.

- هنگام گزارش مشکل، لطفاً سند اصلی و (در صورت امکان) بخشی از کد مربوط به مشکل را شامل کنید. اگر نیاز به پیوست چندین فایل دارید، آن‌ها را در یک بسته ZIP قرار دهید. می‌توانید اسناد خود را در Aspose.Forums به‌صورت امن پیوست کنید زیرا **فقط** شما و توسعه‌دهندگان Aspose به فایل‌های پیوست دسترسی دارند.

- برای هر مشکل یک رشته (موضوع) جداگانه ایجاد و استفاده کنید. اگر مشکل دیگری دارید، باید یک رشته جدید ایجاد کنید تا آن را گزارش دهید.
