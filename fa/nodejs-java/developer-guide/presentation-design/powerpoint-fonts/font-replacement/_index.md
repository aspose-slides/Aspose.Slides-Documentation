## **نمای کلی**

Aspose.Slides به شما امکان می‌دهد یک قلم را در سراسر ارائه با قلم دیگری جایگزین کنید. هنگامی که یک قلم جایگزین می‌شود، تمام نمونه‌های قلم اصلی به قلم جدید تغییر می‌کند.

برای انجام جایگزینی قلم، ارائه را بارگذاری کنید، قلم منبع و قلم جایگزین را تعریف کنید، متد جایگزینی قلم را فراخوانی کنید و ارائهٔ اصلاح‌شده را به‌صورت فایل PPTX ذخیره کنید. این روش زمانی مفید است که عمداً بخواهید یک خانوادهٔ قلم را در سراسر ارائه با خانوادهٔ دیگری تعویض کنید.

## **جایگزینی فونت‌ها**

اگر نظر خود را دربارهٔ استفاده از یک قلم عوض کنید، می‌توانید آن قلم را با قلم دیگری جایگزین کنید. تمام نمونه‌های قلم قدیمی توسط قلم جدید جایگزین خواهند شد. 

Aspose.Slides به شما این امکان را می‌دهد تا قلم را به این روش جایگزین کنید:

1. ارائهٔ مربوطه را بارگذاری کنید. 
2. قلم مورد جایگزینی را بارگذاری کنید. 
3. قلم جدید را بارگذاری کنید. 
4. قلم را جایگزین کنید. 
5. ارائهٔ اصلاح‌شده را به‌صورت فایل PPTX بنویسید.

این کد JavaScript جایگزینی قلم را نشان می‌دهد:

```javascript
// یک ارائه را بارگذاری می‌کند
var pres = new aspose.slides.Presentation("Fonts.pptx");
try {
    // قلم منبعی که جایگزین خواهد شد را بارگذاری می‌کند
    var sourceFont = new aspose.slides.FontData("Arial");
    // قلم جدید را بارگذاری می‌کند
    var destFont = new aspose.slides.FontData("Times New Roman");
    // قلم‌ها را جایگزین می‌کند
    pres.getFontsManager().replaceFont(sourceFont, destFont);
    // ارائه را ذخیره می‌کند
    pres.save("UpdatedFont_out.pptx", aspose.slides.SaveFormat.Pptx);
} finally {
    if (pres != null) {
        pres.dispose();
    }
}
```

## **سوالات متداول**

**تفاوت بین «جایگزینی قلم»، «جاگذاری قلم» و «قلم‌های پیش‌فرض» چیست؟**

جایگزینی یک تعویض عمدی از یک خانواده به دیگری در سرتاسر سند است. [جاگذاری](/slides/fa/nodejs-java/font-substitution/) قانونی است مشابه «اگر قلم در دسترس نباشد، از X استفاده کنید». [قلم‌های پیش‌فرض](/slides/fa/nodejs-java/fallback-font/) به صورت هدفمند برای گلیف‌های گمشدهٔ جداگانه اعمال می‌شود هنگامی که قلم پایه نصب شده باشد اما شامل کاراکترهای مورد نیاز نباشد.

**آیا جایگزینی بر اسلایدهای اصلی، قالب‌ها، یادداشت‌ها و نظرات اعمال می‌شود؟**

بله. جایگزینی بر تمام اشیای ارائه که از قلم اصلی استفاده می‌کنند، از جمله اسلایدهای اصلی و یادداشت‌ها تأثیر می‌گذارد؛ نظرات نیز جزئی از سند هستند و توسط موتور قلم در نظر گرفته می‌شوند.

**آیا قلم داخل اشیای OLE توکار (به عنوان مثال Excel) تغییر خواهد کرد؟**

خیر. [محتوای OLE](/slides/fa/nodejs-java/manage-ole/) توسط برنامهٔ خود آن کنترل می‌شود. جایگزینی در ارائه داده‌های داخلی OLE را بازفرمت نمی‌کند؛ ممکن است به‌صورت تصویر یا محتوای قابل ویرایش خارجی نمایش داده شود.

**آیا می‌توانم یک قلم را فقط در بخشی از ارائه (بر حسب اسلایدها یا مناطق) جایگزین کنم؟**

جایگزینی هدفمند ممکن است اگر قلم را در سطح اشیای/محدوده‌های مورد نیاز تغییر دهید به‌جای اعمال جایگزینی سراسری بر کل سند. منطق کلی انتخاب قلم در هنگام رندرینگ همانند قبل باقی می‌ماند.

**چگونه می‌توانم پیشاپیش تعیین کنم که ارائه از چه قلم‌هایی استفاده می‌کند؟**

از [مدیر قلم](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/fontsmanager/) ارائه استفاده کنید: این ابزار فهرستی از [خانواده‌های مورد استفاده](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/fontsmanager/getfonts/) و اطلاعاتی دربارهٔ [جاگذاری‌ها/قلم‌های «نامشخص»](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/fontsmanager/getsubstitutions/) ارائه می‌دهد که برنامه‌ریزی جایگزینی را آسان می‌کند.

**آیا جایگزینی قلم هنگام تبدیل به PDF/تصاویر کار می‌کند؟**

بله. هنگام خروجی، Aspose.Slides همان [دنبالهٔ انتخاب/جاگذاری قلم](/slides/fa/nodejs-java/font-selection-sequence/) را اعمال می‌کند، بنابراین یک جایگزینی که پیشاپیش انجام شده در زمان تبدیل رعایت می‌شود.

**آیا لازم است قلم هدف را در سیستم نصب کنم یا می‌توانم پوشه‌ای از قلم‌ها را پیوست کنم؟**

نصب لازم نیست: کتابخانه امکان [بارگذاری قلم‌های خارجی](/slides/fa/nodejs-java/custom-font/) را از پوشه‌های کاربر برای استفاده در هنگام [رندرینگ و خروجی](/slides/fa/nodejs-java/convert-powerpoint/) فراهم می‌کند.

**آیا جایگزینی مشکل «توفو» (مربعات) به جای کاراکترها را رفع می‌کند؟**

فقط در صورتی که قلم هدف واقعاً گلیف‌های مورد نیاز را داشته باشد. در غیر این صورت، [پیکربندی پیش‌فرض](/slides/fa/nodejs-java/fallback-font/) را برای پوشش کاراکترهای گمشده انجام دهید.
