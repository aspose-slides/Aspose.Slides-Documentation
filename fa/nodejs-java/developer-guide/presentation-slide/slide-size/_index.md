## **مقدمه**

Aspose.Slides ابزارهای کاملی برای تنظیم اندازه اسلاید و نسبت تصویر در ارائه‌های PowerPoint فراهم می‌کند که برای چاپ و نمایش روی صفحه بسیار حیاتی است.

اندازه‌ها و نسبت‌های محبوب اسلاید:

- **استاندارد (نسبت تصویر 4:3)**: مناسب برای صفحه‌نمایش‌ها و دستگاه‌های قدیمی.
- **عریض (نسبت تصویر 16:9)**: برای پروژکتور‌ها و نمایشگرهای مدرن توصیه می‌شود.

اطمینان حاصل کنید که در سراسر ارائه یک اندازه اسلاید و نسبت تصویر ثابت باشد؛ این تنظیم برای تمام اسلایدها اعمال می‌شود. برای دریافت نتایج بهینه، ابعاد اسلاید را در ابتدای فرآیند ایجاد ارائه تنظیم کنید تا از بروز مشکلات جلوگیری کنید.

{{% alert color="primary" %}} 
به‌طور پیش‌فرض، ارائه‌های ایجاد شده با Aspose.Slides از نسبت تصویر استاندارد 4:3 استفاده می‌کنند.
{{% /alert %}}

## **تغییر اندازه اسلاید در ارائه‌ها**

این کد نمونه نشان می‌دهد چگونه اندازه اسلاید را در یک ارائه با JavaScript و استفاده از Aspose.Slides تغییر دهید:

```javascript
var pres = new aspose.slides.Presentation("pres-4x3-aspect-ratio.pptx");
try {
    pres.getSlideSize().setSize(aspose.slides.SlideSizeType.OnScreen16x9, aspose.slides.SlideSizeScaleType.DoNotScale);
    pres.save("pres-4x3-aspect-ratio.pptx", aspose.slides.SaveFormat.Pptx);
} finally {
    if (pres != null) {
        pres.dispose();
    }
}
```

## **مشخص کردن اندازه سفارشی اسلاید در ارائه‌ها**

اگر اندازه‌های رایج اسلاید (4:3 و 16:9) برای کار شما مناسب نیستند، می‌توانید از یک اندازه خاص یا منحصربه‌فرد استفاده کنید. برای مثال، اگر قصد دارید اسلایدهای تمام‌صفحه را بر روی یک قالب صفحه سفارشی چاپ کنید یا ارائه خود را بر روی برخی انواع صفحه‌نمایش‌ها نمایش دهید، تنظیم اندازه سفارشی می‌تواند مفید باشد.

این کد نمونه نشان می‌دهد چگونه با استفاده از Aspose.Slides برای Node.js از طریق Java یک اندازه سفارشی برای اسلاید در JavaScript تعیین کنید:

```javascript
var pres = new aspose.slides.Presentation("pres.pptx");
try {
    pres.getSlideSize().setSize(780, 540, aspose.slides.SlideSizeScaleType.DoNotScale);// اندازه کاغذ A4
    pres.save("pres-a4-slide-size.pptx", aspose.slides.SaveFormat.Pptx);
} finally {
    if (pres != null) {
        pres.dispose();
    }
}
```

## **مقابله با مشکلات هنگام تغییر اندازه اسلایدها در ارائه‌ها**

پس از تغییر اندازه اسلاید یک ارائه، محتویات اسلایدها (مانند تصاویر یا اشیاء) ممکن است دچار اعوجاج شوند. به‌طور پیش‌فرض، اشیاء به‌صورت خودکار برای مطابقت با اندازه جدید اسلاید تغییر اندازه می‌یابند. با این حال، هنگام تغییر اندازه اسلاید یک ارائه می‌توانید تنظیمی را مشخص کنید که تعیین می‌کند Aspose.Slides چطور با محتواهای اسلایدها رفتار کند.

بسته به هدفی که دارید می‌توانید از یکی از این تنظیمات استفاده کنید:

- `DoNotScale`
  
  اگر نمی‌خواهید اشیاء روی اسلایدها تغییر اندازه دهند، از این تنظیم استفاده کنید.

- `EnsureFit`
  
  اگر می‌خواهید به اندازه اسلاید کوچکتر مقیاس کنید و نیاز دارید Aspose.Slides اشیاء را کوچکتر کند تا همه آن‌ها در اسلاید جای گیرند (به این ترتیب از از دست رفتن محتوا جلوگیری می‌شود)، این تنظیم را انتخاب کنید.

- `Maximize`
  
  اگر می‌خواهید به اندازه اسلاید بزرگتر مقیاس کنید و نیاز دارید Aspose.Slides اشیاء را بزرگتر کند تا نسبت به اندازه جدید اسلاید متناسب شوند، این تنظیم را استفاده کنید.

این کد نمونه نشان می‌دهد چگونه از تنظیم `Maximize` هنگام تغییر اندازه اسلاید یک ارائه استفاده کنید:

```javascript
var pres = new aspose.slides.Presentation("pres.pptx");
try {
    pres.getSlideSize().setSize(aspose.slides.SlideSizeType.Ledger, aspose.slides.SlideSizeScaleType.Maximize);
} finally {
    if (pres != null) {
        pres.dispose();
    }
}
```

## **سوالات متداول**

**آیا می‌توانم اندازه سفارشی اسلاید را با واحدهایی غیر از اینچ (مثلاً پوینت یا میلی‌متر) تنظیم کنم؟**

بله. Aspose.Slides به‌صورت داخلی از پوینت استفاده می‌کند که 1 پوینت برابر 1/72 اینچ است. می‌توانید هر واحدی (مانند میلی‌متر یا سانتی‌متر) را به پوینت تبدیل کرده و از مقادیر تبدیل‌شده برای تعریف عرض و ارتفاع اسلاید استفاده کنید.

**آیا اندازه سفارشی خیلی بزرگ اسلاید بر عملکرد و مصرف حافظه هنگام رندرینگ تأثیر می‌گذارد؟**

بله. ابعاد بزرگ‌تر اسلاید (به پوینت) همراه با مقیاس رندرینگ بالاتر منجر به افزایش مصرف حافظه و زمان پردازش می‌شود. سعی کنید اندازه اسلایدی عملی انتخاب کنید و مقیاس رندرینگ را فقط در صورت نیاز برای رسیدن به کیفیت خروجی مطلوب تنظیم کنید.

**آیا می‌توانم یک اندازه اسلاید غیر استاندارد تعریف کنم و سپس اسلایدها را از ارائه‌هایی با اندازه‌های متفاوت ادغام کنم؟**

نمی‌توانید [merge presentations](/slides/fa/nodejs-java/merge-presentation/) را در حالی که اندازه اسلایدها متفاوت است انجام دهید — ابتدا یک ارائه را به اندازه دیگر تغییر اندازه دهید. هنگام تغییر اندازه اسلاید می‌توانید نحوه برخورد با محتواهای موجود را از طریق گزینه [SlideSizeScaleType](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/slidesizescaletype/) مشخص کنید. پس از هم‌راستا کردن اندازه‌ها، می‌توانید اسلایدها را ادغام کنید و قالب‌بندی را حفظ کنید.

**آیا می‌توانم برای اشکال منفرد یا نواحی خاصی از یک اسلاید تصویر بندانگشتی تولید کنم و آیا این تصاویر نسبت به اندازه جدید اسلاید احترام می‌گذارند؟**

بله. Aspose.Slides می‌تواند تصویر بندانگشتی برای [entire slides](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/slide/#getImage) و همچنین برای [selected shapes](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/shape/#getImage) تولید کند. تصاویر حاصل اندازه و نسبت تصویر فعلی اسلاید را منعکس می‌کنند و چارچوب و هندسه‌ی سازگاری را تضمین می‌نمایند.
