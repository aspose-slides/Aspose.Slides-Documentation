## **مقدمه**

در PowerPoint 2019، مایکروسافت تنظیم **Always Open Read-Only** را به عنوان یکی از گزینه‌هایی که کاربران می‌توانند برای محافظت از ارائه‌های خود استفاده کنند، معرفی کرد. ممکن است بخواهید از این تنظیم حالت فقط‑خواندنی برای محافظت از یک ارائه استفاده کنید وقتی

- می‌خواهید از ویرایش‌های تصادفی جلوگیری کنید و محتوای ارائه‌تان را ایمن نگه دارید. 
- می‌خواهید به افراد اطلاع دهید که ارائه‌ای که ارائه داده‌اید نسخه نهایی است. 

بعد از این که گزینه **Always Open Read-Only** را برای یک ارائه انتخاب کردید، وقتی کاربران آن را باز می‌کنند، توصیه **Read-Only** را می‌بینند و ممکن است پیامی به این شکل نمایش داده شود: *برای جلوگیری از تغییرات تصادفی، نویسنده این فایل را برای باز شدن به صورت فقط‑خواندنی تنظیم کرده است.*

توصیه Read-Only یک بازدارنده ساده ولی مؤثر است که از ویرایش منع می‌کند، زیرا کاربران باید کاری انجام دهند تا آن را حذف کنند قبل از اینکه اجازه ویرایش ارائه را داشته باشند. اگر نمی‌خواهید کاربران تغییراتی در ارائه ایجاد کنند و می‌خواهید این را به‌صورت مودبانه به آن‌ها اطلاع دهید، توصیه Read-Only می‌تواند گزینهٔ مناسبی برای شما باشد. 

> اگر یک ارائه با حفاظت **Read-Only** در یک برنامهٔ قدیمی‌تر Microsoft PowerPoint باز شود—که از عملکرد جدید معرفی‌شده پشتیبانی نمی‌کند—توصیه **Read-Only** نادیده گرفته می‌شود (ارائه به‌صورت عادی باز می‌شود).

## **اعمال حالت فقط‑خواندنی**

Aspose.Slides برای Node.js از طریق Java به شما اجازه می‌دهد یک ارائه را به حالت **Read-Only** تنظیم کنید، به این معنا که کاربران (پس از باز کردن ارائه) توصیه **Read-Only** را می‌بینند. این کد نمونه نشان می‌دهد چگونه یک ارائه را با استفاده از Aspose.Slides در JavaScript به حالت **Read-Only** تنظیم کنید:

```javascript
var pres = new aspose.slides.Presentation();
try {
    pres.getProtectionManager().setReadOnlyRecommended(true);
    pres.save("ReadOnlyPresentation.pptx", aspose.slides.SaveFormat.Pptx);
} finally {
    if (pres != null) {
        pres.dispose();
    }
}
```

**نکته**: توصیه **Read-Only** صرفاً برای جلوگیری از ویرایش یا متوقف کردن کاربران از ایجاد تغییرات تصادفی در یک ارائه PowerPoint در نظر گرفته شده است. اگر فردی با انگیزه—که می‌داند چه کاری انجام می‌دهد—تصمیم بگیرد ارائه شما را ویرایش کند، به راحتی می‌تواند تنظیمات فقط‑خواندنی را حذف کند. اگر به‌طور جدی نیاز به جلوگیری از ویرایش غیرمجاز دارید، بهتر است از [حفاظت‌های سخت‌تری که شامل رمزگذاری و گذرواژه‌ها می‌شوند](https://docs.aspose.com/slides/fa/nodejs-java/password-protected-presentation/) استفاده کنید.

## **سؤالات متداول**

**تفاوت «Read-Only recommended» با حفاظت کامل با گذرواژه چیست؟**

«Read-Only recommended» فقط یک پیشنهاد برای باز کردن فایل در حالت فقط‑خواندنی نمایش می‌دهد و به‌راحتی می‌توان از آن عبور کرد. [حفاظت با گذرواژه](/slides/fa/nodejs-java/password-protected-presentation/) در واقع باز کردن یا ویرایش را محدود می‌کند و وقتی به کنترل‌های امنیتی واقعی نیاز دارید مناسب است.

**آیا می‌توان «Read-Only recommended» را با واترمارک‌ها ترکیب کرد تا ویرایش‌ها بیشتر دلسرد شوند؟**

بله. این توصیه می‌تواند با [واترمارک‌ها](/slides/fa/nodejs-java/watermark/) به عنوان یک بازدارندهٔ بصری ترکیب شود؛ آن‌ها مکانیزم‌های جداگانه‌ای هستند و به‌خوبی با هم کار می‌کنند.

**آیا یک ماکرو یا ابزار خارجی همچنان می‌تواند فایل را وقتی توصیه فعال است، تغییر دهد؟**

بله. این توصیه تغییرات برنامه‌نویسی شده را مسدود نمی‌کند. برای جلوگیری از ویرایش‌های خودکار، از [گذرواژه‌ها و رمزگذاری](/slides/fa/nodejs-java/password-protected-presentation/) استفاده کنید.

**«Read-Only recommended» چگونه با پرچم‌های «IsEncrypted» و «IsWriteProtected» مرتبط است؟**

آن‌ها سیگنال‌های متفاوتی هستند. «Read-Only recommended» یک اعلان نرم و اختیاری است؛ [isWriteProtected](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/protectionmanager/iswriteprotected/) و [isEncrypted](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/protectionmanager/isencrypted/) محدودیت‌های واقعی نوشتن یا خواندن را نشان می‌دهند که به گذرواژه‌ها یا رمزگذاری وابسته‌اند.
