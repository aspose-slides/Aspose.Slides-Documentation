## **مقدمه**

زمانی که یک ارائه را با رمز عبور محافظت می‌کنید، به این معنی است که رمزی تنظیم می‌کنید که برخی محدودیت‌ها را بر روی ارائه اعمال می‌کند. برای حذف این محدودیت‌ها باید رمز عبور وارد شود. یک ارائه محافظت‌شده با رمز عبور به عنوان یک ارائه قفل‌شده در نظر گرفته می‌شود.

به طور معمول می‌توانید برای اعمال این محدودیت‌ها بر روی یک ارائه، یک رمز عبور تنظیم کنید:

- **تغییر**

  اگر می‌خواهید فقط کاربران مشخصی بتوانند ارائه شما را تغییر دهند، می‌توانید محدودیت تغییر را تنظیم کنید. این محدودیت مانع از تغییر، اصلاح یا کپی کردن موارد در ارائه شما می‌شود (مگر این که رمز عبور را ارائه دهند).

  با این حال، در این حالت حتی بدون رمز عبور، کاربر می‌تواند به سند شما دسترسی پیدا کند و آن را باز کند. در این حالت فقط‑خواندنی، کاربر می‌تواند محتوا یا موارد—لینک‌ها، انیمیشن‌ها، افکت‌ها و غیره—درون ارائه شما را ببیند، اما نمی‌تواند موارد را کپی کند یا ارائه را ذخیره کند.

- **باز کردن**

  اگر می‌خواهید فقط کاربران مشخصی بتوانند ارائه شما را باز کنند، می‌توانید محدودیت باز کردن را تنظیم کنید. این محدودیت مانع از مشاهده محتویات ارائه شما می‌شود (مگر این که رمز عبور را ارائه دهند).

  از لحاظ فنی، محدودیت باز کردن همچنین مانع کاربران از تغییر ارائه‌های شما می‌شود: وقتی افراد نمی‌توانند ارائه‌ای را باز کنند، نمی‌توانند آن را تغییر دهند یا اصلاحی روی آن انجام دهند.

  **Note** وقتی یک ارائه را با رمز عبور محافظت می‌کنید تا باز کردن آن را منع کنید، فایل ارائه رمزگذاری می‌شود.

## **محافظت با رمز عبور برای ارائه‌ها در Aspose.Slides**
**فرمت‌های پشتیبانی‌شده**

Aspose.Slides از محافظت با رمز عبور، رمزگذاری و عملیات مشابه برای ارائه‌ها در این فرمت‌ها پشتیبانی می‌کند:

- PPTX و PPT - ارائهٔ مایکروسافت پاورپوینت
- ODP - ارائهٔ OpenDocument
- OTP - قالب ارائهٔ OpenDocument

**عملیات پشتیبانی‌شده**

Aspose.Slides به شما امکان می‌دهد تا با استفاده از محافظت با رمز عبور از تغییرات ارائه‌ها جلوگیری کنید به روش‌های زیر:

- رمزگذاری یک ارائه
- تنظیم حفاظت نوشتن برای یک ارائه

**سایر عملیات**

Aspose.Slides به شما امکان می‌دهد تا سایر وظایف مربوط به محافظت با رمز عبور و رمزگذاری را به روش‌های زیر انجام دهید:

- رمزگشایی یک ارائه؛ باز کردن یک ارائهٔ رمزگذاری‌شده
- حذف رمزگذاری؛ غیرفعال کردن محافظت با رمز عبور
- حذف حفاظت نوشتن از یک ارائه
- دریافت خصوصیات یک ارائهٔ رمزگذاری‌شده
- بررسی اینکه آیا یک ارائه رمزگذاری شده است یا نه
- بررسی اینکه آیا یک ارائه با رمز عبور محافظت شده است یا نه.

## **رمزگذاری یک ارائه**

می‌توانید با تنظیم یک رمز عبور، یک ارائه را رمزگذاری کنید. سپس برای تغییر ارائهٔ قفل‌شده، کاربر باید رمز عبور را ارائه دهد.

برای رمزگذاری یا محافظت با رمز عبور از یک ارائه، باید از متد encrypt (از [ProtectionManager](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/ProtectionManager)) برای تنظیم رمز عبور استفاده کنید. رمز عبور را به متد encrypt پاس می‌دهید و با استفاده از متد save ارائهٔ اکنون رمزگذاری‌شده را ذخیره می‌کنید.

این کد نمونه نشان می‌دهد چگونه یک ارائه را رمزگذاری کنید:

```javascript
var presentation = new aspose.slides.Presentation("pres.pptx");
try {
    presentation.getProtectionManager().encrypt("123123");
    presentation.save("encrypted-pres.pptx", aspose.slides.SaveFormat.Pptx);
} finally {
    if (presentation != null) {
        presentation.dispose();
    }
}
```

## **تنظیم حفاظت نوشتن برای یک ارائه**

می‌توانید یک علامت «تغییر نکنید» به یک ارائه اضافه کنید. به این طریق می‌توانید به کاربران بگویید که نمی‌خواهید آن‌ها تغییراتی در ارائه اعمال کنند.

**Note** فرآیند حفاظت نوشتن ارائه را رمزگذاری نمی‌کند. بنابراین، کاربران—اگر بخواهند—می‌توانند ارائه را تغییر دهند، اما برای ذخیرهٔ تغییرات باید ارائه‌ای با نام متفاوت ایجاد کنند.

برای تنظیم حفاظت نوشتن، باید از متد [setWriteProtection](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/ProtectionManager#setWriteProtection-java.lang.String-) استفاده کنید. این کد نمونه نشان می‌دهد چگونه حفاظت نوشتن را برای یک ارائه تنظیم کنید:

```javascript
var presentation = new aspose.slides.Presentation("pres.pptx");
try {
    presentation.getProtectionManager().setWriteProtection("123123");
    presentation.save("write-protected-pres.pptx", aspose.slides.SaveFormat.Pptx);
} finally {
    if (presentation != null) {
        presentation.dispose();
    }
}
```

## **رمزگشایی یک ارائه؛ باز کردن یک ارائهٔ رمزگذاری‌شده**

Aspose.Slides به شما امکان می‌دهد تا یک فایل رمزگذاری‌شده را با پاس دادن رمز عبور بارگذاری کنید. برای رمزگشایی یک ارائه، باید متد [removeEncryption](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/ProtectionManager#removeEncryption--) را بدون پارامتر فراخوانی کنید. سپس باید رمز عبور صحیح را وارد کنید تا ارائه بارگذاری شود.

این کد نمونه نشان می‌دهد چگونه یک ارائه را رمزگشایی کنید:

```javascript
var loadOptions = new aspose.slides.LoadOptions();
loadOptions.setPassword("123123");
var presentation = new aspose.slides.Presentation("pres.pptx", loadOptions);
try {
    // کار با ارائهٔ رمزگشایی‌شده
} finally {
    if (presentation != null) {
        presentation.dispose();
    }
}
```

## **حذف رمزگذاری؛ غیرفعال کردن محافظت با رمز عبور**

می‌توانید رمزگذاری یا محافظت با رمز عبور یک ارائه را حذف کنید. به این ترتیب کاربران می‌توانند بدون هیچ محدودیتی به ارائه دسترسی پیدا کنند یا آن را modify کنند.

برای حذف رمزگذاری یا محافظت با رمز عبور، باید متد [removeEncryption](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/ProtectionManager#removeEncryption--) را صدا بزنید. این کد نمونه نشان می‌دهد چگونه رمزگذاری یک ارائه را حذف کنید:

```javascript
var loadOptions = new aspose.slides.LoadOptions();
loadOptions.setPassword("123123");
var presentation = new aspose.slides.Presentation("pres.pptx", loadOptions);
try {
    presentation.getProtectionManager().removeEncryption();
    presentation.save("encryption-removed.pptx", aspose.slides.SaveFormat.Pptx);
} finally {
    if (presentation != null) {
        presentation.dispose();
    }
}
```

## **حذف حفاظت نوشتن از یک ارائه**

می‌توانید از Aspose.Slides برای حذف حفاظت نوشتن استفاده‌شده بر روی یک فایل ارائه بهره بگیرید. به این ترتیب، کاربران می‌توانند همان‌طور که می‌خواهند تغییر ایجاد کنند—و هنگام انجام چنین کاری هشدار دریافت نمی‌کنند.

می‌توانید حفاظت نوشتن را با استفاده از متد [removeWriteProtection](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/ProtectionManager#removeWriteProtection--) حذف کنید. این کد نمونه نشان می‌دهد چگونه حفاظت نوشتن را از یک ارائه حذف کنید:

```javascript
var presentation = new aspose.slides.Presentation("pres.pptx");
try {
    presentation.getProtectionManager().removeWriteProtection();
    presentation.save("write-protection-removed.pptx", aspose.slides.SaveFormat.Pptx);
} finally {
    if (presentation != null) {
        presentation.dispose();
    }
}
```

## **دریافت خصوصیات یک ارائهٔ رمزگذاری‌شده**

معمولاً کاربران برای دریافت خصوصیات سند یک ارائهٔ رمزگذاری‌شده یا محافظت‌شده با رمز عبور مشکل دارند. با این حال، Aspose.Slides یک مکانیسم ارائه می‌دهد که به شما اجازه می‌دهد ارائه‌ای را با رمز عبور محافظت کنید در حالی که کاربران همچنان می‌توانند به خصوصیات آن دسترسی داشته باشند.

**Note** وقتی Aspose.Slides یک ارائه را رمزگذاری می‌کند، خصوصیات سند ارائه نیز به‌صورت پیش‌فرض با رمز عبور محافظت می‌شوند. اما اگر نیاز داشته باشید تا خصوصیات ارائه پس از رمزگذاری همچنان دسترس‌پذیر باشند، Aspose.Slides این امکان را برای شما فراهم می‌کند.

اگر می‌خواهید کاربران قابلیت دسترسی به خصوصیات یک ارائهٔ رمزگذاری‌شده را حفظ کنند، می‌توانید خصوصیت [encryptDocumentProperties](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/ProtectionManager#getEncryptDocumentProperties--) را روی `true` تنظیم کنید. این کد نمونه نشان می‌دهد چگونه یک ارائه را رمزگذاری کنید در حالی که راهی برای دسترسی کاربران به خصوصیات سند آن فراهم می‌شود:

```javascript
var presentation = new aspose.slides.Presentation("pres.pptx");
try {
    presentation.getProtectionManager().setEncryptDocumentProperties(true);
    presentation.getProtectionManager().encrypt("123123");
} finally {
    if (presentation != null) {
        presentation.dispose();
    }
}
```

## **بررسی اینکه آیا یک ارائه قبل از بارگذاری با رمز عبور محافظت شده است**

قبل از بارگذاری یک ارائه، ممکن است بخواهید بررسی کنید و اطمینان حاصل کنید که ارائه با رمز عبور محافظت نشده است. به این ترتیب می‌توانید از خطاها و مشکلات مشابهی که هنگام بارگذاری یک ارائهٔ محافظت‌شده بدون رمز عبور رخ می‌دهد، جلوگیری کنید.

این کد JavaScript نشان می‌دهد چگونه یک ارائه را بررسی کنید تا ببینید آیا با رمز عبور محافظت شده است (بدون بارگذاری خود ارائه):

```javascript
var presentationInfo = aspose.slides.PresentationFactory.getInstance().getPresentationInfo("example.pptx");
console.log("The presentation is password protected: " + presentationInfo.isPasswordProtected());
```

## **بررسی اینکه آیا یک ارائه رمزگذاری شده است**

Aspose.Slides به شما امکان می‌دهد تا بررسی کنید که آیا یک ارائه رمزگذاری شده است یا نه. برای انجام این کار می‌توانید از خصوصیت [isEncrypted](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/ProtectionManager#isEncrypted--) استفاده کنید که `true` برمی‌گرداند اگر ارائه رمزگذاری شده باشد یا `false` اگر رمزگذاری نشده باشد.

این کد نمونه نشان می‌دهد چگونه بررسی کنید آیا یک ارائه رمزگذاری شده است:

```javascript
var presentation = new aspose.slides.Presentation("pres.pptx");
try {
    var isEncrypted = presentation.getProtectionManager().isEncrypted();
} finally {
    if (presentation != null) {
        presentation.dispose();
    }
}
```

## **بررسی اینکه آیا یک ارائه حفاظت نوشتن دارد**

Aspose.Slides به شما امکان می‌دهد تا بررسی کنید که آیا یک ارائه حفاظت نوشتن دارد یا نه. برای انجام این کار می‌توانید از خصوصیت [isWriteProtected](https://reference.aspose.com/slides/fa/nodejs-java/aspose.slides/ProtectionManager#isWriteProtected--) استفاده کنید که `true` برمی‌گرداند اگر ارائه رمزگذاری شده باشد یا `false` اگر رمزگذاری نشده باشد.

این کد نمونه نشان می‌دهد چگونه بررسی کنید آیا یک ارائه حفاظت نوشتن دارد:

```javascript
var presentation = new aspose.slides.Presentation("pres.pptx");
try {
    var isEncrypted = presentation.getProtectionManager().isWriteProtected();
} finally {
    if (presentation != null) {
        presentation.dispose();
    }
}
```

## **اعتبارسنجی یا تأیید اینکه رمز عبور خاصی برای محافظت از یک ارائه استفاده شده است**

ممکن است بخواهید بررسی کنید و تأیید کنید که رمز عبور خاصی برای محافظت از سند ارائه استفاده شده است. Aspose.Slides ابزارهایی را برای اعتبارسنجی رمز عبور در اختیار شما قرار می‌دهد.

این کد نمونه نشان می‌دهد چگونه یک رمز عبور را اعتبارسنجی کنید:

```javascript
var presentation = new aspose.slides.Presentation("pres.pptx");
try {
    // بررسی کنید آیا "pass" با ... مطابقت دارد
    var isWriteProtected = presentation.getProtectionManager().checkWriteProtection("my_password");
} finally {
    if (presentation != null) {
        presentation.dispose();
    }
}
```

این مقدار `true` برمی‌گرداند اگر ارائه با رمز عبور مشخص شده رمزگذاری شده باشد. در غیر این صورت، `false` برمی‌گرداند.

## **سوالات متداول**

**کدام روش‌های رمزگذاری توسط Aspose.Slides پشتیبانی می‌شود؟**

Aspose.Slides از روش‌های رمزگذاری مدرن، از جمله الگوریتم‌های مبتنی بر AES، پشتیبانی می‌کند و سطح بالایی از امنیت داده‌ها را برای ارائه‌های شما تضمین می‌نماید.

**اگر هنگام تلاش برای باز کردن یک ارائه، رمز عبور نادرست وارد شود، چه اتفاقی می‌افتد؟**

در صورت استفاده از رمز عبور نادرست، یک استثنا رخ می‌دهد که اعلام می‌نماید دسترسی به ارائه رد شده است. این امر به جلوگیری از دسترسی غیرمجاز و حفاظت از محتوای ارائه کمک می‌کند.

**آیا هنگام کار با ارائه‌های محافظت‌شده با رمز عبور، تأثیرات عملکردی وجود دارد؟**

فرآیند رمزگذاری و رمزگشایی ممکن است کمی بار اضافی بر عملیات باز کردن و ذخیره‌سازی ایجاد کند. در اکثر موارد، این تأثیر عملکردی جزئی است و به‌طور قابل توجهی بر زمان کلی پردازش وظایف شما تأثیر نمی‌گذارد.
