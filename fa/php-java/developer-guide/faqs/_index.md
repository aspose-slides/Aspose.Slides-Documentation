## **نمای کلی**

این سؤالات متداول پاسخ‌های سوالات رایج درباره Aspose.Slides را ارائه می‌دهد. این راهنما شامل فرمت‌های فایل پشتیبانی‌شده، مدیریت استثناها هنگام کار با ارائه‌های بزرگ، تغییر اندازه اسلایدها، پیش‌نمایش اسلایدها، استخراج متن از ارائه‌ها، قالب‌بندی حاشیه‌های جدول، قرار دادن تصویرها و حل مشکلات مربوط به قلم‌ها هنگام تبدیل ارائه‌ها به PDF یا تصاویر است.

## **فرمت‌های فایل پشتیبانی‌شده**

**Q: چه قالب‌های فایلی Aspose.Slides برای PHP از طریق Java پشتیبانی می‌کند؟**

**A**: Aspose.Slides برای PHP از طریق Java فرمت‌های فایلی که در [Supported File Formats](/slides/fa/php-java/supported-file-formats/) توصیف شده‌اند را پشتیبانی می‌کند.

## **استثناها**

**Q: هنگام بارگذاری یک فایل PPT بزرگ با تصویرها، استثنای out of memory دریافت می‌کنم. آیا محدودیتی در Aspose.Slides برای اندازه فایل وجود دارد؟**

**A**: هیچ فرمول خاصی برای محاسبهٔ اندازهٔ ارائه‌ای که Aspose.Slides پشتیبانی می‌کند وجود ندارد. باید فضای کافی برای نگهداری کل ساختار ارائه و تصویرها در حافظه وجود داشته باشد. معمولاً تصویرها در حافظه فضای بیشتری نسبت به دیسک سخت اشغال می‌کنند، به‌ویژه وقتی تصویرها افکت‌های اضافی داشته باشند.

به‌طور کلی، Aspose.Slides برای PHP از طریق Java می‌تواند به راحتی فایل‌های ارائه حدود ۳۰۰ مگابایت را در سروری با ۴ گیگابایت RAM مدیریت کند.

## **کار با اسلایدها**

**Q: آیا می‌توانم اندازهٔ اسلایدها را در یک ارائه تغییر دهم؟**

**A**: می‌توانید از متد `getSlideSize` که توسط کلاس [Presentation](https://reference.aspose.com/slides/fa/php-java/aspose.slides/presentation/) ارائه شده است، برای تعریف اندازهٔ اسلایدها در یک ارائه استفاده کنید.

**Q: آیا راهی برای تعریف اسلایدهای با اندازه‌های مختلف در یک ارائه وجود دارد؟**

**A**: از آنجا که اندازهٔ اسلایدها در اسناد Microsoft PowerPoint در سطح ارائه تعریف می‌شود، راهی برای انجام این کار وجود ندارد.

**Q: آیا Aspose.Slides برای PHP از طریق Java پیش‌نمایش اسلاید قبل از ذخیره‌سازی را پشتیبانی می‌کند؟**

**A**: می‌توانید اسلایدهای ارائه را به تصویر رندر کنید و از این تصاویر برای پیش‌نمایش اسلایدها استفاده کنید.

## **کار با متن**

**Q: آیا امکان استخراج تمام متن از یک ارائه وجود دارد؟**

**A**: Aspose.Slides برای PHP از طریق Java کلاس [SlideUtil](https://reference.aspose.com/slides/fa/php-java/aspose.slides/slideutil/) را ارائه می‌دهد که روش‌های متنوعی برای استخراج کل متن از ارائه‌ها فراهم می‌کند.

**Q: چرا اندازه‌های پاراگراف در سیستم‌عامل‌های Windows و Linux متفاوت است؟**

**A**: محاسبهٔ اندازهٔ پاراگراف بر پایهٔ محاسبهٔ اندازهٔ متن نمایانگر پاراگراف داده‌شده است. این محاسبهٔ اندازهٔ متن بر مبنای معیارهای (متریک‌های) فونت مشخص‌شده در ارائه PowerPoint انجام می‌شود. اگر فونت مشخص‌شده موجود نباشد، با نزدیک‌ترین فونت جایگزین می‌شود، اما این فونت معیارهای متفاوتی نسبت به فونت اصلی دارد. بنابراین، محاسبهٔ اندازهٔ پاراگراف‌ها در سیستم‌های مختلف منجر به نتایج متفاوتی می‌شود که بستگی به مجموعهٔ فونت‌های نصب‌شده دارد. برای دستیابی به همان نتیجه در سیستم‌عامل‌های مختلف، باید همان فونت‌ها را بر روی سیستم‌ها نصب کنید یا در زمان اجرا آنها را به‌عنوان [external fonts](/slides/fa/php-java/custom-font/) بارگذاری کنید.

## **قالب‌بندی و تصاویر**

**Q: چگونه می‌توانم رنگ حاشیهٔ جدول را تنظیم کنم؟**

**A**: می‌توانید رنگ تمام حاشیه‌های جدول یا فقط حاشیهٔ کلی جدول را تغییر دهید. برای تغییر همهٔ حاشیه‌ها، لطفاً از متد `getCellFormat` در کلاس [Cell](https://reference.aspose.com/slides/fa/php-java/aspose.slides/cell/) استفاده کنید. برای حاشیهٔ کلی جدول، باید بر روی سلول‌ها پیمایش کنید و رنگ حاشیه‌های بیرونی را تغییر دهید.

**Q: Aspose.Slides برای PHP از طریق Java برای قرار دادن تصویرها از چه مقیاسی استفاده می‌کند؟**

**A**: مختصات و ابعاد تمام اشکال روی اسلایدها بر حسب نقطه (points) (۷۲ dpi) اندازه‌گیری می‌شوند.

## **کار با فونت‌ها**

**Q: هنگام تبدیل PPT به PDF یا تصویرها، چرا فونت‌ها در اسناد خروجی متفاوت هستند؟**

**A**: این مشکل ممکن است نشان دهد که فونت‌های استفاده‌شده در ارائه بر روی سیستم‌عامل اجرایی کد موجود نیستند. باید فونت‌ها را بر روی سیستم‌عامل نصب کنید یا با استفاده از کلاس [FontsLoader](https://reference.aspose.com/slides/fa/php-java/aspose.slides/fontsloader/) به‌عنوان فونت‌های خارجی بارگذاری کنید همان‌طور که در زیر نشان داده شده است:
```php
$folders = ["path_to_a_folder_with_fonts"];
FontsLoader::loadExternalFonts($folders);
```
