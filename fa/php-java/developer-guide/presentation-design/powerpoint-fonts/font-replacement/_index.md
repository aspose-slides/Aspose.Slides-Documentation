## **مرور کلی**

Aspose.Slides به شما امکان می‌دهد یک قلم را در سراسر ارائه با قلم دیگری جابجا کنید. هنگامی که یک قلم جابجا می‌شود، تمام موارد قلم اصلی به قلم جدید تغییر می‌یابند.

برای انجام جایگزینی قلم، ارائه را بارگذاری کنید، قلم منبع و قلم جایگزین را تعریف کنید، متد جایگزینی قلم را فراخوانی کنید و ارائهٔ تغییر یافته را به عنوان فایل PPTX ذخیره کنید. این روش زمانی مفید است که عمداً بخواهید از یک خانوادهٔ قلم به خانوادهٔ دیگری در کل ارائه سوئیچ کنید.

## **جایگزینی قلم‌ها**

اگر نظرتان در مورد استفاده از یک قلم عوض شد، می‌توانید آن قلم را با قلم دیگری جایگزین کنید. تمام موارد قلم قدیمی توسط قلم جدید جایگزین می‌شود.

Aspose.Slides به شما امکان می‌دهد قلم را به این شکل جایگزین کنید:

1. ارائهٔ مربوطه را بارگذاری کنید.  
2. قلمی را که قرار است جایگزین شود بارگذاری کنید.  
3. قلم جدید را بارگذاری کنید.  
4. قلم را جایگزین کنید.  
5. ارائهٔ اصلاح‌شده را به عنوان فایل PPTX ذخیره کنید.

این کد PHP جایگزینی قلم را نشان می‌دهد:

```php
  # بارگذاری یک ارائه
  $pres = new Presentation("Fonts.pptx");
  try {
    # بارگذاری قلم منبعی که جایگزین می‌شود
    $sourceFont = new FontData("Arial");
    # بارگذاری قلم جدید
    $destFont = new FontData("Times New Roman");
    # جایگزینی قلم‌ها
    $pres->getFontsManager()->replaceFont($sourceFont, $destFont);
    # ذخیره ارائه
    $pres->save("UpdatedFont_out.pptx", SaveFormat::Pptx);
  } finally {
    if (!java_is_null($pres)) {
      $pres->dispose();
    }
  }
```

برای تنظیم قوانینی که تعیین می‌کند در شرایط خاص چه اتفاقی می‌افتد (به عنوان مثال اگر قلم قابل دسترسی نباشد)، به [**جایگزینی قلم**](/slides/fa/php-java/font-substitution/) مراجعه کنید.

## **سؤالات متداول**

**تفاوت بین «جایگزینی قلم»، «جایگزینی جایگزین» و «قلم‌های پیش‌فرض» چیست؟**

جایگزینی یک تغییر عمدی از یک خانوادهٔ قلم به خانوادهٔ دیگر در تمام سند است. [جایگزینی](/slides/fa/php-java/font-substitution/) قاعده‌ای مانند «اگر قلم در دسترس نباشد، از X استفاده شود» است. [قلم‌های پیش‌فرض](/slides/fa/php-java/fallback-font/) به‌صورت هدفمند برای گلیف‌های گم‌شدهٔ فردی اعمال می‌شود هنگامی که قلم پایه نصب شده باشد اما کاراکترهای مورد نیاز را نداشته باشد.

**آیا جایگزینی بر اسلایدهای اصلی، طرح‌بندی‌ها، یادداشت‌ها و نظرات اعمال می‌شود؟**

بله. جایگزینی بر تمام اشیاء ارائه که از قلم اصلی استفاده می‌کنند تأثیر می‌گذارد، از جمله اسلایدهای اصلی و یادداشت‌ها؛ نظرات نیز بخشی از سند هستند و توسط موتور قلم مدنظر قرار می‌گیرند.

**آیا قلم داخل اشیای OLE توکار (مثلاً Excel) تغییر می‌کند؟**

خیر. [محتوای OLE](/slides/fa/php-java/manage-ole/) توسط برنامهٔ خود آن کنترل می‌شود. جایگزینی در ارائه داده‌های داخلی OLE را دوباره قالب‌بندی نمی‌کند؛ ممکن است به‌صورت تصویر یا به‌عنوان محتوای قابل ویرایش خارجی نمایش داده شود.

**آیا می‌توانم قلم را فقط در بخشی از ارائه (بر حسب اسلایدها یا مناطق) جایگزین کنم؟**

جایگزینی هدفمند امکان‌پذیر است اگر قلم را در سطح اشیاء/محدوده‌های مورد نیاز تغییر دهید به جای اعمال جایگزینی سراسری بر کل سند. منطق کلی انتخاب قلم هنگام رندر همانند قبل باقی می‌ماند.

**چگونه می‌توانم از پیش تعیین کنم که ارائه از چه قلم‌هایی استفاده می‌کند؟**

از [مدیریت قلم](https://reference.aspose.com/slides/fa/php-java/aspose.slides/fontsmanager/) ارائه استفاده کنید: این روش فهرستی از [خانواده‌های مورد استفاده](https://reference.aspose.com/slides/fa/php-java/aspose.slides/fontsmanager/getfonts/) و اطلاعات درباره [جایگزینی‌ها/قلم‌های «نامشخص»](https://reference.aspose.com/slides/fa/php-java/aspose.slides/fontsmanager/getsubstitutions/) را فراهم می‌کند که به برنامه‌ریزی جایگزینی کمک می‌کند.

**آیا جایگزینی قلم هنگام تبدیل به PDF/تصاویر کار می‌کند؟**

بله. در هنگام خروجی، Aspose.Slides همان [دنبالهٔ انتخاب/جایگزینی قلم](/slides/fa/php-java/font-selection-sequence/) را اعمال می‌کند، بنابراین یک جایگزینی که پیشاپیش انجام شده باشد در طول تبدیل لحاظ می‌شود.

**آیا نیاز است قلم هدف را در سیستم نصب کنم یا می‌توانم پوشه‌ای از قلم‌ها ضمیمه کنم؟**

نیازی به نصب قلم هدف در سیستم نیست: کتابخانه امکان [بارگذاری قلم‌های خارجی](/slides/fa/php-java/custom-font/) را از پوشه‌های کاربر برای استفاده در [رندر و خروجی](/slides/fa/php-java/convert-powerpoint/) فراهم می‌کند.

**آیا جایگزینی مشکل «توفو» (مربعات) به جای کاراکترها را رفع می‌کند؟**

فقط در صورتی که قلم هدف واقعاً گلیف‌های مورد نیاز را داشته باشد این مشکل رفع می‌شود. در غیر این صورت، برای پوشش کاراکترهای گمشده، [پیکربندی fallback](/slides/fa/php-java/fallback-font/) را انجام دهید.
