## **مقدمه**

در PowerPoint 2019، مایکروسافت تنظیم **Always Open Read-Only** را به عنوان یکی از گزینه‌های محافظت از ارائه‌ها معرفی کرد. ممکن است بخواهید از این تنظیم فقط‌خواندنی برای محافظت از یک ارائه استفاده کنید زمانی که

- می‌خواهید از ویرایش‌های تصادفی جلوگیری کنید و محتوای ارائه‌تان را ایمن نگه دارید. 
- می‌خواهید به افراد اطلاع دهید که ارائه‌ای که ارائه کرده‌اید نسخه نهایی است. 

پس از انتخاب گزینه **Always Open Read-Only** برای یک ارائه، وقتی کاربران ارائه را باز می‌کنند، توصیه **Read-Only** را می‌بینند و ممکن است پیام زیر را مشاهده کنند: *برای جلوگیری از تغییرات تصادفی، نویسنده این فایل را برای باز شدن به‌صورت فقط‌خواندنی تنظیم کرده است.*

توصیه فقط‌خواندنی یک مانع ساده اما مؤثر است که از ویرایش جلوگیری می‌کند زیرا کاربران باید کاری انجام دهند تا قبل از امکان ویرایش، این توصیه را حذف کنند. اگر نمی‌خواهید کاربران تغییراتی در ارائه ایجاد کنند و بخواهید این موضوع را به‌صورت مؤدبانه بیان کنید، توصیه فقط‌خواندنی می‌تواند گزینه مناسبی باشد. 

> اگر یک ارائه با حفاظت **Read-Only** در یک نسخه قدیمی‌تر از Microsoft PowerPoint که این قابلیت جدید را پشتیبانی نمی‌کند باز شود، توصیه **Read-Only** نادیده گرفته می‌شود (ارائه به‌صورت معمولی باز می‌شود).

## **اعمال حالت فقط‌خواندنی**

Aspose.Slides for PHP via Java به شما امکان می‌دهد یک ارائه را به **Read-Only** تنظیم کنید، به این معنی که کاربران (پس از باز کردن ارائه) توصیه **Read-Only** را می‌بینند. این نمونه کد نشان می‌دهد چگونه یک ارائه را با Aspose.Slides به **Read-Only** تنظیم کنید:

```php
  $pres = new Presentation();
  try {
    $pres->getProtectionManager()->setReadOnlyRecommended(true);
    $pres->save("ReadOnlyPresentation.pptx", SaveFormat::Pptx);
  } finally {
    if (!java_is_null($pres)) {
      $pres->dispose();
    }
  }
```

{{% alert color="primary" %}} 

**توجه**: توصیه **Read-Only** صرفاً برای جلوگیری از ویرایش یا توقف تغییرات ناخواسته در یک ارائه PowerPoint است. اگر فردی که می‌داند چه کاری انجام می‌دهد تصمیم به ویرایش ارائه شما بگیرد، می‌تواند به سادگی تنظیم فقط‌خواندنی را حذف کند. اگر واقعاً نیاز به جلوگیری از ویرایش غیرمجاز دارید، بهتر است از [محافظت‌های سخت‌گیرانه‌تری که شامل رمزگذاری‌ها و گذرواژه‌ها می‌شوند](https://docs.aspose.com/slides/fa/php-java/password-protected-presentation/) استفاده کنید.

## **سوالات متداول**

**«Read-Only recommended» چه تفاوتی با محافظت کامل با گذرواژه دارد؟**

«Read-Only recommended» فقط یک پیشنهاد برای باز کردن فایل در حالت فقط‌خواندنی نشان می‌دهد و به راحتی می‌توان از آن عبور کرد. [Password protection](/slides/fa/php-java/password-protected-presentation/) در واقع باز کردن یا ویرایش را محدود می‌کند و زمانی مناسب است که نیاز به کنترل‌های امنیتی واقعی دارید.

**آیا می‌توان «Read-Only recommended» را با واترمارک‌ها ترکیب کرد تا ویرایش‌ها بیشتر دلسرد شوند؟**

بله. این توصیه می‌تواند همراه با [watermarks](/slides/fa/php-java/watermark/) به‌عنوان یک عامل دلسردکننده بصری استفاده شود؛ این دو مکانیزم جداگانه هستند و به‌خوبی با هم کار می‌کنند.

**آیا یک ماکرو یا ابزار خارجی هنوز می‌تواند فایل را وقتی توصیه فعال است، تغییر دهد؟**

بله. این توصیه مانعی برای تغییرات برنامه‌نویسی نمی‌شود. برای جلوگیری از ویرایش‌های خودکار، از [passwords and encryption](/slides/fa/php-java/password-protected-presentation/) استفاده کنید.

**«Read-Only recommended» چگونه به متدهای «isEncrypted» و «isWriteProtected» مرتبط است؟**

این‌ها سیگنال‌های متفاوتی هستند. «Read-Only recommended» یک اعلان نرم و اختیاری است؛ [isWriteProtected](https://reference.aspose.com/slides/fa/php-java/aspose.slides/protectionmanager/iswriteprotected/) و [isEncrypted](https://reference.aspose.com/slides/fa/php-java/aspose.slides/protectionmanager/isencrypted/) محدودیت‌های واقعی نوشتن یا خواندن را نشان می‌دهند که به گذرواژه یا رمزگذاری وابسته هستند.
