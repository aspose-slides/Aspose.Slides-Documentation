## **مقدمه**

Aspose.Slides ابزارهای جامع برای تنظیم اندازه اسلاید و نسبت عرض به ارتفاع در ارائه‌های PowerPoint فراهم می‌کند که برای چاپ و نمایش روی صفحه نمایش هر دو حیاتی هستند.  

اندازه‌ها و نسبت‌های رایج اسلاید:

- **استاندارد (نسبت 4:3)**: مناسب برای صفحه‌نمایش‌ها و دستگاه‌های قدیمی.  
- **واید اسکرین (نسبت 16:9)**: برای پروژکتور‌ها و نمایشگرهای مدرن توصیه می‌شود.  

برای حفظ یکپارچگی در سراسر ارائه، یک اندازه اسلاید و نسبت عرض به ارتفاع یکسان برای تمام اسلایدها اعمال می‌شود. برای دریافت بهترین نتایج، ابعاد اسلاید را در ابتدای فرآیند ایجاد ارائه تنظیم کنید تا از بروز مشکلات جلوگیری شود.  

{{% alert color="primary" %}} 
به طور پیش‌فرض، ارائه‌های ایجاد شده با Aspose.Slides از نسبت استاندارد 4:3 استفاده می‌کنند.  
{{% /alert %}}

## **تغییر اندازه اسلاید در ارائه‌ها**

این کد نمونه نشان می‌دهد چگونه با استفاده از Aspose.Slides اندازه اسلاید را در یک ارائه تغییر دهید:

```php
  $pres = new Presentation("pres-4x3-aspect-ratio.pptx");
  try {
    $pres->getSlideSize()->setSize(SlideSizeType::OnScreen16x9, SlideSizeScaleType::DoNotScale);
    $pres->save("pres-4x3-aspect-ratio.pptx", SaveFormat::Pptx);
  } finally {
    if (!java_is_null($pres)) {
      $pres->dispose();
    }
  }
```

## **مشخص کردن اندازه‌های سفارشی اسلاید در ارائه‌ها**

اگر اندازه‌های رایج اسلاید (4:3 و 16:9) برای کار شما مناسب نیستند، ممکن است بخواهید از یک اندازه خاص یا منحصر به‌فرد استفاده کنید. به‌عنوان مثال، اگر قصد دارید اسلایدهای تمام‌ابعاد را از ارائه خود روی یک طرح صفحه سفارشی چاپ کنید یا ارائه خود را روی انواع خاصی از نمایشگرها نمایش دهید، احتمالاً از تنظیم اندازه سفارشی برای ارائه خود بهره‌مند خواهید شد.  

این کد نمونه نشان می‌دهد چگونه از Aspose.Slides برای PHP از طریق Java برای تعیین اندازه سفارشی اسلاید در یک ارائه استفاده کنید:

```php
  $pres = new Presentation("pres.pptx");
  try {
    $pres->getSlideSize()->setSize(780, 540, SlideSizeScaleType::DoNotScale);// اندازه کاغذ A4

    $pres->save("pres-a4-slide-size.pptx", SaveFormat::Pptx);
  } finally {
    if (!java_is_null($pres)) {
      $pres->dispose();
    }
  }
```

## **مدیریت محتوای اسلاید پس از تغییر اندازه**

پس از تغییر اندازه اسلاید برای یک ارائه، محتوای اسلایدها (مانند تصاویر یا اشیاء) ممکن است مخدوش شوند. به‌صورت پیش‌فرض، اشیاء به‌صورت خودکار برای سازگاری با اندازه جدید اسلاید تغییر اندازه می‌یابند. با این حال، هنگام تغییر اندازه اسلاید ارائه، می‌توانید تنظیمی را مشخص کنید که نحوهٔ رفتار Aspose.Slides با محتوای اسلایدها را تعیین می‌کند.  

بسته به آنچه می‌خواهید انجام دهید یا به دست آورید، می‌توانید از هر یک از این تنظیمات استفاده کنید:

- `DoNotScale`

  اگر نمی‌خواهید اشیاء روی اسلایدها تغییر اندازه یابند، از این تنظیم استفاده کنید.

- `EnsureFit`

  اگر می‌خواهید به یک اندازهٔ اسلاید کوچکتر مقیاس بدهید و نیاز دارید Aspose.Slides اشیاء اسلایدها را کوچک کند تا همهٔ آنها در اسلایدها جا بگیرند (بدین ترتیب از از دست رفتن محتوا جلوگیری می‌کنید)، از این تنظیم استفاده کنید.

- `Maximize`

  اگر می‌خواهید به یک اندازهٔ اسلاید بزرگتر مقیاس بدهید و نیاز دارید Aspose.Slides اشیاء اسلایدها را بزرگ کند تا نسبت به اندازهٔ جدید اسلاید متناسب شوند، از این تنظیم استفاده کنید.

این کد نمونه نشان می‌دهد چگونه هنگام تغییر اندازهٔ اسلاید یک ارائه از تنظیم `Maximize` استفاده کنید:

```php
  $pres = new Presentation("pres.pptx");
  try {
    $pres->getSlideSize()->setSize(SlideSizeType::Ledger, SlideSizeScaleType::Maximize);
  } finally {
    if (!java_is_null($pres)) {
      $pres->dispose();
    }
  }
```

## **پرسش‌های متداول**

**آیا می‌توانم اندازه سفارشی اسلاید را با واحدهایی متفاوت از اینچ (مثلاً پوینت یا میلی‌متر) تعیین کنم؟**  

بله. Aspose.Slides به‌صورت داخلی از پوینت استفاده می‌کند؛ یک پوینت برابر است با 1/72 اینچ. می‌توانید هر واحدی (مانند میلی‌متر یا سانتی‌متر) را به پوینت تبدیل کرده و مقادیر تبدیل شده را برای تعیین عرض و ارتفاع اسلید به کار ببرید.  

**آیا اندازهٔ سفارشی بسیار بزرگ اسلاید بر عملکرد و مصرف حافظه هنگام رندرینگ تأثیر می‌گذارد؟**  

بله. ابعاد بزرگ‌تر اسلاید (به پوینت) به‌همراه مقیاس بالاتر رندرینگ منجر به مصرف حافظه بیشتر و زمان پردازش طولانی‌تر می‌شود. سعی کنید اندازهٔ اسلاید معقولی انتخاب کنید و مقیاس رندرینگ را تنها در صورت نیاز برای رسیدن به کیفیت خروجی مطلوب تنظیم کنید.  

**آیا می‌توانم یک اندازهٔ اسلاید غیر استاندارد تعریف کنم و سپس اسلایدها را از ارائه‌هایی که اندازه‌های متفاوتی دارند، ادغام کنم؟**  

در حالی که اندازه‌های اسلاید متفاوت هستند، نمی‌توانید [ادغام ارائه‌ها](/slides/fa/php-java/merge-presentation/) انجام دهید — ابتدا یک ارائه را بازنشانی کنید تا با دیگری هماهنگ شود. هنگام تغییر اندازهٔ اسلاید، می‌توانید با استفاده از گزینهٔ [SlideSizeScaleType](https://reference.aspose.com/slides/fa/php-java/aspose.slides/slidesizescaletype/) تعیین کنید محتویات موجود چگونه مدیریت شوند. پس از هم‌ترازی اندازه‌ها، می‌توانید اسلایدها را ادغام کنید در حالی که قالب‌بندی حفظ می‌شود.  

**آیا می‌توانم برای اشکال فردی یا نواحی خاصی از یک اسلاید تصویر بندانگشتی ایجاد کنم و آیا این تصاویر اندازهٔ جدید اسلاید را رعایت می‌کنند؟**  

بله. Aspose.Slides می‌تواند تصاویر بندانگشتی برای [کل اسلایدها](https://reference.aspose.com/slides/fa/php-java/aspose.slides/slide/#getImage) و همچنین برای [اشکال انتخابی](https://reference.aspose.com/slides/fa/php-java/aspose.slides/shape/#getImage) تولید کند. تصاویر حاصل اندازهٔ فعلی اسلاید و نسبت عرض به ارتفاع را منعکس می‌کنند و چارچوب و هندسهٔ یک‌دست را تضمین می‌نمایند.
