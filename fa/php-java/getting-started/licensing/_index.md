## **ارزیابی Aspose.Slides**
می‌توانید به راحتی Aspose.Slides را برای ارزیابی دانلود کنید. بسته ارزیابی دقیقاً همان بسته خریداری‌شده است. نسخه ارزیابی پس از افزودن چند خط کد برای اعمال مجوز، به سادگی مجوزدار می‌شود.

## **محدودیت نسخه ارزیابی**
نسخه ارزیابی Aspose.Slides (بدون مشخص کردن مجوز) تمام قابلیت‌های محصول را فراهم می‌کند، اما یک واترمارک ارزیابی در بالای سند هنگام باز کردن و ذخیره‌سازی درج می‌کند. همچنین هنگام استخراج متن از اسلایدهای ارائه، به یک اسلاید محدود می‌شوید.

{{% alert color="primary" %}} 

اگر می‌خواهید Aspose.Slides را بدون محدودیت‌های نسخه ارزیابی تست کنید، می‌توانید **مجوز موقت ۳۰ روزه** درخواست کنید. برای اطلاعات بیشتر به [چگونه یک مجوز موقت دریافت کنیم؟](https://purchase.aspose.com/temporary-license) مراجعه کنید.

{{% /alert %}} 

## **دربارهٔ مجوز**
می‌توانید به راحتی یک نسخه ارزیابی Aspose.Slides برای PHP via Java را از [صفحه دانلود](https://packagist.org/packages/aspose/slides) دریافت کنید. نسخه ارزیابی دقیقاً **همان قابلیت‌ها** را که نسخه مجوزدار Aspose.Slides دارد، ارائه می‌دهد. علاوه بر این، پس از خرید یک مجوز و افزودن چند خط کد برای اعمال مجوز، نسخه ارزیابی به سادگی مجوزدار می‌شود.

مجوز یک فایل XML متنی ساده است که شامل جزئیاتی مانند نام محصول، تعداد توسعه‌دهندگانی که مجوز دارند، تاریخ انقضای اشتراک و غیره می‌باشد. این فایل به‌صورت دیجیتالی امضا شده است، بنابراین آن را تغییر ندهید. حتی افزودن یک خط جدید ناخواسته به محتویات فایل نیز موجب نامعتبر شدن آن می‌شود.

برای رفع محدودیت‌های مرتبط با نسخه ارزیابی، باید قبل از استفاده از **Aspose.Slides** یک مجوز تنظیم کنید. شما فقط یک بار در هر برنامه یا فرآیند نیاز به تنظیم مجوز دارید.

{{% alert color="primary" %}} 

ممکن است بخواهید به [مجوز متری](https://docs.aspose.com/slides/fa/php-java/metered-licensing/) نگاهی بیندازید.

{{% /alert %}}

### **تنظیم مجوز در Aspose.Slides برای PHP via Java**

مجوزها می‌توانند از این مکان‌ها اعمال شوند:

* مسیر واضح
* جریان
* به‌عنوان مجوز متری – یک مکانیزم جدید مجوزدهی

{{% alert color="primary" %}}

از روش **setLicense** برای مجوزدهی یک مؤلفه استفاده کنید.

در حالی‌که فراخوانی‌های متعدد **setLicense** ضرری ندارند، اما منبع (پردازنده) را هدر می‌دهند.

{{% /alert %}}

{{% alert color="warning" %}}

مجوزهای جدید می‌توانند Aspose.Slides را فقط از نسخه ۲۱.۴ به بعد فعال کنند. نسخه‌های قدیمی‌تر از سیستم مجوزدهی متفاوتی استفاده می‌کنند و این مجوزها را شناسایی نمی‌کنند.

{{% /alert %}}

#### **اعمال مجوز با استفاده از فایل**

این قطعه کد برای تنظیم یک فایل مجوز استفاده می‌شود:

**PHP**

```php
<?php
require_once("http://localhost:8080/JavaBridge/java/Java.inc");
require_once("lib/aspose.slides.php");

use aspose\slides\License;

$license = new License();
$license->setLicense("Aspose.Slides.lic");
?>
```

هنگام فراخوانی متد setLicense، نام مجوز باید همانند نام فایل مجوز شما باشد. به عنوان مثال، می‌توانید نام فایل مجوز را به "Aspose.Slides.lic.xml" تغییر دهید. سپس در کد خود باید نام جدید مجوز (Aspose.Slides.lic.xml) را به متد setLicense پاس کنید.

#### **اعمال مجوز از یک جریان**

این قطعه کد برای اعمال مجوز از یک جریان استفاده می‌شود:

```php
<?php
require_once("http://localhost:8080/JavaBridge/java/Java.inc");
require_once("lib/aspose.slides.php");

use aspose\slides\License;

$license = new License();
$license->setLicense($stream);

```

## **پرسش‌های متداول**

**آیا می‌توانم مجوز را در یک محیط کاملاً آفلاین (بدون دسترسی به اینترنت) اعمال کنم؟**

بله. اعتبارسنجی مجوز به‌صورت محلی با استفاده از فایل مجوز انجام می‌شود؛ نیازی به اتصال اینترنتی نیست.

**بعد از پایان دوره یک‌سالهٔ اشتراک چه اتفاقی می‌افتد؟ آیا کتابخانه متوقف می‌شود؟**

خیر. مجوز دائمی است: می‌توانید از نسخه‌های منتشر شده پیش از تاریخ پایان اشتراک خود استفاده کنید؛ فقط بدون تجدید، امکان استفاده از نسخه‌های جدیدتر وجود ندارد.
