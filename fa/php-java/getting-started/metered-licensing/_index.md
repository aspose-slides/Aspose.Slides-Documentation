## **مقدمه**

مجوز متری یک مکانیزم صدور مجوز است که می‌تواند همراه با روش‌های موجود مجوزدهی استفاده شود. اگر می‌خواهید بر اساس استفاده‌تان از ویژگی‌های API Aspose.Slides هزینه دریافت کنید، مجوز متری را انتخاب می‌کنید.

## **اعمال کلیدهای متری**

هنگامی که یک مجوز متری خریداری می‌کنید، کلیدها (و نه فایل مجوز) دریافت می‌کنید. این کلید متری می‌تواند با استفاده از کلاس [Metered](https://reference.aspose.com/slides/fa/php-java/aspose.slides/metered/) که Aspose برای عملیات متری ارائه داده است، اعمال شود. برای جزئیات بیشتر، به [Metered Licensing FAQ](https://purchase.aspose.com/faqs/licensing/metered) مراجعه کنید.

1. یک نمونه از کلاس [Metered](https://reference.aspose.com/slides/fa/php-java/aspose.slides/metered/) ایجاد کنید.
2. کلیدهای عمومی و خصوصی خود را به متد [setMeteredKey](https://reference.aspose.com/slides/fa/php-java/aspose.slides/metered/#setMeteredKey-java.lang.String-java.lang.String-) پاس دهید.
3. مقداری پردازش انجام دهید (وظایف را اجرا کنید).
4. متد [getConsumptionQuantity](https://reference.aspose.com/slides/fa/php-java/aspose.slides/metered/#getConsumptionQuantity--) از کلاس `Metered` را فراخوانی کنید.

باید مقدار/تعداد درخواست‌های API که تا الآن مصرف کرده‌اید را مشاهده کنید.

این نمونه کد نشان می‌دهد چگونه از مجوز متری استفاده کنید:

```php
// یک نمونه از کلاس Metered ایجاد می‌کند
$metered = new Metered();

try {
    // کلیدهای عمومی و خصوصی را به شیء Metered می‌گذارد
    $metered->setMeteredKey("<valid pablic key>", "<valid private key>");

    // مقدار مقدار مصرف شده را قبل از تماس‌های API دریافت می‌کند
    $amountBefore = Metered::getConsumptionQuantity();
    echo("Amount consumed before: " . $amountBefore);

    // کاری با API Aspose.Slides اینجا انجام دهید
    // ...

    // مقدار مقدار مصرف شده را پس از تماس‌های API دریافت می‌کند
    $amountAfter = Metered::getConsumptionQuantity();
    echo("Amount consumed after: " . $amountAfter);
} catch (JavaException $ex) {
  $ex->printStackTrace();
}
```

برای استفاده از مجوز متری، به یک اتصال اینترنتی پایدار نیاز دارید زیرا مکانیزم مجوزدهی برای تعامل مستمر با سرویس‌های ما و انجام محاسبات از اینترنت استفاده می‌کند.

## **سؤالات متداول**

**آیا می‌توانم یک مجوز متری را همراه با یک مجوز عادی (دائمی یا موقت) در همان برنامه استفاده کنم؟**

بله. مجوز متری یک مکانیزم اضافی است که می‌تواند همراه با [روش‌های مجوزدهی](/slides/fa/php-java/licensing/) موجود استفاده شود. شما هنگام شروع برنامه، مکانیزم مورد نظر را انتخاب می‌کنید.

**دقتاً چه چیزی تحت یک مجوز متری به‌عنوان مصرف شمارش می‌شود: عملیات‌ها یا فایل‌ها؟**

مصرف بر اساس استفاده از API شمرده می‌شود، به این معنی که تعداد درخواست‌ها یا عملیات‌ها. می‌توانید مصرف فعلی را از طریق [روش‌های ردیابی مصرف](https://reference.aspose.com/slides/fa/php-java/aspose.slides/metered/) به‌دست آورید.

**آیا مجوز متری برای میکروسرویس‌ها و محیط‌های سرورلس که نمونه‌ها به‌طور مکرر ریستارت می‌شوند مناسب است؟**

بله. از آنجا که حسابداری در سطح فراخوانی API انجام می‌شود، سناریوهای با شروع سرد مکرر سازگار هستند، به شرط آنکه دسترسی شبکه‌ای پایدار برای محاسبات متری فراهم باشد.

**آیا عملکرد کتابخانه هنگام استفاده از مجوز متری نسبت به مجوز دائمی متفاوت است؟**

خیر. این صرفاً درباره مکانیزم مجوزدهی و صورتحساب است؛ قابلیت‌های محصول یکسان هستند.

**مجوز متری چگونه با نسخه آزمایشی و مجوز موقت مرتبط است؟**

نسخه آزمایشی دارای محدودیت‌ها و علامت‌های آبکی است، [مجوز موقت](https://purchase.aspose.com/temporary-license/) محدودیت‌ها را به مدت ۳۰ روز حذف می‌کند، و مجوز متری محدودیت‌ها را حذف کرده و بر اساس استفاده واقعی هزینه دریافت می‌کند.

**آیا می‌توانم بودجه را با واکنش خودکار وقتی آستانه مصرف عبور می‌کند، کنترل کنم؟**

بله. یک روش معمول این است که به‌طور دوره‌ای مصرف فعلی را از طریق [روش‌های ردیابی](https://reference.aspose.com/slides/fa/php-java/aspose.slides/metered/) خوانده و محدودیت‌ها یا هشدارهای خود را در سطح برنامه یا مانیتورینگ پیاده‌سازی کنید.
