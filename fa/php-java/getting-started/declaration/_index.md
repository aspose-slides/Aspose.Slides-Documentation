{{% alert color="primary" %}} 

تمام اجزای Aspose Java به مجموعهٔ مجوز Full Trust نیاز دارند. دلیل این است که اجزای Aspose Java برای انجام برخی عملیات مثل تجزیهٔ قلم‌ها و غیره، نیاز به دسترسی به تنظیمات رجیستری و فایل‌های سیستمی خارج از دایرکتوری مجازی دارند. علاوه بر این، اجزای Aspose Java بر پایهٔ کلاس‌های اصلی سیستم Java ساخته شده‌اند که در بسیاری از موارد نیز به مجموعهٔ مجوز Full Trust نیاز دارند. 

{{% /alert %}} 

ارائه‌دهندگان خدمات اینترنتی که چندین برنامه از شرکت‌های مختلف را میزبانی می‌کنند، عمدتاً سطح امنیتی Medium Trust را اعمال می‌نمایند: 

- OleDbPermission در دسترس نیست. این به این معناست که نمی‌توانید از فراهم‌کنندهٔ دادهٔ OLE DB مدیریت‌شدهٔ ADO.NET برای دسترسی به پایگاه‌های داده استفاده کنید.  
- EventLogPermission در دسترس نیست. این به این معناست که نمی‌توانید به گزارش رویدادهای ویندوز دسترسی پیدا کنید.  
- ReflectionPermission در دسترس نیست. این به این معناست که نمی‌توانید از Reflection استفاده کنید.  
- RegistryPermission در دسترس نیست. این به این معناست که نمی‌توانید به رجیستری دسترسی داشته باشید.  
- WebPermission محدود شده است. این به این معناست که برنامهٔ شما فقط می‌تواند با آدرسی یا محدوده‌ای از آدرس‌ها که در عنصر <trust> تعریف می‌کنید، ارتباط برقرار کند.  
- FileIOPermission محدود شده است. این به این معناست که فقط می‌توانید به فایل‌هایی در سلسله‌مراتب دایرکتوری مجازی برنامهٔ خود دسترسی داشته باشید.  

{{% alert color="primary" %}} 

به دلایل مذکور، اجزای Aspose Java نمی‌توانند بر روی سرورهایی که مجموعهٔ مجوزی غیر از Full Trust صادر می‌کنند، استفاده شوند. 

{{% /alert %}}
