## **Was ist das Open XML SDK?**
Manchmal erhalten wir diese Frage: *Warum sollten wir Aspose-Produkte statt des kostenlosen Open XML SDK verwenden?* 

Wir finden es einfach, diese Frage in Bezug auf Funktionen und Merkmale zu beantworten. 

Laut der [MSDN Library](https://docs.microsoft.com/en-us/office/open-xml/open-xml-sdk) ist das Open XML SDK wie folgt definiert: 

> "Das Open XML SDK 2.0 vereinfacht die Aufgabe, Open XML Pakete und die zugrunde liegenden Open XML Schema Elemente innerhalb eines Pakets zu manipulieren. Das Open XML SDK 2.0 fasst viele gängige Aufgaben, die Entwickler mit Open XML Paketen ausführen, zusammen, sodass Sie komplexe Vorgänge mit nur wenigen Codezeilen erledigen können. OOXML Dokumente sind im Wesentlichen gezippte XML Dateien und das Open XML SDK ist eine Sammlung von Klassen, die Ihnen ermöglicht, mit dem Inhalt von OOXML Dokumenten typensicher zu arbeiten. Das bedeutet, anstatt eine Datei zu entzippen, um XML zu extrahieren, das XML in einen DOM Baum zu laden und direkt mit XML Elementen und Attributen zu arbeiten, stellt das Open XML SDK Klassen zur Verfügung, die das erledigen."

## **Was ist Aspose.Slides?**
Aspose.Slides ist eine Klassenbibliothek, die Anwendungen ermöglicht, die folgenden Präsentationsverarbeitungsaufgaben auszuführen: 

- Programmierung mit einem Präsentations-Objektmodell.
- Hochwertige Konvertierungen, die alle gängigen unterstützten PowerPoint‑Präsentationsformate umfassen, einschließlich Konvertierung zu PDF, XPS, TIFF und Drucken.
- Erzeugen von Folien‑Thumbnails in bekannten Formaten wie PNG, JPEG und BMP sowie Export von Folien nach SVG.
- Erstellen von Präsentationen von Grund auf oder durch Kombinieren von Elementen aus einem oder mehreren Dokumenten.
- Hinzufügen von Animationen, OLE‑Frames, Tabellen, Erstellen und Verwalten von Diagrammen.
- Steuerung (umfangreiche Kontrolle) und Verwaltung der Textformatierung auf Ebene von TextFrames, Absätzen und Portionen. 

Für weitere Details zu den verfügbaren Funktionen siehe die [Aspose.Slides Features](/slides/de/net/product-overview/) Seite.

## **Fazit**
Open XML SDK und Aspose.Slides stehen nicht in direkter Konkurrenz, da sie deutlich unterschiedliche Bedürfnisse adressieren und unterschiedliche Zielgruppen ansprechen. 

{{% alert color="primary" %}} 

Open XML SDK ist eine Klassenbibliothek, die eine typensichere Methode zum Arbeiten mit OOXML‑Dokumenten bietet, während Aspose.Slides eine äußerst nützliche Bibliothek zur Präsentationsverarbeitung ist, die umfassende Unterstützung für fast alle Microsoft PowerPoint‑Dateiformate bereitstellt. 

{{% /alert %}} 

Wenn Ihr Workflow eine einfache Programmieroperation an einem PPTX‑Dokument ist, kann Open XML SDK eine gute Wahl sein. Mit Open XML SDK sollten Sie in der Lage sein, einfache Aufgaben wie das Erzeugen eines einfachen PPTX‑Dokuments oder das Entfernen von Kommentaren, Kopf‑/Fußzeilen, das Extrahieren von Bildern usw. auszuführen. Bestimmte Aufgaben können mit Open XML SDK durchgeführt werden, jedoch nicht mit Aspose.Slides. Beispielsweise sollten Sie Open XML SDK verwenden, wenn Sie direkten Zugriff auf die XML‑Elemente und -Attribute eines OOXML‑Dokuments benötigen. 

Wenn Sie komplexe Aufgaben an Dokumenten ausführen müssen – wie die unten aufgeführten Aufgaben – ist Aspose.Slides die beste Option. 

- Operationen, die ältere PowerPoint‑Formate (und auch PPTX) betreffen.
- Kopieren oder Klonen von Shapes innerhalb von Folien auf eine Weise, die Objekte, Stile und andere Formatierungselemente angemessen kombiniert.
- Ersetzen von formatiertem oder unformatiertem Text.
- Anwenden von Animationen und Verwenden von Verbindern mit Shapes.
- Konvertieren eines Dokuments zu PDF, TIFF oder XPS, sodass es wie von Microsoft PowerPoint konvertiert erscheint.
- Entwickeln einer .NET‑ oder Java‑Anwendung sowohl für Desktop‑ als auch für webbasierte Umgebungen.
