## **PPT-zu-PPTX-Konvertierung**
### **Über die PPT-zu-PPTX-Konvertierung**
Aspose.Slides unterstützt jetzt die Konvertierung von PPT nach PPTX.
### **PPT nach PPTX mit Aspose.Slides konvertieren**
Aspose.Slides für .NET bietet teilweise Unterstützung für die Konvertierung von Präsentationen im PPT‑Dateiformat in das PPTX‑Dateiformat. Da die Unterstützung für diese Präsentationskonvertierungsfunktion gerade erst in Aspose.Slides für .NET eingeführt wurde, ist sie derzeit nur eingeschränkt verfügbar und funktioniert nur für einfache Präsentationen. Der Hauptvorteil, den die Aspose.Slides für .NET API‑Bibliothek bei der Konvertierung einer PPT‑Präsentation in das PPTX‑Format bietet, ist die einfache Handhabung der API zur Erreichung des gewünschten Ziels. Bitte gehen Sie zu diesem [link](/slides/de/net/convert-presentation/) im Abschnitt Codebeispiele für weitere Details. Der folgende Abschnitt zeigt deutlich, welche Funktionen bei der Konvertierung von PPT‑Präsentationen in PPTX‑Präsentationen unterstützt bzw. nicht unterstützt werden.

## **Präsentation zu PDF-Konvertierung**
### **Über PDF**
Das [Portable Document Format](https://en.wikipedia.org/wiki/PDF) ist ein Dateiformat, das von Adobe Systems erstellt wurde, um den Austausch von Dokumenten zwischen verschiedenen Organisationen zu ermöglichen. Der Zweck dieses Formats ist, dass der Inhalt von Dokumenten so dargestellt werden kann, dass das visuelle Erscheinungsbild nicht von der Plattform abhängt, auf der es angezeigt wird.
### **PDF in Aspose.Slides für .NET**
[PDF ](https://docs.fileformat.com/pdf/) ist ein Dateiformat, das von Adobe Systems erstellt wurde, um den Austausch von Dokumenten zwischen verschiedenen Organisationen zu ermöglichen. Der Zweck dieses Formats ist, dass das visuelle Erscheinungsbild der Dokumentinhalte nicht von der Plattform abhängt, auf der es angezeigt wird.  

Jedes Präsentationsdokument, das in Aspose.Slides für .NET geladen werden kann, lässt sich in ein PDF‑Dokument konvertieren. Sie können die Präsentationsdokumente direkt mit der Aspose.Slides für .NET‑Komponente in PDF exportieren. Dafür benötigen Sie keine anderen Drittanbieter‑Bibliotheken oder die Aspose.PDF‑Komponente.

## **Präsentation zu XPS-Konvertierung**
### **Über XPS**
Die [XML Paper Specification](https://en.wikipedia.org/wiki/Open_XML_Paper_Specification) ist eine Seitenspezifikationssprache und ein festes Dokumentformat, das ursprünglich von Microsoft entwickelt wurde. Wie PDF ist XPS ein festes Layout‑Dokumentformat, das entwickelt wurde, um die Treue zum Originaldokument zu bewahren und ein geräteunabhängiges Erscheinungsbild zu bieten.
### **Präsentation zu XPS mit Aspose.Slides konvertieren**
Jedes Präsentationsdokument, das von Aspose.Slides für .NET geladen werden kann, lässt sich in das XPS‑Format konvertieren. Aspose.Slides für .NET verwendet die hochpräzise Seitenlayout‑ und Rendering‑Engine, um Ausgaben im festgelegten XPS‑Dokumentformat zu erzeugen. Erwähnenswert ist, dass Aspose.Slides für .NET XPS direkt erzeugt, ohne die mit .NET Framework 3.5 ausgelieferten Windows Presentation Foundation (WPF)-Klassen zu verwenden, wodurch Aspose.Slides für .NET XPS‑Dokumente auch auf Maschinen mit .NET Framework‑Versionen vor 3.5 erstellen kann. Sie können mehr darüber erfahren, wie Sie Präsentationsdokumente mit Aspose.Slides für .NET in XPS‑Dokumente exportieren, in [diesem Thema](/slides/de/net/convert-powerpoint-ppt-and-pptx-to-microsoft-xps-document/).

## **Präsentation zu TIFF-Konvertierung**
### **Über TIFF**
Das [Tagged Image File Format](https://en.wikipedia.org/wiki/TIFF) , das für das Speichern mehrerer Bilder in einem Dokument bekannt ist, wurde ursprünglich von Aldus erstellt. Dieses Format wird von Scan‑, Fax‑ und anderen Bildbearbeitungsanwendungen breit unterstützt.
### **Präsentation zu TIFF mit Aspose.Slides konvertieren**
Jedes Dokument, das in Aspose.Slides für .NET geladen werden kann, lässt sich ebenfalls direkt mit Aspose.Slides für .NET in ein TIFF‑Dokument konvertieren, wodurch die Notwendigkeit einer Drittanbieter‑Komponente entfällt. Zusätzlich können Sie optional die Größe der Bilder im resultierenden TIFF‑Dokument festlegen. Informationen zum Export von Präsentationsdokumenten in TIFF‑Dokumente mit Aspose.Slides für .NET finden Sie in [diesem Thema](/slides/de/net/convert-powerpoint-ppt-and-pptx-to-tiff/).

## **FAQ**

**Kann ich ausgeblendete Folien beim Export nach PDF/XPS einbeziehen?**

Ja. Der Export unterstützt das Einbeziehen ausgeblendeter Folien über die entsprechende Option in den [PDF](https://reference.aspose.com/slides/net/aspose.slides.export/pdfoptions/showhiddenslides/)/[XPS](https://reference.aspose.com/slides/net/aspose.slides.export/xpsoptions/showhiddenslides/)-Einstellungen.

**Wird das Speichern im PDF/A-Format (für die Archivierung) unterstützt?**

Ja, PDF/A‑Konformitätsstufen sind beim Export [verfügbar](https://reference.aspose.com/slides/net/aspose.slides.export/pdfcompliance/) (einschließlich A-2a/A-2b/A-2u und A-3a/A-3b).

**Was passiert mit Schriftarten bei der Konvertierung: werden sie eingebettet oder ersetzt?**

Es gibt flexible Optionen: Sie können alle Glyphen oder nur die verwendeten Teilmengen [einbetten](/slides/de/net/embedded-font/), eine [Fallback‑Schriftart](/slides/de/net/fallback-font/) angeben und das [Verhalten steuern](/slides/de/net/font-substitution/), wenn einer Schriftart bestimmte Stile fehlen.

**Wie kann ich die Qualität und Größe des resultierenden PDFs steuern?**

Optionen sind verfügbar für die [JPEG‑Qualität](https://reference.aspose.com/slides/net/aspose.slides.export/pdfoptions/jpegquality/), die [Textkomprimierung](https://reference.aspose.com/slides/net/aspose.slides.export/pdfoptions/textcompression/), und einen Schwellenwert für [ausreichende Auflösung](https://reference.aspose.com/slides/net/aspose.slides.export/pdfoptions/sufficientresolution/) bei Bildern, sowie einen Modus, der die [beste Kompression für Bilder](https://reference.aspose.com/slides/net/aspose.slides.export/pdfoptions/bestimagescompressionratio/) auswählt.

**Kann ich nur einen Folienbereich exportieren (z. B. 5–12)?**

Ja, der Export unterstützt die Auswahl eines Teilbereichs von Folien.

**Wird die Multi‑Core‑Verarbeitung mehrerer Dateien gleichzeitig unterstützt?**

Es ist zulässig, verschiedene Präsentationen parallel in separaten Prozessen zu verarbeiten. Wichtig: Das gleiche [Presentation]‑Objekt darf nicht von mehreren [Threads](/slides/de/net/multithreading/) geladen oder gespeichert werden.

**Gibt es Risiken beim Anwenden der Lizenz aus verschiedenen Threads?**

Ja, Aufrufe zum [Lizenz‑Setzen](/slides/de/net/licensing/) sind nicht threadsicher und erfordern eine Synchronisation.
