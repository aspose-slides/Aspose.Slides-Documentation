## **PPT-zu-PPTX-Konvertierung**
### **Über die PPT-zu-PPTX-Konvertierung**
Aspose.Slides unterstützt jetzt die Konvertierung von PPT zu PPTX.
### **PPT mit Aspose.Slides zu PPTX konvertieren**
Aspose.Slides für .NET bietet Teilunterstützung für die Konvertierung von Präsentationen im PPT-Dateiformat in das PPTX-Dateiformat. Da die Unterstützung für die genannte Präsentationskonvertierungsfunktion gerade erst in Aspose.Slides für .NET eingeführt wurde, ist sie momentan noch eingeschränkt und funktioniert nur für einfache Präsentationen. Der Hauptvorteil, den die Aspose.Slides für .NET API-Bibliothek bei der Konvertierung von PPT-Präsentationen in das PPTX-Format bietet, ist die einfache Verwendung der API, um das gewünschte Ergebnis zu erzielen. Bitte gehen Sie zu diesem [link](/slides/de/net/convert-presentation/) im Abschnitt Codebeispiele für weitere Details. Der folgende Abschnitt illustriert klar, welche Funktionen beim Konvertieren von PPT-Format-Präsentationen in PPTX-Format-Präsentationen unterstützt bzw. nicht unterstützt werden.

## **Präsentation zu PDF-Konvertierung**
### **Über PDF**
Die [Portable Document Format](https://en.wikipedia.org/wiki/PDF) ist ein Dateiformat, das von Adobe System erstellt wurde, um den Austausch von Dokumenten zwischen verschiedenen Organisationen zu ermöglichen. Der Zweck dieses Formats war es, zu gewährleisten, dass der Inhalt der Dokumente so dargestellt werden kann, dass das visuelle Erscheinungsbild nicht von der Plattform abhängt, auf der es angezeigt wird.
### **PDF in Aspose.Slides für .NET**
[PDF](https://docs.fileformat.com/pdf/) ist ein Dateiformat, das von Adobe System erstellt wurde, um den Austausch von Dokumenten zwischen verschiedenen Organisationen zu ermöglichen. Der Zweck dieses Formats war es, zu gewährleisten, dass das visuelle Erscheinungsbild der Dokumentinhalte nicht von der Plattform abhängt, auf der es angezeigt wird.

Jedes Präsentationsdokument, das in Aspose.Slides für .NET geladen werden kann, kann in ein PDF-Dokument konvertiert werden. Sie können Präsentationsdokumente direkt mit der Aspose.Slides für .NET-Komponente in PDF exportieren. Sie benötigen dafür keine anderen Drittanbieterbibliotheken oder die Aspose.PDF-Komponente.

## **Präsentation zu XPS-Konvertierung**
### **Über XPS**
Die [XML Parser Specification](https://en.wikipedia.org/wiki/Open_XML_Paper_Specification) ist eine Seitenbeschreibungssprache und ein festes Dokumentformat, das ursprünglich von Microsoft entwickelt wurde. Ähnlich wie PDF ist XPS ein festes Layout-Dokumentformat, das entwickelt wurde, um die Treue des Dokuments zu bewahren und ein geräteunabhängiges Dokumentenerscheinungsbild zu bieten.
### **Präsentation zu XPS mit Aspose.Slides konvertieren**
Jedes Präsentationsdokument, das von Aspose.Slides für .NET geladen werden kann, kann in das XPS-Format konvertiert werden. Aspose.Slides für .NET nutzt die hochpräzise Seitenlayout- und Rendering-Engine, um Ausgaben im festgelegten XPS-Dokumentformat zu erzeugen. Erwähnenswert ist, dass Aspose.Slides für .NET XPS direkt generiert, ohne von den Windows Presentation Foundation (WPF)-Klassen abhängig zu sein, die mit .NET Framework 3.5 geliefert werden, wodurch Aspose.Slides für .NET XPS-Dokumente auf Maschinen mit .NET Framework-Versionen vor Version 3.5 erstellen kann. Sie können mehr über den Export von Präsentationsdokumenten zu XPS-Dokumenten über Aspose.Slides für .NET in [diesem Thema](/slides/de/net/convert-powerpoint-to-xps/) erfahren.

## **Präsentation zu TIFF-Konvertierung**
### **Über TIFF**
Das [Tagged Image File Format](https://en.wikipedia.org/wiki/TIFF), das für das Speichern mehrerer Bilder in einem Dokument bekannt ist, wurde ursprünglich von Aldus erstellt. Dieses Format wird von Scan-, Fax- und anderen Bildbearbeitungsanwendungen breit unterstützt.
### **Präsentation zu TIFF mit Aspose.Slides konvertieren**
Jedes Dokument, das in Aspose.Slides für .NET geladen werden kann, kann ebenfalls direkt von Aspose.Slides für .NET in ein TIFF-Dokument konvertiert werden, wodurch die Notwendigkeit einer Drittanbieterkomponente entfällt. Darüber hinaus können Sie optional die Größe der Bilder im resultierenden TIFF-Dokument festlegen. Sie finden Informationen zum Export von Präsentationsdokumenten zu TIFF-Dokumenten über Aspose.Slides für .NET in [diesem Thema](/slides/de/net/convert-powerpoint-to-tiff/).

## **FAQ**

**Kann ich versteckte Folien beim Export nach PDF/XPS einbeziehen?**

Ja. Der Export unterstützt das Einbeziehen versteckter Folien über die entsprechende Option in den [PDF](https://reference.aspose.com/slides/net/aspose.slides.export/pdfoptions/showhiddenslides/)/[XPS](https://reference.aspose.com/slides/net/aspose.slides.export/xpsoptions/showhiddenslides/)-Einstellungen.

**Wird das Speichern im PDF/A-Format (für Archivierung) unterstützt?**

Ja, PDF/A-Konformitätsstufen sind [verfügbar](https://reference.aspose.com/slides/net/aspose.slides.export/pdfcompliance/) (einschließlich A-2a/A-2b/A-2u und A-3a/A-3b) während des Exports.

**Was passiert mit Schriftarten während der Konvertierung: werden sie eingebettet oder ersetzt?**

Es gibt flexible Optionen: Sie können [alle Glyphen oder nur die verwendeten Teilmengen einbetten](/slides/de/net/embedded-font/), eine [Fallback-Schriftart](/slides/de/net/fallback-font/) festlegen und das [Verhalten](/slides/de/net/font-substitution/) steuern, wenn einer Schriftart bestimmte Stile fehlen.

**Wie kann ich die Qualität und Größe des resultierenden PDFs steuern?**

Optionen stehen zur Verfügung für [JPEG-Qualität](https://reference.aspose.com/slides/net/aspose.slides.export/pdfoptions/jpegquality/), [Textkompression](https://reference.aspose.com/slides/net/aspose.slides.export/pdfoptions/textcompression/), und einen [ausreichenden Auflösungs]https://reference.aspose.com/slides/net/aspose.slides.export/pdfoptions/sufficientresolution/ Schwellenwert für Bilder, plus ein Modus, der die [beste Kompression für Bilder](https://reference.aspose.com/slides/net/aspose.slides.export/pdfoptions/bestimagescompressionratio/) auswählt.

**Kann ich nur einen Folienbereich exportieren (z. B. 5–12)?**

Ja, der Export unterstützt die Auswahl eines Teilbereichs von Folien.

**Wird die Multi-Core-Verarbeitung mehrerer Dateien gleichzeitig unterstützt?**

Es ist zulässig, verschiedene Präsentationen parallel in separaten Prozessen zu verarbeiten. Wichtig: Das gleiche [Präsentations](https://reference.aspose.com/slides/net/aspose.slides/presentation/)-Objekt darf nicht von [mehreren Threads](/slides/de/net/multithreading/) geladen oder gespeichert werden.

**Gibt es Risiken bei der Lizenzanwendung aus verschiedenen Threads?**

Ja, Aufrufe zum [Lizenzsetzen](/slides/de/net/licensing/) sind nicht threadsicher und erfordern Synchronisation.
