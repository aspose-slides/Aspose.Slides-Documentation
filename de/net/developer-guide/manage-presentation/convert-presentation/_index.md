## **PPT-zu-PPTX-Konvertierung**
### **Über PPT-zu-PPTX-Konvertierung**
Aspose.Slides unterstützt jetzt die Konvertierung von PPT nach PPTX.
### **PPT mit Aspose.Slides nach PPTX konvertieren**
Aspose.Slides für .NET bietet teilweise Unterstützung für die Konvertierung von Präsentationen im PPT‑Dateiformat in das PPTX‑Dateiformat. Da die Unterstützung für die genannte Präsentationskonvertierungsfunktion gerade erst in Aspose.Slides für .NET eingeführt wurde, ist sie derzeit noch begrenzt und funktioniert nur für einfache Präsentationen. Der Hauptvorteil der Aspose.Slides für .NET API‑Bibliothek bei der Konvertierung von PPT‑Präsentationen in das PPTX‑Format ist die einfache Handhabung der API, um das gewünschte Ergebnis zu erzielen. Bitte gehen Sie zu diesem [Link](/slides/de/net/convert-presentation/) im Abschnitt Code‑Beispiele für weitere Details. Der folgende Abschnitt zeigt deutlich, welche Funktionen beim Konvertieren von PPT‑Format‑Präsentationen zu PPTX‑Format‑Präsentationen unterstützt bzw. nicht unterstützt werden.

## **Präsentation‑zu‑PDF-Konvertierung**
### **Über PDF**
Das [Portable Document Format](https://en.wikipedia.org/wiki/PDF) ist ein Dateiformat, das von Adobe System erstellt wurde, um Dokumente zwischen verschiedenen Organisationen auszutauschen. Der Zweck dieses Formats besteht darin, dass der Inhalt der Dokumente so dargestellt werden kann, dass das visuelle Erscheinungsbild nicht von der Plattform abhängt, auf der es angezeigt wird.
### **PDF in Aspose.Slides für .NET**
[PDF ](https://docs.fileformat.com/pdf/) ist ein Dateiformat, das von Adobe System erstellt wurde, um Dokumente zwischen verschiedenen Organisationen auszutauschen. Der Zweck dieses Formats besteht darin, dass das visuelle Erscheinungsbild der Dokumentinhalte nicht von der Plattform abhängt, auf der es angezeigt wird.  

Jedes Präsentationsdokument, das in Aspose.Slides für .NET geladen werden kann, lässt sich in ein PDF‑Dokument konvertieren. Sie können die Präsentationsdokumente direkt mit der Aspose.Slides für .NET‑Komponente nach PDF exportieren. Weitere Bibliotheken von Drittanbietern oder die Aspose.PDF‑Komponente sind dafür nicht erforderlich.

## **Präsentation‑zu‑XPS-Konvertierung**
### **Über XPS**
Die [XML Paper Specification](https://en.wikipedia.org/wiki/Open_XML_Paper_Specification) ist eine Seitenbeschreibungssprache und ein festes Dokumentformat, das ursprünglich von Microsoft entwickelt wurde. Wie PDF ist XPS ein festes Layout‑Dokumentformat, das die Dokumenttreue bewahrt und ein geräteunabhängiges Erscheinungsbild bietet.
### **Präsentation mit Aspose.Slides nach XPS konvertieren**
Jedes Präsentationsdokument, das von Aspose.Slides für .NET geladen werden kann, lässt sich in das XPS‑Format konvertieren. Aspose.Slides für .NET verwendet die hochpräzise Seitenlayout‑ und Rendering‑Engine, um Ausgaben im festgelegten XPS‑Dokumentformat zu erzeugen. Erwähnenswert ist, dass Aspose.Slides für .NET XPS direkt generiert, ohne die mit .NET Framework 3.5 gelieferten Windows Presentation Foundation (WPF)-Klassen zu benötigen, wodurch XPS‑Dokumente auch auf Maschinen mit .NET Framework‑Versionen vor 3.5 erstellt werden können. Weitere Informationen zum Exportieren von Präsentationsdokumenten nach XPS über Aspose.Slides für .NET finden Sie in [diesem Thema](/slides/de/net/convert-powerpoint-ppt-and-pptx-to-microsoft-xps-document/).

## **Präsentation‑zu‑TIFF-Konvertierung**
### **Über TIFF**
Das [Tagged Image File Format](https://en.wikipedia.org/wiki/TIFF) , das für die Speicherung mehrerer Bilder in einem Dokument bekannt ist, wurde ursprünglich von Aldus erstellt. Dieses Format wird von Scan‑, Fax‑ und anderen Bildbearbeitungsanwendungen weitgehend unterstützt.
### **Präsentation mit Aspose.Slides nach TIFF konvertieren**
Jedes Dokument, das in Aspose.Slides für .NET geladen werden kann, lässt sich auch direkt von Aspose.Slides für .NET in ein TIFF‑Dokument konvertieren, ohne dass ein Drittanbieter‑Komponente erforderlich ist. Zusätzlich können Sie optional die Bildgröße im resultierenden TIFF‑Dokument festlegen. Informationen zum Export von Präsentationsdokumenten nach TIFF über Aspose.Slides für .NET finden Sie in [diesem Thema](/slides/de/net/convert-powerpoint-ppt-and-pptx-to-tiff/).

## **FAQ**

**Kann ich versteckte Folien beim Export nach PDF/XPS einbeziehen?**

Ja. Der Export unterstützt das Einbeziehen versteckter Folien über die entsprechende Option in den [PDF](https://reference.aspose.com/slides/net/aspose.slides.export/pdfoptions/showhiddenslides/)/[XPS](https://reference.aspose.com/slides/net/aspose.slides.export/xpsoptions/showhiddenslides/) Einstellungen.

**Wird das Speichern im PDF/A-Format (für die Archivierung) unterstützt?**

Ja, PDF/A‑Konformitätsstufen [sind verfügbar](https://reference.aspose.com/slides/net/aspose.slides.export/pdfcompliance/) (einschließlich A-2a/A-2b/A-2u und A-3a/A-3b) während des Exports.

**Was passiert mit Schriftarten während der Konvertierung: werden sie eingebettet oder ersetzt?**

Es gibt flexible Optionen: Sie können [alle Glyphen oder nur verwendete Teilmengen einbetten](/slides/de/net/embedded-font/), eine [Fallback‑Schriftart](/slides/de/net/fallback-font/) angeben und das [Verhalten steuern](/slides/de/net/font-substitution/), wenn einer Schriftart bestimmte Stile fehlen.

**Wie kann ich die Qualität und Größe des resultierenden PDFs steuern?**

Optionen stehen für [JPEG‑Qualität](https://reference.aspose.com/slides/net/aspose.slides.export/pdfoptions/jpegquality/), [Textkomprimierung](https://reference.aspose.com/slides/net/aspose.slides.export/pdfoptions/textcompression/), und einen Schwellenwert für [ausreichende Auflösung](https://reference.aspose.com/slides/net/aspose.slides.export/pdfoptions/sufficientresolution/) von Bildern zur Verfügung, sowie ein Modus, der die [beste Bildkompression](https://reference.aspose.com/slides/net/aspose.slides.export/pdfoptions/bestimagescompressionratio/) auswählt.

**Kann ich nur einen Folienbereich exportieren (z. B. 5–12)?**

Ja, der Export unterstützt die Auswahl eines Folienunterbereichs.

**Wird die Mehrkernverarbeitung mehrerer Dateien gleichzeitig unterstützt?**

Es ist zulässig, verschiedene Präsentationen parallel in separaten Prozessen zu verarbeiten. Wichtig: das gleiche [Presentation](https://reference.aspose.com/slides/net/aspose.slides/presentation/)-Objekt darf nicht von [mehreren Threads](/slides/de/net/multithreading/) geladen oder gespeichert werden.

**Gibt es Risiken beim Anwenden der Lizenz aus verschiedenen Threads?**

Ja, Aufrufe zum [Lizenz‑Setzen](/slides/de/net/licensing/) sind nicht thread‑sicher und erfordern Synchronisation.
