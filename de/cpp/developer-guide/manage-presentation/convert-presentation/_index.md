## **PPT-zu-PPTX-Konvertierung**
### **Über die PPT-zu-PPTX-Konvertierung**
Aspose.Slides unterstützt jetzt die Konvertierung von PPT zu PPTX.
### **PPT zu PPTX mit Aspose.Slides konvertieren**
Aspose.Slides für C++ bietet teilweisen Support für die Konvertierung von Präsentationen im PPT‑Dateiformat in das PPTX‑Dateiformat. Da die Unterstützung für die genannte Präsentationskonvertierungs‑Funktion gerade erst in Aspose.Slides für C++ eingeführt wurde, ist sie momentan nur eingeschränkt und funktioniert nur für einfache Präsentationen. Der Hauptvorteil, den die Aspose.Slides für C++‑API‑Bibliothek bei der Konvertierung von PPT‑Präsentationen in das PPTX‑Format bietet, ist die einfache Nutzung der API, um das gewünschte Ziel zu erreichen. Bitte gehen Sie zu diesem [Link](/slides/de/cpp/convert-presentation/) im Abschnitt Code‑Snippets für weitere Details. Der folgende Abschnitt zeigt deutlich, welche Funktionen beim Konvertieren von PPT‑Präsentationen in PPTX‑Präsentationen unterstützt bzw. nicht unterstützt werden.

## **Präsentation zu PDF-Konvertierung**
### **Über PDF**
Das [Portable Document Format](https://en.wikipedia.org/wiki/PDF) ist ein Dateiformat, das von Adobe Systems erstellt wurde, um Dokumente zwischen verschiedenen Organisationen auszutauschen. Der Zweck dieses Formats besteht darin, zu ermöglichen, dass der Inhalt der Dokumente so dargestellt werden kann, dass das visuelle Erscheinungsbild nicht von der Plattform abhängt, auf der es angezeigt wird.
### **PDF in Aspose.Slides für C++**
[PDF](https://docs.fileformat.com/pdf/) ist ein Dateiformat, das von Adobe Systems erstellt wurde, um Dokumente zwischen verschiedenen Organisationen auszutauschen. Der Zweck dieses Formats besteht darin, zu ermöglichen, dass das visuelle Erscheinungsbild des Dokumentinhalts nicht von der Plattform abhängt, auf der es angezeigt wird.

Jedes Präsentationsdokument, das in Aspose.Slides für C++ geladen werden kann, lässt sich in ein PDF‑Dokument konvertieren. Sie können die Präsentationsdokumente direkt mit der Aspose.Slides für C++‑Komponente nach PDF exportieren. Dafür benötigen Sie keine anderen Drittanbieter‑Bibliotheken oder die Aspose.PDF‑Komponente.

## **Präsentation zu XPS-Konvertierung**
### **Über XPS**
Die [XML Parser Specification](https://en.wikipedia.org/wiki/Open_XML_Paper_Specification) ist eine Seitenbeschreibungssprache und ein festes Dokumentformat, das ursprünglich von Microsoft entwickelt wurde. Wie PDF ist XPS ein festes Layout‑Dokumentformat, das dazu dient, die Treue des Dokuments zu bewahren und ein geräteunabhängiges Erscheinungsbild zu ermöglichen.
### **Präsentation mit Aspose.Slides in XPS konvertieren**
Jedes Präsentationsdokument, das von Aspose.Slides für C++ geladen werden kann, lässt sich in das XPS‑Format konvertieren. Aspose.Slides für C++ verwendet die hochpräzise Seitenlayout‑ und Rendering‑Engine, um Ausgaben im festgelegten XPS‑Dokumentformat zu erzeugen. Erwähnenswert ist, dass Aspose.Slides für C++ XPS direkt erzeugt, ohne auf die Windows Presentation Foundation (WPF)‑Klassen zurückzugreifen, die mit dem C++ Framework 3.5 paketiert sind, wodurch Aspose.Slides für C++ XPS‑Dokumente auch auf Rechnern mit C++ Framework‑Versionen vor 3.5 erzeugen kann. Weitere Informationen zum Export von Präsentationsdokumenten in XPS‑Dokumente finden Sie in [diesem Thema](/slides/de/cpp/convert-powerpoint-ppt-and-pptx-to-microsoft-xps-document/).

## **Präsentation zu TIFF-Konvertierung**
### **Über TIFF**
Das [Tagged Image File Format](https://en.wikipedia.org/wiki/TIFF) , das für das Speichern mehrerer Bilder in einem Dokument bekannt ist, wurde ursprünglich von Aldus erstellt. Dieses Format wird von Scan‑, Fax‑ und anderen Bildbearbeitungsanwendungen weitgehend unterstützt.
### **Präsentation mit Aspose.Slides in TIFF konvertieren**
Jedes Dokument, das in Aspose.Slides für C++ geladen werden kann, lässt sich ebenfalls direkt von Aspose.Slides für C++ in ein TIFF‑Dokument konvertieren, wodurch die Notwendigkeit einer Drittanbieter‑Komponente entfällt. Zusätzlich können Sie optional die Größe der Bilder im erzeugten TIFF‑Dokument festlegen. Weitere Informationen zum Export von Präsentationsdokumenten in TIFF‑Dokumente finden Sie in [diesem Thema](/slides/de/cpp/convert-powerpoint-ppt-and-pptx-to-tiff/).

## **FAQ**

**Kann ich ausgeblendete Folien beim Export nach PDF/XPS einbeziehen?**

Ja. Der Export unterstützt das Einbeziehen ausgeblendeter Folien über die entsprechende Option in den [PDF](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfoptions/set_showhiddenslides/)/[XPS](https://reference.aspose.com/slides/cpp/aspose.slides.export/xpsoptions/set_showhiddenslides/)‑Einstellungen.

**Wird das Speichern im PDF/A‑Format (für Archivierung) unterstützt?**

Ja, PDF/A‑Konformitätsstufen sind beim Export [verfügbar](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfcompliance/) (einschließlich A-2a/A-2b/A-2u und A-3a/A-3b).

**Was passiert mit den Schriftarten während der Konvertierung: werden sie eingebettet oder ersetzt?**

Es gibt flexible Optionen: Sie können [alle Glyphen oder nur verwendete Teilmengen einbetten](/slides/de/cpp/embedded-font/), eine [Ersatzschriftart](/slides/de/cpp/fallback-font/) festlegen und das [Verhalten steuern](/slides/de/cpp/font-substitution/), wenn einer Schriftart bestimmte Stile fehlen.

**Wie kann ich die Qualität und Größe des resultierenden PDF steuern?**

Optionen für die [JPEG‑Qualität](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfoptions/set_jpegquality/), die [Textkomprimierung](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfoptions/set_textcompression/), und einen Schwellenwert für eine [ausreichende Auflösung](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfoptions/set_sufficientresolution/) von Bildern stehen zur Verfügung, zusätzlich gibt es einen Modus, der die [beste Bildkompression](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfoptions/set_bestimagescompressionratio/) auswählt.

**Kann ich nur einen Folienbereich exportieren (z. B. 5–12)?**

Ja, der Export unterstützt die Auswahl eines Teilbereichs von Folien.

**Wird die Mehrkernverarbeitung mehrerer Dateien gleichzeitig unterstützt?**

Es ist zulässig, verschiedene Präsentationen parallel in separaten Prozessen zu verarbeiten. Wichtig: Das gleiche [Presentation](https://reference.aspose.com/slides/cpp/aspose.slides/presentation/)‑Objekt darf nicht von [mehreren Threads](/slides/de/cpp/multithreading/) geladen oder gespeichert werden.

**Gibt es Risiken bei der Lizenzanwendung aus verschiedenen Threads?**

Ja, Aufrufe zum [Lizenz‑Setzen](/slides/de/cpp/licensing/) sind nicht threadsicher und erfordern Synchronisation.
