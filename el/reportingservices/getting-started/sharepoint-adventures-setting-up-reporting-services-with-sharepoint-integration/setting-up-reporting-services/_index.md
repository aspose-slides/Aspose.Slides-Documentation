{{% alert color="primary" %}} 

Η πρώτη μας στάση στον RS Server είναι ο Διαχειριστής Διαμόρφωσης Υπηρεσιών Αναφοράς. 

{{% /alert %}} 
## **Λογαριασμός Υπηρεσίας**
Βεβαιωθείτε ότι κατανοείτε ποιον λογαριασμό υπηρεσίας χρησιμοποιείτε για τις Υπηρεσίες Αναφοράς. Αν προκύψουν προβλήματα, ίσως σχετίζονται με τον λογαριασμό υπηρεσίας που χρησιμοποιείτε. Η προεπιλογή είναι το Network Service. Κάθε φορά που αναπτύσσω νέες εκδόσεις, χρησιμοποιώ πάντα Λογαριασμούς Τομέα, επειδή εκεί είναι πιο πιθανό να προκύψουν προβλήματα. Για αυτή τη διαμόρφωση στο διακομιστή μου, χρησιμοποίησα έναν Λογαριασμό Τομέα με το όνομα **RSService**. 
## **Διεύθυνση URL Υπηρεσίας Ιστού**
Θα χρειαστεί να διαμορφώσουμε τη Διεύθυνση URL Υπηρεσίας Ιστού. Πρόκειται για τον εικονικό φάκελο **ReportServer** (vdir) που φιλοξενεί τις Υπηρεσίες Ιστού που χρησιμοποιούν οι Υπηρεσίες Αναφοράς, και με τον οποίο επικοινωνεί το SharePoint. Εκτός εάν θέλετε να προσαρμόσετε τις ιδιότητες του vdir (π.χ. SSL, θύρες, host headers κ.λπ.), θα πρέπει απλώς να πατήσετε Apply εδώ και να είστε έτοιμοι.

## **Βάση Δεδομένων**
Πρέπει να δημιουργήσουμε τη Βάση Δεδομένων Καταλόγου των Υπηρεσιών Αναφοράς. Μπορεί να τοποθετηθεί σε οποιοδήποτε SQL 2008 ή SQL 2008 R2 Database Engine. Το SQL11 λειτουργεί επίσης, αλλά είναι ακόμα σε BETA. Αυτή η ενέργεια θα δημιουργήσει δύο βάσεις δεδομένων, **ReportServer** και **ReportServerTempDB**, προεπιλεγμένα. 
Το άλλο σημαντικό βήμα είναι να επιλέξετε SharePoint Integrated ως τύπο βάσης δεδομένων. Μόλις γίνει αυτή η επιλογή, δεν μπορεί να αλλάξει. Παρακαλούμε δείτε τα Σχήματα 5, 6 και 7 για αναφορά.

**Σχήμα 7**: Ρύθμιση Ονόματος και Λειτουργίας Βάσης Δεδομένων 

Για τα διαπιστευτήρια, έτσι ο Διακομιστής Αναφορών θα επικοινωνεί με τον SQL Server. Όποιος λογαριασμός και αν επιλέξετε, θα λάβει ορισμένα δικαιώματα στη βάση δεδομένων Καταλόγου καθώς και σε μερικές συστημικές βάσεις μέσω του RSExecRole. Η MSDB είναι μία από αυτές τις βάσεις για χρήση Συνδρομών, καθώς χρησιμοποιούμε τον SQL Agent.

## **Διεύθυνση URL Διαχειριστή Αναφορών**
Μπορούμε να παραλείψουμε τη Διεύθυνση URL Διαχειριστή Αναφορών, καθώς δεν χρησιμοποιείται όταν λειτουργούμε σε λειτουργία SharePoint Integrated. Το SharePoint είναι το frontend μας. Ο Διαχειριστής Αναφορών δεν λειτουργεί. 
## **Κλειδιά Κρυπτογράφησης**
Δημιουργήστε αντίγραφα ασφαλείας των Κλειδιών Κρυπτογράφησης και βεβαιωθείτε ότι ξέρετε πού τα αποθηκεύετε. Αν χρειαστεί να μεταφέρετε ή να επαναφέρετε τη Βάση Δεδομένων, θα τα χρειαστείτε. 

![todo:image_alt_text](setting-up-reporting-services_9.png)

Αυτό είναι όλο για τον Διαχειριστή Διαμόρφωσης Υπηρεσιών Αναφοράς. Αν μεταβείτε στη διεύθυνση URL στην καρτέλα Διεύθυνση URL Υπηρεσίας Ιστού, θα πρέπει να εμφανιστεί κάτι παρόμοιο με το παρακάτω σχήμα.

**Σχήμα 12**: Πρόσβαση σε Διακομιστή Αναφορών μετά την εγκατάσταση 

Τι συνέβη; Το SharePoint είναι εγκατεστημένο στο WFE μου και ολοκλήρωσα τη ρύθμιση των Υπηρεσιών Αναφοράς. Σε αυτό το παράδειγμα, οι Υπηρεσίες Αναφοράς και το SharePoint βρίσκονται σε διαφορετικές μηχανές. Αν ήταν στην ίδια μηχανή, δεν θα είχατε δει αυτό το σφάλμα. Πρακτικά, πρέπει να εγκαταστήσουμε το SharePoint στο κουτί RS. Αυτό σημαίνει ότι το IIS θα ενεργοποιηθεί επίσης.
