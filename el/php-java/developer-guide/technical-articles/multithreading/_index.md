## **Εισαγωγή**

Ενώ η παραλληλία εργασίας με παρουσιάσεις είναι δυνατή (εκτός από την ανάλυση/φόρτωση/κλωνοποίηση) και τα πάντα κυμαίνονται καλά (στις περισσότερες περιπτώσεις), υπάρχει μικρή πιθανότητα να λάβετε λανθασμένα αποτελέσματα όταν χρησιμοποιείτε τη βιβλιοθήκη σε πολλαπλά νήματα.

Συνιστούμε έντονα να **μην** χρησιμοποιείτε ένα μόνο αντικείμενο [Presentation](https://reference.aspose.com/slides/el/php-java/aspose.slides/Presentation) σε περιβάλλον πολλαπλών νημάτων, επειδή μπορεί να οδηγήσει σε απρόβλεπτα σφάλματα ή αποτυχίες που δεν εντοπίζονται εύκολα.

Δεν είναι **ασφαλές** να φορτώνετε, αποθηκεύετε και/ή να κλωνοποιείτε ένα αντικείμενο της κλάσης [Presentation](https://reference.aspose.com/slides/el/php-java/aspose.slides/Presentation) σε πολλαπλά νήματα. Τέτοιες λειτουργίες **δεν** υποστηρίζονται.  Εάν χρειάζεστε την εκτέλεση τέτοιων εργασιών, πρέπει να παραλληλίσετε τις λειτουργίες χρησιμοποιώντας αρκετές διαδικασίες μονονηματικής εκτέλεσης — και καθεμία από αυτές τις διαδικασίες πρέπει να χρησιμοποιεί το δικό της αντικείμενο παρουσίασης.

Δεν εγγυόμαστε την πολυνηματική λειτουργία σε PHP όταν χρησιμοποιείτε επεκτάσεις. Εάν τις χρησιμοποιήσετε, κάντε το με δική σας ευθύνη.

## **Συχνές ερωτήσεις**

**Πρέπει να εκτελώ τη ρύθμιση άδειας σε κάθε νήμα;**

Όχι. Αρκεί να το κάνετε μία φορά ανά διαδικασία/περιοχή εφαρμογής πριν ξεκινήσουν τα νήματα. Εάν η [ρύθμιση άδειας](/slides/el/php-java/licensing/) μπορεί να κληθεί ταυτόχρονα (για παράδειγμα, κατά την αργή αρχικοποίηση), συγχρονίστε αυτήν την κλήση, επειδή η μέθοδος ρύθμισης άδειας από μόνη της δεν είναι ασφαλής ως προς τα νήματα.

**Μπορώ να μεταβιβάσω αντικείμενα `Presentation` ή `Slide` μεταξύ νημάτων;**

Η μεταφορά «ζωντανών» αντικειμένων παρουσίασης μεταξύ νημάτων δεν συνιστάται: χρησιμοποιήστε ανεξάρτητα αντικείμενα ανά νήμα ή προδημιουργήστε ξεχωριστές παρουσιάσεις/κοντέινερ διαφανειών για κάθε νήμα. Αυτή η προσέγγιση ακολουθεί τη γενική σύσταση να μην μοιράζεστε ένα μόνο αντικείμενο παρουσίασης μεταξύ νημάτων.

**Είναι ασφαλές να παραλληλοποιήσετε την εξαγωγή σε διαφορετικές μορφές (PDF, HTML, εικόνες) εφόσον κάθε νήμα διαθέτει το δικό του αντικείμενο `Presentation`;**

Ναι. Με ανεξάρτητα αντικείμενα και ξεχωριστές διαδρομές εξόδου, τέτοιες εργασίες συνήθως παραλληλοποιούνται σωστά· αποφύγετε οποιαδήποτε κοινόχρηστα αντικείμενα παρουσίασης και κοινά ρεύματα I/O.

**Τι πρέπει να κάνω με τις παγκόσμιες ρυθμίσεις γραμματοσειρών (φακέλους, υποκαταστάσεις) σε πολυνηματικό περιβάλλον;**

Αρχικοποιήστε όλες τις παγκόσμιες [ρυθμίσεις γραμματοσειρών](/slides/el/php-java/powerpoint-fonts/) πριν ξεκινήσετε τα νήματα και μην τις αλλάξετε κατά τη διάρκεια της παράλληλης εργασίας. Αυτό εξαλείφει τους ανταγωνισμούς κατά την πρόσβαση σε κοινόχρηστους πόρους γραμματοσειρών.
