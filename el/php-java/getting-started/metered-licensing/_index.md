## **Εισαγωγή**

Η αδειοδότηση με μετρήσεις είναι ένας μηχανισμός αδειοδότησης που μπορεί να χρησιμοποιηθεί παράλληλα με τις υπάρχουσες μεθόδους αδειοδότησης. Εάν θέλετε να χρεώνεστε βάσει της χρήσης των λειτουργιών του Aspose.Slides API, επιλέγετε την αδειοδότηση με μετρήσεις.

## **Εφαρμογή Μετρημένων Κλειδιών**

Όταν αγοράζετε μια αδειοδότηση με μετρήσεις, λαμβάνετε κλειδιά (και όχι αρχείο άδειας). Αυτό το μετρημένο κλειδί μπορεί να εφαρμοστεί χρησιμοποιώντας την κλάση [Metered](https://reference.aspose.com/slides/el/php-java/aspose.slides/metered/) που παρέχει η Aspose για λειτουργίες μέτρησης. Για περισσότερες λεπτομέρειες, δείτε το [Metered Licensing FAQ](https://purchase.aspose.com/faqs/licensing/metered).

1. Δημιουργήστε ένα αντικείμενο της κλάσης [Metered](https://reference.aspose.com/slides/el/php-java/aspose.slides/metered/).

1. Περάστε τα δημόσια και ιδιωτικά κλειδιά σας στη μέθοδο [setMeteredKey](https://reference.aspose.com/slides/el/php-java/aspose.slides/metered/#setMeteredKey-java.lang.String-java.lang.String-).

1. Πραγματοποιήστε κάποια επεξεργασία (εκτελέστε εργασίες).

1. Καλέστε τη μέθοδο [getConsumptionQuantity](https://reference.aspose.com/slides/el/php-java/aspose.slides/metered/#getConsumptionQuantity--) της κλάσης `Metered`.

Θα πρέπει να δείτε το ποσό/την ποσότητα των αιτημάτων API που έχετε καταναλώσει μέχρι τώρα.

Αυτό το δείγμα κώδικα σας δείχνει πώς να χρησιμοποιήσετε την αδειοδότηση με μετρήσεις:

```php
// Δημιουργεί ένα αντικείμενο της κλάσης Metered
$metered = new Metered();

try {
    // Περνά τα δημόσια και ιδιωτικά κλειδιά στο αντικείμενο Metered
    $metered->setMeteredKey("<valid pablic key>", "<valid private key>");

    // Λαμβάνει την τιμή της καταναλωθείσας ποσότητας πριν από κλήσεις API
    $amountBefore = Metered::getConsumptionQuantity();
    echo("Amount consumed before: " . $amountBefore);

    // Κάντε κάτι με το API του Aspose.Slides εδώ
    // ...

    // Λαμβάνει την τιμή της καταναλωθείσας ποσότητας μετά τις κλήσεις API
    $amountAfter = Metered::getConsumptionQuantity();
    echo("Amount consumed after: " . $amountAfter);
} catch (JavaException $ex) {
  $ex->printStackTrace();
}
```

{{% alert color="warning" title="NOTE"  %}} 

Για να χρησιμοποιήσετε την αδειοδότηση με μετρήσεις, χρειάζεστε σταθερή σύνδεση στο διαδίκτυο επειδή ο μηχανισμός αδειοδότησης χρησιμοποιεί το διαδίκτυο για συνεχή αλληλεπίδραση με τις υπηρεσίες μας και εκτέλεση υπολογισμών.

{{% /alert %}} 

## **Συχνές Ερωτήσεις**

**Μπορώ να χρησιμοποιήσω μια αδειοδότηση με μετρήσεις μαζί με μια κανονική (μόνιμη ή προσωρινή) στην ίδια εφαρμογή;**

Ναι. Η αδειοδότηση με μετρήσεις είναι ένας πρόσθετος μηχανισμός αδειοδότησης που μπορεί να χρησιμοποιηθεί παράλληλα με τις υπάρχουσες [μεθόδους αδειοδότησης](/slides/el/php-java/licensing/). Επιλέγετε ποιον μηχανισμό θα εφαρμόσετε όταν η εφαρμογή ξεκινά.

**Τι ακριβώς μετράται ως κατανάλωση σε μια αδειοδότηση με μετρήσεις: λειτουργίες ή αρχεία;**

Η χρήση του API καταμετράται, δηλαδή ο αριθμός των αιτημάτων ή των λειτουργιών. Μπορείτε να λάβετε την τρέχουσα κατανάλωση μέσω των [μεθόδους παρακολούθησης κατανάλωσης](https://reference.aspose.com/slides/el/php-java/aspose.slides/metered/).

**Είναι η αδειοδότηση με μετρήσεις κατάλληλη για μικροϋπηρεσίες και περιβάλλοντα χωρίς διακομιστή όπου οι περιπτώσεις επανεκκινούνται συχνά;**

Ναι. Δεδομένου ότι η λογιστική γίνεται σε επίπεδο κλήσης API, τα σενάρια με συχνές κρύες εκκινήσεις είναι συμβατά, εφόσον υπάρχει σταθερή πρόσβαση δικτύου για τους υπολογισμούς της αδειοδότησης με μετρήσεις.

**Διαφέρει η λειτουργικότητα της βιβλιοθήκης όταν χρησιμοποιείται αδειοδότηση με μετρήσεις σε σύγκριση με μόνιμη άδεια;**

Όχι. Αυτό αφορά μόνο τον μηχανισμό αδειοδότησης και χρέωσης· οι δυνατότητες του προϊόντος παραμένουν ίδιες.

**Πώς η αδειοδότηση με μετρήσεις σχετίζεται με την δοκιμαστική έκδοση και την προσωρινή άδεια;**

Η δοκιμαστική έκδοση έχει περιορισμούς και υδατογραφήματα, η [temporary license](https://purchase.aspose.com/temporary-license/) αφαιρεί περιορισμούς για 30 ημέρες, και η αδειοδότηση με μετρήσεις αφαιρεί περιορισμούς και χρεώνει βάσει της πραγματικής χρήσης.

**Μπορώ να ελέγξω τον προϋπολογισμό ενεργώντας αυτόματα όταν ξεπεραστεί ένα όριο κατανάλωσης;**

Ναι. Μία κοινή πρακτική είναι να διαβάζετε περιοδικά την τρέχουσα κατανάλωση μέσω των [tracking methods](https://reference.aspose.com/slides/el/php-java/aspose.slides/metered/) και να εφαρμόζετε τα δικά σας όρια ή ειδοποιήσεις στο επίπεδο της εφαρμογής ή της παρακολούθησης.
