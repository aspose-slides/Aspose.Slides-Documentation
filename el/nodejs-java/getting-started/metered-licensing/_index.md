## **Εισαγωγή**

Η μετρημένη αδειοδότηση είναι ένας μηχανισμός αδειοδότησης που μπορεί να χρησιμοποιηθεί παράλληλα με τις υπάρχουσες μεθόδους αδειοδότησης. Εάν θέλετε να χρεώνεστε με βάση τη χρήση των λειτουργιών του Aspose.Slides API, επιλέγετε τη μετρημένη αδειοδότηση.

## **Εφαρμογή Μετρημένων Κλειδιών**

Όταν αγοράζετε μια μετρημένη άδεια, λαμβάνετε κλειδιά (και όχι αρχείο άδειας). Αυτό το μετρημένο κλειδί μπορεί να εφαρμοστεί χρησιμοποιώντας την κλάση [Metered](https://reference.aspose.com/slides/el/nodejs-java/aspose.slides/metered/) που παρέχει η Aspose για λειτουργίες μέτρησης. Για περισσότερες λεπτομέρειες, δείτε το [Metered Licensing FAQ](https://purchase.aspose.com/faqs/licensing/metered).

1. Δημιουργήστε μια παρουσία της κλάσης [Metered](https://reference.aspose.com/slides/el/nodejs-java/aspose.slides/metered/).

1. Περάστε τα δημόσια και ιδιωτικά κλειδιά σας στη μέθοδο [setMeteredKey](https://reference.aspose.com/slides/el/nodejs-java/aspose.slides/metered/#setMeteredKey).

1. Πραγματοποιήστε κάποια επεξεργασία (εκτελέστε εργασίες).

1. Καλέστε τη μέθοδο [getConsumptionQuantity](https://reference.aspose.com/slides/el/nodejs-java/aspose.slides/metered/#getConsumptionQuantity) της κλάσης `Metered`.

Θα πρέπει να δείτε το ποσό/την ποσότητα των αιτημάτων API που έχετε καταναλώσει μέχρι τώρα.

Αυτό το δείγμα κώδικα σας δείχνει πώς να χρησιμοποιήσετε τη μετρημένη άδεια:

```js
var aspose = aspose || {};
aspose.slides = require("aspose.slides.via.java");

// Δημιουργεί μια παρουσία της κλάσης Metered
var metered = new aspose.slides.Metered();

// Περαίνει τα δημόσια και ιδιωτικά κλειδιά στο αντικείμενο Metered
metered.setMeteredKey("<valid public key>", "<valid private key>");

// Λαμβάνει την τιμή της καταναλωμένης ποσότητας πριν από τις κλήσεις API
var amountBefore = aspose.slides.Metered.getConsumptionQuantity();
console.log("Amount consumed before:", amountBefore);

// Κάντε κάτι με το API του Aspose.Slides εδώ
// ...

// Λαμβάνει την τιμή της καταναλωμένης ποσότητας μετά τις κλήσεις API
var amountAfter = aspose.slides.Metered.getConsumptionQuantity();
console.log("Amount consumed after:", amountAfter);
```

{{% alert color="warning" title="NOTE"  %}} 
Για να χρησιμοποιήσετε τη μετρημένη άδεια, χρειάζεστε μια σταθερή σύνδεση στο διαδίκτυο, επειδή ο μηχανισμός αδειοδότησης χρησιμοποιεί το διαδίκτυο για συνεχή αλληλεπίδραση με τις υπηρεσίες μας και για την εκτέλεση υπολογισμών.
{{% /alert %}} 

## **Συχνές Ερωτήσεις**

**Μπορώ να χρησιμοποιήσω μια μετρημένη άδεια μαζί με μια κανονική (ατέρμονη ή προσωρινή) στην ίδια εφαρμογή;**

Ναι. Η μετρημένη άδεια είναι ένας επιπλέον μηχανισμός αδειοδότησης που μπορεί να χρησιμοποιηθεί παράλληλα με τις υπάρχουσες [licensing methods](/slides/el/nodejs-java/licensing/). Επιλέγετε ποιον μηχανισμό θα εφαρμόσετε όταν εκκινείται η εφαρμογή.

**Τι ακριβώς μετράται ως κατανάλωση υπό μια μετρημένη άδεια: λειτουργίες ή αρχεία;**

Μετράται η χρήση του API, δηλαδή ο αριθμός των αιτημάτων ή των λειτουργιών. Μπορείτε να αποκτήσετε την τρέχουσα κατανάλωση μέσω των [consumption-tracking methods](https://reference.aspose.com/slides/el/nodejs-java/aspose.slides/metered/).

**Είναι η μετρημένη άδεια κατάλληλη για μικροϋπηρεσίες και περιβάλλοντα serverless όπου οι διεργασίες επανεκκινούν συχνά;**

Ναι. Επειδή η καταμέτρηση γίνεται στο επίπεδο κλήσης API, τα σενάρια με συχνές κρύες εκκινήσεις είναι συμβατά, εφόσον υπάρχει σταθερή πρόσβαση δικτύου για τους υπολογισμούς της μετρημένης άδειας.

**Διαφέρει η λειτουργικότητα της βιβλιοθήκης όταν χρησιμοποιείται μια μετρημένη άδεια σε σύγκριση με μια ατέρμονη άδεια;**

Όχι. Πρόκειται μόνο για τον μηχανισμό αδειοδότησης και χρέωσης· οι δυνατότητες του προϊόντος παραμένουν οι ίδιες.

**Πώς σχετίζεται η μετρημένη άδεια με τη δοκιμαστική έκδοση και την προσωρινή άδεια;**

Η δοκιμαστική έκδοση έχει περιορισμούς και υδατογραφήματα, η [temporary license](https://purchase.aspose.com/temporary-license/) αφαιρεί τους περιορισμούς για 30 ημέρες, και η μετρημένη άδεια αφαιρεί τους περιορισμούς και χρεώνει βάσει της πραγματικής χρήσης.

**Μπορώ να ελέγξω τον προϋπολογισμό αντιδρώντας αυτόματα όταν ξεπεραστεί ένα όριο κατανάλωσης;**

Ναι. Ένα συνηθισμένο πρακτικό είναι να διαβάζετε περιοδικά την τρέχουσα κατανάλωση μέσω των [tracking methods](https://reference.aspose.com/slides/el/nodejs-java/aspose.slides/metered/) και να εφαρμόζετε τα δικά σας όρια ή ειδοποιήσεις στο επίπεδο της εφαρμογής ή της παρακολούθησης.
