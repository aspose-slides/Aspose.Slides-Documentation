## **Εισαγωγή**

Η χρέωση με μέτρηση είναι ένας μηχανισμός αδειοδότησης που μπορεί να χρησιμοποιηθεί παράλληλα με τις υπάρχουσες μεθόδους αδειοδότησης. Εάν θέλετε να χρεωθείτε με βάση τη χρήση των δυνατοτήτων του Aspose.Slides API, επιλέγετε τη χρέωση με μέτρηση.

## **Εφαρμογή κλειδιών χρέωσης με μέτρηση**

{{% alert color="primary" %}} 

Η χρέωση με μέτρηση είναι ένας νέος μηχανισμός αδειοδότησης που μπορεί να χρησιμοποιηθεί παράλληλα με τις υπάρχουσες μεθόδους αδειοδότησης. Εάν θέλετε να χρεωθείτε με βάση τη χρήση των δυνατοτήτων του Aspose.Slides API, επιλέγετε τη χρέωση με μέτρηση.

Όταν αγοράζετε μια χρέωση με μέτρηση, λαμβάνετε κλειδιά (όχι αρχείο άδειας). Αυτό το κλειδί χέρησης μπορεί να εφαρμοστεί χρησιμοποιώντας την κλάση [Metered](https://reference.aspose.com/slides/el/java/com.aspose.slides/metered/) που παρέχει η Aspose για λειτουργίες χρέωσης. Για περισσότερες λεπτομέρειες, δείτε το [Metered Licensing FAQ](https://purchase.aspose.com/faqs/licensing/metered).

{{% /alert %}} 

1. Δημιουργήστε μια παρουσία της κλάσης [Metered](https://reference.aspose.com/slides/el/java/com.aspose.slides/metered/).

1. Περάστε τα δημόσια και ιδιωτικά κλειδιά σας στη μέθοδο [setMeteredKey](https://reference.aspose.com/slides/el/java/com.aspose.slides/metered/#setMeteredKey-java.lang.String-java.lang.String-) .

1. Κάντε κάποια επεξεργασία (εκτελέστε εργασίες).

1. Καλέστε τη μέθοδο [getConsumptionQuantity](https://reference.aspose.com/slides/el/java/com.aspose.slides/metered/#getConsumptionQuantity--) της κλάσης `Metered`.

Θα πρέπει να δείτε το ποσό/την ποσότητα των αιτημάτων API που έχετε καταναλώσει μέχρι στιγμής.

Αυτό το δείγμα κώδικα σας δείχνει πώς να χρησιμοποιήσετε τη χρέωση με μέτρηση:

```java
// Δημιουργεί μια παρουσία της κλάσης Metered
com.aspose.slides.Metered metered = new com.aspose.slides.Metered();

try {
    // Μεταβιβάζει τα δημόσια και ιδιωτικά κλειδιά στο αντικείμενο Metered
    metered.setMeteredKey("<valid public key>", "<valid private key>");

    // Λαμβάνει την τιμή της καταναλισκόμενης ποσότητας πριν από τις κλήσεις API
    double amountBefore = com.aspose.slides.Metered.getConsumptionQuantity();
    System.out.println("Amount consumed before: " + amountBefore);

    // Κάντε κάτι με το API του Aspose.Slides εδώ
    // ...

    // Λαμβάνει την τιμή της καταναλισκόμενης ποσότητας μετά τις κλήσεις API
    double amountAfter = com.aspose.slides.Metered.getConsumptionQuantity();
    System.out.println("Amount consumed after: " + amountAfter);
} catch (Exception ex) {
    ex.printStackTrace();
}
```

{{% alert color="warning" title="NOTE"  %}} 

Για να χρησιμοποιήσετε τη χρέωση με μέτρηση, χρειάζεστε σταθερή σύνδεση στο διαδίκτυο επειδή ο μηχανισμός αδειοδότησης χρησιμοποιεί το διαδίκτυο για συνεχή αλληλεπίδραση με τις υπηρεσίες μας και για εκτέλεση υπολογισμών.

{{% /alert %}} 

## **FAQ**

**Μπορώ να χρησιμοποιήσω μια χρέωση με μέτρηση μαζί με μια κανονική (μόνιμη ή προσωρινή) στο ίδιο πρόγραμμα;**

Ναι. Η χρέωση με μέτρηση είναι ένας επιπλέον μηχανισμός αδειοδότησης που μπορεί να χρησιμοποιηθεί παράλληλα με τις υπάρχουσες [μεθόδους αδειοδότησης](/slides/el/java/licensing/). Επιλέγετε ποιος μηχανισμός θα εφαρμοστεί όταν η εφαρμογή ξεκινήσει.

**Τι ακριβώς μετράται ως κατανάλωση σε μια χρέωση με μέτρηση: λειτουργίες ή αρχεία;**

Καταμετρώνεται η χρήση του API, δηλαδή ο αριθμός των αιτημάτων ή των λειτουργιών. Μπορείτε να λάβετε την τρέχουσα κατανάλωση μέσω των [μεθόδων παρακολούθησης κατανάλωσης](https://reference.aspose.com/slides/el/java/com.aspose.slides/metered/).

**Είναι η χρέωση με μέτρηση κατάλληλη για μικροϋπηρεσίες και περιβάλλοντα serverless όπου οι εκτελέσεις επανεκκινούνται συχνά;**

Ναι. Δεδομένου ότι η καταγραφή γίνεται σε επίπεδο κλήσης API, τα σενάρια με συχνές κρύες εκκινήσεις είναι συμβατά, αρκεί να υπάρχει σταθερή πρόσβαση δικτύου για τους υπολογισμούς χρέωσης.

**Διαφέρει η λειτουργικότητα της βιβλιοθήκης όταν χρησιμοποιείται χρέωση με μέτρηση σε σχέση με μόνιμη άδεια;**

Όχι. Αυτό αφορά μόνο τον μηχανισμό αδειοδότησης και χρέωσης· οι δυνατότητες του προϊόντος παραμένουν οι ίδιες.

**Πώς σχετίζεται η χρέωση με μέτρηση με την δοκιμαστική έκδοση και την προσωρινή άδεια;**

Η δοκιμαστική έκδοση έχει περιορισμούς και υδατογραφήματα, η [προσωρινή άδεια](https://purchase.aspose.com/temporary-license/) αφαιρεί περιορισμούς για 30 ημέρες, ενώ η χρέωση με μέτρηση αφαιρεί περιορισμούς και χρεώνει βάσει της πραγματικής χρήσης.

**Μπορώ να ελέγχω τον προϋπολογισμό ώστε να αντιδρώ αυτόματα όταν ξεπεραστεί ένα όριο κατανάλωσης;**

Ναι. Συνηθισμένη πρακτική είναι η περιοδική ανάγνωση της τρέχουσας κατανάλωσης μέσω των [μεθόδων παρακολούθησης](https://reference.aspose.com/slides/el/java/com.aspose.slides/metered/) και η υλοποίηση δικών σας ορίων ή ειδοποιήσεων σε επίπεδο εφαρμογής ή παρακολούθησης.
