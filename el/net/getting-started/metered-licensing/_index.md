## **Εισαγωγή**

Η αδειοδότηση με μέτρηση (metered licensing) είναι ένας μηχανισμός αδειοδότησης που μπορεί να χρησιμοποιηθεί παράλληλα με τις υπάρχουσες μεθόδους αδειοδότησης. Αν θέλετε να χρεώνονται βάσει της χρήσης των λειτουργιών του Aspose.Slides API, επιλέγετε την αδειοδότησή με μέτρηση.

## **Εφαρμογή Μετρικών Κλειδιών**

Όταν αγοράζετε μια άδεια με μέτρηση, λαμβάνετε κλειδιά (και όχι αρχείο άδειας). Αυτό το μετρικό κλειδί μπορεί να εφαρμοστεί χρησιμοποιώντας την κλάση [Metered](https://reference.aspose.com/slides/el/net/aspose.slides/metered/) που παρέχει η Aspose για λειτουργίες μέτρησης. Για περισσότερες λεπτομέρειες, δείτε το [Metered Licensing FAQ](https://purchase.aspose.com/faqs/licensing/metered).

1. Δημιουργήστε ένα στιγμιότυπο της κλάσης [Metered](https://reference.aspose.com/slides/el/net/aspose.slides/metered/).
1. Μεταβιβάστε τα δημόσια και ιδιωτικά κλειδιά σας στη μέθοδο [SetMeteredKey](https://reference.aspose.com/slides/el/net/aspose.slides/metered/setmeteredkey/).
1. Εκτελέστε κάποια επεξεργασία (εκτελέστε εργασίες).
1. Καλέστε τη μέθοδο [GetConsumptionQuantity](https://reference.aspose.com/slides/el/net/aspose.slides/metered/getconsumptionquantity/) της κλάσης `Metered`.

Θα πρέπει να δείτε το ποσό/πλήθος των αιτημάτων API που έχετε καταναλώσει μέχρι τώρα.

Αυτό το δείγμα κώδικα δείχνει πώς να χρησιμοποιήσετε την αδειοδότηση με μέτρηση:

```cs
// Δημιουργεί ένα στιγμιότυπο της κλάσης Metered
Aspose.Slides.Metered metered = new Aspose.Slides.Metered();

// Μεταβιβάζει τα δημόσια και ιδιωτικά κλειδιά στο αντικείμενο Metered
metered.SetMeteredKey("<valid public key>", "<valid private key>");

// Λαμβάνει την ποσότητα των μετρητικών δεδομένων πριν από την κλήση API
decimal amountBefore = Aspose.Slides.Metered.GetConsumptionQuantity();
Console.WriteLine("Amount consumed before: " + amountBefore.ToString());

// Κάντε κάτι με το API του Aspose.Slides εδώ
// ...

// Λαμβάνει την ποσότητα των μετρητικών δεδομένων μετά την κλήση API
decimal amountAfter = Aspose.Slides.Metered.GetConsumptionQuantity();
Console.WriteLine("Amount consumed after: " + amountAfter.ToString());
```

{{% alert color="warning" title="ΣΗΜΕΙΩΣΗ"  %}} 

Για την χρήση αδειοδότησης με μέτρηση, χρειάζεται σταθερή σύνδεση στο διαδίκτυο, επειδή ο μηχανισμός αδειοδότησης χρησιμοποιεί το διαδίκτυο για συνεχή αλληλεπίδραση με τις υπηρεσίες μας και για εκτέλεση υπολογισμών.

{{% /alert %}} 

## **Συχνές Ερωτήσεις**

**Μπορώ να χρησιμοποιήσω μια άδεια με μέτρηση μαζί με μια κανονική άδεια (πάγια ή προσωρινή) στην ίδια εφαρμογή;**

Ναι. Η αδειοδότηση με μέτρηση είναι ένας επιπλέον μηχανισμός αδειοδότησης που μπορεί να χρησιμοποιηθεί παράλληλα με τις υπάρχουσες [licensing methods](/slides/el/net/licensing/). Επιλέγετε ποιος μηχανισμός θα εφαρμοστεί κατά την εκκίνηση της εφαρμογής.

**Τι ακριβώς μετράται ως κατανάλωση υπό μια άδεια με μέτρηση: λειτουργίες ή αρχεία;**

Μετράται η χρήση του API, δηλαδή ο αριθμός των αιτημάτων ή των λειτουργιών. Μπορείτε να λάβετε την τρέχουσα κατανάλωση μέσω των [consumption-tracking methods](https://reference.aspose.com/slides/el/net/aspose.slides/metered/).

**Είναι η αδειοδότηση με μέτρηση κατάλληλη για μικρουπηρεσίες και περιβάλλοντα serverless όπου οι στιγμιότυπα επανεκκινούνται συχνά;**

Ναι. Επειδή η καταγραφή γίνεται σε επίπεδο κλήσεων API, σενάρια με συχνές κρύες εκκινήσεις είναι συμβατά, εφόσον υπάρχει σταθερή πρόσβαση δικτύου για τους υπολογισμούς της αδειοδότησης με μέτρηση.

**Διαφέρει η λειτουργικότητα της βιβλιοθήκης όταν χρησιμοποιείται άδεια με μέτρηση σε σύγκριση με μία παγία άδεια;**

Όχι. Αυτό αφορά μόνο τον μηχανισμό αδειοδότησης και χρέωσης· οι δυνατότητες του προϊόντος παραμένουν ίδιες.

**Πώς σχετίζεται η αδειοδότηση με μέτρηση με την έκδοση δοκιμής και την προσωρινή άδεια;**

Η έκδοση δοκιμής έχει περιορισμούς και υδατογραφήματα, η [temporary license](https://purchase.aspose.com/temporary-license/) αφαιρεί περιορισμούς για 30 ημέρες, και η αδειοδότηση με μέτρηση αφαιρεί περιορισμούς και χρεώνει βάσει πραγματικής χρήσης.

**Μπορώ να ελέγξω τον προϋπολογισμό ανταποκρινόμενος αυτόματα όταν ξεπεραστεί ένα όριο κατανάλωσης;**

Ναι. Συνήθης πρακτική είναι η περιοδική ανάγνωση της τρέχουσας κατανάλωσης μέσω των [tracking methods](https://reference.aspose.com/slides/el/net/aspose.slides/metered/) και η υλοποίηση δικών σας ορίων ή ειδοποιήσεων σε επίπεδο εφαρμογής ή παρακολούθησης.
