## **Επισκόπηση**

Το Aspose.Slides μπορεί να χρησιμοποιηθεί σε λειτουργία αξιολόγησης ή με έγκυρη άδεια. Η έκδοση αξιολόγησης προσφέρει την ίδια λειτουργικότητα με την άδεια έκδοση, αλλά προσθέτει ένα υδατογράφημα αξιολόγησης όταν οι παρουσιάσεις ανοίγονται ή αποθηκεύονται και περιορίζει την εξαγωγή κειμένου σε μία διαφάνεια.

Αυτό το άρθρο εξηγεί πώς λειτουργεί η αδειοδότηση στο Aspose.Slides και πώς να εφαρμόσετε άδεια πριν χρησιμοποιήσετε τη βιβλιοθήκη. Μια άδεια μπορεί να φορτωθεί από αρχείο, ροή ή ενσωματωμένο πόρο χρησιμοποιώντας την κλάση `License`. Το άρθρο δείχνει επίσης πώς να επαληθεύσετε εάν η άδεια εφαρμόστηκε σωστά.

## **Αδειοδότηση στο Aspose.Slides**
* Μια έκδοση αξιολόγησης γίνεται αδειοδοτημένη μετά την αγορά άδειας και την προσθήκη μερικών γραμμών κώδικα (για την εφαρμογή της άδειας).
* Η άδεια είναι ένα αρχείο XML απλού κειμένου που περιέχει λεπτομέρειες όπως το όνομα προϊόντος, ο αριθμός των προγραμματιστών που αδειοδοτούνται, η ημερομηνία λήξης της συνδρομής κ.λπ. 
* Το αρχείο άδειας είναι ψηφιακά υπογεγραμμένο, επομένως δεν πρέπει να το τροποποιήσετε. Ακόμη και η τυχαία προσθήκη μιας επιπλέον αλλαγής γραμμής στο περιεχόμενο του αρχείου θα την ακυρώσει.
* Το Aspose.Slides for .NET συνήθως προσπαθεί να βρει την άδεια στις εξής τοποθεσίες:
  * Ένας ρητός δρόμος
  * Ο φάκελος που περιέχει το dll του στοιχείου (περιλαμβάνεται στο Aspose.Slides)
  * Ο φάκελος που περιέχει τη συναρμολόγηση που κάλεσε το dll του στοιχείου (περιλαμβάνεται στο Aspose.Slides)
  * Ο φάκελος που περιέχει τη συναρμολόγηση εισόδου (το .exe σας)
  * Ένας ενσωματωμένος πόρος στη συναρμολόγηση που κάλεσε το dll του στοιχείου (περιλαμβάνεται στο Aspose.Slides).
* Για να αποφύγετε τους περιορισμούς που σχετίζονται με την έκδοση αξιολόγησης, πρέπει να ορίσετε μια άδεια πριν χρησιμοποιήσετε το Aspose.Slides. Χρειάζεται να ορίσετε άδεια μόνο μία φορά ανά εφαρμογή ή διαδικασία.

Μπορείτε να δείτε το [Metered Licensing](https://docs.aspose.com/slides/el/net/metered-licensing/).

{{% /alert %}} 


## **Εφαρμογή Άδειας**
Μια άδεια μπορεί να φορτωθεί από **αρχείο**, **ροή** ή **ενσωματωμένο πόρο**. 

{{% alert color="primary" %}}

Το Aspose.Slides παρέχει την κλάση [License](https://reference.aspose.com/slides/el/net/aspose.slides/license) για λειτουργίες αδειοδότησης.

{{% /alert %}} 

{{% alert color="warning" %}} 

Οι νέες άδειες μπορούν να ενεργοποιήσουν το Aspose.Slides μόνο από την έκδοση 21.4 και μετά. Παλαιότερες εκδόσεις χρησιμοποιούν διαφορετικό σύστημα αδειοδότησης και δεν θα αναγνωρίσουν αυτές τις άδειες.

{{% /alert %}}

### **Αρχείο**
Η πιο εύκολη μέθοδος ορισμού άδειας απαιτεί να τοποθετήσετε το αρχείο άδειας στον ίδιο φάκελο που περιέχει το DLL του στοιχείου (περιλαμβάνεται στο Aspose.Slides) και να καθορίσετε μόνο το όνομα του αρχείου χωρίς τη διαδρομή του.

Αυτός ο κώδικας C# δείχνει πώς να ορίσετε ένα αρχείο άδειας:

``` csharp
// Δημιουργεί ένα αντικείμενο της κλάσης License 
Aspose.Slides.License license = new Aspose.Slides.License();

// Ορίζει τη διαδρομή του αρχείου άδειας
license.SetLicense("Aspose.Slides.lic");
```

{{% alert color="warning" %}} 

Αν τοποθετήσετε το αρχείο άδειας σε διαφορετικό κατάλογο, όταν καλείτε τη μέθοδο [SetLicense](https://reference.aspose.com/slides/el/net/aspose.slides/license/setlicense/#setlicense_1), το όνομα του αρχείου άδειας στο τέλος της ρητής διαδρομής πρέπει να είναι το ίδιο με το αρχείο άδειας σας.

Για παράδειγμα, μπορείτε να αλλάξετε το όνομα του αρχείου άδειας σε *Aspose.Slides.lic.xml*. Στη συνέχεια, στον κώδικά σας, πρέπει να περάσετε τη διαδρομή προς το αρχείο (τελειώνοντας με *Aspose.Slides.lic.xml*) στη μέθοδο [SetLicense](https://reference.aspose.com/slides/el/net/aspose.slides/license/setlicense/#setlicense_1).

{{% /alert %}}

### **Ροή**
Μπορείτε να φορτώσετε μια άδεια από ροή. Αυτός ο κώδικας C# δείχνει πώς να εφαρμόσετε άδεια από ροή:

``` csharp
// Δημιουργεί ένα αντικείμενο της κλάσης License 
Aspose.Slides.License license = new Aspose.Slides.License();

// Ορίζει την άδεια μέσω ροής
license.SetLicense(myStream);
```

### **Ενσωματωμένος Πόρος**
Μπορείτε να ενσωματώσετε την άδεια στην εφαρμογή σας (για να μην τη χάσετε) προσθέτοντας την άδεια ως ενσωματωμένο πόρο σε μία από τις συναρμολογήσεις που καλούν το DLL του στοιχείου (περιλαμβάνεται στο Aspose.Slides). 

Αυτή είναι η διαδικασία προσθήκης ενός αρχείου άδειας ως ενσωματωμένου πόρου:

1. Στο Visual Studio, προσθέστε το αρχείο άδειας (.lic) στο έργο με τον τρόπο: Πηγαίνετε στο **File** > **Add Existing Item** > **Add**. 
2. Επιλέξτε το αρχείο στον **Solution Explorer**.
3. Στο παράθυρο **Properties**, ορίστε το **Build Action** σε **Embedded Resource**.
4. Για να αποκτήσετε πρόσβαση στην άδεια ενσωματωμένη στη συναρμολόγηση, προσθέστε το αρχείο άδειας ως ενσωματωμένο πόρο στο έργο και, στη συνέχεια, περάστε το όνομα του αρχείου άδειας στη μέθοδο `SetLicense`. 


Η κλάση `License` εντοπίζει αυτόματα το αρχείο άδειας στους ενσωματωμένους πόρους. Δεν χρειάζεται να καλέσετε τις μεθόδους `GetExecutingAssembly` και `GetManifestResourceStream` της κλάσης `System.Reflection.Assembly` στο Microsoft .NET Framework.

Αυτός ο κώδικας C# δείχνει πώς να ορίσετε μια άδεια ως ενσωματωμένο πόρο:

``` csharp
// Δημιουργεί ένα αντικείμενο της κλάσης License
Aspose.Slides.License license = new Aspose.Slides.License();

// Περνά το όνομα αρχείου άδειας ενσωματωμένο στη συναρμολόγηση
license.SetLicense("Aspose.Slides.lic");
```

## **Επικύρωση Άδειας**

Για να ελέγξετε εάν μια άδεια έχει οριστεί σωστά, μπορείτε να την επικυρώσετε. Αυτός ο κώδικας C# δείχνει πώς να επικυρώσετε μια άδεια:

```c#
Aspose.Slides.License license = new Aspose.Slides.License();

license.SetLicense("Aspose.Slides.lic");

if (license.IsLicensed())
{
    Console.WriteLine("License is good!");
    Console.Read();
}
```

Εάν αυτή η μέθοδος πρέπει να κληθεί ταυτόχρονα από πολλά νήματα, ίσως θελήσετε να χρησιμοποιήσετε πρωτογενή σύγχρονη (όπως κλειδί) για την αποφυγή προβλημάτων.
