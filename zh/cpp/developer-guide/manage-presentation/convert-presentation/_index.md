## **PPT 转 PPTX 转换**
### **关于 PPT 转 PPTX 转换**
Aspose.Slides 现在支持将 PPT 转换为 PPTX。
### **使用 Aspose.Slides 将 PPT 转换为 PPTX**
Aspose.Slides for C++ 为将 PPT 文档格式的演示文稿转换为 PPTX 文档格式的演示文稿提供了部分支持。由于此演示文稿转换功能刚刚在 Aspose.Slides for C++ 中引入，因此目前功能有限，仅适用于简单的演示文稿。Aspose.Slides for C++ API 库在将 PPT 演示文稿转换为 PPTX 格式演示文稿时的主要优势是使用 API 轻松实现目标。请前往此[link](/slides/zh/cpp/convert-presentation/)的代码片段部分获取更多详细信息。以下章节清晰说明了在将 PPT 格式演示文稿转换为 PPTX 格式演示文稿时哪些功能受支持，哪些不受支持。

## **演示文稿转 PDF 转换**
### **关于 PDF**
[Portable Document Format](https://en.wikipedia.org/wiki/PDF) 是一种由 Adobe System 创建的文件格式，用于在不同组织之间交换文档。此格式的目的是使文档内容能够以不依赖于查看平台的方式呈现其视觉外观。
### **Aspose.Slides for C++ 中的 PDF**
[PDF ](https://docs.fileformat.com/pdf/) 是一种由 Adobe System 创建的文件格式，用于在不同组织之间交换文档。此格式的目的是使文档内容的视觉外观不依赖于查看平台。
任何可以在 Aspose.Slides for C++ 中加载的演示文稿都可以转换为 PDF 文档。您可以仅使用 Aspose.Slides for C++ 组件直接将演示文稿导出为 PDF，无需任何其他第三方库或 Aspose.PDF 组件。

## **演示文稿转 XPS 转换**
### **关于 XPS**
[XML Parser Specification](https://en.wikipedia.org/wiki/Open_XML_Paper_Specification) 是一种页面描述语言和固定文档格式，最初由 Microsoft 开发。与 PDF 类似，XPS 是一种固定布局的文档格式，旨在保留文档的真实性并提供与设备无关的文档外观。
### **使用 Aspose.Slides 将演示文稿转换为 XPS**
任何可以由 Aspose.Slides for C++ 加载的演示文稿都可以转换为 XPS 格式。Aspose.Slides for C++ 使用高保真页面布局和渲染引擎生成固定布局的 XPS 文档。值得一提的是，Aspose.Slides for C++ 直接生成 XPS，而无需依赖随 C++ Framework 3.5 打包的 Windows Presentation Foundation (WPF) 类，从而使其能够在运行早于 3.5 版本的 C++ Framework 的机器上生成 XPS 文档。您可以在 [this topic](/slides/zh/cpp/convert-powerpoint-ppt-and-pptx-to-microsoft-xps-document/) 中了解通过 Aspose.Slides for C++ 将演示文稿导出为 XPS 文档的方式。

## **演示文稿转 TIFF 转换**
### **关于 TIFF**
[Tagged Image File Format](https://en.wikipedia.org/wiki/TIFF) 是一种最初由 Aldus 创建的、以在单个文档中存储多幅图像而闻名的格式。该格式被扫描、传真及其他图像处理应用广泛支持。
### **使用 Aspose.Slides 将演示文稿转换为 TIFF**
任何可以在 Aspose.Slides for C++ 中加载的文档也可以直接由 Aspose.Slides for C++ 转换为 TIFF 文档，无需任何第三方组件。此外，您还可以自行定义生成的 TIFF 文档中图像的尺寸。您可以在 [this topic](/slides/zh/cpp/convert-powerpoint-ppt-and-pptx-to-tiff/) 中找到通过 Aspose.Slides for C++ 将演示文稿导出为 TIFF 文档的信息。

## **常见问题**

**导出为 PDF/XPS 时，我可以包含隐藏的幻灯片吗？**

是的。导出时可通过 [PDF](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfoptions/set_showhiddenslides/)/[XPS](https://reference.aspose.com/slides/cpp/aspose.slides.export/xpsoptions/set_showhiddenslides/) 设置中的相应选项包含隐藏的幻灯片。

**是否支持保存为 PDF/A 格式（用于归档存储）？**

是的，在导出时可以使用 PDF/A 合规级别[可用](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfcompliance/)（包括 A-2a/A-2b/A-2u 和 A-3a/A-3b）。

**转换期间字体会怎样：是嵌入还是替换？**

有灵活的选项：您可以[嵌入所有字形或仅使用的子集](/slides/zh/cpp/embedded-font/)，指定[回退字体](/slides/zh/cpp/fallback-font/)，以及在字体缺少某些样式时[控制行为](/slides/zh/cpp/font-substitution/)。

**如何控制生成的 PDF 的质量和大小？**

可以使用[JPEG 质量](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfoptions/set_jpegquality/)、[文本压缩](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfoptions/set_textcompression/)，以及图像的[足够分辨率](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfoptions/set_sufficientresolution/)阈值选项，还可以选择[图片的最佳压缩](https://reference.aspose.com/slides/cpp/aspose.slides.export/pdfoptions/set_bestimagescompressionratio/)模式。

**我可以只导出特定范围的幻灯片吗（例如，5–12）？**

是的，导出支持选择幻灯片的子集。

**是否支持同时对多个文件进行多核处理？**

可以在独立进程中并行处理不同的演示文稿。重要提示：同一个[演示文稿](/reference.aspose.com/slides/cpp/aspose.slides/presentation/)对象不能在[多个线程](/slides/zh/cpp/multithreading/)中被加载或保存。

**从不同线程应用许可证时是否存在风险？**

是的，[license-setting](/slides/zh/cpp/licensing/)调用不是线程安全的，需进行同步。
